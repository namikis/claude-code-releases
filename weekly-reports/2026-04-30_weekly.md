## 🚀 最新機能系

### Claude for Creative Work — 9 つの新コネクタ ✅ GA
- **概要**: Adobe Creative Cloud / Blender / Ableton / Splice / Affinity by Canva / Autodesk Fusion / SketchUp / Resolume Arena & Wire の 9 ツールに Claude が直接接続。Photoshop・Premiere・Express、Live/Push、3D モデリング、VJ ライブ制御、音楽プロデュースなど、コーディング以外の **クリエイティブワークフロー全般を Claude 経由で操作可能** に。Blender 統合は MCP ベースで他 LLM からも利用可能（オープンソース原則尊重）。
- **対象**: 全 Claude プランで即時 GA
- **初出**: 2026-04-28（Anthropic 公式 Newsroom）
- **Claude Code への含意**: Blender Python スクリプト生成、Photoshop UXP プラグイン、SketchUp Ruby スクリプト等のクリエイティブ系コーディングで、Claude Code から該当ツールを呼び出すパスが整備された
- **情報源**: [日次レポート 2026-04-29](../reports/2026-04-29_v2.1.121-122-and-claude-creative-work.md) / [Anthropic: Claude for Creative Work](https://www.anthropic.com/news/claude-for-creative-work) / [Blender 公式プレス](https://www.blender.org/press/anthropic-joins-the-blender-development-fund-as-corporate-patron/)

### Claude Cowork on Amazon Bedrock（Claude Code Desktop 含む） ✅ GA
- **概要**: Claude Cowork デスクトップアプリと **Claude Code Desktop** を **Amazon Bedrock 経由で実行可能**に。Anthropic からのシート課金不要、AWS の従量課金のみ。データはユーザー AWS アカウント内に留まる（Bedrock はプロンプト・ファイル・レスポンスを保持しない）。AWS IAM / Bedrock API キー認証、VPC エンドポイント、CloudTrail 監査、OpenTelemetry → CloudWatch 連携、細粒度コスト属性などエンタープライズ機能完備
- **対象**: AWS Bedrock を利用する全エンタープライズ顧客（macOS / Windows / Linux）
- **初出**: 2026-04-21 公開 → 2026-04-27 AWS Weekly Roundup でハイライト
- **対応リージョン**: Claude モデルが Bedrock で利用可能な全リージョン。in-Region / geo cross-Region / global cross-Region 推論プロファイル選択可
- **意義**: Cowork が個人開発者プロダクトから **AWS 顧客のエンタープライズ標準提供チャネル** に拡張。NEC（30,000 人）など大規模顧客の受け皿
- **情報源**: [日次レポート 2026-04-28](../reports/2026-04-28_cowork-on-bedrock-and-arxiv-paper.md) / [AWS ML Blog](https://aws.amazon.com/blogs/machine-learning/from-developer-desks-to-the-whole-organization-running-claude-cowork-in-amazon-bedrock/) / [AWS Weekly Roundup 2026-04-27](https://aws.amazon.com/blogs/aws/aws-weekly-roundup-anthropic-meta-partnership-aws-lambda-s3-files-amazon-bedrock-agentcore-cli-and-more-april-27-2026/)

### Rate Limits API（管理者向け） ✅ GA
- **概要**: 組織と各ワークスペースに設定された **レート制限を Admin API キー（`sk-ant-admin...`）でプログラマティックに照会** できる新 API。Console の Limits ページと同等情報を JSON で取得。`GET /v1/organizations/rate_limits` / `GET /v1/organizations/workspaces/{workspace_id}/rate_limits` の 2 エンドポイント。`group_type`（`model_group` / `batch` / `token_count` / `files` / `skills` / `web_search`）、`models` フィールド、`limits.{type, value}` を返却。**ゲートウェイ・プロキシの自動同期、社内アラート、ワークスペース設定監査** が主要ユースケース
- **対象**: Messages API および関連リソース（Message Batches / Files / Token Counting / Agent Skills / Web Search）。Claude Managed Agents は対象外。Admin ロールメンバーのみ
- **初出**: 2026-04-24
- **制限**: **読み取り専用**（制限値の更新は Console Workspaces → Limits タブから手動）。ページネーションは現在常に単一ページ
- **情報源**: [日次レポート 2026-04-25](../reports/2026-04-25_rate-limits-api-and-nec-partnership.md) / [Rate Limits API ドキュメント](https://platform.claude.com/docs/en/build-with-claude/rate-limits-api)

### `/config` 設定の `~/.claude/settings.json` 永続化 + マルチホスト PR サポート（v2.1.119）✅
- **概要**: `/config` で変更した `theme` / `editor mode` / `verbose` 等が **`~/.claude/settings.json` に永続化**され、project / local / policy のオーバーライド順位に参加。シェル再起動で消える「一時的状態」だった挙動を **managed settings ピラミッドと統合**。同時に `--from-pr` が **GitLab MR・Bitbucket PR・GitHub Enterprise PR URL** に対応し、`owner/repo#N` ショートハンドリンクも **git remote のホストに解決**（社内 GHE/GitLab 利用時に正しいリンク生成）
- **対象**: 全プラン
- **初出**: v2.1.119（2026-04-23 23:24 UTC）
- **付随機能**: `prUrlTemplate` 設定（社内コードレビュー URL）、`CLAUDE_CODE_HIDE_CWD` 環境変数（スクリーンシェア時の cwd 非表示）、PowerShell tool の auto-approve（Windows）、`PostToolUse` hook 入力に `duration_ms` 追加、Subagent / SDK MCP サーバー再構成の並列化
- **情報源**: [日次レポート 2026-04-24（postmortem）](../reports/2026-04-24_v2.1.119-postmortem-and-managed-agents-memory.md)

### MCP `alwaysLoad` 設定 + プラグイン管理拡張（v2.1.121）✅
- **概要**: MCP server config で **`alwaysLoad: true`** を指定すると、そのサーバーの全ツールが tool-search 遅延ロードをスキップして常時利用可能に（tool-search で見落とされがちな MCP ツールを確実にロード）。新規 **`claude plugin prune`** コマンドで orphaned auto-installed プラグイン依存を削除、`plugin uninstall --prune` で連鎖削除。`/skills` には **type-to-filter 検索ボックス**追加、長い skill リストでスクロール不要に。`PostToolUse` hooks の出力置換機能（`hookSpecificOutput.updatedToolOutput`）が **MCP 限定から全ツールへ拡張**
- **対象**: 全プラン
- **初出**: v2.1.121（2026-04-28 00:31 UTC）
- **付随**: フルスクリーンモードのスクロール固定改善、オーバーフローダイアログのスクロール対応、折り返し URL クリック対応、SDK / `claude -p` で `CLAUDE_CODE_FORK_SUBAGENT=1` が non-interactive セッションで動作、`/terminal-setup` の iTerm2 clipboard 有効化、MCP server 起動エラーの自動リトライ最大 3 回、Vertex AI で X.509 証明書ベース Workload Identity Federation（mTLS ADC）対応
- **情報源**: [日次レポート 2026-04-29](../reports/2026-04-29_v2.1.121-122-and-claude-creative-work.md) / [Release v2.1.121](https://github.com/anthropics/claude-code/releases/tag/v2.1.121)

### `ANTHROPIC_BEDROCK_SERVICE_TIER` & `/resume` PR URL 検索（v2.1.122）✅
- **概要**: 環境変数 **`ANTHROPIC_BEDROCK_SERVICE_TIER`** で Bedrock サービス層を `default` / `flex` / `priority` から選択可能に（`X-Amzn-Bedrock-Service-Tier` ヘッダーとして送信）。`priority` で低レイテンシ保証、`flex` でコスト最適化を Claude Code 経由で利用可能。**`/resume` 検索に PR URL 貼り付けサポート**（GitHub / GHE / GitLab / Bitbucket）—コードレビュー時の文脈再開が容易に
- **対象**: 全プラン（Bedrock 利用時は AWS 顧客）
- **初出**: v2.1.122（2026-04-28 22:05 UTC）
- **付随**: `/mcp` で claude.ai connectors の重複表示、OpenTelemetry `api_request` / `api_error` の数値属性が string → number、新ログイベント `claude_code.at_mention`
- **情報源**: [日次レポート 2026-04-29](../reports/2026-04-29_v2.1.121-122-and-claude-creative-work.md) / [Release v2.1.122](https://github.com/anthropics/claude-code/releases/tag/v2.1.122)

### Anthropic 品質劣化 公式 Postmortem 公開
- **カテゴリ**: 透明性・運用ガバナンス
- **概要**: Anthropic が **Claude Code / Agent SDK / Cowork の品質低下を公式に認知**。3 件の独立した変更・バグが原因と認定（Claude API は影響なし）：
  1. **デフォルト努力レベル `high` → `medium`**（3/4〜4/7、v2.1.94 で修正）
