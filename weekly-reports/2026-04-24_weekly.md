## エグゼクティブサマリー

- **2つのモデルID変更が同一週に集中**。Claude Haiku 3（`claude-3-haiku-20240307`）が4/19 にAPIリタイア発効、4/23 には Enterprise pay-as-you-go / Anthropic API で `opus` エイリアスが Opus 4.6 → **Opus 4.7** に自動切替。モデルIDをピンしていない本番APIコールが直接影響を受ける最も注意すべきイベント。
- **Claude Opus 4.7 が Amazon Bedrock で GA（4/20）**。4リージョン、1Mコンテキスト、adaptive thinking 対応。ただし Claude Code の Bedrock エイリアス解決は依然 Opus 4.6 のままで、明示ピン必須／既知不具合 `#49238` 残存。
- **v2.1.118（4/23）で CLI に vim visual / visual-line モード追加、`/cost`＋`/stats` が `/usage` に統合、Hook から MCP ツール直接呼び出し（`type: "mcp_tool"`）、`DISABLE_UPDATES` 環境変数、カスタムテーマ作成・プラグイン配布が一気に投入**。v2.1.117 では macOS/Linux ネイティブバイナリに **`bfs` / `ugrep` が埋め込まれ**、`Glob`/`Grep` 専用ツールを Bash 経由で置換して検索高速化。
- **Claude Design が Pro/Max/Team/Enterprise 向け研究プレビューとして利用条件確定（4/18）**。Opus 4.7 駆動、`claude.ai/design` サーフェス、通常 chat/Code 枠とは独立の**週次クォータ**、Enterpriseではデフォルトオフ。先週の「発表のみ」から実運用条件が揃った。
- **ビジネス面では Freshfields × Anthropic 複数年提携（4/23）**が最大ニュース。Magic Circle 級の法律事務所が33拠点・5,700人にClaudeを全社導入、**Cowork展開予定**、RELX（LexisNexis親会社）株価下落。並行して米連邦政府との関係も大きく動き、**NSAが既にMythos利用中（Axiosスクープ 4/19）／CISAは未アクセス（同 4/21 訂正報道）／Trump大統領がDoD合意を "possible" と軟化発言（CNBC 4/21）**。
- **プライシング側で事件も発生**。Anthropicが**Claude Code を Pro プラン（$20）から除外する A/B テストを新規 prosumer の2%に無告知実施、Webドキュメントを全面書き換え**。48時間以内にrevert、既存ユーザーは影響なしと公式コメント。

対象週のCLIリリース本数は5本／8日で、前週（14本／8日）から減速したものの、1本あたりの変更量は密度が高く、Opus 4.7の実装定着と Pro/Max 向けの effort 既定引き上げ、ネイティブバイナリ化の第二弾（検索ツール埋め込み）が進んだ週となった。

## 🚀 最新機能系

### CLIのネイティブビルド: bfs / ugrep 埋め込み（v2.1.117）✅
- **概要**: macOS / Linux のネイティブバイナリに **`bfs`（fast find 互換）と `ugrep`（高速 grep）を内蔵**し、`Glob` / `Grep` 専用ツールの round-trip を廃止して Bash から直接呼び出す方式へ置換。検索処理が高速化。Windows / npm 版は従来通り。v2.1.113 のネイティブバイナリ移行に続くネイティブ化第二弾。
- **対象**: 全プラン・macOS / Linux のネイティブビルド
- **初出**: v2.1.117（2026-04-22）
- **情報源**: [日次レポート 2026-04-24](../reports/2026-04-24_v2.1.117-118-vim-visual-and-freshfields.md) / [GitHub Releases v2.1.117](https://github.com/anthropics/claude-code/releases)

### Vim Visual / Visual-line モード（v2.1.118）✅
- **概要**: `/config > Editor mode` で既に有効化できた vim normal モードに加え、**`v` で visual（キャラクタ選択）、`V` で visual-line（行選択）モード**を追加。選択範囲への `d` / `y` / `c` 等の operators と視覚フィードバックをサポート。ターミナル内のテキスト編集体験が本格的な vim 互換に近づいた。
- **対象**: 全プラン
- **初出**: v2.1.118（2026-04-23）
- **情報源**: [日次レポート 2026-04-24](../reports/2026-04-24_v2.1.117-118-vim-visual-and-freshfields.md)

### `/cost` + `/stats` → `/usage` 統合（v2.1.118）✅
- **概要**: `/cost` と `/stats` を**単一の `/usage` コマンド配下にタブUIとして統合**。`/cost` / `/stats` はタイピングショートカットとして残り、対応タブをオープン。加えて Settings の Usage タブは5時間・週次使用量を即座に表示、エンドポイントのレート制限時も失敗しないよう堅牢化（v2.1.116）。
- **対象**: 全プラン
- **初出**: v2.1.118（2026-04-23）
- **情報源**: [日次レポート 2026-04-24](../reports/2026-04-24_v2.1.117-118-vim-visual-and-freshfields.md) / [日次レポート 2026-04-21](../reports/2026-04-21_v2.1.116-and-opus-4.7-bedrock.md)

### Hook から MCP tools を直接呼び出し（`type: "mcp_tool"`）（v2.1.118）✅
- **概要**: hook 定義で **`type: "mcp_tool"`** を指定すると、MCP サーバーのツールを直接実行可能に。従来のシェル / HTTP / LLM プロンプトに加わる **4 つめの hook タイプ**。MCPを中心に据えた自動化ワークフローの構築が劇的に簡素化される。
- **対象**: 全プラン
- **初出**: v2.1.118（2026-04-23）
- **情報源**: [日次レポート 2026-04-24](../reports/2026-04-24_v2.1.117-118-vim-visual-and-freshfields.md)

### `DISABLE_UPDATES` 環境変数（v2.1.118）✅
- **概要**: 手動 `claude update` を含む**すべてのアップデートパスを完全にブロック**する環境変数。従来の `DISABLE_AUTOUPDATER`（自動更新のみ遮断）より厳格。規制対応や厳密なバージョン固定を要する企業・政府環境で有用。
- **対象**: 全プラン
- **初出**: v2.1.118（2026-04-23）
- **情報源**: [日次レポート 2026-04-24](../reports/2026-04-24_v2.1.117-118-vim-visual-and-freshfields.md)
