## 🚀 最新機能系

### Agent View（並行セッション管理 UI） 🔬 研究プレビュー
- **概要**: `claude agents` で **複数の Claude Code 並行セッションを一画面管理**する新インターフェース。Peek / Reply / Attach / Detach / フィルタ・グルーピング・ピン留め、バックグラウンドセッションは per-user supervisor プロセスがホストし、ファイル編集は自動で `.claude/worktrees/` 配下に git worktree を作成して隔離。`/bg`（`/background`）で既存セッションをバックグラウンド化、`claude --bg "<prompt>"` で直接起動。
- **対象**: Pro / Max / Team / Enterprise / Claude API（管理者は `disableAgentView` managed setting で組織全体を無効化可）
- **戦略的位置づけ**: 5/6 SF キーノートで予告された「並行エージェントオーケストレーション」戦略の中核機能。TestingCatalog 評: **"making Claude Code less like a chat-driven coding tool and more like an agent operations layer for software teams"**。
- **初出**: v2.1.139（2026-05-11 18:43 UTC）
- **情報源**: [日次レポート 2026-05-12](../reports/2026-05-12_v2.1.139-agent-view-and-claude-platform-aws.md) / [Anthropic Blog - Agent view in Claude Code](https://claude.com/blog/agent-view-in-claude-code) / [Agent View Docs](https://code.claude.com/docs/en/agent-view)

### `/goal` コマンド（完了条件駆動の自律実行） 🔬 研究プレビュー
- **概要**: 完了条件を `<objective>` タグで宣言し、Claude が満たすまで複数ターン自律実行を続けるコマンド。**runaway guard デフォルト 500 回 Stop-hook continuation**、`/goal status` / `pause` / `resume` / `clear` サブコマンド、`--tokens 250K` でトークン上限指定。Interactive / `-p` フラグ / Remote Control 全モード対応、elapsed/turns/tokens overlay 常時表示。Managed Agents 側「Outcomes」（5/6 公開β）の Claude Code クライアント側相当。
- **対象**: v2.1.139 以降の全プラン
- **初出**: v2.1.139（2026-05-11 18:43 UTC）
- **情報源**: [日次レポート 2026-05-12](../reports/2026-05-12_v2.1.139-agent-view-and-claude-platform-aws.md) / [Claude Code Changelog](https://code.claude.com/docs/en/changelog)

### Claude Platform on AWS ✅ GA
- **概要**: **Anthropic のネイティブ Claude Platform**（Messages API / Managed Agents β / Advisor β / Web Search・Fetch / MCP connector β / Agent Skills β / Code Execution / Files API β / Prompt Caching / Citations / Batch Processing）を **AWS IAM / CloudTrail / 統合請求を経由して利用可能**にする新経路。**Amazon Bedrock とは別経路**で、Anthropic が運用しデータは AWS セキュリティ境界外で処理される。AWS は「Anthropic のネイティブ Claude Platform を提供する最初のクラウド」。
- **対象プラットフォーム**: 18 リージョン（US/Canada/南米/欧州/アジア太平洋）。既存 AWS commitments に課金充当可能

### Fast mode for Opus 4.7 🔬 研究プレビュー
- **概要**: `speed: "fast"` + `model: "claude-opus-4-7"` + beta header `fast-mode-2026-02-01` で **標準 Opus 4.7 比 最大 2.5x の出力トークン生成速度**を実現。Opus 4.6 fast mode（2026-02 launch）と同等の **$30 入力 / $150 出力 per MTok**（標準 Opus 4.7 の 6 倍）。Console API は waitlist 制、Claude サブスクリプションは extra usage 残高経由でアクセス。Opus 4.7 GA から約 4 週間後の対応で、5/6 SpaceX Colossus 1 / 5/8 Akamai / 5/11 Claude Platform on AWS によるコンピュート供給確保完了直後のタイミング。
- **Claude Code への影響**: 現状 `/fast` トグルは Opus 4.6 fast mode 対象、Opus 4.7 fast mode 対応は今後の課題（公式言及なし）。**Pro/Max の compute crunch ナラティブへの一つの解**として位置付け可能。
- **発表**: 2026-05-12（Platform Release Notes）

### `settings.autoMode.hard_deny`（Auto Mode セキュリティ強化） ✅ GA
- **概要**: Auto Mode 分類器のルールセットに **「無条件ブロック」階層**を新設。従来は `permissions.deny`（managed settings 最強ブロック）と `autoMode.soft_deny`（分類器がコンテキスト判断で許可しうる）の二段だったところに、**Auto Mode 専用の最強拒否枠**を追加。エンタープライズ管理者が Auto Mode を許容しつつ「絶対に通過させたくないコマンド」を明示宣言できる。**5/8 Dragos 報告書 + 4/24 PocketOS DB 9 秒消去事件**と同タイミングで投入され、Auto Mode セーフガード強化として Cursor 等の第三者ツールとの差別化材料に。
