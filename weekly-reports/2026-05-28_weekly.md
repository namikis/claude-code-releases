## 🚀 最新機能系

### `/simplify` → `/code-review` rename + エフォート + GitHub PR コメント投稿 ✅ GA
- **概要**: `/simplify` コマンドを **`/code-review`** に rename し、旧自動修正動作を **`--fix` フラグの明示指定方式**へ整理。`/code-review low|medium|high` でエフォートレベル指定、**`--comment` フラグで GitHub PR にインラインコメントとして findings を投稿**。5/6 Code with Claude SF キーノートで GA 化された **GitHub Code Review（重大度ランク付き複数エージェント並行 PR レビュー）**との位置付け統一。
- **対象**: 全プラン
- **初出**: v2.1.146（2026-05-21 01:51 UTC）/ v2.1.147 で再確認
- **情報源**: [日次レポート 2026-05-21〜23](../reports/2026-05-21_to_2026-05-23_v2.1.146-150-code-review-rename-compliance-api-and-30b-close.md) / [GitHub Releases v2.1.146](https://github.com/anthropics/claude-code/releases/tag/v2.1.146) / [Code Simplifier Plugin](https://claude.com/plugins/code-simplifier)

### `/code-review --fix` 自動適用モード ✅ GA
- **概要**: コードレビュー findings の修正提案を **作業ツリーに直接適用**する `--fix` フラグを追加。`/simplify` は **`/code-review --fix` を呼び出すエイリアス**として残存（v2.1.146 の rename と統合された設計）。**5/6 GitHub Code Review GA + 5/21 `/code-review` rename**に続く一連の再編の最終段で、「レビュー＝レポートのみ」と「自動修正適用」を**ユーザーが明示的に切り替えられる**設計に到達。
- **対象**: 全プラン
- **初出**: v2.1.152（2026-05-27 01:30 UTC）
- **情報源**: [GitHub Releases v2.1.152](https://github.com/anthropics/claude-code/releases/tag/v2.1.152) / [DevelopersIO Claude Code v2.1.152 解説](https://dev.classmethod.jp/en/articles/20260524-claude-code-updates-v2-1-152/)

### Pinned background sessions のライフサイクル改善 ✅ GA
- **概要**: `claude agents` で `Ctrl+T` ピン留めしたバックグラウンドセッションが (1) **idle でも生き残る**（通常は idle で daemon が auto-retire）、(2) **アップデート適用のためにその場で再起動**（restart in place、コンテキスト復元）、(3) **メモリ圧迫時に non-pinned を先に shed、pinned は最後**、という 3 つの保護を獲得。v2.1.143 でピン留め機能が追加されたが、daemon の idle-exit / memory pressure / binary upgrade 時にピン留めが期待通り保護されない issue を修正。**長時間走行する Goal-driven セッション / multi-day refactoring セッションのピン留めが信頼できるレベルに到達**。
- **対象**: 全プラン（5/12 v2.1.139 Agent View / 5/15 v2.1.143 Pinned 機能の継続改善）
- **初出**: v2.1.147（2026-05-21 20:39 UTC）
- **情報源**: [日次レポート 2026-05-21〜23](../reports/2026-05-21_to_2026-05-23_v2.1.146-150-code-review-rename-compliance-api-and-30b-close.md) / [GitHub Releases v2.1.147](https://github.com/anthropics/claude-code/releases/tag/v2.1.147)

### `/usage` per-category breakdown ✅ GA
- **概要**: `/usage` コマンドが **カテゴリ別内訳**で limits 利用状況を表示。**Skills** 利用量 / **Subagents** 利用量 / **Plugins** 利用量 / **per-MCP-server cost**（各 MCP サーバーが消費したコスト）の 4 区分。5/14 サブスクリプション課金構造変更（third-party agent harness の別建てメーター化、6/15 施行）、5/19 `extra-usage` → `usage-credits` rename と連動した **コスト可視性の本格的強化**。**「どの MCP サーバーが課金を圧迫しているか」「どの subagent が rate-limit を消費しているか」を visibility 化**、コスト最適化判断が可能に。
- **対象**: 全プラン（特にエンタープライズのコスト最適化）
- **初出**: v2.1.149（2026-05-22 22:09 UTC）
- **情報源**: [日次レポート 2026-05-21〜23](../reports/2026-05-21_to_2026-05-23_v2.1.146-150-code-review-rename-compliance-api-and-30b-close.md) / [GitHub Releases v2.1.149](https://github.com/anthropics/claude-code/releases/tag/v2.1.149)

### スキル管理拡張（`/reload-skills` / `disallowed-tools` / SessionStart `reloadSkills`） ✅ GA
- **概要**: **`/reload-skills`** コマンド追加でセッション再起動なしにスキルディレクトリを再スキャン可能。**スキルのフロントマターで `disallowed-tools`** を設定し、特定スキル実行中に使用禁止ツールを宣言可能（個別スキル単位のセキュリティ境界）。**`SessionStart` フックで `reloadSkills: true`** を返すとセッション開始時にスキル再スキャン、**`hookSpecificOutput.sessionTitle`** でセッションタイトルを動的設定可能。エンタープライズのスキルガバナンスと開発時イテレーション両面のニーズを同時に解消。
- **対象**: 全プラン
- **初出**: v2.1.152（2026-05-27 01:30 UTC）
- **情報源**: [GitHub Releases v2.1.152](https://github.com/anthropics/claude-code/releases/tag/v2.1.152) / [DevelopersIO Claude Code v2.1.152 解説](https://dev.classmethod.jp/en/articles/20260524-claude-code-updates-v2-1-152/)

### `MessageDisplay` フック ✅ GA

- **対象**: 全プラン
- **初出**: v2.1.152（2026-05-27 01:30 UTC）

### Claude Code サンドボックスバイパス silent パッチ開示（SecurityWeek 5/20）
- **カテゴリ**: セキュリティインシデント（既修正の回顧的開示）
- **概要**: SecurityWeek が **Anthropic が Claude Code のネットワーク egress（送信）フィルタを回避できる 2 件のサンドボックスバイパス**を告知なくパッチしたと開示。(1) **設定誤解釈バグ（CVE-2025-66479）**: 「すべてのアウトバウンドトラフィックをブロック」設定が「すべて許可」と誤解釈される。(2) **SOCKS5 ホスト名ヌルバイト挿入（CVE 未割り当て）**: `attacker-host.com\x00.google.com` のようなホスト名でフィルタが末尾の `.google.com` を許可と判定する一方、OS はヌルバイトで切り詰めて攻撃者ホストへ接続。**プロンプトインジェクションと組み合わせで環境変数・認証情報の流出可能**。
- **発生日**: 2026-05-20（SecurityWeek 開示）/ 修正自体は過去の 2.1.x で適用済み（具体バージョン要一次確認）
- **影響**: 現行版（v2.1.150 / v2.1.152）には影響なし。ただし「silently patched」の透明性が業界で議論。Claude Code を本番運用するエンタープライズの **「ネットワーク egress フィルタを最強防御線と見なしてはいけない」**教訓として記録価値。
- **情報源**: [日次レポート 2026-05-26](../reports/2026-05-26_microsoft-pulls-claude-code-and-mythos-public-toggle.md) / [SecurityWeek (5/20)](https://www.securityweek.com/anthropic-silently-patches-claude-code-sandbox-bypass/)
