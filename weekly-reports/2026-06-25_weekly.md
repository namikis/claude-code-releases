# Claude Code 週次レポート: 2026-06-18 〜 2026-06-25

**作成日:** 2026-06-25
**対象期間:** 2026-06-18 〜 2026-06-25（日次レポート 7 件を統合）
**対象バージョン:** v2.1.181 〜 v2.1.191（期間中のCLIリリース。v2.1.180/182/184/188/189 は欠番）

---

## エグゼクティブサマリー

- **CLI は安全・ガバナンス・安定化に全振りの一週間**: v2.1.183 で **Auto mode の破壊的 git / IaC コマンドガードレール**（`git reset --hard` / `terraform destroy` 等を破棄指示なしにブロック）が GA。v2.1.187 で **`sandbox.credentials`**（サンドボックスからの資格情報読み取りブロック）と**組織モデル制限の UI 露出**、v2.1.191 で **`/clear` 前からの `/rewind`** と**ストリーミング中 CPU 約 37% 削減**。新スラッシュコマンド・新 Hook は期間中ゼロで、既存機能の堅牢化が主軸。
- **エンタープライズ向けプラットフォーム機能が 6/18 に 2 本ベータ投入**: **Claude Code Artifacts**（セッション成果物をライブ更新の共有 HTML 化）と**エンタープライズ管理 MCP 認可（EMA / Okta）**（IdP 経由の zero-touch MCP コネクタ）。さらに 6/23 に **Claude Tag for Slack**（チーム共有型 AI ティームメイト、Claude in Slack を置換）が公式発表。
- **Anthropic ソウルオフィス開設（6/18）** と韓国企業の大型採用（Naver エンジニア全部門＝アジア最大の利用事例、Nexon / Samsung SDS 等）。
- **Claude Fable 5 / Mythos 5 は週を通じてアクセス停止が継続**（6/12 米政府輸出管理指令）。6/18 に国際担当 MD Ciauri が「数日以内に復旧と確信」と公式発言した一方、週末にかけて予測市場の復旧見通しは「近日」→「年内後ろ倒し」へ大幅後退し、公式発言と市場が乖離。
- **プラットフォーム障害は沈静化後に再燃**: 週前半は沈静化傾向だったが、6/22→6/23 と連日の大規模障害（Claude.ai/Console/API/Code/Cowork に影響、約 2.5 時間で解消）。

全体トーンとしては、新機能の派手なローンチよりも「自律実行の安全装置」「エンタープライズ・ガバナンス」「MCP 信頼性」「パフォーマンス」の地固めが目立つ週で、ビジネス面では韓国市場拡大という前進と、輸出規制によるフラッグシップモデル停止の長期化・インフラ障害という逆風が同居した。

## 🚀 最新機能系

### Auto mode の破壊的 git / IaC コマンドガードレール ✅ GA
- **概要**: ユーザーが破棄を指示していない限り、破壊的 git コマンド（`git reset --hard` / `git checkout -- .` / `git clean -fd` / `git stash drop`）をブロック。エージェント自身が当該セッションで作成していない commit への `git commit --amend` もブロック。`terraform destroy` / `pulumi destroy` / `cdk destroy` も特定スタック指定がない限りブロック。
- **対象**: 全プラン（Auto mode 利用時）
- **初出**: v2.1.183 (2026-06-19)
- **情報源**: [日次レポート](../reports/2026-06-20_v2.1.183-auto-mode-git-guardrails-artifacts-and-okta-mcp.md) / [Claude Code changelog](https://code.claude.com/docs/en/changelog)

### Claude Code Artifacts 🧪 ベータ
- **概要**: Claude Code セッションの成果物を、ライブで更新される共有可能な対話的 HTML ページに変換。PR ウォークスルー・インシデントページ・ダッシュボード・チェックリスト等が用途。作成者・組織内に限定（公開不可）。
- **対象**: Claude Team / Enterprise、Claude Code CLI + デスクトップアプリ
- **初出**: 2026-06-18（報道）
- **情報源**: [日次レポート](../reports/2026-06-20_v2.1.183-auto-mode-git-guardrails-artifacts-and-okta-mcp.md) / [VentureBeat](https://venturebeat.com/data/anthropics-claude-code-artifacts-update-brings-live-shared-dashboards-and-interactive-workspaces-to-enterprises) / [Claude Help Center](https://support.claude.com/en/articles/9487310-what-are-artifacts-and-how-do-i-use-them)

### エンタープライズ管理 MCP 認可（EMA / Okta）🧪 ベータ
- **概要**: 管理者が ID プロバイダ経由で MCP コネクタを一度プロビジョニングすれば、ユーザーは初回ログイン時に zero-touch でアクセスを継承。SSO で得た署名済み ID-JAG JWT を MCP サーバーの access token と交換する仕組み。MCP の EMA 拡張が 6/18 に "stable" 化した初の本番実装で、Okta が最初の対応 IdP。
- **対象**: Claude Team / Enterprise、Claude / Claude Code / Cowork を横断（launch 時点で Asana / Atlassian / Canva / Figma / Linear / Supabase 等が対応、Slack も追加中）
- **初出**: 2026-06-18
- **情報源**: [日次レポート](../reports/2026-06-20_v2.1.183-auto-mode-git-guardrails-artifacts-and-okta-mcp.md) / [TestingCatalog](https://www.testingcatalog.com/anthropic-launches-managed-connector-access-with-okta/) / [Okta Newsroom](https://www.okta.com/en-ca/newsroom/press-releases/okta-becomes-a-featured-identity-provider-powering-secure-ai-agent-connections-for-claude-enterprise/)

### Claude Tag for Slack 🧪 ベータ
- **概要**: Slack で `@Claude` をタグ付けしタスクを委任できるチーム共有型 AI ティームメイト。タスクを段階に分解して実行し Slack スレッドへ成果を投稿、チャンネル内では 1 つの Claude を全員が共有しチームの作業文脈を記憶。既存の「Claude in Slack」アプリを置換（管理者は 30 日以内にオプトイン移行）。
- **対象**: Claude Enterprise / Team、Slack、Opus 4.8 で動作
- **初出**: 2026-06-23（Anthropic 公式ニュース「Introducing Claude Tag」）
- **情報源**: [日次レポート](../reports/2026-06-25_v2.1.190-191-rewind-before-clear-and-claude-tag-slack.md) / [TechCrunch](https://techcrunch.com/2026/06/23/anthropics-claude-tag-is-learning-your-company-one-slack-message-at-a-time/) / [Fortune](https://fortune.com/2026/06/23/anthropic-claude-tag-virtual-employee-tool-slack/)

### `claude mcp login` / `claude mcp logout`（MCP CLI 認証コマンド）✅ GA
- **概要**: インタラクティブな `/mcp` メニューを開かず、ターミナルから直接 MCP サーバーを認証 / ログアウト。`--no-browser` で stdin リダイレクトをサポートし SSH 越しでも認証フロー完了が可能。
- **対象**: 全プラン
- **初出**: v2.1.186 (2026-06-22)
- **情報源**: [日次レポート](../reports/2026-06-23_v2.1.186-mcp-cli-auth-and-bash-autorespond.md) / [Claude Code changelog](https://code.claude.com/docs/en/changelog)

### `!` Bash コマンド出力への Claude 自動応答 ✅ GA
- **概要**: 従来 `!` で実行した Bash 出力はコンテキストに追加されるだけだったが、v2.1.186 から Claude が自動的にその出力へ応答するよう挙動変更。`settings.json` の `respondToBashCommands: false` で従来挙動に戻せる。
- **対象**: 全プラン
- **初出**: v2.1.186 (2026-06-22)
- **情報源**: [日次レポート](../reports/2026-06-23_v2.1.186-mcp-cli-auth-and-bash-autorespond.md) / [Claude Code changelog](https://code.claude.com/docs/en/changelog)

### `/clear` 実行前の地点からの `/rewind` 対応 ✅ GA
- **概要**: これまで `/clear`（会話履歴クリア）後はその前の文脈に戻れなかったが、v2.1.191 で `/rewind` が `/clear` 前のチェックポイントへの巻き戻し・会話再開をサポート。誤って `/clear` したケースの救済になる。
- **対象**: 全プラン
- **初出**: v2.1.191 (2026-06-24)
- **情報源**: [日次レポート](../reports/2026-06-25_v2.1.190-191-rewind-before-clear-and-claude-tag-slack.md) / [CHANGELOG.md](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)

### `/config key=value` 構文 ✅ GA
- **概要**: プロンプトから任意の設定値を直接変更できる構文（例 `/config thinking=false`）。interactive・`-p`（print）・Remote Control の全モードで動作。v2.1.183 で `/config --help`（shorthand キー一覧）とトグル挙動の改善（Esc が「保存して閉じる」に変更）が追加された。
- **対象**: 全プラン
- **初出**: v2.1.181 (2026-06-17)、`--help` 追加は v2.1.183 (2026-06-19)
- **情報源**: [日次レポート](../reports/2026-06-18_v2.1.181-config-keyvalue-and-fable5-still-suspended.md) / [日次レポート](../reports/2026-06-20_v2.1.183-auto-mode-git-guardrails-artifacts-and-okta-mcp.md)

### 組織モデル制限のモデル選択 UI / CLI への露出 ✅ GA
- **概要**: 組織が設定したモデル制限（`availableModels` / `enforceAvailableModels`）を `/model` ピッカー・`--model`・`ANTHROPIC_MODEL` に反映。制限対象を選ぶと「restricted by your organization's settings」と表示。これまでバックエンドで固めてきたモデルガバナンスを、ユーザーが操作する入口に可視化した。
- **対象**: Team / Enterprise（モデルガバナンス利用組織）
- **初出**: v2.1.187 (2026-06-23)
- **情報源**: [日次レポート](../reports/2026-06-24_v2.1.187-sandbox-credentials-and-org-model-restrictions.md) / [Claude Code changelog](https://code.claude.com/docs/en/changelog)

### Claude Design 大型アップデート（Claude Code 連携強化）🧪 ベータ
- **概要**: デザインシステムの GitHub / デザインファイル / 生アップロードからのインポート、キャンバス直接編集、エクスポート拡充に加え、`/design`・`/design-sync` でターミナルからデザインプロジェクトを作成・編集・同期する Claude Code 連携を導入。chat / Cowork / Claude Code 間で使用量上限を共有。
- **対象**: Claude Pro / Max / Team / Enterprise のベータ
- **初出**: 2026-06-17
- **情報源**: [日次レポート](../reports/2026-06-19_seoul-office-fable5-restore-outlook-and-claude-design-update.md) / [TechRepublic](https://www.techrepublic.com/article/news-anthropic-claude-design-overhaul-enterprise-teams/)

> 注: `/design`・`/design-sync` は Claude Design ベータ側の機能として案内されており、Claude Code 公式 CHANGELOG（Layer 1）には未掲載。

## 💼 ビジネス系

### Anthropic ソウルオフィス開設と韓国企業の大型採用
- **カテゴリ**: 組織・市場拡大 / エンタープライズ採用
- **概要**: 6/18 にソウルで韓国オフィスを開設（代表理事 Choi Ki-young）。Naver がエンジニアリング部門全体に Claude Code を導入し「アジア最大のエンタープライズ利用事例」と位置づけられた。Nexon（ゲーム）、Samsung SDS（Cowork + Claude Code）、LG CNS / WRTN / Law&Company / 国家 AI 研究ラボ等も発表。
- **発生日**: 2026-06-18
- **影響**: アジア最大級の Claude Code エンタープライズ採用事例が出現し、北米偏重からの市場分散が進む。
- **情報源**: [日次レポート](../reports/2026-06-19_seoul-office-fable5-restore-outlook-and-claude-design-update.md) / [Korea JoongAng Daily](https://www.koreajoongangdaily.com/business/anthropic-confident-of-reenabling-mythos-fable-5-access-in-coming-days-executive/12727522) / [The Korea Times](https://www.koreatimes.co.kr/business/tech-science/20260618/anthropic-opens-seoul-office-to-expand-ties-with-korean-ai-ecosystem)

### Claude Fable 5 / Mythos 5 アクセス停止の継続と復旧見通しの乖離
- **カテゴリ**: 規制（輸出管理）
- **概要**: 6/12 の米商務省輸出管理指令（外国籍者アクセス停止要求）により全ユーザー停止が週を通じて継続。6/18 にソウルで国際担当 MD Chris Ciauri が「数日以内（in the coming days）に復旧と非常に確信」とオン・レコードで明言（❓噂 → 📢公式幹部発言へ格上げ）。しかし週末にかけて予測市場（Polymarket / Kalshi）の復旧見通しは「7/1 が最有力（約 50%）」から「12/31 = 87% / 7/31 = 68% / 7/1 = 20%」へと大幅に後ろ倒しし、公式発言と市場が乖離。規制の発端は「Mythos アクセス予定先に中国と潜在的関係を疑われる韓国通信会社（SK Telecom と報道）が含まれる」点とされる。Opus 4.8 含む他の全モデルは無影響。
- **発生日**: 2026-06-12（停止開始）〜 期間中継続
- **影響**: フラッグシップ Mythos クラスの一般提供が長期停止し、Glasswing 参加組織や国際ユーザーへの影響が継続。Claude Code の通常ワークフローはデフォルト Opus 4.8 のため影響は限定的。
- **情報源**: [日次レポート(6/19)](../reports/2026-06-19_seoul-office-fable5-restore-outlook-and-claude-design-update.md) / [日次レポート(6/24)](../reports/2026-06-24_v2.1.187-sandbox-credentials-and-org-model-restrictions.md) / [Korea JoongAng Daily](https://www.koreajoongangdaily.com/business/anthropic-confident-of-reenabling-mythos-fable-5-access-in-coming-days-executive/12727522) / [Polymarket](https://polymarket.com/event/claude-fable-5-restored-for-us-customers-by-20260613193753196)

### Fable 5 の課金パッケージ変更（無料同梱 → usage credits → 恒久同梱シグナル）
- **カテゴリ**: 価格 / 商用条件
- **概要**: 6/9〜6/22 の「サブスク同梱無料」期間が 6/22 で終了し、6/23 から usage credits 建てへ移行する建付け。さらに 6/25 に v2.1.190 の文字列差分から、Fable 5 をサブスクに恒久同梱し週次 usage 上限を課す方向の準備が検出された（"You've used your Fable 5 usage for this week" 追加、"purchased separately from your plan" 削除）。ただしモデル自体が停止中のため実質保留で、課金シグナルは復旧の確定材料ではない。
- **発生日**: 2026-06-22（無料終了）〜 2026-06-25（恒久同梱シグナル、❓データマイン）
- **影響**: プレミアムアドオン → サブスク標準同梱＋週次上限という料金体系の転換シグナル。
- **情報源**: [日次レポート(6/22)](../reports/2026-06-22_v2.1.185-stream-stall-message-and-fable5-still-suspended.md) / [日次レポート(6/25)](../reports/2026-06-25_v2.1.190-191-rewind-before-clear-and-claude-tag-slack.md)

### プラットフォーム大規模障害（6/22→6/23 連日）
- **カテゴリ**: 障害 / インフラ
- **概要**: 週前半は沈静化傾向（6/18 短時間障害は解決済み）だったが、6/23 10:02 AM ET 頃から大規模障害が発生（10:13 ET にピーク約 7,119 件報告、12:44 ET 解決）。Claude.ai / Console / API / Claude Code / Cowork に影響し、6/22 の障害から 14 時間未満での再発。一部メディア（Cybernews）が障害後に「顧客データ漏洩の主張を Anthropic が調査中」と報じたが、公式確認は取れておらず ❓ 未確認。
- **発生日**: 2026-06-22 / 2026-06-23
- **影響**: 6 月を通じたインフラ逼迫・不安定さのパターンが継続。Claude Code 利用にも断続的に影響。
- **情報源**: [日次レポート](../reports/2026-06-24_v2.1.187-sandbox-credentials-and-org-model-restrictions.md) / [TechRadar](https://www.techradar.com/news/live/claude-down-june-23-2026) / [Claude Status](https://status.claude.com/)
