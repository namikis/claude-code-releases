## エグゼクティブサマリー

- **Claude Opus 4.7 GA**（4/16）がウィークのハイライト。SWE-bench Verified 87.6% でフロンティアリーダーシップを確立し、Claude API／Bedrock／Vertex／Foundry／Snowflake Cortex／GitHub Copilotの6プラットフォームに同日展開。4月23日にはAPIの `opus` エイリアスが自動的に Opus 4.7 を指すデフォルト切替が予定されている。
- **v2.1.113（4/17）でCLIがネイティブバイナリへ移行**。Node.jsバンドルの呼び出しを廃止し、Bun/JSCベースのネイティブプロセスを spawn する約100MBのアーキテクチャ転換。起動性能・Node.js依存排除を狙う近年最大級の構造変更。
- **Routines 研究プレビュー公開（4/14）**。スケジュール／API／GitHubイベント駆動の自動化を Anthropic クラウドで実行する新機能で、Claude Code の「CLI + 自動実行バックエンド」化が進んだ。デスクトップアプリも全面再設計され、統合ターミナル・サイドチャット・再配置ペインなど IDE 相当のワークスペースに進化。
- **Claude Design（Anthropic Labs）発表（4/17）**。デザイン自動化領域への戦略的拡張でFigma・Adobe・Wix株が下落。Microsoft Office三大アプリ（Word Add-in GA）との統合完了と合わせ、Claudeの射程が開発者領域を越えて拡大。
- **ビジネス面では規制・政治・インフラ**が同時進行。ホワイトハウス首席補佐官とDario Amodei CEOの面会で米連邦機関へのMythosアクセス認可が報じられる一方、Mythosの自律サイバー攻撃能力（TLO 32ステップ完走）がUK AI安全性研究所により評価され、IPO噂（$380B評価額、10月予定）・Claude.ai大規模障害（4/15、約3時間）・Claudeパフォーマンス低下へのユーザー反発など複数の火種が並列化している。

期間中のCLIリリース本数（14本／8日）は近年最高水準で、大型機能（Opus 4.7統合／ネイティブバイナリ／Routines／`/ultrareview`／`xhigh`努力レベル）と同時にセキュリティハードニングが大量投入されており、開発速度・商用展開・安全性対応のすべてが加速している週だった。

- **概要**: SWE-bench Verified 87.6%（Opus 4.6比+6.8pt）、SWE-bench Pro 64.3%（GPT-5.4・Gemini 3.1 Proを上回る）、2,576pxビジョン、`xhigh`新努力レベルと「タスク予算」機能を搭載。価格はOpus 4.6据え置き（入力$5／出力$25 per 1M tokens）。トークナイザー更新によるAPIブレーキングチェンジあり。
- **対象**: Claude API／Amazon Bedrock／Google Vertex AI／Microsoft Foundry／Snowflake Cortex AI（Public Preview）／GitHub Copilot の6プラットフォーム同日展開。

### CLIのネイティブバイナリ移行（v2.1.113） ✅
- **概要**: CLIがバンドルJavaScriptを呼び出す代わりに、プラットフォーム別 optional dependency として配布される**ネイティブClaude Codeバイナリ**を spawn する構造に変更。Bun + JavaScriptCore ベースで約100MB、Node.jsバージョン依存を段階的に排除。既存の `npm install` 経路は維持。
- **対象**: 全プラン・全プラットフォーム（macOS/Linux/Windows）の全ユーザー。インストールパスに直接影響する近年最大級のアーキテクチャ変更。
- **初出**: v2.1.113（2026-04-17）
- **情報源**: [日次レポート 2026-04-18（v2.1.113）](../reports/2026-04-18_v2.1.113-native-binary-and-default-model-shift.md) / [frr.dev 解説](https://www.frr.dev/posts/claude-code-native-build-bun/)

- **対象**: Pro / Max / Team / Enterprise。Web（claude.ai/code/routines）／Desktop App／CLI（`/schedule`）から作成。
- **初出**: 2026-04-14
- **情報源**: [日次レポート 2026-04-15](../reports/2026-04-15_v2.1.107-108-routines-and-desktop-redesign.md) / [Anthropic公式](https://claude.com/blog/introducing-routines-in-claude-code) / [Docs](https://code.claude.com/docs/en/routines)

### `xhigh` 努力レベルと `/effort` インタラクティブスライダー ✅
- **概要**: Opus 4.7 専用の新しい努力レベル `xhigh`（`high` と `max` の間）をCLIに統合。Claude Codeは全プランで `xhigh` をデフォルト推奨。`/effort` を引数なしで呼び出すと矢印キーで選択できるインタラクティブスライダーが表示される。他モデルでは `high` にフォールバック。
- **対象**: 全プラン（Opus 4.7 使用時）
- **初出**: v2.1.111（2026-04-16）
- **情報源**: [日次レポート 2026-04-18（v2.1.111-112）](../reports/2026-04-18_v2.1.111-112-opus-4.7-and-claude-design.md)

### `/ultrareview` コマンド ✅
- **概要**: クラウドで並列マルチエージェント分析による包括的コードレビューを実行。引数なしで現在のブランチ、`/ultrareview <PR#>` で特定 GitHub PR をフェッチ。v2.1.113 で並列化による高速起動、diffstat 表示、起動中アニメーションが追加。
- **対象**: 全プラン
- **初出**: v2.1.111（2026-04-16）
- **情報源**: [日次レポート 2026-04-18（v2.1.111-112）](../reports/2026-04-18_v2.1.111-112-opus-4.7-and-claude-design.md) / [日次レポート 2026-04-18（v2.1.113）](../reports/2026-04-18_v2.1.113-native-binary-and-default-model-shift.md)

## 💼 ビジネス系

### 4月23日デフォルトAPIモデル切替（Opus 4.6 → Opus 4.7）📢
- **カテゴリ**: 価格・API挙動変更
- **概要**: Enterprise pay-as-you-go および Anthropic API（第一者）で、`opus` エイリアスが自動的に Claude Opus 4.7 を指すように切り替わる。Opus 4.6 は `claude-opus-4-6-*` で引き続き利用可能（自動リタイアではない）。Opus 4.7 はトークナイザー・API挙動でブレーキングチェンジを含むため、モデルIDを明示ピンしていない本番APIコールは事前検証が必須。
- **発生日**: 2026-04-23（予定）／告知は 2026-04-17
- **影響**: モデルピン未設定の全APIユーザー。本番トラフィックに直接影響
- **情報源**: [日次レポート 2026-04-18（v2.1.113）](../reports/2026-04-18_v2.1.113-native-binary-and-default-model-shift.md) / [APIリリースノート](https://platform.claude.com/docs/en/release-notes/overview)

- **概要**: Dario Amodei CEO がホワイトハウス首席補佐官 Susie Wiles 氏と面会（4/17）。トランプ政権が国防総省の反対を押し切り、CISA・財務省・インテリジェンスコミュニティ等の米連邦機関に Mythos アクセスを認可する方向で調整中。3月の国防総省ブラックリスト指定（控訴中）とレッドラインポリシーをめぐる紛争解消交渉。インド金融インフラ保護目的や、カナダ国家AI大臣からの評価など、政府顧客の需要が同時並行で拡大。
- **発生日**: 2026-04-17
- **影響**: エンタープライズ／政府契約市場の開拓、Mythos商用化の道筋、政策リスクの解消

### Claude.ai 大規模障害（2026-04-15）

- **概要**: 4/15 10:53 AM ET発生、11:03 AM に一時修正、11:40 AM 再発、1:42 PM 完全解決（約3時間）。影響範囲は Claude.ai／Claude API／Claude Code 全体。15,000件超の報告。公式原因は「ユーザー需要の急増」。3〜4月に繰り返される障害パターンで、4月6-7日の2日連続障害に続くもの。

### Claude パフォーマンス低下問題とユーザー反発
- **カテゴリ**: ブランド・品質論争
- **概要**: Anthropic がデフォルト努力レベルを medium に引き下げたトークン節約策に対して、AMD AI部門シニアディレクター Stella Laurenzo が6,852セッション／234,760ツール呼び出しの分析で67%のパフォーマンス低下を文書化。Microsoft 研究者 Dimitris Papailiopoulos も効力 max 設定時の杜撰さを指摘。Boris Cherny（Claude Code責任者）が Team/Enterprise デフォルトの high effort 化テストを認め、v2.1.94 で API-key／Bedrock／Vertex／Foundry／Team／Enterprise 向けに medium → high 変更済み。透明性の欠如が最大の批判点。
- **発生日**: 報道集中は 2026-04-13〜15
- **影響**: 「透明性重視」ブランドの毀損、IPO 投資家評価、顧客離れリスク
- **情報源**: [日次レポート 2026-04-16](../reports/2026-04-16_v2.1.109-110-tui-fullscreen-and-outage.md) / [Fortune](https://fortune.com/2026/04/14/anthropic-claude-performance-decline-user-complaints-backlash-lack-of-transparency-accusations-compute-crunch/) / [VentureBeat](https://venturebeat.com/technology/is-anthropic-nerfing-claude-users-increasingly-report-performance)
