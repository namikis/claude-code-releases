## 🚀 最新機能系

### Code Review ✅ GA 昇格（Claude Code）
- **概要**: PR がオープンされると複数の AI エージェントが並行で精査し、severity ランク付けでインラインコメントを残す。Anthropic 自社では「every team at Anthropic で利用」（CPO Ami Vora）。1,000 行以上の大規模 PR で **84% に平均 7.5 件の指摘**、平均レビュー時間 **約 20 分**。価格は **$15-25 / レビュー**（PR 規模・複雑度依存）。
- **対象**: Team / Enterprise（GA 後の対象は今後 Help Center で要確認）
- **初出**: 5/6 Code with Claude SF キーノートで GA 宣言（研究プレビューは 2026-03-09〜）
- **情報源**: [日次レポート 2026-05-07](../reports/2026-05-07_code-with-claude-sf-keynote-and-v2.1.129-132.md) / [Code Review Docs](https://code.claude.com/docs/en/code-review) / [Simon Willison Live blog](https://simonwillison.net/2026/May/6/code-w-claude-2026/)

### Security Reviews ✅ GA 昇格（Claude Code 内蔵）
- **概要**: PR / コミットに対して自動的にセキュリティ脆弱性検出を実行。`/security-review` コマンドでセッション中のペンディング変更を包括的レビューも可能。Claude Security（5/1 公開ベータ、claude.ai/security のリポジトリスキャン製品）とは**別製品**で、CLI / IDE 内のフローで完結。
- **対象**: Claude Code 全プラン（具体プラン制限は今後 Help Center 更新待ち）
- **初出**: 5/6 Code with Claude SF キーノートで GA 宣言（`/security-review` スラッシュコマンドは 2025-08〜提供）
- **情報源**: [日次レポート 2026-05-07](../reports/2026-05-07_code-with-claude-sf-keynote-and-v2.1.129-132.md) / [Automated Security Reviews in Claude Code (Help Center)](https://support.claude.com/en/articles/11932705-automated-security-reviews-in-claude-code) / [claude-code-security-review (GitHub)](https://github.com/anthropics/claude-code-security-review)

### CI Auto-Fix ✅ GA 昇格（Claude Code on Web）
- **概要**: PR の **CI 失敗とレビューコメントを継続監視** → ログ / フィードバックを読んで何を直すか判断 → 同一ブランチに自動 push。**ループしない設計**で、解決できない場合は試行内容を添えて人間にエスカレーション。Code Review（指摘）+ Security Reviews（脆弱性指摘）+ CI Auto-Fix（自動修正）で **「PR の品質を Claude が一気通貫で担保」** ストーリーが完成。
- **対象**: Claude Code on Web の利用者（PR ベースのワークフロー）
- **初出**: 5/6 Code with Claude SF キーノートで GA 宣言（研究プレビューは 2026-03〜）
- **情報源**: [日次レポート 2026-05-07](../reports/2026-05-07_code-with-claude-sf-keynote-and-v2.1.129-132.md) / [PR Autofix with Claude Code (GitHub Marketplace)](https://github.com/marketplace/actions/pr-autofix-with-claude-code)

### Remote Agents ✅ GA 昇格（旧 Remote Control の後継 / 名称統一）
- **概要**: ローカル PC で Claude Code エージェントを走らせたまま、**iOS / Android / ブラウザから状態確認・承認・指示・キュー投入**が可能。エージェントはローカルマシン上で動作（フルツールアクセス保持）し、モバイルからはチャットレベルの制御のみ。コード・ファイルはローカルに留まる暗号化チャネル設計。Vora が「control your laptop from your phone」と紹介。
- **対象**: Claude Pro / Max / Team / Enterprise（プラン拡大の有無は今後の Help Center で要確認）
- **初出**: 5/6 Code with Claude SF キーノートで GA 宣言（研究プレビューは 2026-01〜、旧称 "Remote Control"）
- **情報源**: [日次レポート 2026-05-07](../reports/2026-05-07_code-with-claude-sf-keynote-and-v2.1.129-132.md) / [VentureBeat](https://venturebeat.com/orchestration/anthropic-just-released-a-mobile-version-of-claude-code-called-remote)

### Outcomes 🧪 Public Beta（Claude Managed Agents、Claude Code から利用可能）
- **概要**: 「**成功とはどんな状態か」を明示的に定義**し、Claude が反復しながら自律的にゴールを目指す機能。`explicit success criteria`（テスト全パス / 元 PR 説明と一致 等）を渡すと達成まで何回でも自動 iterate。Anthropic 計測で自己評価成功率が **最大 +10pt 向上**。

- **Claude Code への含意**: Auto Mode（Pro / Max）と組み合わせると、ロングホライズン作業（リファクタ / 大規模ライブラリ移行 / バグ調査）で「人間が逐次承認しなくてもゴール条件を満たすまで走る」運用が現実味を帯びる
- **情報源**: [日次レポート 2026-05-07](../reports/2026-05-07_code-with-claude-sf-keynote-and-v2.1.129-132.md) / [Anthropic Blog - New in Claude Managed Agents](https://claude.com/blog/new-in-claude-managed-agents) / [Claude Managed Agents Overview Docs](https://platform.claude.com/docs/en/managed-agents/overview)

### Multi-Agent Orchestration 🧪 Public Beta（Managed Agents + Claude Code Agent Teams）
- **概要**: 1 つのリードエージェントが複数のサブエージェントに**並列でタスクを分解 / 委任 / 統合**するオーケストレーション。各サブエージェントは独立コンテキストで動作。Claude Code 側 Agent Teams（lead/teammate モデル、tmux/in-process 表示、TaskCreated / TaskCompleted / TeammateIdle hooks）は引き続き `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` フラグ要 experimental（公開ベータ昇格は Managed Agents 側 API での状態）。
- **対象**: Claude Managed Agents API + Claude Code（Code 側は experimental 継続）
- **初出**: 2026-05-06（研究プレビュー → 公開ベータ昇格）
- **使用例**: PR レビューを 3 体（セキュリティ / パフォーマンス / テストカバレッジ）並列、デバッグで複数仮説を別エージェントが互いに反証しながら検証、リファクタを 4 体並列で別モジュール担当
- **情報源**: [日次レポート 2026-05-07](../reports/2026-05-07_code-with-claude-sf-keynote-and-v2.1.129-132.md) / [Agent Teams Docs](https://code.claude.com/docs/en/agent-teams) / [Multi-agent Sessions Docs](https://platform.claude.com/docs/en/managed-agents/multi-agent)

### Dreaming 🔬 Research Preview（Claude Managed Agents・新機能）
- **概要**: **エージェントが過去のセッションを再点検し、自分が見落とした点・繰り返したミス・反復パターンを抽出して memory artifact を再構成**する自己改善機能。Anthropic 公式は「人間が経験から学ぶように、Claude も眠る間に整理する」と説明。**自動モード**（バックグラウンドで stale notes prune / 重複 merge / 矛盾解消 / CLAUDE.md リオーガナイズ）+ **手動モード**（`/dream` コマンドで明示起動）。Memory（既存 Public Beta、学んだことを記録）+ Dreaming（学んだことを精緻化・統合）の 2 層構成。
- **対象**: Claude Managed Agents 利用者の中で[アクセス申請](https://claude.com/form/claude-managed-agents)通過した開発者
- **初出**: 2026-05-06 launch
- **想定ユースケース**: 法務 drafting・大規模ログ分析・ドキュメント品質チェック・writing automation。Claude Code では「同じバグパターンを毎週踏むチームのメモリを夜間に整理」用途
- **情報源**: [日次レポート 2026-05-07](../reports/2026-05-07_code-with-claude-sf-keynote-and-v2.1.129-132.md) / [Anthropic Blog - New in Claude Managed Agents](https://claude.com/blog/new-in-claude-managed-agents)
