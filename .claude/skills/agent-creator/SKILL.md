# Agent Creator

Claude Codeのカスタムサブエージェント（`.claude/agents/*.md`）を設計・作成・編集する。

## サブエージェントとは

独立したコンテキストウィンドウで動作する専門AIアシスタント。メインセッションを汚さずにタスクを処理でき、試行錯誤の過程がメインセッションに残らない利点がある。YAMLフロントマター付きMarkdownファイルとして定義する。

## ワークフロー

### Step 1: 要件ヒアリング

ユーザーの指示から以下を特定する。不明確な点は作成前に必ず確認する：

| 項目 | 確認内容 | デフォルト |
|------|----------|------------|
| 目的 | エージェントが何をするか | — （必須） |
| スコープ | プロジェクト or ユーザー | 確認する |
| 権限レベル | 読み取り専用 or 編集権限あり | 目的から推測 |
| モデル | haiku / sonnet / opus / inherit | 目的から推測 |

**特に確認すべき場面：**
- スコープの指定がない場合（プロジェクト `.claude/agents/` vs ユーザー `~/.claude/agents/`）
- エージェントが具体的にどんなタスクを行うか不明瞭な場合

### Step 2: エージェント定義の設計

以下のフロントマターフィールドを決定する：

```yaml
---
name: <ケバブケースの識別子>
description: <具体的な用途。自動委任の精度に直結するため最重要>
tools: <必要最小限のツールをカンマ区切りで指定>
skills: <プリロードするスキル名のYAMLリスト（任意）>
model: <haiku | sonnet | opus | inherit>
permissionMode: <dontAsk | acceptEdits | bypassPermissions>
---
```

#### フィールド設計の指針

| フィールド | 指針 |
|-----------|------|
| **name** | ケバブケースで簡潔に。役割が一目でわかる名前 |
| **description** | 最重要フィールド。具体的なユースケースを記載。必要に応じて「MUST BE USED when...」を含める |
| **tools** | 最小権限の原則。読み取り専用→`Read, Grep, Glob, Bash`。編集あり→`Read, Edit, Write, Bash, Grep, Glob` |
| **skills** | 既存スキルをエージェントのコンテキストにプリロードする |
| **model** | 探索・定型→`haiku`、実装→`sonnet`、複雑な判断→`opus`、親に合わせる→`inherit` |
| **permissionMode** | 読み取り専用→`dontAsk`、編集あり→`acceptEdits` |

#### 利用可能なツール

基本: `Read`, `Edit`, `Write`, `Bash`, `Grep`, `Glob`, `Agent`, `WebFetch`, `WebSearch`, `TodoWrite`, `NotebookEdit`
MCPツール: プロジェクトで利用可能なMCPツールも指定可能

### Step 3: システムプロンプトの作成

フロントマターの下に以下の構成で記述する。プロンプトは必要最小限にすること：

```markdown
# Role（役割）
専門領域と振る舞いの定義。1〜2文で簡潔に。

# When Invoked（呼び出し時のアクション）
具体的な手順を番号付きリストで記述。

# Key Practices（守るべき方針）
- 方針をリストで記述

# Important Notes（避けるべき行動）
- やってはいけないことをリストで記述
```

**設計の4原則：**
1. **1エージェント1責務** - 複数の役割を混在させない
2. **共通ルールはCLAUDE.mdへ** - エージェント定義に重複記述しない
3. **プロンプトは必要最小限** - 長ければ良いわけではない
4. **モデルのコスト最適化** - 定型タスクはhaikuで十分

### Step 4: ファイルの作成

1. 配置先ディレクトリが存在しない場合は作成する
2. ファイルパス: `<配置先>/<name>.md`
   - プロジェクトスコープ: `.claude/agents/<name>.md`
   - ユーザースコープ: `~/.claude/agents/<name>.md`
3. 作成前にユーザーに設計内容（フロントマター + プロンプト概要）を提示し、確認を取る
4. 確認後、Writeツールでファイルを作成する

### Step 5: 案内

作成後、以下を伝える：
- 作成したファイルのパス
- `@<name> <指示>` で明示的に呼び出し可能なこと
- descriptionに合致するタスクで自動委任されること
- 新しい会話で反映されること

## 既存エージェントの編集

1. `.claude/agents/` および `~/.claude/agents/` 内の既存エージェントを一覧表示
2. 対象エージェントの現在の定義を読み込む
3. 改善点をユーザーに提案し、確認後にEditツールで修正

## 完了条件

エージェント定義ファイルが適切な場所に作成（または編集）され、呼び出し方法がユーザーに案内された時点で完了。
