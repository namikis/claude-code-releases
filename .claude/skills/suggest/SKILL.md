# Suggest スキル

タスク完了後に、関連する改善提案を簡潔に提示するスキル。

---

## 改善提案の観点

直前に完了したタスクの内容を踏まえ、以下の観点から改善点を検討する（該当するもののみ）：

### コード・ファイル品質
- 重複の排除、構造の改善
- 命名の一貫性
- 不要なファイル・コードの削除

### プロジェクト構成
- ディレクトリ構造の整理
- 設定ファイルの最適化
- 依存関係の見直し

### ワークフロー・自動化
- 手作業の自動化可能性
- スクリプト・スキル・エージェントの改善
- CI/CDやジョブの最適化

### Claude Code機能の活用
- 今のタスクやワークフローをClaude Codeの機能でもっと上手くできないかを検討する
- **メインコンテキストの圧迫を避けるため、サブエージェントに委任する**

  以下のように `Agent` ツールを呼び出す。`{タスクの要約}` には直前に完了したタスクの内容を1〜3文で簡潔に記述する：

  ```
  Agent(
    description: "Claude Code機能の活用提案",
    subagent_type: "Explore",
    model: "sonnet",
    prompt: "以下のタスクに対して、CURRENT_FEATURES.md に記載されているClaude Codeの機能で活用できるものがないか調査してください。

タスク内容: {タスクの要約}

手順:
1. Grep で CURRENT_FEATURES.md の機能名一覧を抽出する（ファイル全体は読まない）:
   pattern: '^#{1,4} |^\\| [`/\\*]|^- \\*\\*'  path: 'CURRENT_FEATURES.md'  output_mode: 'content'
2. 結果から上記タスクに関連しそうな機能を特定し、その行番号周辺のみ Read（offset/limit指定）で詳細を確認する
3. まだ活用されていない機能や、新しく追加された機能で適用可能なものがあれば、以下の形式で報告する（最大2件）:
   - **{機能名}**: なぜこのタスクに有効か、どう活用するか（1〜2文）
   該当なしの場合は「該当なし」とだけ返す。200字以内で簡潔に。"
  )
  ```

- サブエージェントの結果が「該当なし」の場合、この観点からの提案はスキップする
- 提案の例: 手動で繰り返している作業をHooksで自動化、MCPサーバーとの連携、スキルやエージェントの新規作成・改善

### ドキュメント
- READMEやCLAUDE.mdの更新漏れ
- 既存ドキュメントとの整合性

---

## 出力ルール

1. **件数**: 1〜3件に絞る。改善点がなければ「特に改善提案はありません」と一言で終了する
2. **フォーマット**: 以下の形式で出力する

```
### 改善提案

- **{提案タイトル}**: {1〜2文の説明。なぜ改善すべきか、どう改善するか}
- ...
```

3. **トーン**: 提案であり指示ではない。「〜すると良いかもしれません」「〜を検討してみてください」のような表現を使う
4. **スコープ**: 直前のタスクに関連する範囲に限定する。無関係な改善は出さない
5. **実行しない**: 提案のみ行い、実際の変更は行わない。ユーザーが「やって」と言った場合のみ実行する

---

## 注意事項

- 些末な指摘（空行の調整、コメントの追加など）は避ける
- ユーザーが急いでいる様子の場合は提案を控えめにする
- 同じ提案を繰り返さない
