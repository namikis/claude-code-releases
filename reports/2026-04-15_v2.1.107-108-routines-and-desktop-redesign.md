# 調査レポート: v2.1.107-108・Routines研究プレビュー・デスクトップ再設計

**調査日**: 2026-04-15
**調査モード**: フルモード（Layer 1〜3）
**対象期間**: 2026年4月14〜15日
**新CLIリリース**: v2.1.107（2026年4月14日）、v2.1.108（2026年4月14日）

---

## 概要

v2.1.107-108がリリースされ、1時間プロンプトキャッシュTTL環境変数、セッション復帰時のrecap機能、組み込みスラッシュコマンドのSkillツール経由呼び出し等が追加された。プラットフォームレベルでは、**Routines（ルーティン）** が研究プレビューとして公開され、スケジュール・API・GitHubイベント駆動の自動化が可能になった。また、**Claude Codeデスクトップアプリが全面再設計**され、統合ターミナル、サイドチャット、再配置可能なペイン、ファイルプレビュー等が追加された。APIプラットフォームではClaude Sonnet 4とOpus 4の廃止予告が発表された。

---

## Layer 1: CLIリリース

### v2.1.108（2026年4月14日）

#### 新機能

| 機能 | 詳細 |
|------|------|
| **`ENABLE_PROMPT_CACHING_1H` 環境変数** | API key、Bedrock、Vertex、Foundry全プラットフォームで1時間プロンプトキャッシュTTLを有効化。`ENABLE_PROMPT_CACHING_1H_BEDROCK` は非推奨だが引き続き有効。`FORCE_PROMPT_CACHING_5M` で5分TTLを強制可能 |
| **Recap機能** | セッション復帰時に文脈サマリーを自動提供。`/config` で設定可能、`/recap` で手動呼び出し。`CLAUDE_CODE_ENABLE_AWAY_SUMMARY` でテレメトリ無効時にも強制有効化 |
| **組み込みスラッシュコマンドのSkillツール発見** | モデルがSkillツール経由で `/init`、`/review`、`/security-review` を発見・呼び出し可能に |
| **`/undo` エイリアス** | `/rewind` のエイリアスとして `/undo` を追加 |

#### 改善

| 改善内容 | 詳細 |
|---------|------|
| `/model` 警告 | 会話中のモデル切替時に「次のレスポンスが全履歴を未キャッシュで再読み込みする」旨を警告 |
| `/resume` ピッカー | デフォルトでカレントディレクトリのセッションを表示。`Ctrl+A` で全プロジェクト表示 |
| エラーメッセージ改善 | サーバーレート制限とプラン使用制限を区別。5xx/529エラーに status.claude.com リンク表示。未知のスラッシュコマンドに最も近い候補を提案 |
| メモリフットプリント削減 | 言語グラマーをオンデマンドロードし、ファイル読み取り・編集・シンタックスハイライトのメモリ使用量を削減 |
| verbose インジケータ | 詳細トランスクリプト表示時（`Ctrl+O`）に "verbose" 表示 |
| プロンプトキャッシュ無効警告 | `DISABLE_PROMPT_CACHING*` 環境変数設定時に起動時警告 |

#### バグ修正（主要なもの）

| 修正内容 |
|---------|
| `/login` コード入力でペーストが動作しない問題（v2.1.105リグレッション） |
| `DISABLE_TELEMETRY` 設定のサブスクライバーが1時間ではなく5分キャッシュTTLにフォールバックする問題 |
| Agentツールがautoモード時に安全分類器のトランスクリプトがコンテキストウィンドウを超えると権限確認を表示する問題 |
| `CLAUDE_ENV_FILE` が `#` コメント行で終わるとBashツールが出力しない問題 |
| `--resume <session-id>` でセッションのカスタム名・色が失われる問題 |
| セッションタイトルが短い挨拶の場合にプレースホルダーテキストが表示される問題 |
| `--teleport` 後にターミナルエスケープコードがゴミテキストとして表示される問題 |
| `/feedback` 再試行で失敗後のEnterキー再送信が動作しない問題 |
| `--teleport`/`--resume <id>` の前提条件エラーがサイレント終了する問題 |
| Remote Controlセッションタイトルが3番目のメッセージ後に上書きされる問題 |
| `--resume` が自己参照メッセージを含むトランスクリプトで切り詰められる問題 |
| トランスクリプト書き込み失敗（ディスク満杯等）がサイレントドロップされる問題 |
| `language` 設定時にダイアクリティカルマーク（アクセント、ウムラウト等）がレスポンスから消える問題 |
| ポリシー管理プラグインが別プロジェクトからインストールされた場合に自動更新されない問題 |

## Layer 2: プラットフォーム発表

### Routines（ルーティン）🔬 研究プレビュー

**発表日**: 2026年4月14日
**公式ブログ**: [Introducing routines in Claude Code](https://claude.com/blog/introducing-routines-in-claude-code)
**ドキュメント**: [Automate work with routines](https://code.claude.com/docs/en/routines)

Claude Codeの自動化をAnthropicクラウドインフラで実行する新機能。プロンプト、リポジトリ、コネクタを一度設定すれば、スケジュール・API・GitHubイベントで自動実行される。PCがオフラインでも動作する。

#### トリガータイプ

| トリガー | 説明 |
|---------|------|
| **Scheduled** | 時間単位・日次・平日・週次の定期実行。カスタムcron式対応（最小間隔1時間） |
| **API** | ルーティンごとのHTTPエンドポイント + ベアラートークン。`POST /fire` で実行、セッションURL返却 |
| **GitHub** | PRオープン、プッシュ、Issue、ワークフロー実行等の17種類のイベントカテゴリに対応。PRフィルタ（作者、タイトル、ラベル、ドラフト状態等）対応 |

1つのルーティンに複数トリガーを組み合わせ可能。

#### 日次実行制限

| プラン | 日次上限 |
|--------|---------|
| Pro | 5回/日 |
| Max | 15回/日 |
| Team / Enterprise | 25回/日 |

Extra Usage有効時は上限超過で従量課金。

#### ユースケース例

- **バックログ整理**: 毎晩Issue trackerを読み、ラベル付与・アサイン・Slack投稿
- **アラートトリアージ**: 監視ツールからAPI呼び出し→スタックトレース分析→修正PR作成
- **カスタムコードレビュー**: PR.opened時にチーム固有チェックリストでインラインコメント
- **デプロイ検証**: CD後にスモークテスト→エラーログスキャン→Go/No-goをSlack投稿
- **ドキュメントドリフト検出**: 週次でマージ済みPRからAPI変更を検出→ドキュメント更新PR作成
- **ライブラリポート**: あるSDKのPRマージ時に他言語SDKに自動ポート

#### 技術詳細

- **作成方法**: Web（claude.ai/code/routines）、Desktop App（New remote task）、CLI（`/schedule`）
- **APIベータヘッダー**: `experimental-cc-routine-2026-04-01`
- **リポジトリ**: デフォルトブランチからクローン。`claude/` プレフィックスブランチのみプッシュ可（制限解除オプションあり）
- **コネクタ**: MCP connectors（Slack、Linear、Google Drive等）を利用可能
- **実行**: フルClaude Codeクラウドセッションとして実行。権限プロンプトなし（自律実行）
- **環境**: Cloud environmentで設定（ネットワークアクセス、環境変数、セットアップスクリプト）

### Claude Codeデスクトップアプリ全面再設計 ✅

**発表日**: 2026年4月14日
**情報源**: [The New Stack](https://thenewstack.io/claude-code-desktop-redesign/)

Claude Code Desktop App（Cowork）が全面的に再設計された。

#### 新機能

| 機能 | 詳細 |
|------|------|
| **統合ターミナル** | アプリ内でコマンド実行可能。ウィンドウ切替不要 |
| **サイドチャット** | `⌘+;`（Mac）/ `Ctrl+;`（Windows/Linux）でメインエージェントの文脈を参照する副チャットを開始。エージェントの作業を中断しない |
| **改善されたDiffビューア** | 大規模changesetでのパフォーマンス向上。現在はunifiedビューのみ |
| **ファイルエディタ** | アプリ内で素早くファイル編集 |
| **ファイルプレビュー** | HTML・PDFをアプリ内で直接表示 |
| **再配置可能なペイン** | Preview、Plan、Diff、Tasks、Terminal、Chatペインを自由に配置 |
| **複数セッション並行** | 1ウィンドウ内で複数Claude Codeセッションを並行実行 |

モーダルスイッチャーがサイドバーに移動し、Chat/Cowork/Codeモードの切替がアイコンベースに。ストリーミングレスポンスでの高速化も実現。
