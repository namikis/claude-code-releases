## 概要

前回レポート（4/29 フルモード）以降、**Claude Code CLI は v2.1.123（4/29 03:29 UTC）が連続 4/28 リリース 3 連発の最後尾**として登場。1 件のみの緊急バグ修正で、`CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS=1` 設定下で OAuth 認証が 401 retry loop に陥る問題を解消した。Layer 2 公式は 4/29-30 新規エントリなし。Layer 3 は **Goldman Sachs が Hong Kong 拠点バンカーの Anthropic Claude アクセスを遮断**（FT スクープ）という米中 AI ガバナンス・地域提供範囲に直結するエンタープライズ運用シグナルが最大級トピック。

### 4/29〜4/30 の主要トピック

1. **Claude Code v2.1.123**（4/29 03:29 UTC）— **OAuth 認証 401 retry loop fix**：`CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS=1` 設定時に OAuth が 401 → retry → 401 を繰り返す現象を修正。1 件のみのホットフィックス
2. **Goldman Sachs Hong Kong 拠点 Claude 遮断**（4/29、FT スクープ） — 香港バンカーが Anthropic Claude にアクセス不可に。**Anthropic の契約・地域提供範囲を厳格解釈**した結果。Anthropic 公式の API/Claude.ai 利用可能リージョンリストに香港は含まれていない。**Gemini・ChatGPT は引き続き利用可能**

4. **4/29 Anthropic 障害（Haiku 4.5 / Opus 4.7 別個 elevated errors）** — Haiku 4.5: 05:37–06:58 PT（81 分）、Opus 4.7: 16:25–16:33 PT（8 分）。4/28 Major Outage（78 分）に続く別個インシデント。**4 月の Claude/API/Code 関連障害は Major クラス + Partial を合算で 10 件超**

## Layer 1: CLI チェンジログ

### Claude Code v2.1.123（2026-04-29 03:29 UTC）✅

公式 GitHub Releases / `code.claude.com/docs/en/changelog` に正式掲載。**1 件のみの緊急 fix**。

#### バグ修正

- **OAuth 認証 401 retry loop fix** — `CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS=1` 環境変数が設定されている環境で、OAuth 認証が 401 エラーを受けて retry し再び 401 を返す無限ループに陥る問題を修正
  - **影響範囲**: 実験的ベータ機能を無効化している企業/コンプライアンス環境のユーザー
  - **背景推測**: v2.1.121 / v2.1.122 で導入された claude.ai connector OAuth フロー強化（`mcp_authenticate` の `redirectUri` 対応 / Microsoft 365 MCP OAuth 修正）の副作用と推察される。実験的ベータパスを無効化すると新しい OAuth コードパスが本来不要なベータヘッダーを要求して認証ループに陥っていた可能性が高い

#### 注記

- v2.1.123 は **新機能の追加なし**。エンタープライズ運用環境で `CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS=1` を運用している組織（金融・公共セクター等）に対する hot-fix
- v2.1.121（4/28 00:31 UTC）→ v2.1.122（4/28 22:05 UTC）→ v2.1.123（4/29 03:29 UTC）と **約 27 時間で 3 リリース連続展開**。v2.1.121 リリース直後の OAuth/MCP 関連修正を畳みかけている

### Sonnet 4.5 / Sonnet 4 の 1M context window beta retirement 発効（2026-04-30）

- **発効日**: 2026-04-30（**本日**）
- **対象モデル**: `claude-sonnet-4-5-20250929`、`claude-sonnet-4-20250514`
- **対象ベータヘッダー**: `context-1m-2025-08-07`
- **挙動変更**:
  - 上記ベータヘッダーをこれら 2 モデルに付与しても **silent fallback で 200K context window**
  - 200K を超えるリクエストは **API error**（rejected）
- **移行先**:
  - **Claude Sonnet 4.6**: 1M context GA、標準価格、ベータヘッダー不要
  - **Claude Opus 4.6**: 1M context GA、標準価格、ベータヘッダー不要
  - 上記いずれも **long-context surcharge なし**（2026-03-13 GA 化時点で long-context premium 完全撤廃済み）
- **Claude Code への影響**:
  - Claude Code 内でモデルピンを `claude-sonnet-4-5-*` / `claude-sonnet-4-*` にして 1M を使っていた場合、**今日からピンを `sonnet`（Sonnet 4.6 alias）または full Sonnet 4.6 model name に変更しないと 200K 超でエラー**
  - `/setup-vertex` / `/setup-bedrock` が v2.1.111 で「with 1M context」オプションを追加した際、Sonnet 4.5/4 をピンしたユーザーは `/model` で再設定が必要
- **タイムライン整理**:
  - 2025-08-12: Sonnet 4 で 1M ベータ launch
  - 2025-09-29: Sonnet 4.5 launch、1M ベータ拡張
  - 2026-02-05: Opus 4.6 launch、1M ベータ追加
  - 2026-03-13: **Sonnet 4.6 / Opus 4.6 の 1M GA**、long-context premium 撤廃
  - 2026-03-30: Sonnet 4.5/4 の 1M ベータ廃止予告（4 月 30 日発効）
  - 2026-04-30: **本日 retirement 発効**

## Layer 3: ニュースメディア

### Goldman Sachs Hong Kong 拠点で Claude アクセス遮断（FT スクープ）

- **報道日**: 2026-04-28（米国時間） / 2026-04-29（アジア時間）
- **一次ソース**: Financial Times（有料記事、Reuters / Bloomberg / Yahoo Finance / U.S. News 等が二次報道）
- **概要**:
  - **Goldman Sachs が Hong Kong 拠点のバンカー全員に対し Anthropic Claude へのアクセスを遮断**
  - **直接の理由**: Anthropic との **契約を厳格解釈**した結果。同社内のコンサルティングを経て「Hong Kong の従業員は Anthropic 製品を一切使うべきでない」と結論
  - **背景**: **Anthropic 公式の API/Claude.ai 利用可能市場リストに Hong Kong は含まれていない**（Anthropic の "Where Claude is available" リスト未掲載）
  - **競合**: **Goldman の社内プラットフォームでは Gemini と ChatGPT は引き続き利用可能** — 香港制限は Anthropic 製品に固有
- **米中 AI 緊張のコンテキスト**:
  - 米中 AI 技術 / データセキュリティ / 先進コンピューティングへのアクセスを巡る緊張高まりの中での動き
  - 銀行業界全体で AI ツールに対する scrutiny 強化（データセキュリティ・サイバーリスクへの感度上昇）
  - 香港の地政学的位置（中国本土への玄関）が Anthropic の export-control コンプライアンス上の懸念対象
- **Claude Code への含意**:
  - **Hong Kong 拠点で Claude Code を使用していた組織は、Anthropic 利用規約上の準拠リスクを再評価する必要**がある
  - Claude Code は内部的に Anthropic API / Claude.ai 認証経路を使用するため、Anthropic 提供地域外での運用は契約違反リスクあり
  - 大規模採用済み法人（NEC・Freshfields など）の地域分散運用にとって、**地域別ガバナンス整備の必要性**が浮上
- **Anthropic からの公式コメント**: 報道時点で Anthropic はコメント未提供
