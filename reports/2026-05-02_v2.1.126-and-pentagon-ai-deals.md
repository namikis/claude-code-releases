## ① v2.1.126 大型リリース（2026-05-01 02:05 UTC）✅ GA

### バージョンスキップの背景

GitHub Releases では **v2.1.123（4/29）→ v2.1.126（5/1）** とジャンプ。`v2.1.124` / `v2.1.125` の releases ページは **404**。npm 上での公開状況も同様に v2.1.124 / v2.1.125 はトレースされておらず、内部ビルドで一般公開前にホールドされた可能性が高い。

### 新機能・新コマンド

#### 1. `claude project purge [path]`（新コマンド） ✨
プロジェクトの全 Claude Code 状態を一括削除する新コマンド。Auto Memory / Recap 等の永続データが累積する課題への正式な掃除手段が登場。

**削除対象:**
- セッショントランスクリプト
- スケジュールタスク
- ファイル変更履歴
- `~/.claude/projects/` 配下の config エントリ

**フラグ:**
| フラグ | 動作 |
|--------|------|
| `--dry-run` | 削除対象をプレビューのみ（実削除しない） |
| `-y` / `--yes` | 確認プロンプトをスキップ |
| `-i` / `--interactive` | 対話的に選択削除 |
| `--all` | 全プロジェクトの状態を一括削除 |

**運用ユースケース:** リポジトリリネーム後の旧データ整理、機密プロジェクトのオフボーディング、Auto Memory が肥大化した際のリセット。

#### 2. `/model` ピッカーがゲートウェイ対応 ✨
`ANTHROPIC_BASE_URL` が Anthropic 互換ゲートウェイ（LiteLLM、OpenRouter、内製プロキシ等）を指している場合、ゲートウェイの **`/v1/models` エンドポイントから動的にモデル一覧を取得**して `/model` ピッカーに表示。従来の固定リスト + `ANTHROPIC_DEFAULT_*_MODEL_NAME` 環境変数によるオーバーライドより柔軟。

#### 3. `--dangerously-skip-permissions` の挙動拡張
従来 `.claude/skills/` / `.claude/agents/` / `.claude/commands/` への書き込みは v2.1.121 で確認プロンプトを残していたが、本リリースで **`.claude/`、`.git/`、`.vscode/`、シェル設定ファイル（`.bashrc` 等）への書き込みプロンプトもバイパス**するように拡張。**ただし危険な削除コマンド（`rm`/`rmdir` の `/` や `$HOME` 対象）は引き続きプロンプト**を表示し、安全性を維持。

⚠️ 設計意図: `--dangerously-skip-permissions` は CI / 自動化向けで、保護パスへの書き込みプロンプト残存はスクリプト実行を阻害していた。安全性は危険削除のチェックで担保。

#### 4. `claude auth login` の WSL2 / SSH / コンテナ対応
ブラウザコールバックが localhost に到達できない環境（**WSL2 / SSH 経由 / Docker コンテナ / リモート開発環境**）で、ターミナルに OAuth コードを **直接貼り付けて認証完了**できるように。従来は手動でコールバック URL のフォワーディングを設定する必要があった。

#### 5. `claude_code.skill_activated` OpenTelemetry イベント拡張
`invocation_trigger` 属性を新規追加し、スキル発動の文脈を観測可能に：

| `invocation_trigger` 値 | 意味 |
|-------------------------|------|
| `"user-slash"` | ユーザーがスラッシュコマンドで明示的に呼び出し |
| `"claude-proactive"` | Claude がプロアクティブにスキルを起動 |
| `"nested-skill"` | 別スキルからネスト呼び出し |

ユーザーが `/skill-name` をタイプした際にも `skill_activated` が発火するように変更（従来はモデル経由の起動のみ）。

#### 6. Auto Mode UI: スピナー赤化
パーミッションチェック遅延時にスピナーが **赤色に変化** — ツールが実際に実行中に見える誤認を防ぐ視覚フィードバック。

#### 7. PowerShell 7 検出強化（Windows）
**Microsoft Store / MSI（PATH未設定）/ .NET Global Tool 経由でインストールされた pwsh** を検出するように。さらに **PowerShell ツールが有効な場合は PowerShell をプライマリシェル**として扱う（従来は Bash デフォルト）。

#### 8. ホスト管理デプロイメントのアナリティクス制御変更
`CLAUDE_CODE_PROVIDER_MANAGED_BY_HOST` 環境下で Bedrock / Vertex / Foundry のアナリティクス自動無効化が **廃止**。ホスト管理運用でも明示的に opt-out する必要に。

#### 9. その他改善
- 読み取りツールのファイルごとマルウェア評価リマインダーを削除（誤拒否と過剰コメントを防止）
- リモートセッションのタイトル: 設定された `language` で生成
- 画像処理: 2000px 超のペースト画像を**自動ダウンスケール**、履歴中の過大画像も自動削除して再試行（従来はセッション破損）

### セキュリティ修正

#### `allowManagedDomainsOnly` / `allowManagedReadPathsOnly` の優先度回帰修正 🔒
高優先度の管理設定ソースに `sandbox` ブロックが存在しない場合、これらの設定が**サイレントに無視されていた**問題を修正。Enterprise の policy enforcement に直結する不具合で、企業 IT 運用上の重要な fix。

## ② Pentagon が 7 社 + Oracle と classified AI 契約・Anthropic 除外継続（2026-05-01）

### スクープ概要

**米国防総省（DoD / "Department of War"）が 2026-05-01 金曜日に発表**。 **classified network 内での deployment（"lawful operational use"）を目的とした AI 調達契約**を 7 社と締結し、後に Oracle を加えて 8 社に拡大。**Anthropic は契約から完全除外**された。Pentagon CTO Emil Michael は CNBC インタビューで「Anthropic はサプライチェーンリスクのまま、Mythos は別件として評価」と明確化。

### 契約 7 社（+ Oracle で 8 社目）

| 企業 | 領域 |
|------|------|
| **Google** | AI モデル / クラウド |
| **OpenAI** | 基盤モデル |
| **Microsoft** | クラウド / AI |
| **Amazon Web Services** | クラウド / AI |
| **Nvidia** | AI ハードウェア / モデル |
| **SpaceX**（xAI 統合済み） | AI モデル / 衛星通信 |
| **Reflection AI** | オープンウェイトモデル |
| **Oracle**（同日追加） | クラウド / AI |

注: 一部報道（Defense News）は 7 社、別報道（Implicator / DefenseScoop）は Oracle 追加で 8 社と報じる。

### Anthropic 排除の経緯

| 日付 | イベント |
|------|---------|

| **2025 年末〜2026 年初頭** | Pentagon が再交渉で同条項を**撤廃要求**、Anthropic は譲らず |

| **2026-04-21** | Trump が CNBC で「合意は possible、shaping up」と発言 |

### Pentagon CTO Emil Michael の重要発言（CNBC インタビュー）

> **"Anthropic remains a supply chain risk."**
>
> **"Mythos is a separate national security moment that needs evaluation for its capabilities in finding cyber vulnerabilities."**

→ ブラックリスト指定は**継続**だが、**Mythos については別枠で評価**する姿勢を明言。NSA はすでに Mythos を環境内脆弱性スキャンに利用中（4/19 Axios スクープ）、ホワイトハウスが連邦機関への Mythos アクセス整備中（4/17 Bloomberg スクープ）の文脈と整合。

### Anthropic 訴訟の現状

- Anthropic は SF（San Francisco）と DC（Washington）の連邦地裁に提訴済み
- 一時差止め命令は有効、Pentagon の「全 contractor が Anthropic と取引禁止」効力は**停止中**
- 控訴審は係属中で、5/1 の契約発表は**訴訟係争中の "事実上の確定"** という性格

### 契約からの除外が示すもの

1. **5/1 契約発表のタイミング**: Trump 発言（"deal possible"）の 10 日後にも関わらず DoD は除外を継続 → Trump-Hegseth/Pentagon 間の温度差を示唆
2. **Mythos の "別枠評価"**: Anthropic 製品全体の禁輸ではなく、Mythos のみ controlled access で許容する**部分的解除モデル**が形成中
3. **Reflection AI の選定**: オープンウェイトモデル提供の若手企業を含めた点に注目 — Pentagon が "any lawful purpose" を許容する provider を優先する姿勢

### Claude Code への影響

- **直接の影響なし**: Claude Code は商用 / Enterprise / Pro / Max プランで通常通り利用可能
- **間接的影響**: Pentagon サプライチェーンに含まれる**米国防衛関連企業**（Lockheed Martin / Northrop Grumman / RTX 等）は依然として Hegseth 通達のリスクを抱える可能性。ただし差止め命令で当面は通達自体が機能停止
- **Mythos 個別解禁トラック**: Mythos の controlled access が NSA / 商務省 AI 標準化センター / エネルギー省等に整備中という流れと整合。Mythos 駆動 OSS ツール（Project Glasswing、Claude Security の Cyber Verification Program）が**実質的な政府機関導入チャネル**として機能する可能性
