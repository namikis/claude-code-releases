## 概要

前回（5/4 朝）レポート以降、Claude Code は **v2.1.128（5/4 23:01 UTC）** をリリースし、約 45 項目の改善・修正を投入した（**v2.1.127 は欠番**で GitHub Releases から 404、CHANGELOG.md にも記載なし）。注目度の高い変更は **`--channels` のコンソール（API key）認証対応**で、これまで claude.ai ログインのみだった Channels（Telegram/Discord/iMessage 等のチャネル統合）がコンソールキー認証下でも有効化可能になった（managed settings で `channelsEnabled: true` 必須）。その他、`/color` ランダムカラー、`/mcp` ツール数表示、`--plugin-dir` の `.zip` 対応、サブプロセスへの `OTEL_*` 環境変数の非継承、MCP の `workspace` 予約名、`EnterWorktree` のブランチ作成元修正、`/model` ピッカーの Opus 表示整理など、多くのユーザビリティ改善・回帰修正が含まれる。

**Layer 2（公式）の最大の動きは 5/4 の `Anthropic + Blackstone + Hellman & Friedman + Goldman Sachs` による Enterprise AI サービス会社の正式発表**。総額 **$1.5B（Anthropic / Blackstone / H&F が各 $300M コミット）**、GIC / Apollo / General Atlantic / Leonard Green / Sequoia Capital がコンソーシアムに参加、PE 傘下の中規模企業を主要ターゲットに **エンジニアを派遣して Claude 中心のワークフロー再設計を行う**新会社を立ち上げる。**Anthropic 自身が「コンサルティング業界に切り込む」と複数メディアが報じる**戦略的ピボット。同日 OpenAI も類似のエンタープライズ AI サービス JV を発表しており、両社が AI システム実装の "誰がやるか" 領域へ垂直展開する流れが明確化。

**Layer 3（メディア）では TestingCatalog の Jupiter-v1-p レッドチームレポート（5/1 公開）**が継続注目。明日 5/6 の Code with Claude SF キーノート（9:00 AM PT、CPO Ami Vora 登壇）で **Sonnet 4.8 / Claude Jupiter / Computer Use ベータ昇格 / Routines GA 化** などのいずれかが発表される蓋然性が高い。本日 5/5 は **The Briefing: Financial Services（11:00 AM EST、NYC livestream、C-suite 向け）** が予定通り開催される。

## Layer 1: Claude Code CLI v2.1.128（2026-05-04 23:01 UTC）

**注**: 本リリースは前回 5/4 朝レポートの**後**にデプロイされた。`v2.1.127` は GitHub Releases / CHANGELOG.md ともに欠番（4/24 の `v2.1.124` / `v2.1.125` 同様、内部スキップと推察）。

### 新機能・挙動変更

#### コマンド・フラグ

- **`--channels` がコンソール（API key）認証をサポート** — これまで Channels 機能（Telegram/Discord/iMessage 等）は claude.ai login 限定だったが、コンソール（Anthropic Console）API キー認証下でも利用可能に。**ただし console org で managed settings がある場合は `channelsEnabled: true` を明示設定する必要あり**（既定はオフ）。これで Channels をエージェントワークフローに組み込んだ自動化用途（CI/サーバ常駐 bot 等）の選択肢が広がる
- **`--plugin-dir` が `.zip` プラグインアーカイブをサポート** — ディレクトリ指定に加え `.zip` ファイル直接指定が可能に。配布・テスト用途で展開不要
- **`/color`（引数なし）でランダムなセッションカラーを選択** — これまで色指定必須だったところ、引数なし呼び出しでランダム選択

#### 表示・UX

- **`/mcp` が接続中サーバーの tool 数を表示**、ツール 0 件で接続したサーバーは警告マーク付きで表示
- **`/model` ピッカー改善** — Opus 4.7 の重複エントリを集約、現行の Opus を「Opus 4.7」ではなく単に「Opus」表示に
- **MCP 再接続時にツール名一覧を会話に flood しない** — re-announce されたツールはサーバー prefix でサマライズ表示
- **`/config` のタブナビゲーション修正** — フォーカスがタブヘッダーから外れない、矢印・Esc が正常動作
- **markdown リンクラベルの保持** — OSC 8 ハイパーリンク非対応ターミナルでも `label (url)` 形式で表示（従来は URL のみ）
- **OSC 9;4 進捗インジケータがツール呼び出し間でちらつかなくなる**（ターン全体で表示維持）
- **ロングURL の各折り返し行が個別にクリック可能**（フルスクリーンモード）

#### MCP

- **`workspace` が MCP の予約サーバー名に** — 既存の `workspace` 名サーバーは警告ログとともにスキップされる
- **`installed_plugins.json` の stale entry 掃除** — 削除済みキャッシュディレクトリを指す古いエントリが PATH を汚染しないよう

#### サブプロセス・分離

- **Bash / Hooks / MCP / LSP のサブプロセスが `OTEL_*` 環境変数を継承しない** — Bash ツール経由で OTEL 計装アプリを実行しても、CLI 自身の OTLP エンドポイントを誤って取得しなくなる（重要なテレメトリ分離改善）
- **MCP stdio サーバーへの引数破損修正** — `CLAUDE_CODE_SHELL_PREFIX` 設定下でスペース・シェルメタ文字を含む引数が正しく渡される

#### SDK / Auto Mode / Worktree

- **SDK ホストが Bash パーミッションプロンプトに `localSettings` 提案を受信** — "Always allow" 選択時に `.claude/settings.local.json` へ書き込まれる
- **Auto mode のエラーヒント** — classifier がアクション評価不能な場合、リトライ／`/compact`／`--debug` 起動を提案
- **`EnterWorktree` のブランチ作成元を `origin/<default-branch>` から local HEAD に修正** — ドキュメント記載通り。**未 push のコミットがドロップされなくなる重要な回帰修正**
- **Headless `--output-format stream-json` の `init.plugin_errors`** に `--plugin-dir` 読み込み失敗が含まれるように

### 主要バグ修正（45項目以上のうち抜粋）

- フォーカスモードが新プロンプト送信時に直前応答を一瞬ディマーする問題
- Kitty 等 OSC 9 解釈ターミナルで `/exit` ごとに「4;0;」デスクトップ通知が出る問題
- Remote Control がレートリミット時に空の "Opening your options…" を表示
- ドラッグ&ドロップ画像アップロードで読み込み失敗時に "Pasting text…" でハング
- 10 MB 超の入力を `claude -p` に stdin パイプするとクラッシュループ
- `/plugin` Components パネルで `--plugin-dir` ロードプラグインに「Marketplace 'inline' not found」表示
- MCP ツール結果で server が structured content と content blocks 両方返したとき画像がドロップされる問題
- リスト項目内のフェンスドコードブロックがコピー時に先頭空白を持ち越す問題
- 1M コンテキストモデル（autocompact ウィンドウが小さい場合）で実 API 上限到達前に "Prompt is too long" で誤ブロックされる問題
- 並列シェルツール呼び出しで read-only コマンド（`grep`/`git diff`/`ls`）の失敗が兄弟呼び出しをキャンセルする問題
- バナーが effort 非対応モデルでも "with X effort" を表示
- `/fast` がサードパーティプロバイダで関連ないスキルにファジーマッチ（"not available" を返さない）
- **Bedrock のデフォルトモデル解決が `global.*` になっていた回帰** → リージョン適切な prefix に修正
- vim NORMAL モードで `Space` がカーソル右移動になる（標準 vi/vim 挙動）
- `/rename`（引数なし）がコンパクト境界が末尾のセッションでフェイル
- `--resume`/`--continue` 後に前セッションの "remote-control is active" stale ステータス行が表示される問題
- **サブエージェント進捗サマリーがプロンプトキャッシュをミス → `cache_creation` が約 3 倍** → 修正により 1/3 に
- `/plugin update` が npm 由来プラグインの新バージョンを検出できない
- アイドルなサブエージェントの transcript 静止中もサマリーが繰り返し発火 → ワーストケースのトークンコストにキャップ
- Bash 並列ツール呼び出しの不正形式ツール名で SDK ハング修正

GitHub Releases に `v2.1.127` タグなし（404）、CHANGELOG.md にもエントリなし。**4/24 の v2.1.124 / v2.1.125 と同パターンの内部スキップ**と推察。npm 発行も v2.1.126 → v2.1.128 に直接遷移。

### Anthropic × Blackstone × Hellman & Friedman × Goldman Sachs Enterprise AI Services 会社（2026-05-04 正式発表）

#### 概要

Anthropic と **Blackstone**（オルタナティブ資産運用最大手）、**Hellman & Friedman**（PE）、**Goldman Sachs**（投資銀行）の 4 者が、**AI ネイティブのエンタープライズサービス会社**を共同設立すると発表。「**Claude を企業の中核業務に高速で組み込むための専門会社**」と位置づけ。

| 項目 | 詳細 |
|------|------|
| **発表日** | 2026-05-04 |
| **総調達額** | **$1.5 billion** |
| **コアコミット** | Anthropic / Blackstone / Hellman & Friedman が **各 $300M** |
| **追加バッキング** | General Atlantic / Leonard Green / Apollo Global Management / GIC（シンガポール政府投資公社）/ Sequoia Capital |
| **法務アドバイザリー** | Kirkland & Ellis LLP（Blackstone 側）|
| **位置づけ** | スタンドアロン新会社（Anthropic 完全子会社ではない）。Anthropic のエンジニアリング・パートナーシップリソースを内部に embed |
| **運用モデル** | エンジニアを中規模企業内部に派遣 → ワークフローを **エージェント中心に再設計** |
| **初期ターゲット** | PE 傘下のヘルスケア / 製造業 / 金融サービス / リテール / 不動産（**partners の portfolio companies が proving ground**）|
| **戦略的意図** | Goldman / Blackstone は自社ポートフォリオ企業を初期テストベッドに使い、その後より広範な中規模企業市場へ展開 |

#### 重要なポイント

1. **Anthropic がコンサルティング業界へ垂直展開** — Fortune は本件を「Anthropic takes shot at consulting industry」と報じる。Accenture / Deloitte / Infosys 等の AI 実装サービスが取り扱う「AI を企業に embed する作業」自体を、Anthropic 主導の専門会社として垂直統合する戦略
2. **同日 OpenAI も類似 JV を発表** — TechCrunch「Anthropic and OpenAI are both launching joint ventures for enterprise AI services」が**両社の同日発表を比較整理**。AI モデル提供だけでなく "実装サービス" まで囲い込みに進む業界トレンド
3. **PE / 投資銀行が JV パートナー** — Goldman / Blackstone / H&F の組成は、**買収済み企業群を直接顧客に変換**する仕掛け（PE 傘下のミドルマーケット企業数千社が即座にパイプライン化）
4. **5/5 The Briefing: Financial Services との接続** — 翌日 5/5 11:00 AM EST の Anthropic 主催「The Briefing: Financial Services」（金融機関 C-suite 向け NYC livestream）と同じ流れ。**5/4 JV 発表 → 5/5 金融セクター C-suite ブリーフィング → 5/6 Code with Claude SF キーノート**で **3 日連続のエンタープライズ・金融戦略コミュニケーション週**を構成

#### 既存の Services Partners との関係

Claude Security 公開ベータ（5/1 反映）の Services Partners として **Accenture / BCG / Deloitte / Infosys / PwC** が記載されているが、本 JV はこれら既存の SI/コンサル提携と並列に位置する Anthropic 自社ブランドの新会社。**つまり「Accenture / Deloitte と組む既存パスを残しつつ、自前のエンタープライズサービス専業会社も持つ」**二段構え。
