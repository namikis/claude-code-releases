# Claude Code 調査レポート: v2.1.100-101・CoreWeave提携

**調査日:** 2026-04-11
**対象期間:** 2026-04-10 〜 2026-04-11
**調査モード:** フルモード

---

## 概要

前回調査（2026-04-10）以降、以下の重要な動きがあった：

1. **v2.1.100**（4月10日）: チェンジログ更新のみのパッチリリース
2. **v2.1.101**（4月10日）: `/team-onboarding`コマンド、OS CA証明書ストア信頼、`/ultraplan`自動クラウド環境作成、コマンドインジェクション脆弱性修正、メモリリーク修正、多数のバグ修正・改善
3. **CoreWeave-Anthropic提携**（4月10日）: 複数年のGPUクラウドインフラ契約を締結
4. **Anthropicカスタムチップ検討**（4月10日）: 自社AIチップ設計を検討中との報道

---

## 主要な新機能

### v2.1.100（2026年4月10日） ✅

チェンジログ更新のみのメンテナンスリリース。機能変更なし。

### v2.1.101（2026年4月10日） ✅

#### 新機能

| 機能 | 詳細 |
|------|------|
| **`/team-onboarding`コマンド** | ローカルのClaude Code使用履歴からチームメイト向けオンボーディングガイドを自動生成 |
| **OS CA証明書ストア信頼** | デフォルトでOSの認証局証明書ストアを信頼。エンタープライズTLSプロキシが追加設定なしで動作。`CLAUDE_CODE_CERT_STORE=bundled`でバンドルCA限定に戻せる |
| **`/ultraplan`自動クラウド環境作成** | `/ultraplan`等のリモートセッション機能がWeb側での事前セットアップ不要で、デフォルトクラウド環境を自動作成 |

#### 改善

| 改善項目 | 詳細 |
|---------|------|
| **Brief mode リトライ** | 構造化メッセージではなくプレーンテキストで応答した場合に1回リトライ |
| **Focus mode 自己完結型サマリー** | ユーザーが最終メッセージしか見ないことを考慮し、より自己完結型のサマリーを生成 |
| **ツール利用不可エラー改善** | ツールが存在するが現在のコンテキストで利用不可な場合、理由と対処法を表示 |
| **レート制限リトライメッセージ** | どの制限に抵触したか・リセット時刻を表示（不透明な秒数カウントダウンの代替） |
| **拒否エラーメッセージ** | API提供の説明文を含むように改善 |
| **`claude -p --resume <name>`** | `/rename`や`--name`で設定したセッションタイトルを受け入れ |
| **settings.json 堅牢性** | 未認識のhookイベント名があってもファイル全体が無視されなくなった |
| **プラグインフック** | managed settingsで強制有効化されたプラグインのフックが`allowManagedHooksOnly`設定時に実行 |
| **`/plugin` 更新警告** | マーケットプレイスのリフレッシュ失敗時にサイレントに古いバージョンを報告せず警告を表示 |
| **Plan mode Ultraplanオプション非表示** | orgや認証設定がWebに到達できない場合に「Refine with Ultraplan」オプションを非表示 |
| **OTELトレーシング改善** | `OTEL_LOG_USER_PROMPTS`、`OTEL_LOG_TOOL_DETAILS`、`OTEL_LOG_TOOL_CONTENT`を尊重。オプトインなしでは機密スパン属性を送信しない |
| **SDK `query()`クリーンアップ** | `break`や`await using`時にサブプロセスと一時ファイルを適切にクリーンアップ |

#### セキュリティ修正

| 修正 | 詳細 |
|------|------|
| **POSIX `which`コマンドインジェクション** | LSPバイナリ検出のPOSIX `which`フォールバックにおけるコマンドインジェクション脆弱性を修正 |
| **`permissions.deny`ルールのバイパス** | PreToolUseフックの`permissionDecision: "ask"`がdenyルールを下回れる問題を修正。フックがdenyをプロンプトにダウングレードできなくなった |

#### 主要なバグ修正

| 修正 | 詳細 |
|------|------|
| **メモリリーク** | 長いセッションで仮想スクローラーにメッセージリストの過去コピーが数十件残留する問題を修正 |
| **`--resume`/`--continue`コンテキスト喪失** | 大規模セッションでローダーがデッドエンドブランチにアンカーし会話コンテキストが失われる問題を修正 |
| **`--resume`チェーン復旧** | 無関係なサブエージェント会話にブリッジする問題を修正 |
| **5分ハードコードタイムアウト** | `API_TIMEOUT_MS`に関わらずスローバックエンドを中断する5分タイムアウトを修正 |
| **Bedrock SigV4認証** | `ANTHROPIC_AUTH_TOKEN`等でAuthorizationヘッダーが設定される場合の403エラーを修正 |
| **`claude -w <name>`** | 前回のworktreeクリーンアップが残留した場合の「already exists」エラーを修正 |
| **サブエージェントMCPツール継承** | 動的注入されたサーバーからのMCPツールをサブエージェントが継承しない問題を修正 |
| **worktree内サブエージェントアクセス** | 分離worktreeで実行されるサブエージェントがworktree内ファイルへのRead/Editアクセスを拒否される問題を修正 |
| **Grep ENOENT自己修復** | 埋め込みripgrepバイナリパスが古くなった場合にシステム`rg`にフォールバックし、セッション中に自己修復 |
| **`/btw`会話全体書き込み** | 使用ごとに会話全体をディスクに書き込む問題を修正 |
| **`/context`表示不一致** | Free spaceとMessages内訳がヘッダーの割合と不一致の問題を修正 |
| **プラグイン複数修正** | スラッシュコマンド解決の重複name問題、`ENAMETOOLONG`、Discover表示、staleキャッシュ、`context: fork`等 |
| **`/mcp` OAuthメニュー** | `headersHelper`設定のMCPサーバーにOAuth固有アクションが表示される問題を修正 |
| **キーバインド修正** | `ctrl+]`、`ctrl+\`、`ctrl+^`がC0制御バイトを送信するターミナルで動作しない問題を修正 |
| **レンダリング修正** | 非フルスクリーンモードのフリッカー、長いセッションでのスクロールバック消去、マウススクロールエスケープシーケンスのリーク |
| **Remote Control複数修正** | worktree削除、接続失敗の永続化、不正な「Disconnected」表示、SSH経由`/remote-control`失敗 |
| **`/resume`ピッカー複数修正** | 狭いデフォルトビュー、Windows Terminalプレビュー到達不能、worktreeのcwd不正、セッション未発見、タイトル未設定、resumeヒントのプロンプト重複 |
| **VSCode** | 最後のエディタタブを閉じた時にチャット入力下のファイル添付がクリアされない問題を修正 |

### CoreWeave-Anthropic 複数年GPUクラウド契約（4月10日） ❓

- **概要**: AnthropicがCoreWeaveのGPUクラウドインフラを複数年契約で利用。Claude AIモデルの本番推論ワークロードに活用
- **詳細**:
  - 金額: 非公開
  - 期間: 複数年
  - チップ: Nvidia GPU（具体的アーキテクチャ非公開）
  - 段階的インフラ展開と拡張オプション付き
- **インフラ戦略の3本柱**:
  1. AWS Trainium（Project Rainier）: プライマリ学習ワークロード
  2. Google/Broadcom TPU（~3.5GW、2027年稼働予定）: 次世代容量
  3. CoreWeave Nvidia GPU: エンタープライズ規模の本番推論
- CoreWeaveのプラットフォームに主要AIモデルプロバイダー10社中9社が参加
- CoreWeave株が11%上昇
