# 調査レポート: Claude Code v2.1.201（Sonnet 5 セッションでハーネスリマインダーの mid-conversation system ロール利用を廃止）

**調査日:** 2026-07-05

## 概要

- **フルモード調査**（7/4 レポート以降の差分）。**Layer 1 — 新規 CLI は v2.1.201（7/3 23:50）の 1 本のみ**。前回 7/4 レポートは v2.1.199／v2.1.200 までを扱っており、**v2.1.201 が未捕捉だった**ため本レポートで反映する。Layer 1 最新は v2.1.201。
- **v2.1.201 は単一エントリの極小リリース** = **Claude Sonnet 5 セッションが、ハーネスリマインダー（system-reminder）の配信に会話途中（mid-conversation）の system ロールを使わなくなった**。Sonnet 5 のプロンプト取り扱いに合わせた内部配線の変更で、ユーザー操作・設定・UI 変更を伴わない。**新機能・新 CLI フラグ・新スラッシュコマンド・新 Hook イベント・バグ修正エントリはいずれもなし**。
- **Layer 2/3 — 7/4〜7/5 に Claude Code 固有の新機能・新モデル・新ティアの公式発表なし**。Anthropic 公式ニュースルームは **7/2『More details on Fable 5's cyber safeguards and our jailbreak framework』以降、7/3〜7/5 に新規投稿なし**。検索で浮上した「Claude's new constitution」は **2026-01-22** の既出記事で本期間とは無関係。

## 主要な新機能

### v2.1.201（2026-07-03）— Sonnet 5 のハーネスリマインダー配線変更 ✅ GA

- **できること / 変更内容**: Claude Sonnet 5 を使うセッションで、Claude Code がこれまで会話の途中に **system ロールのメッセージ**として差し込んでいた「ハーネスリマインダー」（`<system-reminder>` 形式の内部注意喚起＝ツール使用ルール・コンテキスト方針などをモデルに再提示する仕組み）を、**mid-conversation の system ロールとしては送らなくなった**。
- **背景（推定）**: v2.1.201 の 1 つ前のリリース群では Sonnet 5 のプロンプト取り扱い（adaptive thinking デフォルト ON・新トークナイザ等、6/30 の Sonnet 5 GA 由来の挙動）が導入されており、本変更はそれに合わせてリマインダーの注入経路を最適化したものと見られる。会話履歴中に system ロールが混入することによるモデル側の解釈ノイズ低減が狙いと考えられる。
- **リリース状態**: ✅ GA（v2.1.201 に含まれ全ユーザーへ配信）。
- **対象**: Claude Sonnet 5 を選択しているセッション（Claude Code のデフォルトモデルは 7/1 の v2.1.197 以降 Sonnet 5）。
- **利用条件**: v2.1.201+。ユーザー側の設定・操作は不要。
- **制限事項 / 注意**: 挙動は内部的で可視の UI 変更はない。Sonnet 5 以外のモデル（Opus 4.8 等）を選択している場合の従来配線への影響は changelog に明記なし。**新スラッシュコマンド・新 Hook・新 CLI フラグの追加はなし**。
- **情報源**: [CHANGELOG.md](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) / [Releases](https://github.com/anthropics/claude-code/releases)
