# 調査レポート: Claude Code v2.1.196（組織デフォルトモデル・ストリーム監視ウォッチドッグ標準化）と Microsoft Foundry GA

**調査日:** 2026-06-30

## 概要

- **Layer 1（CLI）**: 前回レポート（6/27＝v2.1.195）以降、新規リリース **v2.1.196**（6/29）が1本登場。**組織デフォルトモデル**対応、**読みやすいセッション既定名**、**チャット内の添付ファイルをクリックで Finder/Explorer 表示**、**ストリーム監視ウォッチドッグの全プロバイダ標準有効化**、**非 Anthropic ホスト時の Remote Control 無効化**などを追加。残りはバックグラウンドジョブ／`claude agents` パネル／MCP OAuth 周辺の堅牢化バグ修正が中心。
- **Layer 2（公式）**: **6/29 — Claude が Microsoft Foundry で一般提供（GA）開始**。Azure ネイティブの認証・課金・ガバナンス、US データゾーン対応、Opus 4.8 / Haiku 4.5 を Messages API で提供、NVIDIA GB300 Blackwell Ultra 上で稼働。あわせて **Claude Managed Agents がユーザー管理のセルフホスト Sandbox ＋プライベート MCP サーバーで動作可能に（公開ベータ）**。
- **Layer 2（モデル退役）**: **6/30 — `claude-mythos-preview`（Claude Mythos Preview）が退役**。
- リリース状態の判定: v2.1.196 は npm 公開済み ✅ GA。Microsoft Foundry は ✅ GA。Managed Agents セルフホスト Sandbox は 🧪 公開ベータ。

---

## 主要な新機能

### 1. 組織デフォルトモデル（Organization Default Models）✅ GA
- **できること**: 管理者が組織コンソールでデフォルトモデルを設定でき、ユーザーが自分でモデルを選んでいない場合に `/model` 上で「Org default」（またはロール単位なら「Role default」）として表示される。
- **対象**: Team / Enterprise（組織コンソール管理者設定）。
- **位置づけ**: 6/13（v2.1.174-175）の「利用可能モデルの強制（enforce available models）」、6/24 の「組織モデル制限」に続く、組織ガバナンス系の拡張。今回はモデルの「既定値」を組織側で指定できる点が新しい。
- **初出**: v2.1.196（2026-06-29）

### 2. ストリーム監視ウォッチドッグを全プロバイダで標準有効化 ✅ GA
- **できること**: レスポンスストリームが **5分間イベントを生成しない**場合に中断・再試行する「ストリーム idle ウォッチドッグ」が、**全プロバイダでデフォルト ON** になった。`CLAUDE_ENABLE_STREAM_WATCHDOG=0` で無効化可能。
- **背景**: 6/22（v2.1.185）で導入された「ストリーム停止（stream stall）メッセージ」の流れを汲む信頼性改善。ストリームのハング検知を標準動作に格上げした。
- **新環境変数**: `CLAUDE_ENABLE_STREAM_WATCHDOG`
- **初出**: v2.1.196（2026-06-29）

### 3. 読みやすいセッション既定名 ✅ GA
- **できること**: セッション開始時に**人間が識別しやすい既定名**が自動付与され、一覧での識別やメッセージ送信が容易になった。
- **初出**: v2.1.196（2026-06-29）

### 4. チャット内の添付ファイルをクリックで表示 ✅ GA
- **できること**: チャット内のファイル添付が**クリック可能**になり、**Cmd/Ctrl + クリック**で Finder / Explorer 上の該当ファイルを開ける。
- **初出**: v2.1.196（2026-06-29）

### 5. 非 Anthropic ホスト時の Remote Control 無効化（セキュリティ）✅ GA
- **できること**: `ANTHROPIC_BASE_URL` が非 Anthropic ホストを指している場合、**Remote Control を無効化**。これは既存の `CLAUDE_CODE_USE_BEDROCK` / `_VERTEX` / `_FOUNDRY` 下での挙動に揃えたもの。
- **補足**: 環境変数 `CLAUDE_CODE_USE_FOUNDRY`（Microsoft Foundry プロバイダ）が公式に言及されており、後述の Foundry GA と整合する。
- **初出**: v2.1.196（2026-06-29）

### 6. MCP の自己承認サーバー起動を抑止（セキュリティ）✅ GA
- **できること**: `claude mcp list` / `get` が、リポジトリにコミットされた `.claude/settings.json` で**自己承認された `.mcp.json` サーバーを起動しなくなった**。信頼されていないワークスペースでは `⏸ Pending approval` と表示される。
- **位置づけ**: 6/23（v2.1.186）の MCP CLI 認証、6/24 の sandbox 資格情報に続く、サプライチェーン／信頼境界の強化。
- **初出**: v2.1.196（2026-06-29）

### 7. Microsoft Foundry で Claude が一般提供（GA）✅ GA【Layer 2】
- **発表日**: 2026-06-29（Anthropic 公式ブログ / Microsoft Azure ブログ）
- **できること**: Claude モデルが Microsoft Foundry（Azure ホスト）で **GA**。Azure 既存の認証・課金・ガバナンスをそのまま利用でき、データレジデンシー要件向けに**推論処理先を選択可能（US データゾーン対応）**。
- **提供モデル**: Claude Opus 4.8 / Claude Haiku 4.5（Messages API、prompt caching・extended thinking 対応）。
- **インフラ**: NVIDIA **GB300 Blackwell Ultra** GPU 上で稼働。
- **2つの実行形態**: ①「Azure ホスト」（Azure 環境内・US データゾーン）、②「Anthropic ホスト」（旧 Foundry Preview。フル API 機能や Azure 未提供モデルが必要な場合）。
- **Claude Code との関係**: CLI 側は `CLAUDE_CODE_USE_FOUNDRY` で Foundry プロバイダを指定可能。Bedrock / Vertex と同列のエンタープライズ・プロバイダ選択肢として整備が進む。
- **情報源**: [Claude in Microsoft Foundry is now generally available](https://claude.com/blog/claude-in-microsoft-foundry) / [Azure Blog](https://azure.microsoft.com/en-us/blog/introducing-anthropics-claude-models-in-microsoft-foundry-bringing-frontier-intelligence-to-azure/)

### 8. Claude Managed Agents のセルフホスト Sandbox ＋プライベート MCP 🧪 公開ベータ【Layer 2】
- **発表日**: 2026-06-29 前後
- **できること**: **Claude Managed Agents** が**ユーザー管理の Sandbox**で動作し、**プライベート MCP サーバー**に接続可能に。ツール実行をユーザー設定の環境（自社インフラ、または Cloudflare / Daytona / Modal / Vercel 等のマネージドプロバイダ）へ移せる。
- **位置づけ**: 5/19（Code with Claude London）で発表された「Self-hosted sandboxes（公開ベータ）」を Managed Agents へ統合・拡張する動き。
- **リリース状態**: 🧪 Claude Platform 公開ベータ。
- **情報源**: [Releasebot: Claude updates June 2026](https://releasebot.io/updates/anthropic/claude)

## バグ修正・パフォーマンス改善（v2.1.196 主要分）

- **バックグラウンドジョブのデータ保全**: トランスクリプト誤判定で会話を恒久削除し元プロンプトを再実行する不具合を修正（ファイルは削除せず退避）。
- **バックグラウンドセッションの堅牢化**: プロセス停止・再起動・更新をまたいで長時間コマンド／ワークフローが生存。Windows ではバックグラウンドシェルを kill せず引き継ぎ。daemon 再起動で kill された worker は次回 agents ビュー表示時に自動再開。
- **`claude agents` サイドパネル**: キーボードフォーカスのスタック、開くたびにサブエージェント型が失われる問題、実行中の誤ステータス表示を修正。完了行が「Done / Needs your input」で点滅する問題、停滞エージェントの「Needs attention」表示、PR 言及時のクリック可能リンク化。
- **`claude agents --dangerously-skip-permissions`** が無言で auto モードへフォールバックしていた問題を修正（バイパス免責表示＋生成エージェントへバイパス適用）。
- **Remote セッションのクラッシュ復旧**: サーバー再起動で中断したセッションが次の worker で自動再開。
- **MCP OAuth**: scope 未指定時に認可サーバーの `scopes_supported` 全カタログを要求し、GitLab セルフホスト等で `invalid_scope` になる不具合を修正。
- **PowerShell**: `git diff`/`git grep`、`egrep`/`fgrep`、`|` を含むクオート検索が exit 1 で失敗扱いになる問題を Bash 挙動に合わせて修正。
- **`/code-review`**: 5つのクリーンアップ finder を1つに統合し、トークン使用量を約25%削減。
- **`/context`**: Bedrock で全ツールグループが 0 トークン表示になる問題を修正。
- **`/deep-research`**: verifier 失敗を「all claims refuted」ではなく `unverified` と正しく報告。
- **UI/レンダリング**: ストリーミング中の no-op サブツリー走査スキップでフレーム描画負荷を軽減。
- **rewind**: アイドルプロンプトでの Esc Esc が rewind メニューを開かない退行を修正。
- **agents ビュー起動操作**: フォアグラウンドセッションから `←` 2回 → **1回**に変更（バックグラウンドセッションと統一）。
- **レート制限**: 並列リクエスト中にレート制限到達時の警告点滅・テレメトリ過大計上を修正。
