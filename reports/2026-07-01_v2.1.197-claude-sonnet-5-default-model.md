# 調査レポート: Claude Code v2.1.197 と Claude Sonnet 5 のデフォルトモデル化

**調査日:** 2026-07-01

## 概要

- **Layer 1（CLI）**: 前回レポート（6/30＝v2.1.196）以降、新規リリース **v2.1.197**（6/30）が1本登場。中身は **Claude Sonnet 5 を Claude Code のデフォルトモデルに切り替える**ためのリリースで、機能追加・バグ修正のエントリはなくモデル切替が単独の目玉。
- **Layer 2（公式・重大）**: **6/30 — Claude Sonnet 5 が GA。**Anthropic の最新 Sonnet で、**1M トークンコンテキストをデフォルト搭載**、Opus 4.8 に迫る品質を大幅に低い価格で提供。**Claude Code のデフォルトモデルが Opus 4.8 → Sonnet 5 に変更**（v2.1.197）。導入価格 **$2/$10 per MTok（8/31 まで）**、通常 $3/$15。
- **Layer 2（挙動変更）**: Sonnet 5 は **adaptive thinking がデフォルト ON**、**手動 extended thinking は 400 エラーで廃止**、**sampling パラメータ（temperature/top_p/top_k）非デフォルト値は 400 エラー**、**新トークナイザで同一テキストが約 30% 多いトークン数**になる（Opus 4.7 で導入されたトークナイザ）。
- **Layer 3（継続）**: **Claude Fable 5（公開版）は 7/1 時点も ⛔ アクセス停止継続**（6/12 米政府輸出管理指令）。Mythos 5 は 6/26 に 100+ 米国機関向け輸出規制部分解除フェーズ。Sonnet 5 を含む通常モデルは無影響。
- リリース状態の判定: v2.1.197 は npm 公開済み ✅ GA。Claude Sonnet 5 は ✅ GA（Microsoft Foundry のみ preview 扱い）。

## 主要な新機能

### 1. Claude Sonnet 5 GA ✅【Layer 2・最重要】
- **発表日**: 2026-06-30（Anthropic 公式ブログ / Claude Platform Docs）
- **位置づけ**: Sonnet 系の次世代モデル。**Claude Sonnet 4.6 のドロップイン後継**。「速度と知能の最良の組み合わせ」を謳い、**エージェント能力を業界標準の価格帯に引き下げる**戦略モデル。Opus 4.8 に迫る品質を大幅に低いコストで提供。
- **モデル ID**: `claude-sonnet-5`
- **コンテキスト / 出力**: **1M トークンコンテキストがデフォルト（かつ最大。小容量バリアントは存在しない）**、**最大出力 128k トークン**。
- **価格**: **導入価格 $2/$10 per MTok（入力/出力、2026-08-31 まで）**、以降は**標準 $3/$15 per MTok**（Sonnet 4.6 から据え置き）。⚠️ ただし新トークナイザで同一テキストのトークン数が増えるため、同等リクエストの実コストは 4.6 と異なり得る。
- **対象プラン**: **Free / Pro のデフォルトモデル**、Max / Team / Enterprise でも利用可。
- **提供面**: Claude API（全顧客）、Claude Code、Cursor、VS Code、GitHub Copilot で launch 時点から利用可。AWS（Amazon Bedrock ＋ Claude Platform on AWS。legacy Bedrock の `InvokeModel`/`Converse` は非対応）、Google Cloud（Vertex AI）、**Microsoft Foundry（preview）**。ZDR 契約組織は zero data retention 対応。
- **ベンチマーク**（報道・公式資料ベース）:
  - **SWE-bench Pro（エージェント型コーディング）: 63.2%**（Sonnet 4.6: 58.1% / Opus 4.8: 69.2% / GPT-5.5: 58.6% / Gemini 3.5 Flash: 55.1%）— Sonnet 5 がこのカテゴリで首位。
  - **Terminal-Bench 2.1: 80.4%**（GPT-5.5: 83.4% がわずかに上回る）。
  - **GDPval-AA v2（ナレッジワーク）: 1,618**（**Opus 4.8 の 1,615 をわずかに上回る**）。
- **安全性**: 欺瞞・ハルシネーション率が前世代より低下、悪意あるリクエストの拒否とプロンプトインジェクション耐性が向上。**Sonnet ティア初のリアルタイムサイバーセキュリティセーフガード**搭載（禁止・高リスクなサイバー話題は拒否＝HTTP 200 + `stop_reason: "refusal"`）。
- **Priority Tier**: Sonnet 5 では**非対応**（4.6 では対応）。
- **情報源**: [Introducing Claude Sonnet 5（Anthropic）](https://www.anthropic.com/news/claude-sonnet-5) / [What's new in Claude Sonnet 5（Platform Docs）](https://platform.claude.com/docs/en/about-claude/models/whats-new-sonnet-5)

### 2. Claude Code のデフォルトモデルが Sonnet 5 に切替 ✅【Layer 1】
- **できること**: **v2.1.197** で **Claude Code のデフォルトモデルが Opus 4.8 → Claude Sonnet 5 に変更**。ネイティブ 1M トークンコンテキストと導入価格（$2/$10、8/31 まで）が適用される。アクセスには **v2.1.197 へのアップデートが必要**。
- **意味合い**: 5/28（v2.1.154）以来 Claude Code のデフォルトだった Opus 4.8（プレミアム Opus）から、より安価な Sonnet ティアへデフォルトが移行。コスト効率重視の方向転換。Opus 4.8 は引き続き `/model` で選択可能（高難度タスク向けエスカレーション）。
- **初出**: v2.1.197（2026-06-30）
- **情報源**: [Claude Code changelog](https://code.claude.com/docs/en/changelog)

### 3. Sonnet 5 の API 挙動変更（Claude Code / API 共通）⚠️【Layer 2】
Sonnet 4.6 からの移行時に注意が必要な 3 点の挙動変更＋トークナイザ変更：
- **adaptive thinking がデフォルト ON**: `thinking` フィールドなしのリクエストが 4.6 では thinking なしで動いたが、Sonnet 5 では adaptive thinking で動く。オフにするには `thinking: {type: "disabled"}`。`max_tokens` は thinking ＋応答の合計上限のため、thinking なし運用だったワークロードは見直しが必要。
- **手動 extended thinking の廃止**: `thinking: {type: "enabled", budget_tokens: N}` は 400 エラー（Opus 4.8/4.7 と同様）。effort パラメータ＋ adaptive thinking へ移行。
- **sampling パラメータ非対応**: `temperature` / `top_p` / `top_k` を非デフォルト値に設定すると 400 エラー。Sonnet ティアでは新制約（Opus 4.7 で先行導入）。
- **新トークナイザ**: 同一テキストが **約 30% 多いトークン**になる（Opus 4.7 導入のトークナイザ）。API の形は不変でコード変更は不要だが、`usage` フィールド・token counting 結果・1M コンテキストの実テキスト量・`max_tokens` 予算・リクエスト単価が影響を受ける。旧モデル基準のトークン計測値は再計測が必要。
- **その他**: assistant メッセージの prefill は引き続き非対応（4.6 から不変）。
