# 調査レポート: Claude Code v2.1.190 / v2.1.191（`/clear` 前からの `/rewind`・MCP 信頼性改善）+ Claude Tag for Slack（6/23 公式）+ Fable 5 復帰のサブスク同梱シグナル

**調査日:** 2026-06-25
**調査モード:** フルモード（Layer 1〜3）

## 概要

6/24 レポート（フルモード、Layer 1 最新は v2.1.187）以降の差分を Layer 1〜3 で調査。

- **Layer 1**: CLI が **v2.1.190（6/24 15:53）** と **v2.1.191（6/24 21:58）** の 2 本進行。**v2.1.188 / v2.1.189 はともに欠番**（v2.1.171・v2.1.180・v2.1.182・v2.1.184 に続く 5・6 つ目の欠番）。**v2.1.190 はバグ修正・信頼性改善のみ**の安定化リリース。**v2.1.191 が機能追加あり** — 目玉は **`/clear` を実行する前の地点から会話を再開できる `/rewind` 対応**。加えて **MCP サーバー信頼性の複数改善**（capability discovery〔`tools/list`/`prompts` 等〕の自動リトライ、MCP OAuth の discovery/token リクエスト一時失敗時の 1 回リトライ、HTTP 404 エラーメッセージに URL 表示＋誘導追加）、**サンドボックスのネットワーク権限ダイアログ改善**（"Yes" 許可ホストの扱い改善）、**ストリーミング応答中の CPU 使用量を約 37% 削減**（テキスト更新の coalesce）、**長時間セッションのターミナル出力キャッシュによるメモリ増加を抑制**、`/voice`・`/login`・`/permissions`・hooks のカンマ区切り matcher・`claude agents` 関連など多数のバグ修正。**新スラッシュコマンド・新 Hook イベントの追加なし**（`/rewind` は既存コマンドの機能拡張）。
- **Layer 2（🆕 6/24 レポートで未捕捉）**: **Claude Tag（6/23 Anthropic 公式ニュース「Introducing Claude Tag」）** — Slack 上で `@Claude` をタグ付けしてタスクを委任できる **チーム共有型 AI ティームメイト**。**Claude Enterprise / Team 向けベータ**、**Opus 4.8** で動作。チャンネル内で 1 つの Claude を全員が共有し、非同期で段階的にタスクを実行→Slack スレッドに結果を投稿、チームの作業文脈を記憶（メモリ）、ambient モードで能動的に情報提示。**既存の「Claude in Slack」アプリを置換**（管理者は 30 日以内にオプトインで移行）。Claude Code の直接機能ではないが、ツール・データ・コードベースに接続するエージェント運用の Slack 面への拡張で、Claude Code エコシステムに隣接。
- **Layer 2/3 — Fable 5 復帰のサブスク同梱シグナル（❓ データマイン）**: ユーザーが **v2.1.190 の文字列差分**から、Fable 5 をサブスクリプションに **恒久同梱（週次 usage 上限付き）** する方向の準備を検出。新文字列 **"You've used your Fable 5 usage for this week"** が追加され、旧来の **"purchased separately from your plan"（プランとは別購入）** 表現が削除された。これは 6/23 に「サブスク同梱無料終了→ usage credits 建て」へ移行した直後の方針転換シグナルだが、**米政府輸出管理指令によるアクセス停止（6/12〜）は依然継続**で、公式の復旧アナウンスは 6/20〜25 の窓では出ていない（Anthropic ニュースルームの当該期間の新規投稿は 6/23 の Claude Tag のみ）。課金パッケージのシグナルと輸出規制の停止は別レイヤーの問題で、本シグナルは復旧の確定材料ではない。

## 主要な新機能

### `/rewind` が `/clear` 実行前の地点からの会話再開に対応 ✅
- **リリース状態**: ✅ GA（v2.1.191）
- **対象プラン**: 全プラン
- **概要**: これまで `/clear`（会話履歴クリア）を実行すると、その前の文脈には戻れなかったが、**v2.1.191 で `/rewind` が `/clear` 前のチェックポイントへの巻き戻し・会話再開をサポート**。誤って `/clear` した、あるいはクリア前の状態に戻したいケースの救済になる。`/rewind`（`/undo` エイリアス、v2.1.108 導入）のチェックポイント巻き戻し機能の適用範囲拡大。
- **初出**: v2.1.191 (2026-06-24)
- **情報源**: [Claude Code CHANGELOG.md](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)

### MCP サーバー信頼性の複数改善 ✅
- **リリース状態**: ✅ GA（v2.1.191）
- **概要**:
  - **capability discovery のリトライ** — `tools/list` / `prompts` 等の能力探索が一時失敗時にリトライされるようになり接続安定性が向上。
  - **MCP OAuth のリトライ** — discovery / token リクエストが一時的エラーで失敗した場合に 1 回リトライ。
  - **エラーメッセージ改善** — HTTP 404 エラーに **URL を表示**し、原因の切り分け先へ誘導。
- **意義**: v2.1.186 の `claude mcp login`/`logout`、v2.1.187 の `CLAUDE_CODE_MCP_TOOL_IDLE_TIMEOUT`（リモート MCP の 5 分無応答アボート）に続く MCP まわりの安定化系譜。
- **初出**: v2.1.191 (2026-06-24)
- **情報源**: [Claude Code CHANGELOG.md](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)

### ストリーミング応答中の CPU 使用量を約 37% 削減 ✅
- **リリース状態**: ✅ GA（v2.1.191）
- **概要**: ストリーミング応答中の **テキスト更新を coalesce（まとめて反映）** することで CPU 使用量を約 37% 削減。あわせて長時間セッションでの**ターミナル出力キャッシュによるメモリ増加を抑制**。長尺の応答や長時間稼働セッションでの体感負荷・発熱・バッテリ消費の改善。
- **初出**: v2.1.191 (2026-06-24)
- **情報源**: [Claude Code CHANGELOG.md](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)

### Claude Tag for Slack 🧪（Layer 2・6/23 公式）
- **リリース状態**: 🧪 ベータ
- **対象プラン**: Claude Enterprise / Team
- **対象プラットフォーム**: Slack
- **動作モデル**: Opus 4.8
- **概要**: Slack のチャンネルで `@Claude` をタグ付けし、自然言語でタスクを委任できる **チーム共有型 AI ティームメイト**。タスクを段階に分解して順に実行し、保有ツールを使って作業、完了時に **Slack スレッドへ成果を投稿**。**チャンネル内では 1 つの Claude を全員が共有**するため、誰が何を依頼・進行中かが可視化され、他メンバーが続きを引き継げる。**チームの作業文脈を時間をかけて記憶**し、ambient モードで能動的に情報を surface。**既存の「Claude in Slack」アプリを置換**し、管理者は **30 日以内にオプトイン**で移行。Anthropic 自社でも製品チームのコードの 65% を社内版 Claude Tag で生成と説明。Claude Code の CLI 機能ではないが、ツール・データ・コードベースに接続するマルチプレイヤー型エージェント運用を Slack 面に拡張する隣接プロダクト。
- **初出**: 2026-06-23（Anthropic 公式ニュース）
- **情報源**: [Anthropic - Introducing Claude Tag](https://www.anthropic.com/news) / [TechCrunch（6/23）](https://techcrunch.com/2026/06/23/anthropics-claude-tag-is-learning-your-company-one-slack-message-at-a-time/) / [Fortune（6/23）](https://fortune.com/2026/06/23/anthropic-claude-tag-virtual-employee-tool-slack/)

## バグ修正・安定化（v2.1.190 / v2.1.191、抜粋）

- **v2.1.190**: バグ修正・信頼性改善のみ（詳細な changelog 項目なし）。
- **v2.1.191（バグ修正）**:
  - ストリーミング応答中に過去出力を読んでいるとスクロール位置が最下部にジャンプする問題を修正
  - 停止したバックグラウンドエージェントが復活（resurrect）する問題を修正
  - `/voice` が無効化時に汎用的な "not available" メッセージを出す問題を修正
  - Windows Terminal で `/login` URL が折り返し時に途切れて開く問題を修正
  - Ghostty over ssh/tmux のフルスクリーンで Cmd+click が効かない問題を修正
  - `claude agents` が `/usage` 等の組み込みスラッシュコマンドをバックグラウンドへ誤送信する問題を修正
  - hooks のカンマ区切り matcher（例 `"Bash,PowerShell"`）がサイレントに無効化される問題を修正
  - `/permissions` の Recently-denied タブで承認した拒否が永続化されない問題を修正
  - managed settings の `forceRemoteSettingsRefresh` が設定時に効く問題を修正
  - サンドボックスのネットワーク権限ダイアログ改善（"Yes" 許可ホストの扱い）
  - vim モードのプロンプト履歴検索（NORMAL `/`）にスラッシュコマンドへの到達ヒントを追加
