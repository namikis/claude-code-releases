## Layer 1: 公式チェンジログ — v2.1.146 / v2.1.147（2026-05-21）

### v2.1.146（2026-05-21 01:51 UTC）

**位置づけ**: v2.1.145（5/19 21:31 UTC）から約 28 時間後の **5/21 早朝ホットフィックス**。Windows PowerShell の重大 regression（v2.1.124 から滞留）+ MCP / Auto mode / Plugin の集中修正。新規機能は `/simplify` → `/code-review` rename のみ。

#### `/simplify` → `/code-review` rename ✅（**動作変更注意**）

- **rename**: `/simplify` コマンドが **`/code-review`** に rename
- **動作変更**: 旧 `/simplify` の「cleanup-and-fix」自動修正動作は **廃止**。新 `/code-review` は **correctness bug をレポートのみ**（修正は別途指示が必要）
- **エフォートレベル指定**: `/code-review high` のように **low / medium / high** を指定可能
- **GitHub PR コメント投稿**: `--comment` フラグで **GitHub PR にインラインコメントとして findings を投稿**
- **背景**: 5/6 Code with Claude SF キーノートで GA 化された **GitHub Code Review（重大度ランク付き複数エージェント並行 PR レビュー）**との位置付け統一。`/simplify` の自動修正は dangerous な silent change を引き起こすことが多く、**「人間がレビューを読んでから修正適用」**フローへの転換
- **情報源**: [GitHub Releases v2.1.146](https://github.com/anthropics/claude-code/releases/tag/v2.1.146) / [Claude Code Changelog](https://code.claude.com/docs/en/changelog) / [Code Simplifier Plugin](https://claude.com/plugins/code-simplifier)

#### Auto-updater 改善 ✅

- 一時的ネットワーク障害をリトライ
- 失敗時に **specific error categories + OS error codes** を報告
- 失敗画面で **current version を表示**（以前は何もない）

#### 主要バグ修正

| 修正項目 | 詳細 |
|---------|------|
| **Windows PowerShell regression（v2.1.124 由来）** | `pwsh` を winget/Microsoft Store でインストールした環境で「command line is invalid」エラー（重要、長期滞留） |
| MCP server pagination | `resources/list` / `resources/templates/list` / `prompts/list` が page 1 以降を drop |
| Windows Terminal | attached background sessions で fullscreen strobing |
| 状況: Auto-updater | failure 時に current version 非表示 |
| Windows background-job worktree | NTFS junctions を辿って main repo を削除 |
| `/background` | skill / custom slash command のみの input でセッション拒否 |
| Backgrounded sessions | already-granted permission を再度プロンプト |
| `/theme` dialog | Esc 非反応 |
| Agent SDK | streaming session 終了時の uncaught exception |
| `forceLoginOrgUUID` / `forceLoginMethod` | third-party / API-key sessions で enforced されない |
| GNOME Terminal | 右クリック / 中クリック paste が text insert しない |
| `CLAUDE_CODE_SUBAGENT_MODEL` | multi-agent session で child process に forward されない |

### v2.1.147（2026-05-21 20:39 UTC）

**位置づけ**: v2.1.146 の **約 19 時間後**、5/21 同日 2 本目の連続リリース。`Ctrl+T` ピン留めセッションのライフサイクル管理、`/simplify` → `/code-review` 再確認、Auto-updater 改善、diff レンダリング性能向上。

#### Pinned background sessions の lifecycle 改善 ✅

- **変更**: `claude agents` で `Ctrl+T` でピン留めしたバックグラウンドセッションが
  - **idle でも生き残る**（通常は idle で daemon が auto-retire）
  - **アップデート適用のためにその場で再起動**（restart in place、コンテキスト復元）
  - **メモリ圧迫時の shed 順序**: non-pinned セッションを先に shed、pinned は最後
- **背景**: v2.1.143 でピン留め機能が追加されたが、daemon の idle-exit / memory pressure / binary upgrade 時にピン留めが「期待通り保護」されない issue を修正
- **エンタープライズ影響**: 長時間走行する Goal-driven セッション / multi-day refactoring セッションのピン留めが信頼できるレベルに到達

#### `/simplify` → `/code-review` rename 確認 ✅

- v2.1.146 で投入された rename を再度 changelog に記載（v2.1.147 は v2.1.146 を実質的に backport / 強化したリリース）

#### Diff レンダリング性能向上 ✅

- 大規模ファイル編集時の diff レンダリングを最適化（特に 1,000 行超の diff で体感差）
- 5/6 GA 化された GitHub Code Review（複数エージェント並行 PR レビュー、1,000 行超 PR で 84% / 平均 7.5 件指摘）の運用負荷に対応

#### その他改善・修正

- **Prompt history が連続重複を記録しない**ように
- **PowerShell tool 出力が default formatter 依存コマンドで欠落**する問題修正
- **「Yes, and don't ask again」on Windows** で matching rules が書かれない
- **`/effort`** で開く slider が間違ったレベル
- **Backgrounded sessions** で already-granted permission の再プロンプト（v2.1.146 の修正の再投入）
- **Plugin agents** で複数 `Agent(...)` 型宣言時に最後以外 drop
- **Hook `if` conditions** の `PowerShell(git push*)` 形式が never matching
- 30+ 件のその他バグ修正

## Layer 1: 公式チェンジログ — v2.1.148（2026-05-22 01:16 UTC）

### Bash exit code 127 regression ホットフィックス

**位置づけ**: v2.1.147（5/21 20:39 UTC）の **約 5 時間後**の緊急ホットフィックス。v2.1.147 が **全 Bash コマンドで exit code 127（command not found）を返す regression** を投入していたため、緊急 revert。

- **修正内容**: **Bash tool が全コマンドで exit code 127 を返す問題**（v2.1.147 で導入された regression）を修正
- **背景**: v2.1.147 で `claude agents` のピン留めライフサイクル改善や PowerShell tool の出力修正等が同梱されたが、Bash tool のコマンド解決パスに副作用で regression が混入
- **影響**: v2.1.147 を使ったユーザーは Bash コマンド実行が事実上不可能だったため、緊急アップデート必須
- **教訓**: 連続リリース運用での **回帰テストカバレッジ強化**の必要性を示す事例（5/21 中に 2 本リリース → 翌日緊急修正）

- TODO リスト / acceptance criteria の視認性大幅向上

- **新規追加**: enterprise managed setting に **`allowAllClaudeAiMcps`** を追加
- **動作**: claude.ai の cloud MCP connectors を **`managed-mcp.json` と並んで一括ロード**
- **背景**: enterprise admin が `managed-mcp.json` で個別 MCP サーバーを enumeration する手間を省き、**claude.ai 標準コネクタ群を一括許可**

#### `/feedback` の context 拡張 ✅

- `/feedback` レポートが **context compaction 前の会話**を含むように
- バグ報告 / フィードバックの再現性が向上

#### 重要バグ修正（PowerShell セキュリティ問題複数）

| 修正項目 | 詳細 |

| Thinking spinner | tool calls を跨いで amber 状態 stuck |

| Slash command arg hint | 末尾入力で clipping |

| Ctrl+O transcript view | 開いて freeze（tail せず） |
| Edit prompt-history | 矢印キーで navigate するとエディット内容を失う |
| `/config` exit summary | auto-compact と theme の phantom 変更を報告 |
