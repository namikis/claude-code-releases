## 主要なリリース内容

### 1. ✅ Claude Code v2.1.139（2026-05-11 18:43 UTC リリース、大型機能リリース）

#### 1-A. 🔬 Agent View（研究プレビュー、Layer 1 最重要機能）

**概要**: 並行する複数の Claude Code セッションを **`claude agents` 一画面で管理**する新インターフェース。5/6 Code with Claude SF キーノートで予告されていた **「並行エージェントオーケストレーション」戦略** の中核機能。

**起動**:
- `claude agents` でシェルから起動
- 任意の `claude` セッションから **`←`（左矢印、空プロンプトで）** → 現在のセッションをバックグラウンド化 + Agent View 起動
- `/bg`（`/background` エイリアス）で既存セッションをバックグラウンド化

**主要機能**:
- **State 表示**（行ごと）: ✽ Working（アニメーション） / 🟡 Needs input / Dimmed Idle / 🟢 Completed / 🔴 Failed / ⚫ Stopped
- **Icon 形状**: `✻` プロセス稼働中 / `∙` プロセス終了済み（peek/reply/attach で再起動）/ `✢` `/loop` セッション sleeping 中
- **Peek**: `Space` で行を選択して状態確認、`↑`/`↓` で隣接セッション、`→` で attach
- **Reply**: peek パネル内で reply 送信、`!` プレフィックスで Bash コマンド、multiple-choice の場合は数字キー、`Tab` で suggested reply
- **Attach**: `Enter` または `→`、`Alt+1` ～ `Alt+9` で N 番目セッション、attach 時に「what happened while you were away」リキャップ表示
- **Detach**: `←`（空プロンプト）/ `Ctrl+C` / `Ctrl+D` / `Ctrl+Z` / `/exit` — どれもセッションは止まらない、`/stop` で明示終了
- **Organize**: `Ctrl+S`（state ↔ directory グルーピング切替）、`Ctrl+T` ピン留め、`Shift+↑/↓` 並び替え、`Ctrl+X` 停止 →2 秒内に再度 `Ctrl+X` で削除
- **Filter**: `a:<name>` / `s:<state>` / `#<PR番号>`
- **One-line summary**: 各行のサマリは **Haiku 級モデル** が生成（最大 15 秒に 1 回 + ターン終了時、課金は通常セッションと同じ料金）
- **PR トラッキング**: セッションが PR をオープンすると、その行に PR リンク + CI ステータスインジケータ表示

**バックグラウンドセッションのアーキテクチャ**:
- **per-user supervisor process** が全バックグラウンドセッションをホスト（初回 `claude agents` または `--bg` で自動起動、ターミナル独立）
- セッションがアイドル 1 時間でプロセス停止（transcript と state は disk 保持、attach 時に再起動）
- バイナリ自動更新を検出して自動再起動（local file watch）
- **マシンスリープ/シャットダウンで停止** → `claude respawn --all` で復元
- 設定ディレクトリ:
  - `~/.claude/daemon.log` supervisor ログ
  - `~/.claude/daemon/roster.json` セッションリスト
  - `~/.claude/jobs/<id>/state.json` セッション状態

**ファイル編集の隔離**:
- バックグラウンドセッションは作業ディレクトリへの書き込みをデフォルトでブロック
- 書き込み必要時は自動で **`.claude/worktrees/` 配下に git worktree を作成** → 並行セッションが同じチェックアウトを参照しながら各自書き込み可能
- subagent frontmatter で `isolation: worktree` を設定すると常時 worktree 起動

**Permission Mode**:
- ディスパッチセッションは起動ディレクトリの settings / permission mode を継承
- `bypassPermissions` / `auto` モード は **一度 interactive で `claude` を起動して accept していないと拒否される**（unattended でのアクションを防ぐセーフガード）

**シェル管理コマンド**:
| コマンド | 機能 |
|---------|-----|
| `claude agents` | Agent View を開く |
| `claude --bg "<prompt>"` | バックグラウンドセッション直接起動 |
| `claude --agent <name> --bg "<prompt>"` | subagent をメインエージェントとして起動 |
| `claude attach <id>` | セッションに attach |
| `claude logs <id>` | recent output 表示 |
| `claude stop <id>` / `claude kill <id>` | 停止 |
| `claude respawn <id>` / `--all` | 停止セッション復元 |
| `claude rm <id>` | リスト削除 |

**プラットフォーム要件**:
- Claude Code v2.1.139 以降
- 対象プラン: Pro / Max / Team / Enterprise / Claude API
- 管理者は `disableAgentView` managed setting または `CLAUDE_CODE_DISABLE_AGENT_VIEW` 環境変数で組織全体を無効化可能

**制限事項（研究プレビュー）**:
- **レート制限が独立に消費される**（10 並行で 10 倍速で quota 消費）
- セッションはローカル実行（マシンスリープで停止）
- worktree はセッション削除時に消える（merge/push を事前に）

**戦略的位置づけ**: 5/6 SF キーノートで宣言された「control your laptop from your phone」（Remote Agents GA）と接続する基盤機能。**Subagent / Agent Teams / Worktrees / Remote Agents** の上位 UI レイヤとして機能し、Claude Code を **CLI チャットツール → エージェント運用基盤** に再定義する。TestingCatalog 5/11 評: 「making Claude Code less like a chat-driven coding tool and more like an agent operations layer for software teams」。

**情報源**: [Anthropic Blog - Agent view in Claude Code](https://claude.com/blog/agent-view-in-claude-code) / [Manage multiple agents with agent view (Docs)](https://code.claude.com/docs/en/agent-view) / [TestingCatalog - Anthropic adds Agent View](https://www.testingcatalog.com/anthropic-adds-agent-view-for-claude-code-for-parralel-work/) / [ClaudeWorld - v2.1.139 Release](https://claude-world.com/articles/claude-code-21139-release/)

#### 1-B. 🔬 `/goal` コマンド（研究プレビュー）

**概要**: 完了条件を宣言し、Claude が満たすまで複数ターンにわたって自律実行を続けるコマンド。Codex の `/goal` に対応する Anthropic 側の制御ループ機能。

**動作**:
- 完了条件を `<objective>` タグでラップし、各ターン後に completion audit を要求
- **runaway guard**: デフォルトで 1 つのアクティブゴールあたり最大 **500 回の Stop-hook continuation** を許可
- `/goal status` / `/goal pause` / `/goal resume` / `/goal clear` でサブコマンド制御
- `/goal --tokens 250K do deep research and build the full prototype` のようにトークン上限指定可能

**動作モード**:
- ✅ **Interactive モード**: 通常の `claude` セッション内
- ✅ **`-p` フラグモード**: ノンインタラクティブパイプライン用途
- ✅ **Remote Control モード**: モバイル / Web からのリモート操作セッション

**Overlay 表示**: live elapsed time / turn count / token usage が overlay panel に常時表示

**位置づけ**: Agent View が **「複数セッションの空間並列性」** を扱うのに対し、`/goal` は **「単一セッションの時間自律性」** を扱う。組み合わせると「複数のゴール駆動セッションを並列で長時間走らせる」運用が成立する。Anthropic Outcomes（Managed Agents 側、5/6 SF キーノートで public β 化）の Code クライアント側相当機能。

**情報源**: [GitHub Releases v2.1.139](https://github.com/anthropics/claude-code/releases) / [Claude Code Changelog](https://code.claude.com/docs/en/changelog) / [Codex /goal vs Claude Code Comparison](https://devtoolpicks.com/blog/codex-goal-command-vs-claude-code-agents-2026)

#### 1-C. ✅ その他の新コマンド・機能

- ✅ **`/scroll-speed` コマンド**: マウスホイールのスクロール速度を **live preview** 付きで調整（Cowork デスクトップ / VSCode 拡張 / JetBrains プラグイン向け）
- ✅ **`claude plugin details <name>`**: 指定プラグインの **コンポーネント inventory**（hooks / MCP servers / skills / commands）と **per-session 予想トークンコスト** を表示。プラグインインストール前のコスト評価に有用
- ✅ **Transcript view navigation 強化**:
  - `?` でキーボードショートカット一覧
  - `{` / `}` でユーザープロンプト間ジャンプ
  - `v` でショートカットパネル切替

#### 1-D. ✅ Hook 関連の強化

- ✅ **Hook `args: string[]` フィールド（exec form 新設）**: コマンドを **シェルを介さず直接 spawn**。path placeholder の quoting 問題が解消。従来の `command: "..."` 文字列方式と併存
- ✅ **Hook `continueOnBlock` config option for `PostToolUse`**: `true` に設定すると **rejection reason を Claude にフィードバックしてターンを継続**できる。`PostToolUse` で block しても会話が止まらない柔軟な拒否制御を実現

#### 1-E. ✅ MCP / 設定強化

- ✅ **MCP stdio servers が `CLAUDE_PROJECT_DIR` 環境変数を受け取る**: プラグイン configs から `${CLAUDE_PROJECT_DIR}` を参照可能に
- ✅ **`/mcp` Reconnect が `.mcp.json` 編集を再起動なしで反映**: 失敗時は HTTP status / URL も表示
- ✅ **`/context all` が per-skill token estimates をモデル tokenizer 込みで表示**（rounded values）
- ✅ **`claude plugin install <name>@<marketplace>` が marketplace 自動 refresh + retry してから「見つからない」エラー**
- ✅ **Compaction prompt が sensitive user instructions を保持**するよう改善
- ✅ **Remote MCP server reconnect retry が全ユーザー有効化**（以前は段階的展開）

#### 1-F. ✅ Subagent / OTEL 強化

- ✅ **Subagent API requests が以下のヘッダを保持**:
  - `x-claude-code-agent-id`
  - `x-claude-code-parent-agent-id`
- ✅ **`claude_code.llm_request` OTEL span に agent 属性追加**

#### 1-G. 🔒 セキュリティ強化（重要）

- 🔒 **API キー / auth token 設定時の機能制限**: `ANTHROPIC_API_KEY` / `apiKeyHelper` / `ANTHROPIC_AUTH_TOKEN` が設定されている場合、以下が無効化される:
  - **Remote Control** （旧 Remote Agents）
  - **`/schedule`**
  - **claude.ai MCP connectors**
  - **通知設定**

  **意図**: API キーベース利用は「機械対機械」スクリプト用途を想定し、ユーザー対話必要機能や claude.ai 認証必要機能を分離。セキュリティ境界の明確化。エンタープライズの誤設定リスク削減

#### 1-H. ✅ 主要バグ修正（50+ 項目から抜粋）

- ✅ **credential deadlock**: `forceRemoteSettingsRefresh` で認証コマンドがブロックされる問題を修正
- ✅ **`autoAllowBashIfSandboxed`**: `$VAR` 等のシェル展開を auto-approve しない問題を修正
- ✅ **Hook terminal access corruption**: フック実行時のターミナル状態破壊
- ✅ **MCP HTTP/SSE servers 無制限メモリ成長**: **フレーム当たり 16 MB cap** を導入
- ✅ **`Skill(name *)` 権限ワイルドカードマッチング**: スキル名ワイルドカードルールが正しく評価されない問題
- ✅ **設定 hot-reload とシンボリックリンク**: シンボリックリンクされた settings ファイルでホットリロードが機能しない問題
- ✅ **Model picker "Default" 行が環境変数オーバーライドを反映しない問題**
- ✅ **"Stream idle timeout" 偽陽性**: レスポンス完了後の偽エラー
- ✅ **10+ MCP servers 利用時の silent failure**（cache 書き込み不可時）
- ✅ **Bash history カーソル問題**
- ✅ **画像ペーストバグとレンダリング問題**

#### 1-I. ✅ VS Code 拡張

- ✅ **`Cmd/Ctrl+Shift+T`**: 直近に閉じた session タブを再オープン（`claudeCode.enableReopenClosedSessionShortcut` で設定可能）

### 2. ✅ Claude Platform on AWS GA（Layer 2、5/11）

**公開元**: [Anthropic Blog（claude.com/blog/claude-platform-on-aws）](https://claude.com/blog/claude-platform-on-aws) + [AWS Machine Learning Blog 同時公開](https://aws.amazon.com/blogs/machine-learning/introducing-claude-platform-on-aws-anthropics-native-platform-through-your-aws-account/) + [AWS What's New](https://aws.amazon.com/about-aws/whats-new/2026/05/claude-platform-aws/)

**概要**: AWS と Anthropic は 5/11 に **「Claude Platform on AWS」** を **GA 提供**開始。**Anthropic ネイティブの Claude Platform 機能を、AWS アカウント・IAM・CloudTrail・統合請求を経由して利用可能**にする。**AWS は Anthropic のネイティブ Claude Platform を提供する最初のクラウド**となる。

**Amazon Bedrock との関係**:
- これまでの **Amazon Bedrock 経路は継続提供**（Claude モデルを Bedrock API で利用、AWS 内データ処理境界を維持）
- **Claude Platform on AWS は別経路**: **Anthropic が運用、データは AWS セキュリティ境界外で Anthropic が処理**（Bedrock との根本的相違）
- **位置づけ**: 「ネイティブ Claude Platform の全機能（Bedrock では未提供の最新機能）を AWS の請求・IAM 統合で利用したい組織向け」

**提供される Claude Platform 機能**:
- Messages API
- **Claude Managed Agents（β）**
- **Advisor Tool（β）**
- Web Search / Web Fetch
- **MCP connector（β）**
- **Agent Skills（β）**
- Code Execution
- **Files API（β）**
- Prompt Caching
- Citations
- Batch Processing

**AWS 統合**:
- ✅ **AWS IAM 認証**（Anthropic API キー管理不要）
- ✅ **CloudTrail 監査ログ**
- ✅ **AWS 統合請求書**（既存 AWS commitments に課金充当）
- ✅ AWS タグ / least-privilege パターン適用

**リージョン展開**（18 リージョン）:
- US East (N. Virginia, Ohio)
- US West (Oregon)
- Canada (Central)
- South America (São Paulo)
- Europe (Dublin, London, Frankfurt, Milan, Zurich, Paris, Stockholm)
- Asia Pacific (Tokyo, Seoul, Melbourne, Jakarta, Sydney)

**Claude Code との関係**:
- **直接の Claude Code 統合は今回のアナウンスでは詳述されていない**が、AWS ブログでは **Claude Code エンジニアが「key users」** として言及される
- 既存 [Cowork on Bedrock GA（4/21〜）](../reports/2026-04-21_v2.1.116-and-opus-4.7-bedrock.md) は **Bedrock 経路の Claude Code 利用**を提供（AWS 境界内データ処理）
- **将来的な Claude Code on Claude Platform on AWS の可能性**: ネイティブ Claude Platform 機能（Managed Agents / Skills / Advisor 等）を AWS アカウント経由で Claude Code セッションから直接呼び出すパス。Anthropic 公式は現時点で言及なし

**戦略的意義**:
- 5/6 SpaceX Colossus 1 提携（300MW / 220K NVIDIA GPU、$ undisclosed）、5/8 Akamai $1.8B エッジコンピュート契約と並ぶ **インフラストラクチャ多角化の一環**

- Anthropic にとっては **エンタープライズ AWS 顧客への直接アクセス**（既存 commitments を流用可能 → 調達ハードル激減）
