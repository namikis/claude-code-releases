# Claude Code 調査レポート: v2.1.113・ネイティブバイナリ・4月23日デフォルトモデル移行

**調査日:** 2026-04-18
**対象期間:** 2026-04-17〜2026-04-18（前回4月18日午前レポート以降の差分）
**対象バージョン:** v2.1.113

---

## 概要

前回レポート（2026-04-18、v2.1.111-112対象）以降、CLIで**1リリース（v2.1.113）**が4月17日に出荷された。v2.1.113は規模・インパクトともに近年最大級のリリースであり、**CLIを「ネイティブClaude Codeバイナリ」へ移行**（バンドルJavaScriptの呼び出しを廃止、プラットフォーム別の optional dependency として配布）するアーキテクチャ的な大転換を含む。加えて `sandbox.network.deniedDomains` 設定の追加、macOSにおける `/private/{etc,var,tmp,home}` の危険削除ターゲット化、Bash denyルールの `env`/`sudo`/`watch`/`ionice`/`setsid` 等の exec ラッパー対応、`Bash(find:*)` allow ルールが `-exec`/`-delete` を自動承認しなくなる、など**セキュリティハードニングが大量**に入った。また長らく残っていた `Bash` の `dangerouslyDisableSandbox` サンドボックスバイパスやMCP並行呼び出しのwatchdog解除バグも修正。プラットフォーム面では、**2026年4月23日（来週木曜）から、Enterprise pay-as-you-go と Anthropic API のデフォルトモデルが Claude Opus 4.6 から Opus 4.7 へ切り替わる**ことが告知された（APIエイリアス `opus` が Opus 4.7 を指すようになる）。Opus 4.7 は同期間で Snowflake Cortex AI と GitHub Copilot にも即日展開され、提供プラットフォーム数が拡大した。

## Layer 1: CLIリリース

### v2.1.113（2026年4月17日）

#### アーキテクチャ変更（最重要）

- **ネイティブClaude Codeバイナリへの移行** ✅
  - CLIが**バンドルJavaScriptを呼び出す代わりに、ネイティブClaude Codeバイナリを spawn** するように変更
  - プラットフォーム別の optional dependency（platform-specific optional dep）として配布
  - 起動時間の改善とNode.js依存の段階的な排除を狙う（外部分析では約100MBのバイナリ、大半はJavaScriptCoreとBunのランタイム）
  - 既存の npm インストール経路は維持されるが、内部エンジンが Bun ベースのネイティブ実行に置き換わる

#### 新機能 — セキュリティ設定

- **`sandbox.network.deniedDomains`** 設定: 広い `allowedDomains` ワイルドカードがあっても、個別ドメインをブロックできる拒否リスト
- **macOS: `/private/{etc,var,tmp,home}` を危険削除ターゲット扱いに**: `Bash(rm:*)` allow ルールの下でも、これらのパスは明示的にダンジャラスとしてハンドル
- **Bash deny ルールの exec ラッパー対応**: `env`, `sudo`, `watch`, `ionice`, `setsid` 等でラップされたコマンドもマッチするように
- **`Bash(find:*)` allow ルールで `find -exec` / `-delete` を自動承認しなくなった**: 破壊的な find オプションの untrusted auto-approve をクローズ

#### 新機能 — UI/UX

- **Fullscreen mode: Shift+↑/↓**: 選択を可視領域の外へ拡張した際にビューポートがスクロール
- **`Ctrl+A` / `Ctrl+E` で論理行の先頭/末尾へ**: マルチライン入力で readline の挙動にマッチ
- **Windows: `Ctrl+Backspace` で直前の単語を削除**
- **OSC 8 ハイパーリンク対応ターミナルで、折り返された長いURLがクリック可能に**: レスポンス・bash出力の両方
- **プロンプト入力中のカーソルが `NO_COLOR` 環境下でも表示される**ように修正

#### 新機能 — コマンド改善

- **`/loop` 改善**: Esc で保留中の wakeup をキャンセル可能、wakeup 時の表示が "Claude resuming /loop wakeup" に統一
- **`/ultrareview` 改善**: 並列化されたチェックで起動が高速化、起動ダイアログに diffstat を表示、起動中アニメーション
- **`/extra-usage` が Remote Control（モバイル/Web）クライアントから利用可能に**
- **Remote Control クライアントで `@`-file オートコンプリート候補をクエリ可能に**
- **`/effort auto` の確認表示**: ステータスバー表示に合わせて "Effort level set to max" と表示
- **mid-stream でストールしたサブエージェント**: 10分後に明確なエラーで失敗（従来は無音ハング）

#### バグ修正（主要）

- **🔒 セキュリティ: `Bash` の `dangerouslyDisableSandbox`** がパーミッションプロンプトなしにサンドボックス外でコマンド実行していた問題を修正
- **🔒 セキュリティ: Bashツールで、1行目がコメントの複数行コマンド**のトランスクリプト表示がUIスプーフィング可能だった問題を修正（全文表示に）
- **`cd <current-directory> && git ...`** の no-op な `cd` がパーミッションプロンプトをトリガーしなくなった
- **MCP並行呼び出しのタイムアウトハンドリング**: あるツール呼び出しへのメッセージが、別の呼び出しの watchdog を silently disarm するバグを修正
- **Cmd-backspace / `Ctrl+U`** が再びカーソルから行頭まで削除するように（v2.1.111のリグレッション修正）
- Markdownテーブル: セル内インラインコードスパンにパイプ文字を含むとテーブルが壊れる問題を修正
- セッションrecapが未送信テキストを入力中でも発火してしまう問題を修正
- `/copy` "Full response" のMarkdownテーブル列がGitHub/Notion/Slackへのペースト時に整列しない問題を修正
- 実行中サブエージェント閲覧中にタイプしたメッセージが、そのトランスクリプトに出ず親AIに誤帰属する問題を修正
- 「コピーしたN文字」トーストが絵文字等のマルチコードユニット文字をオーバーカウントする問題を修正
- `/insights` が Windows で `EBUSY` クラッシュする問題を修正
- 終了確認ダイアログが一発スケジュールタスクを recurring として誤ラベルしていた問題を修正（カウントダウン表示へ）
- Fullscreen mode でスラッシュ/@ 補完メニューがプロンプト境界に flush しない問題を修正
- `CLAUDE_CODE_EXTRA_BODY` の `output_config.effort` が effort 非対応モデルや Vertex AI 上のサブエージェント呼び出しで400エラーになる問題を修正
- `ToolSearch` ランキング: ペーストされたMCPツール名で説明文一致の兄弟ツールが上位に出ていたのを修正
- 長コンテキストセッションの resume 後のコンパクションが "Extra usage is required for long context requests" で失敗する問題を修正
- `plugin install` が既インストールのプラグインとバージョン競合があっても成功していた問題を修正（`range-conflict` をレポート）
- "Refine with Ultraplan" がトランスクリプトにリモートセッションURLを表示しない問題を修正
- SDK のイメージコンテンツブロック処理失敗でセッションがクラッシュ → テキストプレースホルダーへデグレード
- **Remote Control**: サブエージェントのトランスクリプトがストリーミングされない、終了時にセッションがアーカイブされない問題を修正
- Bedrock Application Inference Profile ARN 経由で Opus 4.7 を使った際の `thinking.type.enabled is not supported` 400エラーを修正

## Layer 2: プラットフォーム・公式発表

### 2026年4月23日予定: デフォルトモデルが Opus 4.7 へ切り替え 📢 → ✅

| 項目 | 詳細 |
|------|------|
| **発効日** | 2026年4月23日（木） |
| **対象** | Enterprise pay-as-you-go、Anthropic API（第一者）ユーザー |
| **変更内容** | デフォルトモデルが Claude Opus 4.6 → **Claude Opus 4.7** |
| **エイリアス解決** | API 上で `opus` → Opus 4.7、`sonnet` → Sonnet 4.6（ `opus` エイリアスが自動で新モデルを指す） |
| **情報源** | [APIリリースノート（platform.claude.com）](https://platform.claude.com/docs/en/release-notes/overview) |

#### 影響

- 明示的にモデルIDをピンしていないAPIコール（`opus` エイリアス使用）は、4月23日以降に自動的に Opus 4.7 で応答する
- Opus 4.6 は引き続き `claude-opus-4-6-*` で利用可能（自動リタイアではない）
- **Opus 4.7 はトークナイザー・API挙動でブレーキングチェンジあり**: 本番利用者は [マイグレーションガイド](https://platform.claude.com/docs/en/about-claude/models/migration-guide#migrating-to-claude-opus-4-7) に従い事前検証が必要
- Claude Code では既に全プランで Opus 4.7 の `xhigh` がデフォルト推奨（v2.1.111〜）

### Claude in Amazon Bedrock 全面開放（2026年4月16日）✅ GA

- **全 Amazon Bedrock 顧客** が Claude をセルフサーブ利用可能に（従来は限定招待）
- Opus 4.7 と Haiku 4.5 を Bedrock コンソールから直接有効化可能
- 新エンドポイント: `/anthropic/v1/messages`（27 AWS リージョン、グローバル/リージョナル両対応）
- **情報源**: [APIリリースノート](https://platform.claude.com/docs/en/release-notes/overview) / [AWS announcement](https://aws.amazon.com/blogs/aws/introducing-anthropics-claude-opus-4-7-model-in-amazon-bedrock/)

### Claude Opus 4.7 — 追加プラットフォーム展開（2026年4月16日）

4月16日のGA発表日に、**Claude API／Bedrock／Vertex AI／Microsoft Foundry** に加えて、以下の2つのプラットフォームでも同日利用可能になったことが外部発表で判明：

- **Snowflake Cortex AI**（Public Preview、US/EU）
  - Cortex Code、Cortex AI Functions、Cortex REST API、Snowflake Intelligence の各サーフェスで利用可能
  - Snowflake セキュリティ境界の内側で動作
  - **情報源**: [Snowflake Blog](https://www.snowflake.com/en/blog/claude-opus-4-7-snowflake-cortex-ai/)
- **GitHub Copilot**（GA）
  - GitHub の Opus 4.7 対応アナウンスでは "generally available" の表現
  - **情報源**: [GitHub Changelog](https://github.blog/changelog/2026-04-16-claude-opus-4-7-is-generally-available/)

これにより、Opus 4.7 はGA日に**6プラットフォーム同時展開**となり、Anthropicのモデル提供範囲が大幅に拡大した。

### v2.1.113 ネイティブバイナリ移行への技術的反応

開発者コミュニティ（frr.dev 等の技術ブログ）では、ネイティブバイナリ化の狙いを以下のように分析：

- **起動性能**: Node.js 起動オーバーヘッド（コールドスタート、V8 初期化、Zod/大型スキーマのJIT）を Bun/JSC ネイティブで吸収
- **配布サイズ**: バイナリは約100MB（大半は JavaScriptCore + Bun APIランタイム）で、npm + node_modules 構成より安定
- **Node.jsバージョン依存の解消**: CI・企業環境での Node バージョンスキューが起きにくくなる

**情報源**: [frr.dev - Claude Code Native Build: 100MB Binary to Ditch Node for Good](https://www.frr.dev/posts/claude-code-native-build-bun/) / [claudenotes - faster startup, Windows ARM64](https://claudenotes.co/newsletter/claude-code-adds-cli-auth-management-faster-startup-windows-arm64)
