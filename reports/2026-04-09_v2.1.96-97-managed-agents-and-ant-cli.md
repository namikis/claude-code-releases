# Claude Code 調査レポート: v2.1.96-97・Managed Agents・ant CLI

**調査日:** 2026-04-09
**対象期間:** 2026-04-08 〜 2026-04-09

## 概要

前回調査（2026-04-08）以降、以下の重要な動きがあった：

1. **v2.1.96**（4月8日）: v2.1.94で発生したBedrock認証リグレッションのホットフィックス

### v2.1.97（2026年4月8日） ✅

#### 新機能・改善

| 機能 | 詳細 |
|------|------|
| **Focus View トグル（`Ctrl+O`）** | `NO_FLICKER` モードでプロンプト、1行ツールサマリー（edit diffstats付き）、最終レスポンスのみを表示するフォーカスビュー |
| **`refreshInterval` ステータスライン設定** | N秒ごとにステータスラインコマンドを再実行する設定 |
| **`workspace.git_worktree`** | ステータスラインJSON入力にgit worktree情報を追加。リンクされたworktree内で自動設定 |
| **`● N running` インジケータ** | `/agents` でライブサブエージェントインスタンス数をエージェントタイプ横に表示 |
| **Cedar ポリシーファイルのシンタックスハイライト** | `.cedar`、`.cedarpolicy` ファイルの構文強調表示 |
| **Accept Edits安全コマンド自動承認** | 安全な環境変数やプロセスラッパー付きのファイルシステムコマンドを自動承認（例: `LANG=C rm foo`、`timeout 5 mkdir out`） |
| **Auto Mode ネットワークアクセス自動承認** | Auto Mode / bypass-permissions モードでサンドボックスネットワークアクセスプロンプトを自動承認 |
| **画像圧縮の統一** | ペースト/添付画像がReadツールと同じトークンバジェットに圧縮 |
| **CJK句読点後のコマンド補完** | スラッシュコマンドと`@`メンション補完がCJK句読文字（句点・読点等）後にトリガー |
| **Bridgeセッションの改善** | claude.aiセッションカードにローカルgitリポジトリ・ブランチ・作業ディレクトリを表示 |
| **フッターレイアウト改善** | Focus・通知インジケータがモード行に統合。コンテキスト低下警告が一時通知に変更 |
| **Markdownブロック引用のバー改善** | 折り返し行をまたぐ連続した左バー表示 |
| **Bash OTEL トレーシング強化** | サブプロセスにW3C `TRACEPARENT` 環境変数を継承（トレーシング有効時） |
| **`/claude-api` スキル更新** | Managed Agentsのカバレッジを追加 |
| **`sandbox.network.allowMachLookup`** | macOSでMach lookupネットワーク制御が有効に |

#### 主要バグ修正

| 修正 | 詳細 |
|------|------|
| **`--dangerously-skip-permissions` ダウングレード問題** | 保護パスへの書き込み承認後にaccept-editsモードに暗黙ダウングレードされる問題を修正 |
| **Bashツール権限の強化** | 環境変数プレフィックスとネットワークリダイレクトのチェックを厳格化。一般的なコマンドでの誤プロンプトを削減 |
| **JavaScript プロトタイプ名パーミッション問題** | `toString` 等のプロトタイププロパティ名に一致する権限ルールで`settings.json`が無視される問題を修正 |
| **managed-settings allowルールの残留** | 管理者がルールを削除してもプロセス再起動まで有効のままだった問題を修正 |
| **additionalDirectories 中間セッション反映** | `permissions.additionalDirectories` の変更がセッション中に反映されない問題を修正 |
| **additionalDirectories と --add-dir の競合** | ディレクトリを設定から削除すると `--add-dir` で渡した同じディレクトリのアクセスも失われる問題を修正 |
| **MCP HTTP/SSE メモリリーク** | サーバー再接続時にHTTP/SSE接続が~50 MB/時の未解放バッファを蓄積する問題を修正 |
| **MCP OAuth トークンリフレッシュ** | `oauth.authServerMetadataUrl` が再起動後のトークンリフレッシュで無視される問題を修正（ADFS等に影響） |
| **429リトライの改善** | 小さな `Retry-After` 値でリトライが~13秒で枯渇する問題を修正。指数バックオフを最小値として適用 |
| **レート制限アップグレードオプション消失** | コンテキストコンパクション後にアップグレードオプションが消える問題を修正 |
| **`/resume` ピッカー複数修正** | `--resume <name>` で編集不可、Ctrl+Aリロードで検索消去、空リストでナビゲーション不能、タスクステータスが会話サマリーを上書き、プロジェクト間での古い情報表示を修正 |
| **`/resume` diff消失** | 10KB超のファイル編集diffが`--resume`時に消える問題を修正 |
| **`/resume` キャッシュミス** | 添付メッセージがトランスクリプトに保存されない問題を修正 |
| **処理中メッセージの未永続化** | Claudeが処理中に入力したメッセージがトランスクリプトに保存されない問題を修正 |
| **Stop/SubagentStop フック失敗** | 長いセッションでフックが失敗し、「JSON validation failed」と表示される問題を修正 |
| **サブエージェントworktree作業ディレクトリリーク** | worktree分離/cwd:オーバーライドのサブエージェントが親セッションのBashツールに作業ディレクトリを漏洩する問題を修正 |
| **コンパクション時のトランスクリプト重複** | prompt-too-longリトライ時にマルチMBのサブエージェントトランスクリプトが重複書き込みされる問題を修正 |
| **`claude plugin update` 偽最新報告** | gitベースのマーケットプレイスプラグインでリモートに新しいコミットがあっても「already at the latest version」と報告される問題を修正 |
| **スラッシュコマンドピッカーのYAMLブール問題** | プラグインのフロントマター`name`がYAMLブールキーワードの場合にピッカーが壊れる問題を修正 |
| **NO_FLICKER URLコピー時のスペース挿入** | 折り返しURLのコピー時に改行位置にスペースが挿入される問題を修正 |
| **NO_FLICKER zellijスクロールアーティファクト** | zellij内でのスクロールレンダリングアーティファクトを修正 |
| **NO_FLICKER MCP ホバークラッシュ** | MCPツール結果上にホバーした際のクラッシュを修正 |
| **NO_FLICKER メモリリーク** | APIリトライで古いストリーミング状態が残留するリークを修正 |
| **NO_FLICKER Windows Terminalスクロール** | マウスホイールスクロールが遅い問題を修正 |
| **NO_FLICKER 短いターミナル** | 24行未満のターミナルでカスタムステータスラインが表示されない問題を修正 |
| **NO_FLICKER Warpショートカット** | Shift+Enter、Alt/Cmd+矢印が動作しない問題を修正 |
| **NO_FLICKER Windows Unicode文字化け** | Korean/Japanese/Unicodeテキストがno-flickerモードのWindows上でコピー時に文字化けする問題を修正 |
| **Bedrock SigV4認証** | `AWS_BEARER_TOKEN_BEDROCK`/`ANTHROPIC_BEDROCK_BASE_URL` が空文字列の場合に認証失敗する問題を修正 |
| **トランスクリプトサイズ最適化** | 空のフックエントリをスキップ、pre-editファイルコピーのサイズ上限を設定 |
| **トランスクリプト精度** | ブロック単位エントリがストリーミングプレースホルダーではなく最終トークン使用量を記録するように改善 |

### Claude Managed Agents 🧪

- **リリース状態**: 🧪 パブリックベータ（全APIアカウントにデフォルト有効）
- **発表日**: 2026年4月8日
- **ベータヘッダー**: `managed-agents-2026-04-01`
- **公式ドキュメント**: [platform.claude.com/docs/en/managed-agents/overview](https://platform.claude.com/docs/en/managed-agents/overview)

#### 概要

Anthropicが提供するクラウドホスト型AIエージェント基盤。Messages APIとは異なり、エージェントループ・ツール実行・ランタイムを完全に管理されたインフラで実行。ファイル読み書き、コマンド実行、Web検索、コード実行をセキュアなコンテナ内で自律的に実行する。

#### コアコンセプト

| 概念 | 説明 |
|------|------|
| **Agent** | モデル、システムプロンプト、ツール、MCPサーバー、スキルの定義 |
| **Environment** | パッケージ（Python、Node.js、Go等）、ネットワークアクセス、ファイルマウントを設定したコンテナテンプレート |
| **Session** | エージェントとEnvironmentの実行インスタンス。特定のタスクを遂行 |
| **Events** | アプリケーションとエージェント間のメッセージ（ユーザーターン、ツール結果、ステータス更新）。SSEストリーミング |

#### 組み込みツール

- **Bash** — コンテナ内シェルコマンド実行
- **File operations** — Read/Write/Edit/Glob/Grep
- **Web search and fetch** — Web検索・URLコンテンツ取得
- **MCP servers** — 外部ツールプロバイダー接続

#### 研究プレビュー機能（要アクセス申請）

- **Outcomes** — 成果定義
- **Multi-agent** — マルチエージェント協調
- **Memory** — セッション間メモリ

#### アーキテクチャ（Anthropic Engineering Blogより）

- **3コンポーネント分離設計**: Brain（Claude + ハーネス）/ Hands（サンドボックス & ツール）/ Session（追記専用イベントログ）
- 各コンポーネントが独立して失敗・交換可能
- ステートレススケーリング: 多数のBrainをHands接続なしで起動可能
- **パフォーマンス**: p50 TTFT（Time to First Token）約60%削減、p95で90%以上改善
- **セキュリティ**: クレデンシャルはサンドボックス内に存在しない。OAuth トークンはMCPプロキシ経由

#### レート制限

| 操作 | 制限 |
|------|------|
| 作成系エンドポイント | 60リクエスト/分 |
| 読み取り系エンドポイント | 600リクエスト/分 |

#### 早期採用企業

Notion、Rakuten、Asana、Vibecode、Sentry

## バグ修正・パフォーマンス改善

### v2.1.97 主要改善

- **Write tool diff計算**: タブ/`&`/`$`を含むファイルで60%高速化
- **MCP HTTP/SSE メモリリーク修正**: 再接続時の~50 MB/時のバッファ蓄積を解消
- **429リトライ**: 指数バックオフの最小値適用で高速枯渇を防止
- **トランスクリプト最適化**: 空フックエントリのスキップ、pre-editコピーのサイズ上限
- **NO_FLICKER モード**: 8件以上のレンダリング・互換性修正（zellij、Warp、Windows Terminal、CJK等）
