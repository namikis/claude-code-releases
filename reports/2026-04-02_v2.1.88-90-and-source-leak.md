# Claude Code v2.1.88〜v2.1.90 & ソースコードリーク事件（2026年4月1日〜2日）

**調査日:** 2026-04-02
**対象期間:** 2026-03-30 〜 2026-04-02
**対象バージョン:** v2.1.88, v2.1.89, v2.1.90

---

## 概要

2026年4月1日に3つのバージョン（v2.1.88〜v2.1.90）がリリースされた。v2.1.88はnpmパッケージにソースマップが誤って含まれ、512,000行以上のTypeScriptソースコードが流出する重大インシデントが発生。v2.1.89とv2.1.90では、PreToolUseフックの`defer`決定、`PermissionDenied`フックイベント、`/powerup`インタラクティブレッスン、Auto Modeの改善など多数の新機能が追加された。プラットフォーム側では、Message Batches APIの`max_tokens`上限が300kに引き上げられ、Sonnet 4.5/4の1Mコンテキストウィンドウベータが4月30日で終了予定と発表された。

---

## 重大インシデント: ソースコードリーク

### 経緯

- **2026年3月30日**: v2.1.88がnpmに公開。パッケージ内に59.8MBのJavaScriptソースマップファイル（.map）が含まれていた
- **2026年3月31日 08:23 UTC頃**: セキュリティ研究者 Chaofan Shou（@Fried_rice、Solayer Labs）がX（旧Twitter）で発見を公表
- **同日中**: Anthropicがv2.1.88をnpmから削除。数時間以内にGitHubにミラーされ数千人の開発者が分析
- **Anthropicの声明**: 「本日初め、Claude Codeのリリースに内部ソースコードの一部が含まれました。機密性の高い顧客データや認証情報は一切関与・露出していません。これはヒューマンエラーによるリリースパッケージングの問題であり、セキュリティ侵害ではありません」

### 流出内容

| 項目 | 詳細 |
|------|------|
| **規模** | 約2,000ファイル、512,000行以上のTypeScriptコード |
| **主な内容** | エージェントアーキテクチャ、ツール統合、実行ロジック |
| **フィーチャーフラグ** | 44個。未出荷の完全実装済み機能を含む（セッションレビュー改善、バックグラウンド永続アシスタント等） |
| **メモリアーキテクチャ** | 3層構造。MEMORY.mdを軽量インデックス（1行約150文字）として常時コンテキストにロード |
| **未発表モデル参照** | "Claude Capybara" — バージョン2、capybara-v2-fastバリアント（1Mコンテキスト対応） |
| **Buddyディレクトリ** | `src/buddy/` — 5ファイル（後日/buddyとして正式リリース） |

### 報道

- Bloomberg、CNBC、VentureBeat、Fortune、Axios、Cybernews、The Hacker News、SiliconANGLE、Slashdot等が報道
- 3月26日のClaude Mythosリークに続く、数日間で2度目のセキュリティ上の失態として注目された

## 主要な新機能

### PreToolUse `defer` 決定（v2.1.89）

- **リリース状態**: GA
- ヘッドレスセッションがツール呼び出し時に一時停止し、`-p --resume`で再開してフックを再評価可能に
- CI/CDパイプラインでの人間による承認フローの実装に有用

### `PermissionDenied` フックイベント（v2.1.89）

- **リリース状態**: GA
- Auto Modeの分類器が拒否した後に発火
- `{retry: true}`を返すことでモデルにリトライを指示可能
- Auto Modeの拒否コマンドが通知表示され、`/permissions` → Recentタブで`r`でリトライ可能に

### `/powerup` コマンド（v2.1.90）

- **リリース状態**: GA
- Claude Codeの機能をアニメーションデモ付きで学べるインタラクティブレッスン
- 新規ユーザーのオンボーディングに有用

### `/buddy` — ターミナルペット（v2.1.89）

- **リリース状態**: GA（April Fools' Day限定機能）
- コーディング中に見守る小さなクリーチャーを孵化させるバーチャルペット
- 18種族: duck, goose, cat, rabbit, owl, penguin, turtle, snail, dragon, octopus, axolotl, ghost, robot, blob, cactus, mushroom, chonk, capybara
- ターミナル版たまごっち。開発ワークフローに反応

### `CLAUDE_CODE_NO_FLICKER=1` 環境変数（v2.1.89）

- **リリース状態**: GA
- alt-screenレンダリングと仮想化スクロールバックによるフリッカーフリー表示を有効化

### `MCP_CONNECTION_NONBLOCKING=true`（v2.1.89）

- **リリース状態**: GA
- `-p`モードでMCP接続待ちを完全スキップ
- `--mcp-config`サーバー接続を5秒でタイムアウト（最遅サーバーでブロックしない）

### Edit ツールの改善（v2.1.89）

- `Bash`の`sed -n`や`cat`でファイルを閲覧した場合、別途`Read`呼び出しなしで`Edit`が動作可能に

### Thinking Summaries デフォルト無効化（v2.1.89）

- インタラクティブセッションでthinking summaryがデフォルトで生成されなくなった
- `showThinkingSummaries: true`をsettings.jsonに設定すれば復元可能

## Auto Mode 改善

### 明示的ユーザー境界の尊重（v2.1.90）

- 「pushしないで」「Xの前にYを待って」等の明示的指示を、アクション自体が安全でもAuto Modeが尊重するよう修正

### 拒否コマンドのUI改善（v2.1.89）

- 拒否されたコマンドが通知として表示
- `/permissions` → Recentタブに表示され、`r`キーでリトライ可能

## バグ修正・パフォーマンス改善（主要なもの）

### パフォーマンス（v2.1.90）

- MCPツールスキーマのキャッシュキー計算でJSON.stringifyを排除
- SSEトランスポートの大きなストリームフレーム処理がO(n²)→O(n)に
- SDK長時間セッションのトランスクリプト書き込みがO(n²)→O(n)に
- `/resume`の全プロジェクトビューがセッション読み込みを並列化

### セキュリティ（v2.1.90）

- PowerShellツールの権限チェック強化: `&`バックグラウンドジョブバイパス、`-ErrorAction Break`デバッガハング、アーカイブ展開TOCTOU、パース失敗時拒否ルール劣化を修正
- `Get-DnsClientCache`と`ipconfig /displaydns`を自動許可から削除（DNSキャッシュプライバシー）
- `.husky`を保護ディレクトリに追加（acceptEditsモード）

### 安定性（v2.1.89）

- autocompactスラッシュループ検出 — コンパクション後すぐにコンテキストが上限に達する場合、3回繰り返すと停止してエラーメッセージを表示
- 50MB超の大きなセッションファイルからメッセージ削除時のクラッシュを修正
- LSPサーバーのゾンビ状態修正 — クラッシュ後、次のリクエストで自動再起動
- 1GiB超の非常に大きなファイルでEditツール使用時のOOMクラッシュ修正
- CJKや絵文字を含むプロンプト履歴が4KB境界で無音で破棄される問題を修正
- Devanagariや他の結合文字テキストがアシスタント出力で切り詰められる問題を修正

### Windows関連（v2.1.89〜v2.1.90）

- Edit/Writeツールの改行問題修正（CRLFの重複、Markdownハードラインブレーク削除）
- Voice ModeのWindows WebSocket問題修正
- Shift+Enterの送信問題修正（Windows Terminal Preview 1.25）
- PowerShellツールのstderr誤報修正（Windows PowerShell 5.1）

## プラットフォーム更新

### Message Batches API `max_tokens` 300k引き上げ（2026年3月30日）

- **リリース状態**: ベータ
- Claude Opus 4.6とSonnet 4.6で`max_tokens`上限が300kに
- `output-300k-2026-03-24`ベータヘッダーが必要
- 長文コンテンツ、構造化データ、大規模コード生成タスク向け

### Sonnet 4.5/4 の1Mコンテキストウィンドウベータ終了（2026年4月30日予定）

- **リリース状態**: 発表済み（4月30日に実施予定）
- `context-1m-2025-08-07`ベータヘッダーが無効に
- 200kを超えるリクエストはエラーを返す
- **移行先**: Sonnet 4.6またはOpus 4.6（1Mコンテキストが標準料金でGA）

### Claude Haiku 3 モデル廃止（2026年4月19日予定）

- **リリース状態**: 発表済み（2月19日発表、4月19日実施予定）
- `claude-3-haiku-20240307`が廃止
- **移行先**: Claude Haiku 4.5
