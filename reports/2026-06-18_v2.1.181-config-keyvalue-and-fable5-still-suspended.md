# 調査レポート: Claude Code v2.1.181 と Fable 5 / Mythos 5 アクセス停止継続

**調査日:** 2026-06-18

## 概要

- **Layer 1（CLI）**: 2026-06-17 リリースの **v2.1.181** が新規（前回レポートは v2.1.179 まで）。**v2.1.180 は欠番**（v2.1.171 に続く 2 つ目の欠番）。目玉は **プロンプトから任意設定を変更できる `/config key=value` 構文**（例 `/config thinking=false`、interactive / `-p` / Remote Control 全対応）+ **`sandbox.allowAppleEvents` 設定**（macOS サンドボックスから Apple Events 送信を許可）+ **`CLAUDE_CLIENT_PRESENCE_FILE` 環境変数**（PC 在席中はモバイル push 通知を抑止）+ **同梱 Bun ランタイムを 1.4 へ更新**。あわせて約 30 件のバグ修正・体感改善（段落のラインバイライン表示、thinking 中の接続断 auto-retry、サブエージェントパネル整理、MCP OAuth ブラウザページのデザイン統一、フルスクリーン URL オープンを Cmd/Ctrl+click に変更など）。**新スラッシュコマンド・新 Hook イベントの追加はなし**。
- **Layer 2/3**: 新しい Claude Code 製品機能の公式発表はなし。**Claude Fable 5 / Claude Mythos 5 は依然アクセス停止のまま**（6/12 米政府輸出管理指令、6/17 時点も復旧日未定）。SNS 上で「48 時間以内に復旧」とする噂（6/16、第三者アカウント BridgeMind 発、❓ 噂）が出回るが Anthropic は未確認。**プラットフォーム障害は引き続き散発**（6/16 で 6/5 以降「12 日間に 10 回目」、Opus 4.8 / Haiku 4.5）。**本人確認・年齢確認ポリシー（7/8 発効）** が Fable 5 の US 限定再開の枠組みになり得るとの観測あり。

## 詳細

### Layer 1: CLI v2.1.181（2026-06-17）

v2.1.180 は CHANGELOG にエントリなしの欠番。実質的な新規リリースは v2.1.181。

**新機能・新設定:**

- **`/config key=value` 構文** — プロンプトから任意の設定値を直接変更できる（例 `/config thinking=false`）。interactive・`-p`（print）・Remote Control の全モードで動作。従来の `/config`（設定画面 UI）に対する追加で、コマンド一発での設定変更が可能に。
- **`sandbox.allowAppleEvents`（opt-in 設定）** — macOS でサンドボックス化されたコマンドが Apple Events を送信できるようにする。`open` / `osascript` / ブラウザ認証フローが Apple Events entitlement 不足でエラー -600 を返していた問題への対応とセット。
- **`CLAUDE_CLIENT_PRESENCE_FILE` 環境変数** — マーカーファイルを指すよう設定すると、その PC で作業中の間はモバイルへの push 通知を抑止する（在席検知）。
- **同梱 Bun ランタイムを 1.4 へアップグレード**。

**体感改善:**

- 長い段落のストリーミングが、最初の改行を待たず **ラインバイラインで表示**されるように。
- thinking 中の API 接続断が **自動リトライ**されるように（従来は "Connection closed while thinking" を表示）。
- サブエージェントパネル改善: アイドルなサブエージェントは 30 秒で自動的に隠れ、リストは最大 5 行 + スクロールヒント、フッターにキーボードヒント表示。
- MCP OAuth のブラウザ認証ページを Claude Code のデザインに統一し、成功時に自動クローズ。
- フルスクリーンモードでの URL オープンを **Cmd+click（macOS）/ Ctrl+click 必須**に変更（ネイティブターミナルの挙動に合わせる）。
- `Improved N memories` 行が verbose モード外では個別ファイルを列挙しないように。

**主なバグ修正（約 30 件）:**

- カスタム `ANTHROPIC_BASE_URL` / Foundry でリクエストごとの attestation トークンによりプロンプトキャッシュが失敗する問題を修正。
- ネットワークドライブ / クラウド同期フォルダで Write/Edit がファイルを切り詰める問題を修正。
- macOS で Apple Events entitlement 不足により `open` / `osascript` / ブラウザ認証がエラー -600 で失敗する問題を修正。
- MCP サーバー未設定時に最初のプロンプトが managed-settings 取得を待つ起動リグレッション修正。
- account settings 取得が遅いときに空白ターミナルで最大 15 秒起動がブロックされる問題修正。
- `.claude.json` の壊れた null プロジェクトエントリで `TypeError: Cannot read properties of null` クラッシュする問題修正。
- Spotlight 再インデックス中の macOS TUI フリーズ修正。
- フォアグラウンドのサブエージェントが無制限にネストチェーンを生成する問題を修正し **5 階層の深さ上限を遵守**（v2.1.172 のネストサブエージェント関連）。
- モデル切替直後に `/recap` や会話フォークが旧モデルを使う問題修正。
- AWS `awsCredentialExport` クレデンシャルが短寿命で毎分リフレッシュされる問題修正。
- Windows での ExitWorktree / OneDrive エージェント作成 EEXIST / 相対 symlink な `settings.json` の ENOENT など環境固有修正多数。

**新スラッシュコマンド・新 Hook イベントの追加はなし。** v2.1.171・v2.1.180 は欠番。Layer 1 最新は v2.1.181（6/18 当日の新規 CLI なし）。

### Layer 2: Claude Fable 5 / Mythos 5 — アクセス停止継続

6/9 にローンチした Mythos クラス 2 モデルは、6/12 の米国商務省輸出管理指令（外国籍者によるアクセス停止を要求）への即時コンプライアンスのため全ユーザーで無効化されたまま。6/17 時点も状況に変化はなく、**公式の復旧日は未定**。Anthropic は「これは誤解だと考えており、できる限り早く復旧する」との立場を維持。

- **US 市民は 7/8 発効の本人確認（identity verification）を経て再開し得る**との観測（=国別アクセス分離が技術的に可能になる枠組み）。一方、**国際ユーザーの復旧は輸出管理指令そのものの撤回が前提**で別タイムライン・時期未定。
- **Opus 4.8 含む他の全モデルは無影響**。

（出典: [TechCrunch](https://techcrunch.com/2026/06/09/anthropics-claude-fable-5-is-a-version-of-mythos-the-public-can-access-today/) / [VentureBeat](https://venturebeat.com/technology/anthropic-blocks-all-public-access-to-claude-fable-5-mythos-5-following-us-government-order-what-enterprises-should-do) / [National Law Review](https://natlawreview.com/article/ai-company-anthropic-suspends-access-claude-fable-5-claude-mythos-5-following-us)）

### Layer 3: 「48 時間以内に復旧」の噂（❓ 噂・未確認）

6/16 に第三者アカウント **BridgeMind** が「Fable 5 は 48 時間以内に復旧する」との予測を投稿。**これは Anthropic の公式アナウンスではない**。Anthropic と商務省当局者の間で実際の交渉が進んでいるとの報道はあるが、公式の復旧タイムラインは出ていない。社内・公式チャネル以外の予測は鵜呑みにしないこと。

（出典: [pasqualepillitteri.it](https://pasqualepillitteri.it/en/news/5180/claude-fable-5-when-will-it-return) / [Polymarket](https://polymarket.com/event/claude-fable-5-restored-for-us-customers-by-20260613193753196)）

### Layer 3: プラットフォーム障害の継続

6/16 の障害は 6/5 以降「12 日間に 10 回目」の重大障害で、claude.ai / Claude API / Claude Code / Claude Cowork 全体で Opus 4.8 / Haiku 4.5 のエラー率上昇を伴った。Anthropic は前例なき需要（Claude Code の急成長 + モバイルアプリ DL 55% 増）でインフラが逼迫していることを認めている。6/17 時点では直近インシデントは解消済み・全体ステータスは operational。

（出典: [TechTimes](https://www.techtimes.com/articles/318514/20260616/claude-outage-tenth-disruption-12-days-exposes-anthropic-infrastructure-strain.htm) / [status.anthropic.com](https://status.anthropic.com/)）

## バグ修正・パフォーマンス改善

v2.1.181 の主要修正は上記「Layer 1」に記載。特に重要なのは ① 段落ストリーミングのラインバイライン化、② thinking 中接続断の自動リトライ、③ ネットワーク/クラウド同期フォルダでの Write/Edit 切り詰め修正、④ フォアグラウンドサブエージェントの 5 階層深さ上限遵守、⑤ 起動ブロッキング系（managed settings / account settings / Spotlight）の複数修正。
