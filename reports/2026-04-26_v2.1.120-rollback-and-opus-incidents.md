# Claude Code 調査レポート: v2.1.120 事実上ロールバック・Opus 4.7 連続障害・claude.ai 障害

**調査日**: 2026-04-26
**対象期間**: 2026-04-25 前回レポート作成以降 〜 2026-04-26 朝
**調査モード**: ニュースモード（Layer 2 公式ステータス・APIドキュメント / Layer 3 ニュース・GitHub Issues。Layer 1 チェンジログはスキップ）
**備考**: 前回 4/25 レポート時点では「v2.1.119 が依然として最新」と整理していたが、本調査により **v2.1.120 が 4/24 22:11 UTC 頃に npm 公開され、4/25 未明に複数の重大回帰により事実上ロールバックされていた** ことが判明。CHANGELOG.md / GitHub Releases には未掲載のままで、ステータスインシデントとサポート記事のみが公式記録。本レポートは前回の訂正・補足を兼ねる。

## 概要

- **Layer 2: Claude Status「Claude Code v2.1.120 Crashes on Startup」インシデント**（2026-04-25 01:45–02:35 UTC）。`claude --resume` / `--continue` 起動時にクラッシュする回帰のため、自動アップデートで **v2.1.119 へ強制ダウングレード**。ユーザーには `claude install 2.1.119` の手動コマンドも提供
- **Layer 3: GitHub Issues（4/24–25 集中）で v2.1.119/120 の 8 件超の回帰が報告**。`UKH/g9H is not a function` クラッシュ（#53044・#53086）、`sandbox required but unavailable` 回帰（#53085）、Opus 4.7 サイレント 1M モデル切替（#53031）、auto-update マニフェスト 404（#53028）等
- **Layer 2: Claude Status で Claude Opus 4.7 elevated errors が 4/25 に 3 件連続発生**。最長は **08:43–11:58 UTC（約 3 時間 15 分）** で、claude.ai / Console / API / Claude Code / Cowork に影響
- **Layer 2: claude.ai 全体の elevated errors**（4/25 18:42–19:02 UTC、約 20 分）も別途発生。同日中に少なくとも **5 件のインシデント** が記録された
- **Layer 3: Anthropic 公式ニュース**（anthropic.com/news）には **4/25・4/26 付の新規ブログ・発表は確認されず**。Layer 2 のリリースノートも 4/24 の Rate Limits API エントリ以降は更新なし

CLI・モデル両方で本日同時多発的に **品質・配信トラブル** が起きており、4/23 の品質劣化 postmortem 直後に再び信頼性が問われる事態となった。本期間中に **新機能リリース・新発表は確認されていない**。

- 上記 GitHub Issues の多くが **Open のまま**で、Anthropic 公式コメントは確認できず（`area:` ラベル・`has-repro` ラベルが付与され、internal triage は進行中とみられる）

### 2. Claude Opus 4.7 連続障害（2026-04-25）

**情報源**: [Claude Status](https://status.claude.com/)

#### インシデント一覧

| 開始 (UTC) | 終了 (UTC) | 期間 | 影響 |
|-----------|-----------|------|------|
| 04-25 01:35 | 04-25 02:34 | 約 59 分 | Opus 4.7 elevated error rates（v2.1.120 クラッシュと同時刻帯） |
| 04-25 07:48 | 04-25 08:37 | 約 49 分 | Opus 4.7 elevated errors |
| 04-25 08:43 | 04-25 11:58 | **約 3 時間 15 分** | Opus 4.7 elevated errors（claude.ai / Console / API / Claude Code / Cowork すべてに影響） |

3 件目のインシデントが特に長時間で、**テスト時間帯（米西部 1:43 AM PT 開始）から日中（米東部 7:58 AM ET 終了）にかけて**、Opus 4.7 を選択したセッションで広範な失敗が発生した。

#### 影響

- Opus 4.7 を選択した Claude Code セッション、Cowork、Claude.ai、Claude Console、API すべてで `5xx` エラー発生率上昇
- 4/16 GA 後の **大規模 Opus 4.7 障害としては最長クラス**
- 4/15 にも大規模障害（前々回レポート参照）があり、4 月後半は **インフラ側の不安定さが目立つ**

#### Claude Code への直接的影響

- Opus 4.7 をデフォルトモデルに設定しているユーザー（4/23 デフォルトモデル切替対象者を含む）は、対応する時間帯にツール呼び出し失敗・compaction 中断・セッション復旧不能を経験した可能性が高い
- 4.7 → 4.6 への一時切替（`--model claude-sonnet-4-6` 等）が回避策として推奨される状況

### 4. v2.1.119 由来の問題: Opus 4.7 サイレント 1M モデル切替

**情報源**: [GitHub Issue #53031](https://github.com/anthropics/claude-code/issues/53031)

- v2.1.118 では `claude-opus-4-7` (200k) を選択すると 200k で動作
- **v2.1.119 から、内部的に `claude-opus-4-7[1m]` (1M コンテキスト) に silently 切替**。コンテキスト表示が `26.1k/1m tokens` のように 1M ベースになる
- ユーザーが意識せず 1M バリアントを利用するため、**料金計算・品質特性の差異** に気付きにくい
- 4/23 の品質劣化 postmortem を受けて「200k と 1M で出力品質に差があるのでは？」というコミュニティ懸念を再燃させている

#### 回避策

```json
// ~/.claude/settings.json
{
  "env": {
    "CLAUDE_CODE_DISABLE_1M_CONTEXT": "1"
  }
}
```

> v2.1.119 がこの挙動を含む状態で「最新安定版」になっているため、**v2.1.120 ロールバック後も本問題は残存**。次回修正リリースで対応されるか要注視。
