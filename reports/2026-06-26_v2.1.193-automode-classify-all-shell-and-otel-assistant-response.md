## 主要な新機能（v2.1.193）

### 1. `autoMode.classifyAllShell` 設定 ✅ GA
- **できること**: Auto mode において、**全ての Bash / PowerShell コマンド**を auto-mode 分類器に通すよう変更する設定。デフォルトでは「任意コード実行パターン（arbitrary-code-execution）」のみが分類器評価の対象だが、本設定を有効にすると**すべてのシェルコマンド**が分類器によるリスク評価を受ける。
- **位置づけ**: 6/20（v2.1.183）の破壊的 git/IaC コマンドガードレール、6/16（v2.1.178）の「サブエージェント生成前の分類器評価」に続く **Auto mode の安全性強化**の一環。より保守的な自動実行ポリシーを敷きたい組織向け。
- **リリース状態**: ✅ GA（v2.1.193）

### 2. Auto mode の拒否理由を可視化 ✅ GA
- **できること**: Auto mode がアクションをブロックした際の**拒否理由**を、① トランスクリプト ② 拒否トースト ③ `/permissions` の「最近の拒否（recent denials）」タブの 3 箇所に表示。なぜブロックされたかが追跡可能になり、ガードレールのデバッグ・調整がしやすくなる。
- **リリース状態**: ✅ GA（v2.1.193）

### 3. `claude_code.assistant_response` OpenTelemetry ログイベント ✅ GA（⚠️ 挙動変更注意）
- **できること**: モデルの応答テキストを含む新しい OTEL ログイベント `claude_code.assistant_response` を追加。従来からあるプロンプトログ（ユーザー入力）に加え、**アシスタント側の応答内容も可観測性パイプラインに送出**できる。
- **⚠️ アップグレード時の重要な挙動変更**: 応答テキストはデフォルトでリダクト（秘匿）されるが、`OTEL_LOG_ASSISTANT_RESPONSES` 環境変数が**未設定の場合は `OTEL_LOG_USER_PROMPTS` の設定に追従**する。つまり**既にプロンプト内容をログ送出している環境では、アップグレード後に自動的に応答内容も送出され始める**。プロンプトのみに留めたい場合は明示的に `OTEL_LOG_ASSISTANT_RESPONSES=0` を設定する必要がある。
- **リリース状態**: ✅ GA（v2.1.193）

### 4. Bash モード（`!`）のライブファイルパス補完 ✅ GA
- **できること**: `!` で始まる Bash モードでコマンドを打つ際に、ファイルパスのライブ（インライン）オートコンプリートが効くようになった。
- **リリース状態**: ✅ GA（v2.1.193）

### 5. MCP 認証が必要なときの起動通知 ✅ GA
- **できること**: 起動時、認証が必要な MCP サーバーがある場合に通知を表示し、`/mcp` への導線を提示。6/22（v2.1.186）の `claude mcp login`/`logout` CLI コマンド、6/25（v2.1.191）の MCP 信頼性改善に続く MCP オンボーディング体験の改善。
- **リリース状態**: ✅ GA（v2.1.193）

### 6. アイドルなバックグラウンドシェルの自動メモリ圧迫リープ ✅ GA
- **できること**: アイドル状態のバックグラウンドシェルコマンドを、メモリ圧迫時に自動的にリープ（回収）する。`CLAUDE_CODE_DISABLE_BG_SHELL_PRESSURE_REAP=1` で無効化可能。6/25（v2.1.191）の「長時間セッションのメモリ増加抑制」に続くメモリ最適化。
- **リリース状態**: ✅ GA（v2.1.193）

## バグ修正・改善（v2.1.193、主要なもの）

- `/model` 等の client-data 依存 UI が `/login` 直後に古い/空の状態を表示する問題を修正
- バックグラウンド化（←←）が、全 running タスクが新セッションに引き継がれる場合でも誤って「N background tasks would be abandoned」でキャンセルされる問題を修正
- ピン留めバックグラウンドエージェントが自動更新のたびに「Continue from where you left off」と再プロンプトされる問題を修正
- メインターンのバックグラウンド化が、メイン会話を再実行する幻の「general-purpose (resumed)」サブエージェントを生成する問題を修正
- サブエージェント表示時にエージェントパネルが兄弟エージェントを隠す問題を修正
- バックグラウンドエージェント改善: 起動結果が Claude に「end your response」と指示しなくなり、エージェント稼働中も他タスクを継続
- MCP `headersHelper` 認証改善: ツール呼び出しが 401/403 を返した際にヘルパーが自動再実行・再接続
- プラグイン自動リネーム改善: マーケットプレイスの `renames` マップに自動追従し設定を新名称へ更新
- `/add-dir` が既存の作業ディレクトリを指定された際のメッセージ改善

## Layer 2 / Layer 3（プラットフォーム・ニュース）

### 新規公式発表なし（6/24〜26）
Anthropic ニュースルーム・リリースノートとも、**6/23 の「Claude Tag for Slack」以降に新規投稿なし**。6/24〜26 の窓で新しい Claude Code 製品機能・モデル・ティアの公式発表は確認されなかった。

### Claude Fable 5 / Claude Mythos 5 — 停止継続（6/25 時点・ゼロトラフィック確認）
- **6/12 の米政府輸出管理指令**（米国内外問わずあらゆる外国籍者によるアクセス停止を要求）以降、6/25 時点でも**復旧していない**（指令から 13 日経過）。
- バイラルなソーシャル投稿が「アクセスが復旧した」と主張したが、**Anthropic スタッフが「Fable / Mythos のトラフィックはゼロ」と明言**し否定。`claude-fable-5` への API 呼び出しはエラーを返す。
- **復旧メカニズムの観測**: Anthropic は **7/8 発効のプライバシーポリシー改定**で政府発行 ID + バイオメトリクスの収集を導入。これが**米国市民限定の復旧メカニズム**になり得る（輸出管理指令を全面解除せずとも、検証済み米国ユーザーには Fable 5 を再開し、国際ユーザーは Opus 4.8 に留まる）との見方。
- **Opus 4.8 含む他の全モデルは無影響**。

> ⚠️ 前回（6/25）レポートで触れた「v2.1.190 文字列差分による Fable 5 サブスク恒久同梱シグナル（❓ データマイン）」については、6/26 時点でも公式アナウンスはなく、輸出規制による停止が継続しているため**復旧の確定材料ではない**点に変わりなし。
