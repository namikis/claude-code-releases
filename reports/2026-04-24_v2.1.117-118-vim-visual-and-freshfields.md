- **Claude Code 新リリース 2 本**: **v2.1.117（2026-04-22）** と **v2.1.118（2026-04-23）** を連続リリース。CLI 本体の継続的改善フェーズ。Platform リリースノート（Claude API / Claude Apps）は 2026-04-20 以降更新なし
- **v2.1.118 の目玉**: (1) **vim visual / visual-line モード**（`v` / `V`）GA、(2) **`/cost` と `/stats` を `/usage` に統合**、(3) **カスタムテーマ作成・プラグイン配布**、(4) **Hook から MCP tools を直接呼び出せる新イベントタイプ `type: "mcp_tool"`**、(5) **`DISABLE_UPDATES` 環境変数で `claude update` を含む全アップデートパス遮断**、(6) WSL 側の Windows managed settings 継承、(7) Auto mode `"$defaults"` プレースホルダ、(8) `claude plugin tag` と大量の OAuth / MCP / credential 修正

## 主要な新機能

### 1. Claude Code v2.1.118（2026-04-23）✅ GA

**情報源**: [GitHub Releases v2.1.118](https://github.com/anthropics/claude-code/releases) / [Raw CHANGELOG.md](https://raw.githubusercontent.com/anthropics/claude-code/refs/heads/main/CHANGELOG.md)

#### エディタ・UI

- **Vim Visual モード追加**: `v` で visual モード（キャラクタ選択）、`V` で visual-line モード（行選択）に入る。選択範囲への operators（`d`, `y`, `c` 等）と視覚フィードバックをサポート
  - 既存の vim normal モード（`/config > Editor mode` で有効化）に続く機能拡張
- **`/cost` と `/stats` を `/usage` に統合**: 単一の `/usage` コマンド配下にタブ UI として統合。`/cost` と `/stats` はタイピングショートカットとして残存し、対応タブをオープン
- **カスタムテーマの作成・切り替え**: `/theme` から名前付きカスタムテーマを作成・切替可能。`~/.claude/themes/` 配下の JSON を直接編集も可能。**プラグインが `themes/` ディレクトリ配下にテーマを同梱して配布可能**
- **`/color` が claude.ai/code にセッションアクセントカラーを同期**: Remote Control 接続時、セッションのアクセント色が claude.ai/code に同期

#### Hook / MCP 統合

- **Hook から MCP tools を直接呼び出し可能**: hook 定義で **`type: "mcp_tool"`** を指定すると MCP サーバーのツールを直接実行可能。従来のシェル / HTTP / LLM プロンプトに加わる 4 つめの hook タイプ
- **MCP OAuth 修正多数**:
  - `/mcp` メニューで `headersHelper` 設定時に Authenticate / Re-authenticate 動作が隠れる問題を修正
  - HTTP/SSE MCP サーバーが custom header + 一過性の 401 で「needs authentication」状態に固まる問題を修正
  - OAuth トークンレスポンスで `expires_in` を返さないサーバーで毎時再認証要求される問題を修正
  - step-up authorization が `insufficient_scope` 403 で silent refresh せず re-consent を促すように
  - OAuth flow のタイムアウト/キャンセル時の unhandled promise rejection 修正
  - OAuth refresh のクロスプロセスロック競合修正
  - **macOS キーチェーンレース条件**: 並行 MCP token refresh でリフレッシュ済みトークンが上書きされ「Please run /login」プロンプトが表示される問題を修正
  - サーバーがローカル期限前にトークンを revoke した際の refresh 失敗修正
- **Credential save crash** を修正: Linux/Windows で `~/.claude/.credentials.json` が破損する問題

#### 設定・運用

- **`DISABLE_UPDATES` 環境変数**: 手動 `claude update` を含む **すべてのアップデートパスを完全にブロック**。従来の `DISABLE_AUTOUPDATER`（自動更新のみ遮断）より厳格
- **WSL の Windows 側 managed settings 継承**: Windows 上の WSL で、`wslInheritsWindowsSettings` ポリシーキーにより Windows 側の managed settings を継承可能に（企業 IT 運用で WSL と Windows の設定を統一可能）
- **Auto mode `"$defaults"` プレースホルダ**: `autoMode.allow`、`autoMode.soft_deny`、`autoMode.environment` に `"$defaults"` を含めると、ビルトインルールを**置き換えず併存**するカスタムルール追加が可能
- **Auto mode "Don't ask again" オプション**: auto mode opt-in プロンプトに追加

#### プラグイン

- **`claude plugin tag`**: プラグイン用の release git tag を作成するコマンド。バージョン検証機能付き
- **Plugin 関連修正**:
  - auto-update が別プラグインの version constraint で skip された際、`/doctor` と `/plugin` Errors タブに表示
  - `plugin install` で既インストールのプラグインに対しても依存関係再解決実行

#### セッション・再開

- **`--continue` / `--resume`**: `/add-dir` でカレントディレクトリを追加したセッションも発見可能に
- **subagents resumed via SendMessage**: 再開時に spawn 時の explicit `cwd` を復元するように修正

#### 入力・UI 修正

- Alt+K / Alt+X / Alt+^ / Alt+_ でのキーボード入力凍結を修正
- リモートセッション接続時の `~/.claude/settings.json` の `model` 設定上書き問題を修正
- `/` 始まりのファイルパスを貼り付けた際の typeahead "No commands match" エラー修正
- 「new messages」スクロール pill と `/plugin` バッジの可読性改善
- Plan 承認ダイアログが `--dangerously-skip-permissions` 時に誤って "auto mode" を提示する問題を修正
- agent-type hooks が Stop / SubagentStop 以外のイベントに対して設定されると "Messages are required for agent hooks" で失敗する問題を修正
- `prompt` hooks が agent-hook verifier subagent のツール呼び出しで再発火する問題を修正
- **`/fork` の効率化**: 親会話全体をディスク書き込みするのではなく、ポインタを書いて読み込み時に hydrate
- `/model` ピッカーが `ANTHROPIC_DEFAULT_*_MODEL_NAME`/`_DESCRIPTION` の override を honor（カスタム `ANTHROPIC_BASE_URL` gateway 使用時）
- File watcher の invalid path / fd 枯渇時の unhandled error 修正
- Remote Control セッションが CCR 初期化の transient blips でアーカイブされる問題を修正

### 2. Claude Code v2.1.117（2026-04-22）✅ GA

**情報源**: [GitHub Releases v2.1.117](https://github.com/anthropics/claude-code/releases) / [Raw CHANGELOG.md](https://raw.githubusercontent.com/anthropics/claude-code/refs/heads/main/CHANGELOG.md)

#### パフォーマンス・ネイティブ化

- **ネイティブビルドで `bfs` と `ugrep` を埋め込み**: macOS / Linux のネイティブバイナリに `bfs`（fast find 互換）と `ugrep`（高速 grep）を内蔵し、**`Glob` と `Grep` ツールを置換**して Bash 経由で直接呼び出し。**専用ツール round-trip を省略して検索を高速化**。Windows / npm インストール版は従来通り
- **MCP サーバー並列接続起動**: ローカル MCP サーバーと claude.ai MCP サーバーが両方設定されている場合の起動速度改善（並列接続がデフォルト）
- **Windows: `where.exe` 実行ファイル検索をプロセスあたりキャッシュ**: サブプロセス起動の高速化

#### モデル・努力レベル

- **Pro/Max サブスクライバーの既定 effort レベル: Opus 4.6 / Sonnet 4.6 で `medium` → `high`** に変更
  - （Opus 4.7 使用時のデフォルトは既に `xhigh`（v2.1.111〜））
  - API key / Bedrock / Vertex / Foundry / Team / Enterprise の既定は v2.1.94 で既に medium → high 済み、今回は Pro/Max subscription ユーザーに同じ変更を適用
- **Opus 4.7 の `/context` 計算修正**: Claude Code が Opus 4.7 のネイティブ 1M コンテキストウィンドウではなく **200K に対して** `/context` 割合を計算していたため、**インフレされたパーセンテージが表示され早期に autocompact が走る**問題を修正

#### Subagent / Agent

- **Forked subagents を外部ビルドで有効化**: `CLAUDE_CODE_FORK_SUBAGENT=1` 環境変数を設定すると外部ビルド（non-native）でも fork 型サブエージェントが利用可能に
- **Agent frontmatter の `mcpServers`** が `--agent` でメインスレッドエージェントセッションとして起動した場合も読み込まれるように
- **Subagents が別モデル実行時の malware 誤警告** を修正（メインエージェントと異なるモデルを使う subagent がファイル読み取りで誤って malware 警告を出す問題）

#### `/model` と `/resume`

- **`/model` 選択の永続化改善**: プロジェクトが別モデルをピンしていても、ユーザーの選択が再起動を跨いで永続。起動ヘッダーに active モデルの出自（project pin / managed-settings pin）を表示
- **`/resume` で stale な大規模セッションを summarize 可能**: 既存の `--resume` 動作と合わせ、再読み込み前に要約するかを尋ねるように

#### Plugin / Managed Settings

- **Plugin 依存の auto-resolve**:
  - `plugin install` で既インストール plugin に対して依存 missing があれば install
  - Plugin 依存エラーが "not installed" と install hint を提示
  - `claude plugin marketplace add` が設定済みマーケットプレイスから missing 依存を auto-resolve
- **`blockedMarketplaces` / `strictKnownMarketplaces` の enforcement**: plugin install / update / refresh / autoupdate のすべてでポリシー適用

#### Advisor Tool（実験）

- **"experimental" ラベル導入**: ダイアログに experimental ラベル、learn-more リンク、startup notification を追加
- **"Advisor tool result content could not be processed" エラー修正**: セッションが毎プロンプトと `/compact` で永続的にエラーになる問題を解消

#### リテンション / テレメトリ

- **`cleanupPeriodDays` 対象拡大**: `~/.claude/tasks/`、`~/.claude/shell-snapshots/`、`~/.claude/backups/` も retention sweep 対象に
- **OpenTelemetry 属性追加**:
  - `user_prompt` イベントに `command_name` と `command_source` を追加（スラッシュコマンド時）
  - `cost.usage`、`token.usage`、`api_request`、`api_error` に **`effort` 属性**を追加（モデルが effort levels サポート時）
  - カスタム / MCP コマンド名は `OTEL_LOG_TOOL_DETAILS=1` 未設定時は redact

#### 主要バグ修正

- **Plain-CLI OAuth 期限切れの反応的リフレッシュ**: access token が mid-session 期限切れになり "Please run /login" でセッションが死ぬ問題を、401 応答での reactive refresh で解決
- `WebFetch` が超大型 HTML ページで hang する問題を、markdown 変換前の truncate で解決
- プロキシが HTTP 204 No Content を返した際の `TypeError` クラッシュを明確なエラーメッセージに
- `CLAUDE_CODE_OAUTH_TOKEN` env で起動したセッションでトークンが期限切れになった際に `/login` が無効化される問題を修正
- **プロンプト入力の undo (`Ctrl+_`) 修正**: タイピング直後に動作しない、undo step ごとに状態をスキップする問題
- `NO_PROXY` が Bun 配下で remote API リクエストに respect されない問題を修正
- Slow connection で key names が coalesced text として到着した際の spurious escape/return triggers 修正
- SDK `reload_plugins` が全ユーザー MCP サーバーに serial 再接続していた問題を並列化で修正
- **Bedrock application-inference-profile 400 エラー修正**: Opus 4.7 + thinking 無効の組み合わせで失敗する問題
- MCP `elicitation/create` リクエストが print/SDK モードで mid-turn 接続完了時に auto-cancel される問題を修正
- idle re-render loop（バックグラウンドタスクがあると Linux でメモリが成長する問題）を修正
- [VSCode] 「Manage Plugins」パネルが複数の大規模マーケットプレイス設定時に壊れる問題を修正

### 4. Claude Code on Pro 試験的削除事件（2026-04-21〜22 発生、48 時間以内に撤回）📰 Layer 3

**情報源**: [The Register - Anthropic tests reaction to yanking Claude Code from Pro](https://www.theregister.com/2026/04/22/anthropic_removes_claude_code_pro/) / [The New Stack - Anthropic is thinking about removing Claude Code from its cheapest plan](https://thenewstack.io/anthropic-claude-code-limits/) / [Simon Willison - Is Claude Code going to cost $100/month? Probably not—it's all very confusing](https://simonwillison.net/2026/Apr/22/claude-code-confusion/) / [Basic Tutorials](https://basic-tutorials.com/news/anthropic-removes-claude-code-from-the-pro-tariff-on-a-trial-basis/) / [Heise Online](https://www.heise.de/en/news/Anthropic-is-temporarily-removing-Claude-Code-from-the-Pro-tariff-11268189.html) / [Where's Your Ed At - Updated article](https://www.wheresyoured.at/news-anthropic-removes-pro-cc/)

#### 経緯

- **2026-04-21**: 公式告知なしに Anthropic の pricing ページから **Pro プラン（$20/月）の Claude Code アクセス項目が削除**。Claude Help Center の記事タイトルと本文、ランディングページも「Max 5x ($100) が Claude Code 最低プラン」と読める形で更新されていた
- **2026-04-21 深夜 – 2026-04-22**: コミュニティが騒然。The Register、The New Stack、Heise、Simon Willison が報道。「Anthropic が Pro から Claude Code を削除した」という解釈が広がる
- **2026-04-22**: Anthropic の head of growth が公式コメント
  > "For clarity, we're running a small test on ~2 percent of new prosumer signups. Existing Pro and Max subscribers aren't affected."
- **2026-04-22**: pricing ページから Claude Code の Pro 項目チェックボックスが **復活**、Help Center ドキュメントのタイトル変更も **revert**

#### なぜこれが起きたか

- Claude Code の利用パターンが Opus 4.7 以降に**大きく変化**: **Pro ユーザーのセッションが Opus 4.6 比で最大 3 倍長時間化**、バックグラウンドで何時間も自動ワークフローが回るケースが頻発
- 現行の Pro プラン上限はこの利用量を想定しておらず、Anthropic がインフラコストで赤字化した可能性
- **2% だけの A/B テスト**のつもりが、**サイト全体の表示を書き換えてしまった実装ミス**で「グローバル変更」に見えてしまった

#### 含意

- **既存の Pro / Max subscribers は影響を受けない**（明言）
- 短期的な撤回であり、**Pro プランからの Claude Code 恒久削除は現時点で未定**
- ただし、Claude Code 単独プランや段階プラン（Claude Code Max など）の可能性が業界内で議論開始
- **情報ソース間の矛盾**: Anthropic の 2% テスト説明 vs 全面的な Web 更新という実態により、信頼性への疑念が Simon Willison など著名開発者から指摘

この件は Claude Code の**プロダクト / プライシング戦略**に対する注目を集め、Pro 以下プランで CLI / Managed Agents / Cowork ヘビー利用者向けの新プラン検討が進行している可能性を示唆。
