# 調査レポート: Claude Code 新機能（v2.1.195・Alibaba 蒸留疑惑・Fable5 停止継続）

**調査日:** 2026-06-27

## 概要

- **Layer 1 — CLI が v2.1.194（changelog エントリなし＝欠番）と v2.1.195（6/26 21:29）の 2 本進行。** v2.1.194 は欠番（v2.1.171・180・182・184・188・189・192 に続く **8 つ目**）、Layer 1 最新は **v2.1.195**。
- **v2.1.195 はバグ修正・信頼性改善主体**だが、新環境変数 1 件あり = **`CLAUDE_CODE_DISABLE_MOUSE_CLICKS`**（フルスクリーン時のマウスクリック/ドラッグ/ホバーを無効化、ホイールスクロールは維持）。加えて **Hook matcher のハイフン入り識別子の厳密一致化**、**音声ディクテーションの複数修正**（macOS の無音キャプチャ・スペースなし言語〔日本語/中国語/タイ語〕の自動送信）、**外部プラグインの明示同意フロー修正**、**バックグラウンドジョブ/エージェントの信頼性改善**多数。**新スラッシュコマンド・新 Hook なし**。
- **Layer 2/3 — 新 Claude Code 製品機能の公式発表なし。** Anthropic ニュースルームは **6/23 Claude Tag for Slack 以降、6/24〜27 に新規投稿なし**。
- **🆕 Layer 3 — Anthropic が Alibaba を「不正アクセス（蒸留）」で告発（6/24 Bloomberg）**: 4/22〜6/5 に約 25,000 の不正アカウント・2,880 万件超のやり取りで Claude の **ソフトウェアエンジニアリング/エージェント推論能力**を標的にしたと主張。Claude Code 隣接能力が蒸留対象だった点で関連。
- **Claude Fable 5 / Claude Mythos 5 は 6/26 時点も停止継続**（6/12 米政府輸出管理指令、復旧の公式発表なし）。バイラルな「復旧した」主張は UI バグで、Anthropic は「トラフィックはゼロ」と再度否定。

## 主要な新機能

### v2.1.195（2026-06-26 21:29） — マウスクリック無効化・音声/プラグイン/BG 信頼性

**新機能・改善**

- **`CLAUDE_CODE_DISABLE_MOUSE_CLICKS` 環境変数** ✅ GA
  - フルスクリーンモードでのマウスクリック・ドラッグ・ホバーを無効化しつつ、**ホイールスクロールは維持**する。
  - v2.1.187 でフルスクリーンのセレクトメニューにマウスクリック対応が入った後、ターミナルのテキスト選択/コピペとの干渉を嫌うユーザー向けのオプトアウト。
- **`claude agents` UI が利用可能な縦スペースを充填し、ライブセッションを常に可視化**するよう改善。
- **Remote セッション起動にプロビジョニングチェックリスト**を追加（起動進捗の可視化）。
- **Linux の音声モード**が「マイクなし」と「SoX 未インストール」を区別するよう改善。

**バグ修正**

- **Hook matcher のハイフン入り識別子（例 `code-reviewer`、`mcp__brave-search`）が部分一致してしまう問題を修正** → 厳密一致に。
  - v2.1.191 でカンマ区切り matcher が発火しない問題を修正した流れの続き。
- **音声ディクテーション（macOS）**が長時間セッションで入力デバイス変更後に無音をキャプチャする問題を修正。
- **音声ディクテーションの自動送信**が、スペースで区切らない言語（**日本語・中国語・タイ語**）で発火しない問題を修正。
- **プロジェクトの `.claude/settings.json` のみで有効化された外部プラグイン**が明示的なインストール同意を要求していなかった問題を修正（セキュリティ／意図しない有効化の防止）。
- **`/plugin` の Enable/Disable** が、プラグイン名とマーケットプレイスエントリ名が異なる場合に機能しない問題を修正。
- **バックグラウンドジョブ**が新しい Claude Code バージョンで書き込まれた際に `claude agents` から消えたりデータを失う問題を修正。
- **クラッシュしたバックグラウンドタスク**の再オープン時に、再起動せず約 5 秒間ブランク画面を表示する問題を修正。
- **バックグラウンドエージェントのデーモン**が制御ソケット失敗時に到達不能のまま稼働し続ける問題を修正。

**リリース状態**: ✅ GA（全プラットフォーム）。`CLAUDE_CODE_DISABLE_MOUSE_CLICKS` は新規環境変数。

## Layer 2/3: プラットフォーム・外部動向

### 新 Claude Code 製品機能の公式発表なし

- Anthropic ニュースルームは **6/23「Introducing Claude Tag」以降、6/24〜27 に新規投稿なし**（直近は 6/17 ソウルオフィス開設、6/12 Fable5/Mythos5 停止声明、6/11 Claude Corps）。
- Claude Tag for Slack（🧪 ベータ、Enterprise/Team、Opus 4.8）は前回 6/24 レポートで反映済み。

### 🆕 Anthropic が Alibaba を蒸留目的の不正アクセスで告発（6/24、Layer 3）

- Bloomberg 報道。Anthropic は中国の Alibaba Group が **約 25,000 の不正アカウント**を用い、**4/22〜6/5 に 2,880 万件超のやり取り**で Claude へ「不正に（illicitly）」アクセスしたと主張。
- 標的は **Claude のソフトウェアエンジニアリング能力およびエージェント推論能力**＝Claude Code が体現する中核能力。モデル蒸留（distillation）を狙ったキャンペーンとされる。
- 6/12 の Fable 5 jailbreak → 米政府輸出管理指令の流れと並び、フロンティアモデルの能力流出を巡るガバナンス・地政学的緊張の継続を示す文脈。Claude Code 自体の機能変更ではないが、能力保護を巡る背景情報として記録。
- **確度**: 報道ベース（Bloomberg）。Anthropic の公式声明位置づけは要追跡。

### Claude Fable 5 / Claude Mythos 5 — 停止継続（⛔）

- 6/12 の米商務省 BIS（Lutnick 長官署名）による輸出管理指令以降、全ユーザーで停止継続。**復旧の公式発表は 6/26 時点でなし。**
- 6/25 のバイラルな「復旧した」主張に対し、Anthropic スタッフが「**Fable/Mythos のトラフィックはゼロ**」と明言、モデルピッカーでの目撃は **UI バグ**と説明（前回 6/26 レポートで反映済み、6/27 時点も変化なし）。
- Opus 4.8 含む他全モデルは無影響。
