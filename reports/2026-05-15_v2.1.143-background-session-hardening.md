## Layer 1: 公式チェンジログ — v2.1.143（2026-05-15 22:28 UTC）

**位置づけ**: v2.1.142（5/14）の翌日。**v2.1.139 で投入されたバックグラウンドセッション機能群の実運用課題への集中ハードニング + プラグインエコシステムの依存解決強化**。新規機能カテゴリは「plugin dependency enforcement」「worktree.bgIsolation: none」「PowerShell ExecutionPolicy Bypass」の 3 つ。

### プラグイン依存関係管理（重要）

#### Plugin dependency enforcement ✅

- **新規追加**: プラグイン間の依存関係を正式にサポート
  - **`claude plugin disable <name>`**: 他の有効プラグインが target に依存している場合、**拒否してチェーンを提示**（強制無効化には `--force` 等が必要）
  - **`claude plugin enable <name>`**: target の transitive dependencies（推移的依存）を **自動的に force-enable**
- **背景**: v2.1.139 で plugin エコシステムが拡張され、Claude Code Plugin Marketplace（公式 + コミュニティ）に複雑な依存関係を持つプラグイン（共通 skill / hook / MCP server を提供する基盤プラグイン）が増加。**依存関係の手動管理が破綻寸前**だった状況を解消
- **位置付け**: npm / pip / cargo に近い依存解決を Claude Code Plugin Marketplace に持ち込み
- **情報源**: [GitHub Releases v2.1.143](https://github.com/anthropics/claude-code/releases/tag/v2.1.143) / [Claude Code Changelog](https://code.claude.com/docs/en/changelog)

#### `/plugin` marketplace に projected context cost 表示 ✅

- **新規追加**: `/plugin` marketplace browse pane に **「プラグインを有効化すると context window がどれだけ消費されるか（projected context cost）」**を表示
- **意義**: プラグイン乱用による silent context bloat を防止。**特に Claude Code が Anthropic 自社プラグイン + サードパーティを混在運用する場面で重要**
- 自社製品エコシステムの透明性向上策の 1 つ

### Worktree / バックグラウンドセッション

#### `worktree.bgIsolation: "none"` 設定 ✅

- **新規追加**: settings.json の `worktree.bgIsolation` に **`"none"`** を新設
- **動作**: バックグラウンドセッションが従来の `EnterWorktree`（`.claude/worktrees/<id>/` への自動 git worktree 作成）をスキップし、**main working copy を直接編集**
- **トレードオフ**: ファイル編集の隔離が失われる代わりに、**ビルド成果物 / 巨大 binary / IDE 統合との衝突を回避**できる
- **対象ユーザー**: ① 並行セッションを使わない（バックグラウンド 1 個運用）/ ② monorepo で worktree コピーがコスト過大 / ③ Docker / 開発サーバーが main checkout を期待する CI 環境
- **デフォルト**: `worktree` 維持（互換性のため）

#### Background sessions が wake 後に model / effort level を保持 ✅

- バックグラウンドセッションが idle 後に wake された際、**直近の `/model` / `/effort` 選択が保持**されるように修正
- 以前は wake 時に default model（Sonnet 4.6）に reset される regression を解消

#### Shift+Tab で auto mode が cycle に含まれる ✅

- attached agent session で Shift+Tab の permission mode 切替に **auto mode が含まれる**ように
- v2.1.139 で attached session の permission mode 切替が一部 mode を skip していた regression を解消

#### `claude --bg --dangerously-skip-permissions` が retire→wake で persist ✅

- バックグラウンドセッションが daemon の idle-exit → wake サイクルを跨いでも、起動時の `--dangerously-skip-permissions` フラグが保持されるように修正

### PowerShell / Windows 環境

#### PowerShell tool に `-ExecutionPolicy Bypass` デフォルト適用 ✅

- **変更**: Claude Code 内部の PowerShell tool が、PowerShell スクリプトを実行する際に **デフォルトで `-ExecutionPolicy Bypass` を渡す**
- **オプトアウト**: `CLAUDE_CODE_POWERSHELL_RESPECT_EXECUTION_POLICY=1` 環境変数で従来動作（システム ExecutionPolicy に従う）に
- **背景**: Windows のデフォルト ExecutionPolicy は **Restricted** で、Claude Code が自動生成する一時 .ps1 スクリプトが署名検証で実行拒否されるケースが多発。**ローカル開発 UX を優先しつつ、エンタープライズは環境変数で従来挙動 opt-in**
- **影響**: Bedrock / Vertex / Foundry で Windows 上の Claude Code を使う開発者の摩擦低減

#### Windows 環境での PowerShell tool がデフォルト有効化（Bedrock/Vertex/Foundry/Gateway）✅

- 従来は明示的に有効化が必要だった PowerShell tool が、**Bedrock / Vertex / Foundry / カスタム gateway 利用時に Windows でデフォルト有効**
- オプトアウト: `CLAUDE_CODE_USE_POWERSHELL_TOOL=0`

### `/goal` / Stop hooks / `/loop` 修正（重要）

#### `/goal` evaluator が background shells/subagents 走行中に発火する問題修正 ✅

- **症状**: `/goal` が completion check のために Stop-hook を起動するタイミングで、**並行する background shells や subagents がまだ走行中**にもかかわらず goal evaluator が走り、誤って「ゴール未達」と判定したり、走行中 subagent の出力を見ずに completion を確定したりする問題
- **修正**: 全 background shells / subagents の完了を待ってから goal evaluator を起動
- **影響**: `/goal` + Agent Teams + バックグラウンドセッションの組み合わせ運用が安定化

#### Stop hooks の無限ループキャップ（8 連続）✅

- **症状**: Stop hook が `continue: false`（または同等）でブロックすることで、**Claude Code が無限に再起動を試行**して終わらない
- **修正**: 連続して Stop hook がブロックした場合、**8 回**で cap し、それ以降はエラーを返してターン終了
- **環境変数**: `CLAUDE_CODE_STOP_HOOK_BLOCK_CAP=<N>` で上書き可能
- **背景**: hook ロジックのバグ / オペレータ設定ミスで無限ループに陥り、5 時間 rate-limit window を 1 ターンで使い切る事故防止

#### `/loop` の cancel 修正 ✅

- Esc / Ctrl+C で **pending な `/loop` wakeup がキャンセル可能**に（以前はキャンセルできず予定時刻まで待つしかなかった）

### 設定 / UI 修正

| 修正項目 | 詳細 |
|---------|------|
| `.credentials.json` corrupt | non-array `scopes` 値で CLI 起動 hang |
| 右クリック paste | Windows Terminal / WSL の `claude agents` で動作しない |
| `NO_COLOR`/`FORCE_COLOR` | settings に書くと Claude Code UI カラーを strip |
| Agent view PowerShell | Windows で `claude agents` が PowerShell プロセスを repeatedly spawn |
| `/bg` no prompt | プロンプトなし `/bg` が "continue" を送信していた |
| `--agent <name>` | plugin contributed agent を prefix なしで find できない |
| Agent view delete | session 削除時に transcript file が残る |
| Background agents stall detection | host sleep / macOS App Nap 後に false-positive worker-stall |
| 5xx error messages | status.claude.com を指す（gateway 設定でも） |
| Background Bash tasks | subagent が spawn した Bash task が exit 後も "Running" 表示 |
| Plugin advisories | 全 shadowed `plugin.json` キーを naming しない |
