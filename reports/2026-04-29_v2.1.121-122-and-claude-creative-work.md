## 概要

前回レポート（4/28 ニュースモード）以降、**Claude Code CLI は 4/28 1日に 2 リリース連続公開**：v2.1.121（00:31 UTC）と v2.1.122（22:05 UTC）。前回ニュースモードでは Layer 1 をスキップしたため、両リリースともに今回の調査で初めてレポート化する。

### 4/28〜4/29 の主要トピック

1. **Claude Code v2.1.121**（4/28 00:31 UTC）— **MCP `alwaysLoad` 設定**、**`claude plugin prune` コマンド**、**`/skills` type-to-filter 検索**、**PostToolUse hooks が全ツール出力を replace 可能に**（従来 MCP のみ）、フルスクリーン UX 改善多数、メモリリーク 3 件 fix、外部ビルドで `CLAUDE_CODE_FORK_SUBAGENT=1` 動作、Microsoft 365 MCP OAuth 修正など 39 件超の変更
2. **Claude Code v2.1.122**（4/28 22:05 UTC）— **`ANTHROPIC_BEDROCK_SERVICE_TIER` 環境変数**（`default`/`flex`/`priority`）、**`/resume` の PR URL 検索**（GitHub/GHE/GitLab/Bitbucket）、`/mcp` の重複コネクタ表示、OpenTelemetry に `claude_code.at_mention` イベント追加、Voice mode の Caps Lock 警告など
3. **Claude for Creative Work** ✅ — **9 つの新コネクタ**（Adobe Creative Cloud / Blender / Ableton / Splice / Affinity by Canva / Autodesk Fusion / SketchUp / Resolume Arena & Wire）を全 Claude プランで即時 GA。Anthropic は **Blender Development Fund の Corporate Patron**（年間最低 €240,000）に
4. **Claude.ai/API/Code Major Outage**（4/28 17:34–18:52 UTC、約 78 分）— Downdetector 12,000 件超、Claude.ai ログイン不可・Claude Code 起動経路エラー。同日 Sonnet 4.5（17 分）・Haiku 4.5（51 分）でも別個に elevated errors。Code Review が断続的失敗。**4 月の Anthropic 障害は 4/15・4/20・4/25 に続き 4 件目の Major クラス**
5. **Anthropic、Blender Development Fund Corporate Patron 加入**（4/28、€240,000+/年） — Blender Python API 開発支援。MCP コネクタは他 LLM からも利用可能（Blender のオープンソース原則を尊重）
6. **同日 Anthropic Sonnet 4.5 / Haiku 4.5 別障害も発生**（4/28 中の小規模インシデント 3 件）

Anthropic 公式 Newsroom で 4/28 公開のエントリは **"Claude for Creative Work" 1 件のみ**。Help Center リリースノート・Claude Platform Release Notes は新規エントリなし（4/24 の Rate Limits API GA が最新）。

## Layer 1: CLI チェンジログ

### Claude Code v2.1.121（2026-04-28 00:31 UTC）✅

公式 GitHub Releases / `code.claude.com/docs/en/changelog` に正式掲載。

#### 新機能・拡張

- **MCP `alwaysLoad` オプション** — MCP server config で `alwaysLoad: true` を指定すると、そのサーバーの**全ツールが tool-search 遅延ロードをスキップ**して常時利用可能に。tool-search で見落とされがちな MCP ツールを確実にロード可能
- **`claude plugin prune`** — orphaned auto-installed プラグイン依存を削除する新コマンド。`plugin uninstall --prune` で連鎖削除（cascading uninstall）
- **`/skills` type-to-filter 検索ボックス** — 長い skill リストでスクロールせずタイプして絞り込み可能
- **PostToolUse hooks が全ツールでツール出力を置換可能** — `hookSpecificOutput.updatedToolOutput` でツール出力を hook が置換できる機能を **MCP ツール限定から全ツールへ拡張**
- **フルスクリーンモード スクロール固定改善** — プロンプトに入力中、過去の出力を読むためスクロールしていてもボトムに自動ジャンプしなくなった
- **オーバーフローダイアログのスクロール対応** — 端末サイズを超えるダイアログを矢印キー / PgUp/PgDn / Home/End / マウスホイールでスクロール可能（フルスクリーン・非フルスクリーン両モード）
- **折り返し URL クリック対応**（フルスクリーンモード） — 行ラップした長い URL のどの行をクリックしても全体 URL が開く
- **SDK / `claude -p`: `CLAUDE_CODE_FORK_SUBAGENT=1` が non-interactive セッションで動作**
- **`--dangerously-skip-permissions` の保護対象拡大** — `.claude/skills/`、`.claude/agents/`、`.claude/commands/` への書き込みは確認プロンプトを表示
- **`/terminal-setup` が iTerm2 「Applications in terminal may access clipboard」を有効化** — `/copy` が iTerm2（tmux 内含む）で動作
- **MCP server 起動エラーの自動リトライ最大 3 回** — 一時的エラーで切断のままになる問題を解消
- **terminal タブセッションタイトルが設定 `language` 言語で生成**
- **claude.ai connectors の重複排除** — 同じ upstream URL のコネクタが重複表示される問題を修正
- **Vertex AI: X.509 証明書ベース Workload Identity Federation（mTLS ADC）対応**
- **起動高速化** — リリースノート splash の Recent Activity パネルを除去
- **LSP diagnostic summary** — クリック / Ctrl+O で展開、展開ヒント表示
- **SDK: `mcp_authenticate` が `redirectUri` をサポート** — カスタムスキーム完了 / claude.ai connectors 対応
- **OpenTelemetry 拡張** — LLM request span に `stop_reason`、`gen_ai.response.finish_reasons`、`user_system_prompt`（`OTEL_LOG_USER_PROMPTS` ゲート付き）を追加
- **\[VSCode\] 音声入力が `accessibility.voice.speechLanguage` 設定を尊重**（Claude Code 言語未設定時）
- **\[VSCode\] `/context` がネイティブトークン使用量ダイアログを開く**

#### バグ修正

- **メモリ関連 3 件**:
  - 多数画像処理時の RSS 数 GB 級増大を解消
  - `/usage` が大規模トランスクリプト履歴で最大 ~2GB リーク
  - 長時間ツールが progress イベントを発行しない場合のメモリリーク
- **Bash ツール** — Claude 起動時のディレクトリが mid-session で削除/移動された場合に永続的に使用不能になる問題を修正
- **`--resume`** — 外部ビルドで起動時クラッシュ修正、unclean shutdown による壊れた transcript line をスキップして大規模セッションでも復元可能に
- **Bedrock application inference profile ARN** で `thinking.type.enabled is not supported` エラー修正
- **Microsoft 365 MCP OAuth** — 重複/未対応 `prompt` パラメータで失敗していた問題を修正
- **scrollback 重複** — Ctrl+L / 再描画時に tmux / GNOME Terminal / Windows Terminal / Konsole で発生していた問題を修正
- **claude.ai MCP コネクタの自然消滅** — 起動時 connector-list fetch が一時的 auth error を返した際に silently 消える問題を修正
- **「Always allow」ルール** — リモートセッションでビルトインツール用の allow ルールが worker 再起動で失われる問題を修正
- **`NO_PROXY`** — `managed-settings.json` でセットされた場合にネイティブビルドの全 HTTP クライアントで尊重されるよう修正
- **managed settings approval prompt** — 承認しても session が exit する問題を修正、設定を適用して継続
- **`/usage` rate limited 表示** — stale OAuth token を自動 refresh
- **`settings.json` の legacy 列挙値** が settings 全体を invalidate していた問題を修正
- **`/usage` ダイアログ** — no-flicker mode オフ時のクリッピング修正
- **`/focus` Unknown command** — フルスクリーンレンダラー無効時に「Unknown command」ではなく有効化方法を表示
- **embedded grep/find/rg shell wrapper** — running binary が mid-session で削除された場合にインストール済みツールへフォールバック
- **`find` の peak file descriptor 使用量削減**（大規模ディレクトリツリー）

### Claude Code v2.1.122（2026-04-28 22:05 UTC）✅

#### 新機能・拡張

- **`ANTHROPIC_BEDROCK_SERVICE_TIER` 環境変数** — Bedrock サービス層を `default`/`flex`/`priority` から選択。`X-Amzn-Bedrock-Service-Tier` ヘッダーとして送信。Bedrock の優先度別レイテンシ制御（`priority` で低レイテンシ保証、`flex` でコスト最適化）を Claude Code 経由で利用可能に
- **`/resume` 検索に PR URL 貼り付けサポート** — GitHub / GitHub Enterprise / GitLab / Bitbucket の PR URL を貼り付けると、その PR を作成したセッションを発見。コードレビュー時の文脈再開に便利
- **`/mcp` で claude.ai connectors の重複表示** — 同一 URL の手動追加サーバーで隠されている claude.ai コネクタも表示し、重複削除のヒントを提供
- **OpenTelemetry 改善**:
  - `api_request` / `api_error` ログイベントの数値属性が **string ではなく number として送出**（メトリック集計の正確性向上）
  - 新しい `claude_code.at_mention` ログイベント — `@`-mention 解決を観測可能に

#### バグ修正

- **`/branch` のフォーク失敗** — "tool_use ids were found without tool_result blocks" エラーで失敗していた問題を修正
- **Vertex AI / Bedrock の session-title 生成** — `invalid_request_error` を返していた問題を修正
- **画像リサイズ修正** — newer モデル（Opus 4.7 等の長辺 2,576px 対応モデル）で誤って 2,576px ではなく 2,000px に縮小されていた問題を修正（→ 正しく **2,000px max** に。注: changelog の表現は「resized to 2576px instead of correct 2000px maximum」で、**従来 newer モデルでは 2,576px に拡大されていたが、本来は 2,000px max が正しい挙動**だった旨を示す）
- **リモコンセッション** — idle status redraw が毎秒 2 回発生していた無駄を解消
- **Voice mode** — Caps Lock にバインドされたキーバインドにエラー表示を追加（Caps Lock の特殊性により正しく動作しないため）

### Claude for Creative Work（9 つの新コネクタ） ✅

- **公開**: 2026-04-28（[Anthropic 公式 Newsroom](https://www.anthropic.com/news/claude-for-creative-work)）
- **リリース状態**: ✅ GA（**全 Claude プランで即時利用可能**）
- **概要**: Claude を主要なクリエイティブアプリケーション内で動作させる **9 つのコネクタ**を一斉公開。プロのデザイナー・3D アーティスト・音楽プロデューサー・建築家・VJ など多様なクリエイティブワークフローを Claude 経由で操作可能に

#### 9 つのコネクタ

| コネクタ | 機能概要 |
|---------|---------|
| **Adobe for Creativity** | Photoshop / Premiere / Express など Creative Cloud 50+ ツールへのアクセス、画像・動画・デザインの編集 |
| **Affinity by Canva** | バッチ画像調整、レイヤーリネーム、ファイルエクスポート等の繰り返し作業を自動化 |
| **Autodesk Fusion** | 3D モデルを自然言語の対話で作成・修正（サブスクリプション要） |
| **Blender** | Python API への自然言語インタフェース、シーン解析、カスタムスクリプト生成、公式ドキュメントへの grounding |
| **Ableton** | Live / Push の公式ドキュメントに grounded された応答 |
| **Splice** | Claude 会話内から royalty-free サンプルカタログを検索 |
| **SketchUp** | 自然言語の空間説明から 3D モデリングのスタート地点を生成、アプリ内で精緻化 |
| **Resolume Arena & Resolume Wire** | VJ / ライブビジュアルアーティスト向け、ライブパフォーマンス中のリアルタイム制御 |

#### Blender Foundation Corporate Patron 加入

- **Blender Development Fund の Corporate Patron 階層**（年間最低 **€240,000**）に Anthropic が加入
- 用途: Blender Python API のメンテナンス・機能拡張支援（コネクタ実装の基盤）
- **Blender 経由のコネクタ実装は MCP ベース**で、他 LLM からも利用可能 — Blender のオープンソース・他 LLM 非排除原則を尊重
- Blender CEO Francesco Siddi: "in these uncertain and divisive times, the support enables the Blender team to keep pursuing projects independently, and to focus on building tools for artists and creators"
- **Claude Code への影響**: クリエイティブ系コーディングワークフロー（Blender Python スクリプト生成、Photoshop UXP プラグイン、SketchUp Ruby スクリプト等）で Claude Code から該当ツールを呼び出すパスが整備されたことを意味する。MCP サーバー実装は他 LLM CLI（OpenClaw 等）でも転用可能

### Claude.ai/API/Code Major Outage（2026-04-28 17:34–18:52 UTC）

- **発生**: 2026-04-28 17:34 UTC（10:34 PT / 13:34 ET）

- **影響範囲**: Claude.ai / Anthropic API / **Claude Code（ログイン経路）**
- **症状**:
  - Claude.ai ログイン時の認証エラー
  - API 呼び出しの elevated error rate

- **規模**: Downdetector に **12,000 件超**のユーザー報告

  - **Claude Haiku 4.5 elevated errors**（11:53–12:44 UTC、51 分、Partial Outage）
  - **Claude Code Code Review intermittently failing**（解決時刻 06:00 UTC、Partial Outage） — Code Review セッションが断続的に開始しない
