# Claude Code 調査レポート: v2.1.116 & Opus 4.7 on Amazon Bedrock

## 概要

- **CLIリリース**: v2.1.116（2026-04-20 22:18 UTC）が1本公開。v2.1.115は欠番（v2.1.114 → v2.1.116）
- **プラットフォーム**: **Claude Opus 4.7 が Amazon Bedrock で GA**（2026-04-20）。4リージョン同時展開、1Mコンテキスト・高解像度画像・adaptive thinking を搭載。ただし Claude Code 側の Bedrock エイリアス解決は依然として Opus 4.6 を指すため、4.7 利用には明示指定が必要（既知の不具合 #49238 も存在）
- **サイバーセキュリティ報道**: Mythos Preview / Project Glasswing 関連で Foreign Policy、Barracuda Networks、Cloud Security Alliance などが新たな分析・警告を公開。前日 Axios スクープの続報が広がる局面
- **イベント**: Anthropic が 2026-04-21 に「Claude for Legal teams」ウェビナーを開催

v2.1.116 はメジャー機能追加というより**パフォーマンス最適化とバグ修正が主体の品質向上リリース**。`/resume` の大規模セッション処理が最大67%高速化、MCP 起動の遅延ローディング、サンドボックスの危険パス保護強化など、既存機能の堅牢性を底上げする内容。

## 主要な新機能・変更

### 1. Claude Code v2.1.116（2026-04-20） ✅ GA

**リリース状態**: ✅ GA（全プラン）
**リリース日時**: 2026-04-20 22:18 UTC
**情報源**: [GitHub Release v2.1.116](https://github.com/anthropics/claude-code/releases/tag/v2.1.116) / [公式Changelog](https://code.claude.com/docs/en/changelog)

#### パフォーマンス最適化

- **`/resume` の大規模セッション処理が最大67%高速化**（40MB+ セッション向け）。dead-fork エントリが多いセッションも効率的に処理
- **MCP 起動の高速化**: 複数の stdio サーバーを設定している環境で、`resources/templates/list` を初回 `@`-mention まで遅延実行するよう変更
- **フルスクリーンスクロール改善**: VS Code、Cursor、Windsurf ターミナルでスムーズなスクロールを実現。`/terminal-setup` がエディタのスクロール感度を自動設定

#### UI / UX

- **Thinking spinner のインライン進捗表示**: 「still thinking」「thinking more」「almost done thinking」の進捗メッセージがスピナー内に統合。別行のヒントは廃止
- **`/config` 検索が値もマッチ**: 「vim」で検索すると Editor mode 設定が見つかる
- **`/doctor` が応答中でも開けるように**: 現ターンの完了を待つ必要がなくなった
- **スラッシュコマンドメニュー**: フィルタ結果0件時に「No commands match」を表示（従来はメニューが消失）

#### 機能追加

- **`/reload-plugins` と バックグラウンド自動更新**: 既に追加済みのマーケットプレイスから**プラグインの欠落依存を自動インストール**
- **Bash ツール: GitHub API レート制限ヒント**: `gh` コマンドがレート制限に達した際にヒントを表示し、エージェントがリトライ連発ではなくバックオフできるように
- **Usage tab**: Settings の利用状況タブが 5時間・週次の使用量を即座に表示、使用状況エンドポイントのレート制限時も失敗しないように
- **Agent frontmatter `hooks:`** が `--agent` でメインスレッドエージェントとして実行される際にも発火

#### セキュリティ

- **sandbox auto-allow の危険パス保護強化**: `rm`/`rmdir` が `/`、`$HOME`、その他クリティカルなシステムディレクトリを対象とする場合、**サンドボックス auto-allow が dangerous-path safety check をバイパスしないように修正**

#### 主要バグ修正

- Devanagari や他の Indic スクリプトのターミナル UI カラムアライメント崩れを修正
- Kitty キーボードプロトコル使用時（iTerm2、Ghostty、kitty、WezTerm、Windows Terminal）の **Ctrl+- で Undo がトリガーされない問題**を修正
- Kitty キーボードプロトコル使用時（Warp fullscreen、kitty、Ghostty、WezTerm）の **Cmd+Left/Right で行頭・行末にジャンプしない問題**を修正
- **Ctrl+Z で Claude Code がハングする問題**（`npx`、`bun run` 等のラッパープロセス経由起動時）を修正
- インラインモードでのスクロールバック複製（ターミナルリサイズ・大規模出力バーストで過去会話履歴が繰り返される）を修正
- 短い高さのターミナルでモーダル検索ダイアログが画面からはみ出し、検索ボックス・キーボードヒントが隠れる問題を修正
- VS Code 統合ターミナルでのスクロール中のブランクセル散在・コンポーザー chrome 消失を修正
- **API 400 エラー**（cache control TTL ordering 起因、リクエストセットアップ中に並列リクエストが完了した場合の間欠的発生）を修正
- `/branch` が 50MB 超のトランスクリプトを持つ会話を拒否する問題を修正
- `/resume` が大規模セッションファイルでエラーを報告せず無言で空の会話を表示する問題を修正
- `/plugin` Installed タブで「Needs attention」や「Favorites」に表示される項目が重複する問題を修正
- セッション中に worktree へ入った後、`/update` と `/tui` が動作しない問題を修正

### 2. Claude Opus 4.7 on Amazon Bedrock（2026-04-20） ✅ GA

**リリース状態**: ✅ GA（Amazon Bedrock）
**発効日**: 2026-04-20（発表と同時に利用可能）
**情報源**: [AWS Weekly Roundup (2026-04-20)](https://aws.amazon.com/blogs/aws/aws-weekly-roundup-claude-opus-4-7-in-amazon-bedrock-aws-interconnect-ga-and-more-april-20-2026/) / [AWS What's New](https://aws.amazon.com/about-aws/whats-new/2026/04/claude-opus-4.7-amazon-bedrock/) / [Bedrock Model Card](https://docs.aws.amazon.com/bedrock/latest/userguide/model-card-anthropic-claude-opus-4-7.html)

#### 展開内容

- **対応リージョン**: US East (N. Virginia) / Asia Pacific (Tokyo) / Europe (Ireland) / Europe (Stockholm) の4リージョンで launch 時点から利用可能
- **レート制限**: **10,000 requests per minute / アカウント / リージョン**
- **コンテキスト**: 1M トークン（フル）
- **画像**: 高解像度画像サポート（チャート、密度の高い文書、画面 UI でのアクセサリー向上）
- **推論エンジン**: Bedrock の**次世代推論エンジン**上で動作。dynamic capacity allocation、adaptive thinking（リクエストの複雑さに応じた thinking token 予算割当）、full 1M token context window に対応

#### パフォーマンス（Bedrock 上）

- SWE-bench Pro: **64.3%**
- SWE-bench Verified: **87.6%**

#### Claude Code ユーザーへの影響（重要）

- **Claude Code on Bedrock のエイリアス解決は依然として `opus` → Opus 4.6、`sonnet` → Sonnet 4.5**（変更なし）
- Opus 4.7 を Claude Code で利用するには以下のいずれかが必要:
  - full model name を明示指定
  - `ANTHROPIC_DEFAULT_OPUS_MODEL` または `ANTHROPIC_DEFAULT_SONNET_MODEL` 環境変数を設定
- **既知の不具合**: GitHub issue [#49238「[BUG] Opus 4.7 doesn't work with bedrock」](https://github.com/anthropics/claude-code/issues/49238) が存在。Bedrock 経由での Opus 4.7 呼び出しが失敗するケースが報告されている（2026-04-20 時点で継続調査中）
- **2026-04-23 の API デフォルトモデル切替**（Enterprise pay-as-you-go および Anthropic API 第一者向け）は**Anthropic API 側のみ**が対象。Bedrock/Vertex/Foundry のエイリアス解決は影響外

#### ドキュメント補足

- Claude Code の [Model configuration ドキュメント](https://code.claude.com/docs/en/model-config) で、Bedrock/Vertex ユーザーは `ANTHROPIC_DEFAULT_OPUS_MODEL` で Opus 4.7 を明示的にピンニングする必要があると案内

## バグ修正・パフォーマンス改善（v2.1.116 再掲の補足）

v2.1.116 は特にターミナル互換性周辺の **Kitty keyboard protocol**（iTerm2、Ghostty、kitty、WezTerm、Windows Terminal、Warp fullscreen）向け修正が多く、最新ターミナルエミュレータでの操作性が大幅改善。また `Ctrl+Z` ハング（`npx`/`bun run` ラッパー経由起動時）、VS Code スクロール中の表示崩れ、scrollback 複製など、エンタープライズ環境で遭遇しやすい問題が一掃された。

API 400 エラー（cache control TTL ordering 起因）は Managed Agents や並列リクエストヘビーなユースケースで間欠発生していた間欠不具合であり、本番ワークロードを運用している環境は v2.1.116 への更新が推奨される。
