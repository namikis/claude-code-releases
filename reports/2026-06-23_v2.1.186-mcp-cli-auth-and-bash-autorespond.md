## 主要な変更

### 1. CLI v2.1.186 — MCP CLI 認証・Bash 自動応答ほか ✅ GA

- **リリース状態**: ✅ GA（npm 配布、6/22 20:37）
- **欠番**: v2.1.184 は欠番のまま（v2.1.171 / v2.1.180 / v2.1.182 に続く 4 つ目）。v2.1.185→186 が連続リリース。

#### 新コマンド・新設定

- **`claude mcp login <name>` / `claude mcp logout <name>`**（新 CLI サブコマンド）
  - インタラクティブな `/mcp` メニューを開かずに、**ターミナルから直接 MCP サーバーを認証 / ログアウト**できる
  - **`--no-browser`** で stdin リダイレクトをサポートし、**SSH 越し**でも認証フローを完了可能
- **`!` Bash コマンドが Claude の自動応答をトリガー**（挙動変更）
  - これまで `!` で実行した Bash コマンドの出力はコンテキストに追加されるだけだったが、**v2.1.186 から Claude が自動的にその出力に応答する**
  - 従来の「コンテキスト追加のみ」に戻すには `settings.json` で **`"respondToBashCommands": false`** を設定
- **`teammateMode: "iterm2"` 設定**（新設定）
  - エージェントチームの teammate を **iTerm2** で起動。auto mode が `it2` CLI を見つけられない場合は警告を表示
- **`/workflows` エージェント詳細ビューにステータスフィルタ追加**（`f` キーで切替）
- **`/plugin` Installed タブに「Skills」セクション追加**
- **`/login` に「Claude Platform on AWS - refresh credentials」オプション追加**（`awsAuthRefresh` 構成時）
- **`/review <pr>` が `/code-review medium` と同じレビューエンジンを使用**するよう統一
- **`CLAUDE_CODE_MAX_RETRIES` の上限を 15 に変更**。無人セッションでは代わりに **`CLAUDE_CODE_RETRY_WATCHDOG`** を使用するよう案内

#### 権限・サブエージェント挙動の改善

- **バックグラウンドサブエージェントの権限プロンプトをメインセッションに表示**（従来は自動拒否）。どのエージェントが要求しているか表示し、Esc でそのツールだけを拒否
- **`Agent(type)` deny ルール / `Agent(x,y)` allowed-types 制限が、名前付きサブエージェント生成に対して機能していなかった問題を修正**
- メイン turn 終了後もバックグラウンドエージェントが走っている間、**Esc / Ctrl+C が反応しない問題を修正**

#### 主なバグ修正（約 25 件）

- **マシンのスリープ復帰後にストリーミングリクエストが "Content block not found" / JSON パースエラーで失敗する問題を修正**
- サブエージェント transcript のスクロール位置がメイン transcript に滲み出る問題を修正
- バックグラウンドセッションの recap が重複する問題を修正（エージェント自身の end-of-turn サマリーを recap 行として表示）
- `~~strikethrough~~` がリテラルのチルダ表示になり取り消し線がレンダリングされない問題を修正
- 使用量ベースの Enterprise / Team サブスクライバーでセッションコストが表示されない問題を修正
- エージェントチーム: tmux / pane バックエンド経由で生成された teammate がリーダーの `--effort` レベルを継承するよう修正
- Workflow `agent({schema})` サブエージェントがスキーマ検証失敗を繰り返して無限ループする問題を修正（5 回で中断）
- `--tools` がコールドな初回起動時に feature-gate されたツールをすり抜ける問題を修正
- メモリ: `MEMORY.md` インデックスがサイズ上限に近づくとコンパクトを促すリマインド追加
- skill frontmatter: `display-name` / `default-enabled` / `fallback` / `metadata.*` が kebab-case / snake_case / camelCase を許容
- 不正な `SKILL.md` YAML frontmatter はサイレント失敗せず空メタデータで本体をロード
- `claude mcp get` / `claude mcp remove` がタイポ時に最も近いサーバー名を提案
- ほか permission prompt の番号ずれ、ダークテーマフラッシュ、`/plugin` スクロール表示など多数
