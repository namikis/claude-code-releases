# 調査レポート: Claude Code v2.1.169（`--safe-mode` / `/cd` / `disableBundledSkills`）+ Project Glasswing 150 組織拡大・Mythos クリティカルインフラ展開

**調査日:** 2026-06-09

## 概要

前回調査（6/8、v2.1.151〜v2.1.168 + Opus 4.8 デフォルト化）以降の差分を全 Layer で調査した。新規 CLI リリースは **v2.1.169（6/8）** の 1 本で、トラブルシュート用の **`--safe-mode` / `CLAUDE_CODE_SAFE_MODE`**（全カスタマイズ無効化）、**`/cd`**（プロンプトキャッシュを壊さずに作業ディレクトリ変更）、**`disableBundledSkills` / `CLAUDE_CODE_DISABLE_BUNDLED_SKILLS`**（同梱スキル・ワークフロー・組み込みスラッシュコマンドの非表示）を追加し、JetBrains 2026.1+ ターミナルのちらつきや Kitty キーボードプロトコル系の Shift+非 ASCII 文字落ちなど多数を修正した。

プラットフォーム面では、**Project Glasswing が約 150 組織・15+ か国へ大規模拡大**（6/2 公式発表）。電力・水道・医療・通信・ハードウェアのクリティカルインフラ事業者を優先し、NATO / ENISA / Okta / Samsung / SK Hynix / SK Telecom などが含まれる。クリティカルインフラ向けには引き続き **Claude Mythos Preview**（一般提供せず）が、一般向けには **Claude Opus 4.8** ベースの **Claude Security** が使われる。6/1 には EU 諸機関への Mythos アクセス提供（CNBC 報道）も伝えられた。Mythos クラスモデルの一般提供時期は依然として未確定（📢 据え置き）。

- **対象期間**: 2026-06-09（前回 6/8 以降の差分）
- **新規 CLI バージョン**: 1 件（v2.1.169）
- **主要トピック**: `--safe-mode` / `/cd` / `disableBundledSkills`、Project Glasswing 150 組織拡大、Mythos のクリティカルインフラ／EU 展開

## 主要な新機能

### 1. Claude Code v2.1.169（✅ GA、2026-06-08）

#### 新フラグ・コマンド・設定

- **`--safe-mode` フラグ / `CLAUDE_CODE_SAFE_MODE` 環境変数**: CLAUDE.md・プラグイン・スキル・フック・MCP サーバーといった**全カスタマイズを無効化**して起動。問題切り分け（トラブルシュート）用。
- **`/cd` コマンド**: セッション途中で**プロンプトキャッシュを壊さずに作業ディレクトリを変更**。長時間セッションでディレクトリを移動してもキャッシュが効いたまま継続できる。
- **`disableBundledSkills` 設定 / `CLAUDE_CODE_DISABLE_BUNDLED_SKILLS` 環境変数**: 同梱スキル・ワークフロー・組み込みスラッシュコマンドを非表示にする。

#### バグ修正・改善（主なもの）

- 長い入力行での上下矢印ナビゲーション修正
- エンタープライズ MCP ポリシーが再接続時・IDE config で適用されない不具合を修正
- Windows で `claude -p` がスラッシュコマンド/スキルのスキャン中にハングする不具合を修正
- Remote Control の再接続不具合、Windows の Git Credential Manager ポップアップ、`claude agents --json` 出力を修正
- バックグラウンドエージェントがプロジェクトレベル設定を無視する不具合を修正
- Windows での MCPB プラグインキャッシュ無効化、プラグインの `.in_use` PID ロックファイル蓄積を修正
- `TaskCreate` の信頼性向上（自動修復）、ストリーミング/アニメーション時の CPU 使用率低減
- Vertex/Foundry で 5 分アイドルタイムアウトを復活（`API_FORCE_IDLE_TIMEOUT=0` でオプトアウト）
- スキルタグの色コントラスト改善
- バックグラウンドセッションが `--ide` / `--chrome` / `--bare` / `--remote-control` フラグを保持するように
- CLAUDE.md の警告しきい値がモデルのコンテキストウィンドウに応じてスケールするように

- **情報源**: [Claude Code changelog](https://code.claude.com/docs/en/changelog) / [GitHub Releases](https://github.com/anthropics/claude-code/releases)

### 2. Project Glasswing — 約 150 組織・15+ か国へ拡大（✅ 発表、2026-06-02）

- **リリース状態**: ✅ 公式発表（プログラム拡大）。Mythos Preview 自体は引き続き一般提供せず（アクセス制限フェーズ）。
- **規模**: 初期コホート約 50 パートナーに加え、**約 150 の新規組織**が参加。**15+ か国**（オーストラリア、カナダ、フランス、ドイツ、イタリア、スイス、オランダ、スペイン、ベルギー、スウェーデン、インド、日本、ニュージーランド、韓国）。
- **対象組織例**: NATO、ENISA、Okta、Samsung、SK Hynix、SK Telecom。これらは他組織・政府が依存する基盤コードベースを保有し、「侵害されれば 1 億人超に影響しうる」とされる。
- **優先セクター**: 初期コホートで手薄だった**電力・水道・医療・通信・ハードウェア**のクリティカルインフラ。
- **モデル構成**:
  - **Claude Mythos Preview**: パートナー向けの専門サイバーセキュリティモデル。「数週間で数千件のゼロデイ脆弱性を特定可能」。パッチ作成・ペネトレーションテスト・脅威検出に利用。**一般提供はしない**方針を継続。
  - **Claude Security**: 最新の公開フロンティアモデル **Claude Opus 4.8** でコードベースをスキャンしパッチを提案する製品（公開ベータ）。
- **実績（初期コホート）**: Claude Mythos Preview で **10,000 件超の高/重大深刻度の脆弱性**を発見。
- **情報源**: [Anthropic - Expanding Project Glasswing](https://www.anthropic.com/news/expanding-project-glasswing) / [TechCrunch](https://techcrunch.com/2026/06/02/anthropic-scales-claude-mythos-to-critical-infrastructure-in-15-countries/) / [Project Glasswing](https://www.anthropic.com/glasswing)

### 3. Mythos の EU 機関アクセス提供（📢 報道、2026-06-01）

- **概要**: CNBC が、Anthropic が EU 諸機関に対し先進モデル Mythos へのアクセスを提供すると報道。Glasswing 拡大（ENISA を含む欧州機関の参加）と整合する動き。
- **リリース状態**: 📢 報道ベース。Mythos の一般提供（GA）には依然つながらず、アクセスは限定パートナー向け。
- **情報源**: [CNBC](https://www.cnbc.com/2026/06/01/anthropic-eu-ai-mythos-access-advanced-model.html)（※ HTTP 403 で本文未取得、検索スニペットベース）

## バグ修正・パフォーマンス改善（v2.1.169 ハイライト）

- JetBrains IDE（IntelliJ / PyCharm / WebStorm 等）2026.1+ ターミナルのちらつきを synchronized output 有効化で修正
- Kitty キーボードプロトコル系ターミナル（WezTerm / Ghostty / kitty）での Shift+非 ASCII 文字（例: Shift+ä → Ä）の取りこぼしを修正
- Bedrock/Vertex ユーザーが `/model` ピッカーで「Opus (1M context)」を選べない不具合、リモートセッションログインの "Can't access this organization" を修正
- ページネーション付き `tools/list` を返す MCP サーバーが 1 ページ目しか返さない不具合を修正
- Windows の PowerShell コマンド検証、macOS の孤立バックグラウンドプロセス、voice mode 認証、管理設定の不具合を修正
