# 調査レポート: Claude Code v2.1.187（サンドボックス資格情報保護・組織モデル制限のUI露出）

**調査日:** 2026-06-24

## 概要

- 最新 CLI は **v2.1.187**（6/23 リリース、前日 6/22 の v2.1.186 に続く連日リリース）。**バグ修正主体だが 2 件のセキュリティ/ガバナンス系の追加**を含む。
- 目玉は ① **`sandbox.credentials` 設定**（サンドボックス実行コマンドから資格情報ファイル・シークレット環境変数の読み取りをブロック）と ② **組織のモデル制限を `/model` ピッカー・`--model`・`ANTHROPIC_MODEL` に露出**（`availableModels`/`enforceAvailableModels` ガバナンスの UI 反映）。
- そのほか **フルスクリーンのセレクトメニューでマウスクリック対応**、**新環境変数 `CLAUDE_CODE_MCP_TOOL_IDLE_TIMEOUT`**（リモート MCP ツール呼び出しの 5 分無応答アボート）、`--resume`/`--json-schema` の修正、`/install-github-app` のワークフロー設定を任意化、CJK ペースト文字化け修正など約 20 件。**新スラッシュコマンド・新 Hook なし**。
- **v2.1.184 は欠番継続**（v2.1.171 / 180 / 182 に続く 4 つ目）。
- **Claude Fable 5 / Mythos 5 は 6/24 時点も停止継続**。6/22 でサブスク同梱無料が終了し 6/23 から usage credits 移行の建付けだが、モデル自体が停止中で実質保留。**予測市場（Polymarket）の復旧見通しが「近日」から「年内後ろ倒し」へ大幅後退**（6/18-19 の Ciauri「数日以内」発言と矛盾）。
- **6/23 に大規模障害**（6/22 から 14 時間未満で再発、Claude.ai/Console/API/Code/Cowork に影響、約 2.5 時間で復旧）。

## 主要な新機能（v2.1.187 / リリース状態：✅ GA）

### 1. `sandbox.credentials` 設定 — サンドボックスからの資格情報読み取りブロック
- **できること**: サンドボックス実行されたコマンドが、資格情報ファイルやシークレット環境変数を読み取れないようにブロックする。
- **意義**: サンドボックス（Linux/macOS の Bash 隔離実行）に対する追加のデータ流出防止策。プロンプトインジェクションや悪意あるツール呼び出しによる秘密情報の吸い出しリスクを低減する、防御層の強化。
- **リリース状態**: ✅ GA（v2.1.187）

### 2. 組織のモデル制限をモデル選択 UI/CLI に露出
- **できること**: 組織側で設定したモデル制限（`availableModels` / `enforceAvailableModels`）を、`/model` ピッカー・`--model`・`/model`・`ANTHROPIC_MODEL` に反映。制限対象モデルを選ぶと「restricted by your organization's settings（組織の設定により制限されています）」というメッセージを表示する。
- **意義**: これまで（v2.1.175 `enforceAvailableModels`、v2.1.176 alias すり抜け修正、v2.1.178 等）でバックエンドのモデルガバナンスを固めてきたが、本リリースで**ユーザーが選択を試みる UI/CLI の入口に明示的なフィードバックを露出**。管理者のモデル許可リストがエンドユーザーに可視化され、誤選択時の理由が分かるようになった。
- **リリース状態**: ✅ GA（v2.1.187）

### 3. フルスクリーンのセレクトメニューでマウスクリック対応
- **できること**: フルスクリーンモードのセレクトメニュー（権限プロンプト、`/model`、`/config` など）でマウスクリックによる選択が可能に。
- **リリース状態**: ✅ GA（v2.1.187）

### 4. `CLAUDE_CODE_MCP_TOOL_IDLE_TIMEOUT`（新環境変数）
- **できること**: リモート MCP ツール呼び出しが 5 分間無応答の場合に、無限ブロックせずエラーでアボートする。タイムアウト値は `CLAUDE_CODE_MCP_TOOL_IDLE_TIMEOUT` でオーバーライド可能。
- **リリース状態**: ✅ GA（v2.1.187）

## バグ修正・改善（v2.1.187、抜粋）

- **`--resume` の "No conversation found" 修正**: 元の `-p` 実行がモデルターンを生成しなかった場合に resume が失敗する問題を修正。
- **`--json-schema` / workflow `agent({schema})` 構造化出力の修正**: 成功後にモデルが `StructuredOutput` を無限に再呼び出ししないように。後続ターンも確実に構造化出力を返すように。
- **Claude Code Remote セッション起動の ~2.7s 遅延修正**（エージェントプロキシ CA のシステム信頼インストール追加が原因だったもの）。
- **CJK ペースト文字化け修正**: ペーストをバイト単位の拡張キーイベントとして配信する端末で、韓国語/CJK テキストが mojibake になる問題。
- **`/update` over Remote Control のハング修正**（起動時 trust ダイアログが出るケース）。
- **agents view のバックグラウンドジョブが "working" 固着する問題修正**（ターン終了時に構造化出力を生成しなかった場合）。
- **チャネル接続のドロップ修正**（agents view への往復、`/bg`・`/tui`・`/update` 後）。
- **エージェント停止通知の帰属修正**（誰が停止したかを正しく表示、文言を "finished"/"stopped" に改善）。
- **サブエージェント深度トラッキング修正**: resume したサブエージェントは元の spawn 深度を復元、fork したサブエージェントは深度上限にカウント。
- **リーク worktree 登録の自動クリーンアップ**: kill されたエージェントのロック済み `.git/worktrees/` エントリを自動掃除。
- **Ghostty (macOS) フルスクリーンで Cmd+click が URL を開かない問題修正**。
- **`claude --help` に `--bg`/`--background` フラグが未掲載だった問題修正**。
- **`/share` アップロード中に Esc/Ctrl-C/Ctrl-D が効かない問題修正**。
- **`/install-github-app` の改善**: GitHub Actions ワークフロー設定を任意化（App だけ入れてワークフロー/シークレット手順をスキップ可能）。
- **`/btw` の改善**: ←/→ で過去回答をステップ移動。
- **`/plugin` の改善**: 最近使っていないプラグインを提示してクリーンアップを促す。
- **[VSCode] 大きいセッション resume 時に拡張が応答不能になる問題修正**。

> **新スラッシュコマンド・新 Hook イベントなし。** 新環境変数は `CLAUDE_CODE_MCP_TOOL_IDLE_TIMEOUT` の 1 件、新設定は `sandbox.credentials` の 1 件。

## Layer 2/3: プラットフォーム動向

### Claude Fable 5 / Mythos 5 — 6/24 時点も停止継続、復旧見通しが後退
- **状況**: 6/12 の米政府輸出管理指令により全ユーザー停止のまま。6/22 でサブスク同梱無料が終了し 6/23 に Pro/Max/Team/seat ベース Enterprise から Fable 5 が外れ usage credits 移行の建付けだが、**モデル自体が停止中のため実質保留**。
- **復旧見通しの変化（❓ 噂・市場予測）**: 予測市場 Polymarket の「US 顧客向け復旧日」が**近日から大幅に後ろ倒し**。6/22 時点では 7/1 が最有力（約 50%）だったが、6/24 時点では **12/31 = 87% / 8/31 = 75% / 7/31 = 68% / 7/1 = 20% / 6/26 = 11%** と近日復旧の織り込みが崩壊。中国関連のアクセス試行報道・Amazon セキュリティチームの jailbreak 脆弱性発見・商務省の輸出管理厳格姿勢が下押し要因とされる。
- **公式発言との矛盾**: 6/18-19 に国際担当 MD **Chris Ciauri** が「数日以内に復旧と非常に確信」とオン・レコードで述べていたが、6/24 時点で復旧の公式発表はなく、市場予測は逆方向（年内後ろ倒し）に動いている。Anthropic は政府の深刻度評価に不同意・ホワイトハウスとの高官級協議継続中とされる。
- **他モデルへの影響**: Opus 4.8 含む他の全モデルは引き続き無影響。

### 6/23 大規模障害（6/22 から連日）
- **タイムライン**: 6/23 10:02 AM ET 頃に開始、10:13 AM ET に米国でピーク（約 7,119 件報告）、12:44 PM ET に解決。Anthropic は 14:19 UTC に複数モデルの elevated error rate を調査開始 → 14:25 UTC に identified、fix 適用。
- **影響範囲**: Claude.ai / Claude Console / Claude API / **Claude Code** / Claude Cowork。
- **文脈**: 6/22 の障害から 14 時間未満での再発で、6 月の不安定さのパターンの一部。なお一部メディア（Cybernews）は障害後に「顧客データ漏洩の主張を Anthropic が調査中」と報じているが、**公式確認は取れておらず ❓ 未確認**（誇張・誤報の可能性に留意）。
