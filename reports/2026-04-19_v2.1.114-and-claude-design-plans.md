# Claude Code 調査レポート: v2.1.114・Claude Design 利用条件確定・Haiku 3 リタイア

**調査日:** 2026-04-19
**対象期間:** 2026-04-18〜2026-04-19（前回レポート 2026-04-18 v2.1.113 以降の差分）
**対象バージョン:** v2.1.114

---

## 概要

前回レポート（2026-04-18、v2.1.113対象）以降、CLIで**1リリース（v2.1.114）**が4月18日に出荷された。v2.1.114はバグ修正のみの極小リリースで、**Agent Teams のチームメイトがツール許可をリクエストした際に permission dialog がクラッシュする問題**を修正した1項目のみ。ただしこの修正は `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` 実験フラグ下の **Agent Teams 機能が依然としてアクティブに手当てされていること**を示す点で、機能継続シグナルとして意味がある。

プラットフォーム面では、**2026年4月17日に発表された Claude Design の具体的な利用条件**（対象プラン、利用枠、Enterprise におけるデフォルトオフ、通常の Claude chat/Claude Code 枠とは分離した週次クォータ等）が公式ヘルプセンター記事で明文化され、前回レポートで「利用条件未詳細」としていた空白を埋める情報が揃った。また **2026年4月19日（本日）は Claude Haiku 3（`claude-3-haiku-20240307`）の API リタイア発効日**にあたり、旧Haiku 3 への API 呼び出しは本日以降エラー応答となる（要 Haiku 4.5 への移行）。

## Layer 1: CLIリリース

### v2.1.114（2026年4月18日）

#### バグ修正

- **Agent Teams の permission dialog クラッシュ修正**
  - `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` 実験フラグ下で、チームメイト（teammate）エージェントが**ツール許可**をリクエストしたときに permission dialog がクラッシュする問題を修正
  - 多エージェント協調セッションで承認フローがブロックされなくなる

#### このリリースの位置付け

- v2.1.113 の大規模リリース（ネイティブバイナリ移行、セキュリティ強化）から**わずか1日でのフォローアップ**で、1項目のみのパッチ
- 修正対象は `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` という実験フラグ下の機能であるため、**Agent Teams が GA 化前であっても積極的に保守対象**であることが示された
- Agent Teams のパーミッションモデルは「チームメイトの権限要求はリード（lead）エージェントにバブルアップし、リードのUIで承認される」という設計であり、その承認ダイアログがクラッシュしていたことを意味する

#### 参考: Agent Teams の現在の仕様（外部ドキュメント・記事より）

- **有効化**: `settings.json` または環境変数に `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` を設定（デフォルトは無効）
- **lead / teammate**: チームメイトはリードのパーミッション設定から継承する（`--dangerously-skip-permissions` もリードに追従）。ただしspawn後に個別のチームメイトモードを変更可能
- **パーミッションバブルアップ**: チームメイトのツール許可要求はリードのUIに浮上する。頻出操作はあらかじめ `permissions.allow` に登録しておくと摩擦が減る
- **公式ドキュメント**: [Orchestrate teams of Claude Code sessions](https://code.claude.com/docs/en/agent-teams)

## Layer 2: プラットフォーム・公式発表

### Claude Design — 利用条件が公式ヘルプ記事で確定（2026年4月18日公開確認）🔬

前回レポート（2026-04-18 朝、v2.1.111-112）時点では「利用条件未詳細」としていた Claude Design について、公式ヘルプセンター記事「[Get started with Claude Design](https://support.claude.com/en/articles/14604416-get-started-with-claude-design)」と関連発表で具体が明らかになった。

| 項目 | 詳細 |
|------|------|
| **リリース状態** | 🔬 **研究プレビュー** |
| **対象プラン** | **Claude Pro / Max / Team / Enterprise**（無料プランは対象外） |
| **動作モデル** | Claude Opus 4.7（Anthropic の「最も高性能なビジョンモデル」として位置付け） |
| **サーフェス** | `claude.ai/design`（Claude アプリ内のビジュアル制作サーフェス） |
| **利用枠** | 既存プラン枠を使用、**かつ通常の Claude chat / Claude Code 枠とは独立した週次 allowance** |
| **extra usage** | クォータ超過後は extra usage を有効化して継続利用可能 |
| **Enterprise での既定** | **デフォルト オフ**。管理者が Organization settings で明示有効化 |
| **ロールアウト** | 2026年4月17日から「throughout the day」で段階展開 |
| **ユースケース** | プロトタイプ、スライド、1-pager、ランディングページ、ソーシャル/マーケティングアセット生成 |
| **位置付け** | Canva / Figma の「代替ではなく補完」（公式説明） |

#### 用途・対象ユーザー

- デザインツール非使用の **ファウンダー、プロダクトマネージャー、マーケター** がアイディアを素早くビジュアル化する想定
- 会話的なプロンプトと修正制御でデザイン・プロトタイプ・スライドを反復作成
- 既存の Claude Cowork 内「Design」プラグイン（デザインクリティーク、UXライティング、アクセシビリティ監査等）とは別サーフェス

### Claude Haiku 3（`claude-3-haiku-20240307`）API リタイア発効 — 2026年4月19日（本日）✅

- **発効日**: 2026-04-19（本日）
- **対象**: Claude API の `claude-3-haiku-20240307` モデル ID
- **挙動**: 本日以降、このモデル ID への API リクエストは**エラー応答**を返す（完全リタイア）

### 2026年4月23日（4日後）: API デフォルトモデル Opus 4.7 切替（再掲・確認）📢 → ✅予定

前回レポートで詳述済みの通り、Enterprise pay-as-you-go / Anthropic API（第一者）で `opus` エイリアスが Opus 4.6 → Opus 4.7 に自動切替される予定。本日時点で変更はなし。モデル ID ピンニングを行っていない本番 API 利用者は、Haiku 3 リタイアとあわせて**今週中の事前検証ウィンドウ**に注意。
