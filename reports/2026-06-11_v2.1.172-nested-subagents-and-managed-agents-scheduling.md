# 調査レポート: Claude Code v2.1.172（ネストサブエージェント）+ Managed Agents スケジュール実行 / 認証情報 Vault

**調査日:** 2026-06-11

## 概要

- **CLI v2.1.172（6/10 20:44 UTC）が新規リリース**。目玉は **サブエージェントが自身のサブエージェントを生成できる「ネストサブエージェント」（最大 5 階層）**。加えて Amazon Bedrock の AWS リージョン自動検出（`~/.aws` config 読み取り）、`/plugin` マーケットプレイス内の **プラグイン検索バー**、OTEL メトリクスへの `model` 属性追加、そして **30 件超のバグ修正**（1M コンテキスト枯渇時の自動コンパクト、`availableModels` 制限の subagent / dispatch picker / advisor への適用漏れ、WebFetch ワイルドカードドメインルール不一致など）。
- **v2.1.171 は欠番**（GitHub Releases・チェンジログとも未掲載、v2.1.170 → v2.1.172 へスキップ）。
- **Layer 2 — Claude Managed Agents に 2 つの実運用機能（6/9、公開ベータ）**: ① **スケジュール実行（Scheduled Deployments）** = cron スケジュールでエージェントを自動起動、スケジューラの自前構築・ホスト不要。② **Vault の環境変数クレデンシャル** = シークレットをサンドボックスにプレースホルダのみ配置し、実際の鍵はネットワーク境界（allowlist ドメイン宛リクエスト）でのみ注入 → プロンプトインジェクションでも context に実シークレットが載らない。**追加課金なし**（既存 Claude Platform 利用料に内包）。Fable 5 と同日発表だが前回レポートでは未記載のため今回反映。
- **Layer 2/3 — Apple Foundation Models フレームワーク対応**（Swift パッケージ経由、iOS/iPadOS/macOS/visionOS 27 系）が周辺発表として報じられた（📢）。

新規 CLI リリースは **v2.1.172 の 1 本**（前回 6/10 レポートは v2.1.170 まで）。プラットフォーム側で新モデル・新ティアの発表はなし。

## 主要な新機能

### 1. CLI v2.1.172（2026-06-10 20:44 UTC）

**リリース状態**: ✅ GA（CLI 安定版）
**情報源**: [GitHub Releases](https://github.com/anthropics/claude-code/releases) / [Claude Code changelog](https://code.claude.com/docs/en/changelog) / [Releasebot - Claude Code](https://releasebot.io/updates/anthropic/claude-code)

#### 新機能

- **ネストサブエージェント（最大 5 階層）**: サブエージェントが自身のサブエージェントを生成可能になった。従来は「サブエージェントはさらにサブエージェントを生成できない（無限ネスト防止）」という制約があったが、本リリースで **最大 5 階層までのネスト**が公式サポートされた。Dynamic Workflows（5/28 研究プレビュー）の数十〜数百並列オーケストレーションと組み合わせ、より深い階層分解が可能に。
- **Amazon Bedrock のリージョン自動検出**: `AWS_REGION` が未設定の場合、`~/.aws` の config ファイルからリージョンを読み取る（AWS SDK と同じ優先順位）。`/status` に「リージョンの取得元」が表示される。
- **`/plugin` マーケットプレイス検索バー**: マーケットプレイスのプラグイン一覧を閲覧する際に検索バーが追加され、大量プラグインから絞り込み可能に。
- **OTEL メトリクス拡張**: `claude_code.lines_of_code.count` メトリクスに `model` 属性を追加（どのモデルが生成したコード行かを観測可能に）。

#### 主なバグ修正（30 件超）

- **1M コンテキストを usage credits なしで使い続けたセッションが恒久的にスタックする問題を修正** — 標準コンテキスト上限以下へ自動コンパクトするようになった。
- `availableModels` 制限が **subagent のモデルオーバーライド / エージェント dispatch のモデルピッカー / advisor モデル**に適用されていなかった問題を修正。
- `availableModels` の allowlist がバージョン固有 ID（`claude-opus-4-8` 等）使用時に `/model` ピッカーの Opus / Sonnet 1M 行を隠してしまう問題を修正。
- **Bedrock の `/model` ピッカーがプロバイダ未提供のモデルを提示**し、選択するとサイレントにセッションモデルを切り替えていた問題を修正。
- `WebFetch(domain:*.example.com)` のワイルドカードドメインルールが allow/deny/ask いずれでもサブドメインに一致しない問題、`Read(secrets-*/config.json)` 等のパターン途中ワイルドカードが起動時に拒否される問題を修正。
- 会話に複数画像が含まれる際の「画像を処理できず削除した」エラー繰り返しを修正。
- バックグラウンドエージェントが事前ウォームされた worker に dispatch された際、**別ディレクトリのプロジェクト設定（`.mcp.json` 承認・trust）を読み込む**可能性があった問題を修正。
- ネストエージェント停止後にバックグラウンドサブエージェントが「active」のままスタックする問題、agents view が worker 応答後も最大 30 秒スピナーを回し続ける問題を修正。
- `model` ID に 1M コンテキストサフィックスが二重付与される（`[1M][1m]`）問題、`opusplan` がプランモードで 1M コンテキストを伴わない問題を修正。
- メモリリコールがリモートセッションでマウント済みチーム memory store（`CLAUDE_MEMORY_STORES`）を見つけられない問題を修正。
- **長い会話のパフォーマンス改善**（冗長なメッセージ正規化の除去）、**アイドル CPU 使用率削減**（`/goal` ステータスチップの 5Hz 再描画停止）、Claude in Chrome のツール一括ロード。
- **[VSCode]** PowerShell ツール呼び出しが生 JSON で表示される問題、シェル出力の ANSI エスケープ除去。
- `/code-review` の `ultra` オプションを claude.ai 未サインイン時も表示（クラウドレビューには claude.ai アカウントが必要との説明付き）。

> 注: 一般のサブエージェントドキュメントには依然「サブエージェントはさらにサブエージェントを生成できない」と記載されており、コミュニティはこれまで `claude -p` headless モードを使った回避策で擬似ネストを実現していた（[gruckion/nested-subagent](https://github.com/gruckion/nested-subagent) 等）。v2.1.172 はこの制約をネイティブに緩和したもの。

### 2. Claude Managed Agents — スケジュール実行（Scheduled Deployments）🧪

**リリース状態**: 🧪 公開ベータ（Claude Platform）
**発表日**: 2026-06-09
**対象**: Claude Platform / Claude Platform on AWS

- **cron スケジュールでエージェントを自動起動**。スケジュールが発火するたびにエージェントが新規セッションを開始しタスクを完了。**スケジューラを自前で構築・ホストする必要がない**。
- 管理者は実行の **一時停止・再開・手動トリガー**が可能。夜間同期・週次スキャン等の定期業務向け。
- **「人間がいないとき誰がエージェントを起動するのか」というエンタープライズ AI 導入の障壁**（パイロット止まりの主因の 1 つ）を解消するという位置づけ。
- **追加課金なし** — 既存の Claude Platform 利用料に内包。
- 早期採用: Rakuten / Actively AI / Notion / Browserbase。
- **情報源**: [Claude Platform release notes（6/9）](https://platform.claude.com/docs/en/release-notes/overview) / [Anthropic - Scaling Managed Agents](https://www.anthropic.com/engineering/managed-agents) / [TechTimes](https://www.techtimes.com/articles/318163/20260610/claude-managed-agents-add-cron-schedules-credential-vaultsanthropic-beta-puts-agents-autopilot.htm)

### 3. Claude Managed Agents — Vault の環境変数クレデンシャル 🧪

**リリース状態**: 🧪 公開ベータ（Claude Platform）
**発表日**: 2026-06-09

- API キー等のシークレットを **Vault に保管し、エージェントのサンドボックスにはプレースホルダのみ**を配置。実際の鍵は **ネットワーク境界（顧客が allowlist したドメイン宛リクエスト）でのみ注入**される。
- これにより **プロンプトインジェクションでエージェントが操作されても、モデルの context には実シークレットが一切載らない**。
- CLI・SDK・環境変数で認証する各種サービスへセキュアにシークレットを注入できる。
- **情報源**: [Claude Platform release notes（6/9）](https://platform.claude.com/docs/en/release-notes/overview) / [Managed Agents vaults docs](https://platform.claude.com/docs/en/managed-agents/vaults)

### 4. Apple Foundation Models フレームワーク対応 📢

**リリース状態**: 📢 発表のみ（新 Swift パッケージ経由）
**概要**: Apple の Foundation Models フレームワーク経由で Claude を利用可能にする対応が報じられた。iOS 27 / iPadOS 27 / macOS 27 / visionOS 27 系が対象。Claude Code 本体の機能ではなく、Apple 開発者向けの Claude 提供チャネル拡張。
**情報源**: [Releasebot - Anthropic](https://releasebot.io/updates/anthropic)（一次情報の公式ブログ確認は今後の課題、確度は 📢 据え置き）
