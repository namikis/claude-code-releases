# 調査レポート: Claude Code v2.1.183（Auto mode 破壊的 git/IaC ガードレール）+ Claude Code Artifacts ベータ・エンタープライズ管理 MCP 認可（Okta）

**調査日:** 2026-06-20
**調査モード:** フルモード（Layer 1〜3）

## 概要

6/19 レポート（ニュースモード、Layer 1 最新は v2.1.181）以降の差分を Layer 1〜3 で調査。

- **Layer 1**: CLI **v2.1.183（6/19）** が新規（前回フルモード調査でカバーしたのは v2.1.181 まで）。**v2.1.182 / v2.1.180 はともに欠番**（v2.1.171 に続く 3・4 つ目の欠番）、Layer 1 最新は **v2.1.183**。目玉は **Auto mode の安全強化**で、**破壊的 git コマンド**（`git reset --hard` / `git checkout -- .` / `git clean -fd` / `git stash drop`）をユーザーが破棄を指示していない限りブロック、**エージェントが当該セッションで作成していない commit への `git commit --amend` をブロック**、**`terraform destroy` / `pulumi destroy` / `cdk destroy` を特定スタック指定がない限りブロック**。加えて **`attribution.sessionUrl` 設定**（web / Remote Control セッションで commit・PR から claude.ai セッションリンクを省略）、**`/config --help`**（`/config key=value` の shorthand キー一覧表示）、**`/config` トグル挙動の変更**（Enter/Space で変更、**Esc が revert ではなく save して閉じる**に）、**起動時 "setup issues" 行を削除**（→ `/doctor` か `--debug` で確認）、**非推奨/自動更新されたモデルの警告を `-p` の stderr に表示**（agent frontmatter のモデル指定もカバー）。多数のバグ修正（subagent 内 WebSearch が空を返す問題、thinking ブロックのみ返したターンの再プロンプト、Windows Terminal でのネストサブエージェント高負荷時 TUI 破損、tmux teammate ペイン起動失敗、scheduled task / webhook トリガー配信がキーボード入力扱いされ auto mode でアクション承認やタイトル設定をしてしまう問題 等）。**新スラッシュコマンド・新 Hook イベントの追加はなし**。
- **Layer 2（🆕 重要 — 6/19 ニュースモードでは未捕捉）**: **6/18 に 2 つのエンタープライズ向け新機能がベータ提供開始**された。① **Claude Code Artifacts 🧪 ベータ**（Team / Enterprise）— Claude Code セッションの成果物を **ライブで更新される、共有可能な対話的 HTML ページ**に変換。CLI・デスクトップ双方から利用でき、複数データソース/コードベースの変化や Claude の作業に応じてリアルタイム更新。PR ウォークスルー・インシデントページ・ダッシュボード・チェックリスト等が用途。**作成者・組織内に限定（公開不可）**。② **エンタープライズ管理 MCP 認可（Enterprise-Managed Authorization, EMA）🧪 ベータ**（Team / Enterprise）— 管理者が ID プロバイダ（**Okta が最初の対応 IdP**）経由で MCP コネクタを一度プロビジョニングすれば、ユーザーは初回ログイン時に zero-touch でアクセスを継承。SSO セッションで取得した署名済み ID-JAG JWT を MCP サーバーの access token と交換する仕組みで、**Claude / Claude Code / Cowork** を横断。これは IETF/MCP の **Enterprise-Managed Authorization 拡張が 6/18 に「stable」化**した初の本番実装で、launch 時点で Asana / Atlassian / Canva / Figma / Granola / Linear / Supabase 等が対応（Slack も追加中）。
- **Layer 3**: **Claude Fable 5 / Claude Mythos 5 は 6/20 時点も依然アクセス停止のまま**（6/12 米政府輸出管理指令）。Anthropic の上級技術スタッフが米商務省と協議中で、国際担当 MD Chris Ciauri の「数日以内（in the coming days）に復旧と確信」発言（6/18）は維持。予測市場は 7/1 までの復旧を約 57%、7/10 までを約 67%、7/17 までを約 75% と織り込む。**6/20 時点で復旧日・対象範囲は依然未確定**。

## 主要な新機能

### Auto mode の破壊的 git / IaC コマンドガードレール ✅
- **リリース状態**: ✅ GA（v2.1.183）
- **対象プラン**: 全プラン（Auto mode 利用時）
- **概要**: Auto mode の安全性を強化。**ユーザーがローカル作業の破棄を指示していない限り、破壊的 git コマンド**（`git reset --hard` / `git checkout -- .` / `git clean -fd` / `git stash drop`）を**ブロック**。**`git commit --amend` は、当該 commit が今セッションでエージェント自身が作成したものでない限りブロック**。さらに **`terraform destroy` / `pulumi destroy` / `cdk destroy` は、ユーザーが特定スタックを指定して依頼した場合を除きブロック**。Auto mode のサブエージェント事前分類（v2.1.178）に続く自律実行の安全強化系譜。
- **初出**: v2.1.183 (2026-06-19)
- **情報源**: [Claude Code changelog](https://code.claude.com/docs/en/changelog)

### `attribution.sessionUrl` 設定 ✅
- **リリース状態**: ✅ GA（v2.1.183）
- **概要**: web / Remote Control セッションで生成される **commit・PR から claude.ai セッションリンク（attribution）を省略**する設定。機微なセッション URL を公開リポジトリの履歴に残したくないチーム向け。
- **初出**: v2.1.183 (2026-06-19)
- **情報源**: [Claude Code changelog](https://code.claude.com/docs/en/changelog)

### `/config --help` と `/config` トグル挙動の改善 ✅
- **リリース状態**: ✅ GA（v2.1.183）
- **概要**:
  - **`/config --help`** — `/config key=value`（v2.1.181 で導入）で使える **shorthand キーの一覧**を表示。
  - **トグル挙動の変更** — `/config` 設定画面で **Enter と Space の両方で選択中の設定を変更**でき、**Esc が従来の revert ではなく「保存して閉じる」**に変更。
- **初出**: v2.1.183 (2026-06-19)
- **情報源**: [Claude Code changelog](https://code.claude.com/docs/en/changelog)

### 非推奨/自動更新モデルの警告（agent frontmatter 対応） ✅
- **リリース状態**: ✅ GA（v2.1.183）
- **概要**: リクエストされたモデルが**非推奨、または新しいモデルへ自動更新された場合に警告を表示**。print モード（`-p`）の stderr に出るほか、**agent frontmatter で指定したモデルもカバー**するようになった。6/15 の Claude 4（Sonnet 4・Opus 4）リタイア後のモデルガバナンス（`availableModels` / `enforceAvailableModels` / `Tool(model:...)`）の延長線上の改善。
- **初出**: v2.1.183 (2026-06-19)
- **情報源**: [Claude Code changelog](https://code.claude.com/docs/en/changelog)

### 起動時 "setup issues" 行の削除 ✅
- **リリース状態**: ✅ GA（v2.1.183）
- **概要**: ロゴ下の起動時 "setup issues" 行を削除。設定上の問題は **`/doctor`**（v2.1.178 でフラットツリー表示・ステータスアイコン改善）または **`--debug`** で確認する方式に統一。
- **初出**: v2.1.183 (2026-06-19)
- **情報源**: [Claude Code changelog](https://code.claude.com/docs/en/changelog)

## Layer 2: プラットフォーム新機能（6/18 ベータ提供開始）

> いずれも Claude Code 公式 CHANGELOG（CLI リリース単位）ではなく、プラットフォーム全体の発表として外部報道・公式サポート記事で確認。6/19 のニュースモード調査では捕捉されておらず、**チェンジログ単独調査では漏れる典型例**。

### Claude Code Artifacts 🧪
- **リリース状態**: 🧪 ベータ
- **対象プラン**: **Claude Team / Enterprise**（Free / Pro は対象外）
- **対象プラットフォーム**: Claude Code CLI + デスクトップアプリ
- **概要**: Claude Code セッションの作業を、**ライブで更新される対話的・共有可能な HTML ページ（custom artifact）**に変換する機能。自然言語で要件を伝えると Claude がライブコード + 複数データソースを束ねた webpage を組み立て、**プライベート URL でチーム共有**できる。Claude Code が自律的/ユーザー誘導で作業を進めるたび、また接続データソース・コードベースが変化するたびに、**閲覧者全員がリアルタイムで更新を確認**できる。用途は PR ウォークスルー、インシデントページ、ダッシュボード、チェックリスト等。
- **制限事項**: **作成者にプライベートがデフォルト、組織内共有のみ（広範なインターネットへの公開は不可）**。ライブデータ連携ゆえ「break / lag / 誤った可視化」のリスクがあり、出力は権威ある情報源ではなくツールとして扱うべきとの指摘。
- **初出（報道）**: 2026-06-18
- **情報源**: [VentureBeat](https://venturebeat.com/data/anthropics-claude-code-artifacts-update-brings-live-shared-dashboards-and-interactive-workspaces-to-enterprises) / [DevOps.com](https://devops.com/anthropic-brings-live-shareable-artifacts-to-claude-code/) / [Crypto Briefing](https://cryptobriefing.com/anthropic-claude-code-artifacts-enterprise-dashboards/) / [Claude Help Center - Artifacts](https://support.claude.com/en/articles/9487310-what-are-artifacts-and-how-do-i-use-them)

### エンタープライズ管理 MCP 認可（Enterprise-Managed Authorization, EMA / Okta）🧪
- **リリース状態**: 🧪 ベータ
- **対象プラン**: **Claude Team / Enterprise**
- **対象プラットフォーム**: **Claude（chat）/ Claude Code / Cowork** を横断
- **概要**: 管理者が **ID プロバイダ経由で MCP コネクタを一度プロビジョニング**すれば、ユーザーは初回ログイン時に **zero-touch でアクセスを継承**する仕組み。SSO セッション中に IdP が発行する署名済み **ID-JAG JWT** を MCP サーバーの authorization server で access token に交換し、ログイン時の認証を承認済み全 MCP コネクタへ自動伝播。MCP コミュニティが「エンタープライズ規模 MCP 採用の最大の障壁」としていた**ユーザー単位の認証ステップ（中央集権ガバナンス不能）**を解消する。
- **対応状況（launch 時点）**: **Okta が最初の対応 IdP**。クライアントは Anthropic / Visual Studio Code、サーバーは Asana / Atlassian / Canva / Figma / Granola / Linear / Supabase 等が対応（Slack も追加中）。EMA は IETF OAuth WG（2025-09 採択）→ MCP 仕様組み込み（2025-11）を経て、**2026-06-18 に MCP の正式な authorization 拡張として "stable" 化**しており、Anthropic の本実装がその最初の本番実装。
- **初出（報道/公式）**: 2026-06-18
- **情報源**: [TestingCatalog](https://www.testingcatalog.com/anthropic-launches-managed-connector-access-with-okta/) / [MCP Blog - Enterprise-Managed Authorization](https://blog.modelcontextprotocol.io/posts/enterprise-managed-auth/) / [Okta Newsroom](https://www.okta.com/en-ca/newsroom/press-releases/okta-becomes-a-featured-identity-provider-powering-secure-ai-agent-connections-for-claude-enterprise/) / [TechTimes](https://www.techtimes.com/articles/318708/20260619/mcp-enterprise-authorization-goes-stable-zero-touch-sso-okta-anthropic-vs-code.htm)

## バグ修正・パフォーマンス改善（v2.1.183 抜粋）

- subagent 内で **WebSearch が空の結果を返す**問題を修正。
- モデルが **thinking ブロックのみを返してターンが無出力で完了**する問題を修正（Claude が一度だけ再プロンプト）。
- **Windows Terminal でのネストサブエージェント高負荷時のフルスクリーン TUI 破損**（statusline 中段表示・スピナー行重複・テキスト結合）を修正。
- **vim モード + ネイティブカーソル**でヒストリ操作後にカーソルがプロンプト上方に取り残される問題を修正。
- **tmux teammate ペインがシェル rc 初期化の遅延時に起動失敗**する問題、エージェント spawn 中のキー入力が新ペインに漏れる問題を修正。
- **scheduled task / webhook トリガー配信がキーボード入力として扱われ**、auto mode で保留中アクションを承認したりセッションタイトルを設定してしまう問題を修正（タスク通知として分類するように）。
- teammate が起動したバックグラウンドタスクが、teammate のターン終了時に kill される問題を修正。
- 認証が必要な **MCP サーバーが headless/SDK モードで auth-stub ツールをモデルに露出**する問題を修正。
- 複数プラグイン有効時に **user-level スキルがスラッシュコマンド補完に重複表示**される問題を修正。
- `thinking.disabled.display: Extra inputs are not permitted` の 400 エラー（subagent spawn・セッションタイトル生成）を修正。
- focus モードで各応答下に "Ran N PostToolUse hooks" のタイミング行が出る問題を修正。
- 想定外の非リトライ対象エラーで **fallback model を使ってリトライ**するように（検索ソースより）。
