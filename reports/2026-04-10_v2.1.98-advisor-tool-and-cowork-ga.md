# Claude Code 調査レポート: v2.1.98・Advisor Tool・Cowork GA

**調査日:** 2026-04-10
**対象期間:** 2026-04-09 〜 2026-04-10
**調査モード:** フルモード

---

## 概要

前回調査（2026-04-09）以降、以下の重要な動きがあった：

1. **v2.1.98**（4月9日）: Google Vertex AIインタラクティブセットアップウィザード、Perforceモード、Monitorツール、PID名前空間サンドボックス、多数のセキュリティ修正・バグ修正
2. **Advisor Tool**（4月9日）: エグゼキューターモデル（Sonnet/Haiku）とアドバイザーモデル（Opus）を組み合わせるAPIツールがパブリックベータ開始
3. **Claude Cowork GA**（4月9日）: 研究プレビューからGA（一般提供）に昇格。エンタープライズ機能（RBAC、グループ支出制限、利用分析、OpenTelemetry拡張）追加

---

## 主要な新機能

### v2.1.98（2026年4月9日） ✅

#### 新機能

| 機能 | 詳細 |
|------|------|
| **Google Vertex AIセットアップウィザード** | ログイン画面の「3rd-party platform」選択時にアクセス可能。GCP認証・プロジェクト/リージョン設定・クレデンシャル検証・モデルピンニングを対話的にガイド |
| **`CLAUDE_CODE_PERFORCE_MODE`** | 環境変数設定時、Edit/Write/NotebookEditが読み取り専用ファイルで失敗し`p4 edit`ヒントを表示。サイレント上書きを防止 |
| **Monitorツール** | バックグラウンドスクリプトからのイベントストリーミング対応 |
| **サブプロセスPID名前空間サンドボックス** | Linux環境で`CLAUDE_CODE_SUBPROCESS_ENV_SCRUB`設定時にPID名前空間分離を適用 |
| **`CLAUDE_CODE_SCRIPT_CAPS`** | セッション単位のスクリプト呼び出し回数を制限する環境変数 |
| **`--exclude-dynamic-system-prompt-sections`** | printモードでクロスユーザープロンプトキャッシュを改善するフラグ |
| **LSP `clientInfo`** | Claude Codeがlanguage serverに対して`initialize`リクエストで自己識別情報を送信 |

#### セキュリティ修正

| 修正 | 詳細 |
|------|------|
| **Bashツールバックスラッシュフラグバイパス** | バックスラッシュエスケープされたフラグが読み取り専用として自動許可され、任意コード実行に繋がる脆弱性を修正 |
| **複合Bashコマンドの権限バイパス** | auto/bypass-permissionsモードで安全チェックと明示的askルールの強制プロンプトをバイパスする問題を修正 |
| **env-varプレフィックス付き読み取り専用コマンド** | `LANG`、`TZ`、`NO_COLOR`等の既知安全変数以外ではプロンプト表示するよう修正 |
| **`/dev/tcp/...`・`/dev/udp/...`リダイレクト** | ネットワークリダイレクトが自動許可されずプロンプト表示するよう修正 |

#### 主要バグ修正

| 修正 | 詳細 |
|------|------|
| **ストリーミングレスポンス停滞** | タイムアウトする代わりに非ストリーミングモードにフォールバック |
| **429リトライ改善** | 小さな`Retry-After`値で全リトライが約13秒で枯渇する問題を修正。指数バックオフ最小値を適用 |
| **MCP OAuth `authServerMetadataUrl`** | 再起動後のトークンリフレッシュ時にADFS等のIdP設定が反映されない問題を修正 |
| **kittyキーボードプロトコル** | xterm/VS Code統合ターミナルで大文字が小文字に変換される問題を修正 |
| **macOSテキスト置換** | トリガーワードが削除され置換テキストが挿入されない問題を修正 |
| **`--dangerously-skip-permissions`ダウングレード** | 保護パスへの書き込み承認後にaccept-editsモードに暗黙ダウングレードされる問題を修正 |
| **managed-settings allowルール** | 管理者が削除した後もプロセス再起動までアクティブのままだった問題を修正 |
| **`permissions.additionalDirectories`** | セッション中の変更が即座に反映されるよう修正 |
| **Bashワイルドカード権限ルール** | `Bash(cmd:*)`や`Bash(git commit *)`がスペース/タブを含むコマンドにマッチしない問題を修正 |
| **Bash denyルールダウングレード** | `cd`とパイプ混在コマンドでdenyルールがプロンプトにダウングレードされる問題を修正 |
| **JSプロトタイプ名権限** | `toString`等のプロトタイプ名が`settings.json`を無視させる問題を修正 |
| **エージェントチーム権限継承** | チームメンバーがリーダーの`--dangerously-skip-permissions`モードを継承しない問題を修正 |
| **フルスクリーンMCPツール結果** | ホバー時のクラッシュを修正 |
| **URL折り返しコピー** | フルスクリーンモードでスペースが挿入される問題を修正 |
| **`--resume`時のファイル編集diff** | 10KB超ファイルのdiffが消える問題を修正 |
| **`/resume`ピッカー** | 複数の問題修正（編集不可セッション、フィルタリセット、空リストナビゲーション、プロジェクト間staleness等） |
| **`/export`パス処理** | 絶対パスと`~`を正しく処理、拡張子が`.txt`に上書きされる問題を修正 |
| **`/effort max`** | 不明/将来のモデルIDで拒否される問題を修正 |
| **スラッシュコマンドYAMLブール** | プラグインフロントマターの`name`がYAMLブールキーワードの場合にピッカーが壊れる問題を修正 |
| **Voice Modeスペースリーク** | push-to-talkキー再押下時にスペース文字が漏れる問題を修正 |
| **`DISABLE_AUTOUPDATER`** | npm版インストールでのバージョンチェック・symlinkの変更を完全抑制 |
| **Remote Control権限ハンドラメモリリーク** | セッション全体にわたってエントリが保持される問題を修正 |
| **バックグラウンドサブエージェント** | エラー時に部分的進捗が親エージェントに報告されない問題を修正 |
| **Stop/SubagentStopフック** | 長いセッションで失敗する問題、フック評価APIエラーのメッセージ表示を修正 |
| **`grep -f FILE`/`rg -f FILE`** | 作業ディレクトリ外のパターンファイル読み取り時にプロンプトが表示されない問題を修正 |
| **worktreeクリーンアップ** | untracked filesを含むworktreeが削除される問題を修正 |
| **`sandbox.network.allowMachLookup`** | macOSで有効にならない問題を修正 |
| **`CLAUDE_CODE_MAX_CONTEXT_TOKENS`** | `DISABLE_COMPACT`設定時に`/compact`ヒントが表示される問題を修正 |

#### UI・UX改善

| 改善 | 詳細 |
|------|------|
| **`/resume`フィルタ** | プロジェクト/worktree/ブランチ名をフィルタインジケータに表示 |
| **フッターインジケータ** | Focus・通知がナロー端末幅でも折り返さずモード行に表示 |
| **`/agents`タブレイアウト** | Runningタブ（ライブサブエージェント）とLibraryタブ（実行/表示アクション）の2タブ構成に |
| **`/reload-plugins`** | プラグイン提供スキルを再起動なしで取得可能に |
| **Vim mode** | `j`/`k`がNORMALモードで履歴ナビゲーション、入力境界でフッターpillを選択 |
| **フックエラー表示** | トランスクリプトにstderrの最初の行を含め、`--debug`なしで自己診断可能に |
| **OTELトレーシング** | interactionスパンが同時SDK呼び出しで正しくフルターンをラップ |
| **トランスクリプトトークン使用量** | ストリーミングプレースホルダーではなく最終トークン使用量を表示 |
| **VSCode Windows** | `CLAUDE_CODE_GIT_BASH_PATH`設定時やデフォルト位置へのGitインストール時の誤エラーを修正 |

### Advisor Tool（APIパブリックベータ） 🧪

- **発表日**: 2026年4月9日
- **ベータヘッダー**: `advisor-tool-2026-03-01`
- **概要**: 高速なエグゼキューターモデル（Sonnet/Haiku）にアドバイザーモデル（Opus等）を組み合わせ、長期的なエージェントワークロードでアドバイザー単体に近い品質を実現しつつ、トークン生成の大部分はエグゼキューターモデルのレートで処理
- **ユースケース**: コスト管理と高レベル推論の両立が必要なエージェント開発
- **情報源**: [Claude Platform Release Notes](https://platform.claude.com/docs/en/release-notes/overview) / [TestingCatalog](https://www.testingcatalog.com/anthropic-launches-advisor-tool-for-claude-platform-api-users/)

### Claude Cowork GA（一般提供） ✅

- **発表日**: 2026年4月9日
- **状態変更**: 🔬 研究プレビュー → ✅ GA（全有料プラン）
- **概要**: Claude Coworkが研究プレビューからGA（一般提供）に昇格。全有料サブスクリプション（Pro/Max/Team/Enterprise）で利用可能に
- **新エンタープライズ機能**:
  - ロールベースアクセス制御（RBAC）: SCIM統合でIDプロバイダー連携、グループ単位のClaude機能アクセス制御
  - グループ支出制限
  - 利用分析
  - 拡張OpenTelemetryサポート
  - Zoom MCPコネクタ
  - ツールごとのコネクタ制御
- **対応プラットフォーム**: macOS / Windows
- **価格**: Pro ($20/月) / Max ($100/月 5x, $200/月 20x) / Enterprise（カスタム）
- **情報源**: [9to5Mac](https://9to5mac.com/2026/04/09/anthropic-scales-up-with-enterprise-features-for-claude-cowork-and-managed-agents/) / [The New Stack](https://thenewstack.io/anthropic-takes-claude-cowork-out-of-preview-and-straight-into-the-enterprise/) / [TestingCatalog](https://www.testingcatalog.com/anthropic-launches-claude-cowork-in-general-availability/)
