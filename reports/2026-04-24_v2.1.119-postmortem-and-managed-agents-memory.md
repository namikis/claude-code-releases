## 主要な新機能

### 1. Claude Code v2.1.119（2026-04-23 23:24 UTC）✅ GA

**情報源**: [GitHub Releases v2.1.119](https://github.com/anthropics/claude-code/releases) / [Raw CHANGELOG.md](https://raw.githubusercontent.com/anthropics/claude-code/refs/heads/main/CHANGELOG.md)

#### 設定の永続化とオーバーライド

- **`/config` 設定の `~/.claude/settings.json` 永続化**: `theme`、`editor mode`、`verbose` 等の設定が **`~/.claude/settings.json` に永続化**され、**project / local / policy のオーバーライド順位に参加**。従来は一時的状態だったためシェル再起動で消えていたものが、通常の managed settings ピラミッドと同じ扱いに統一
- **`prUrlTemplate` 設定**: footer の PR バッジが github.com 以外の **カスタムコードレビュー URL** を指すように設定可能（社内 code review サービス、Reviewable、独自ホスト GitLab 等）
- **`CLAUDE_CODE_HIDE_CWD` 環境変数**: スタートアップロゴで作業ディレクトリを非表示に。スクリーンシェア時や共有端末運用で cwd の可視化を抑制

#### マルチホスト PR / VCS サポート

- **`--from-pr` が GitLab MR、Bitbucket PR、GitHub Enterprise PR URL をサポート**: 従来の GitHub.com PR に加え、主要な Git ホスティングの PR URL を引数に取れるように
- **`owner/repo#N` ショートハンドリンクが git remote のホストを使用**: 出力内の `owner/repo#N` リンクが**常に github.com ではなく、現在の git remote のホストに解決**されるように。Enterprise GitHub・GitLab・Bitbucket 利用時の正しいリンク生成

#### エージェント実行 / permission

- **`--print` モードが agent の `tools:` / `disallowedTools:` フロントマターを honor**: インタラクティブモードと同じ動作に統一。ヘッドレス実行時のツール制約が効くように
- **`--agent <name>` が built-in agent 定義の `permissionMode` を honor**: Explore / Plan / General-purpose 等のビルトインエージェントの定義済み permission mode がセッション起動時に適用
- **PowerShell tool の auto-approve**: Bash と同じ挙動で、permission mode 下で PowerShell コマンドも自動承認可能に（Windows・`CLAUDE_CODE_USE_POWERSHELL_TOOL=1` 有効時）

#### Hooks / OpenTelemetry

- **`PostToolUse` / `PostToolUseFailure` hook 入力に `duration_ms` 追加**: ツール実行時間（permission prompt と PreToolUse hook の時間は除外）。**ツールレベルのパフォーマンス計測やスローツール検出**に利用可能
- **Subagent / SDK MCP サーバーの再構成が並列接続**: 直列接続だった reconfiguration を並列化。MCP サーバーが多い環境での reload が高速化
- **OpenTelemetry 拡張**:
  - `tool_result` と `tool_decision` イベントに **`tool_use_id`** を追加（特定のツール呼び出しと結果を相関可能）
  - `tool_result` に **`tool_input_size_bytes`** を追加（ペイロードサイズトラッキング）
- **ステータスラインの stdin JSON に `effort.level` と `thinking.enabled` を追加**: 外部ステータスラインスクリプトが現在の努力レベル・thinking 状態を把握可能

#### プラグイン

- **他プラグインの version constraint でピンされたプラグインが、満たす最高の git tag に自動アップデート**: 依存満足解決の精度向上
- **セキュリティ: `blockedMarketplaces` が `hostPattern` と `pathPattern` エントリを正しく enforce**: 従来は URL 完全一致のみだったが、ホスト・パスパターンによるブロックが確実に動作するように修正

#### Vim mode / 入力

- **Vim mode: INSERT で Esc がキュー済みメッセージを入力に戻さない**: Esc で INSERT を抜け、**もう一度 Esc で割り込み**（中断）のフローに変更
- **スラッシュコマンドサジェスト**: **クエリにマッチした文字をハイライト**表示
- **スラッシュコマンドピッカー**: 長い説明が**切り詰めではなく 2 行目に折り返し**表示
- **CRLF コンテンツ貼り付け修正**: Windows クリップボード・Xcode コンソールから貼り付け時に全行間に空行が追加される問題を修正
- **マルチライン貼り付け修正**: kitty keyboard protocol シーケンスを bracketed paste 内で使用するターミナルで改行が失われる問題を修正
- **`@`-file Tab completion 修正**: 絶対パス指定のスラッシュコマンド内で使用した際にプロンプト全体が置換される問題を修正

#### macOS / Linux ネイティブビルド

- **`Glob` と `Grep` ツールが Bash 拒否時に消える問題修正**: ネイティブ macOS / Linux ビルドで Bash ツールが permission で deny されていると Glob/Grep が消えてしまう問題（v2.1.117 で `bfs`/`ugrep` を Bash 経由に置換したことによる副作用）を修正
- **フルスクリーンモードのスクロール修正**: ツール完了ごとにスクロール位置が最下部にスナップバックする問題を修正
- **macOS Terminal.app スタートアップで `p` が残る問題修正**: Docker / SSH 経由で起動した際にプロンプトに `p` の文字が現れる問題を修正

#### MCP OAuth / プロトコル

- **MCP HTTP 接続の "Invalid OAuth error response" 修正**: サーバーが OAuth discovery リクエストに対して non-JSON レスポンスを返した際の接続失敗を修正
- **`${ENV_VAR}` プレースホルダの置換**: HTTP/SSE/WebSocket MCP サーバーの `headers` における `${ENV_VAR}` がリクエスト送信前に置換されない問題を修正
- **MCP OAuth `--client-secret` の `client_secret_post`**: `--client-secret` で保存したクライアントシークレットがトークン交換時に送信されない問題を修正（`client_secret_post` 要求サーバー向け）
- **プラグイン MCP サーバー on Windows**: プラグインキャッシュが不完全な状態でプラグイン提供 MCP サーバーが Windows で spawn しない問題を修正
- **プラグイン MCP サーバーの `${user_config.*}` 空フィールド対応**: optional フィールドを空のままにした際の失敗を修正

#### `/skills`・`/agents`・`/export`・`/usage`・`/plan`

- **`/skills` で Enter がダイアログを閉じる問題修正**: Enter は `/<skill-name>` をプロンプトに pre-fill するように変更
- **`/agents` 詳細ビュー**: サブエージェントが使えない built-in ツールが "Unrecognized" と誤ラベルされる問題を修正
- **`/export` のモデル表示修正**: デフォルトモデルではなく**会話で実際に使用されたモデル**を表示
- **`/usage` のプログレスバー** が "Resets …" ラベルと重なる問題を修正
- **`/plan` と `/plan open`** が plan mode 入場時に**既存の plan を操作対象にしない**問題を修正

#### Auto Mode / Plan Mode 整合

- **Auto mode が plan mode を "Execute immediately" 指示で上書きする問題修正**: Plan mode の矛盾する指示で plan mode が override されないように

#### Subagent / Worktree / Task

- **Agent tool `isolation: "worktree"` が以前のセッションの stale な worktree を再利用する問題修正**: 新規 worktree を毎回作成するように
- **`TaskList` がファイルシステム順ではなく ID ソート**で返すように修正
- **spinner が subagent タスク通知が孤立した際に on のまま残る問題修正**

#### SDK / bridge / Rewind / verbose

- **async `PostToolUse` hook が empty entries を session transcript に書き込む問題修正**: レスポンスペイロードを返さない非同期 hook の扱いを改善
- **Skills が auto-compaction 前に invoke された際に次のユーザーメッセージに対して再実行される問題修正**
- **Rewind オーバーレイ**: 画像添付付きメッセージで "(no prompt)" と表示される問題を修正
- **`/reload-plugins` と `/doctor`** が disabled plugins のロードエラーを報告する問題を修正
- **無効化された MCP サーバーが `/status` で "failed" として表示される問題修正**
- **SDK / bridge `read_file` がファイル成長時にサイズキャップを正しく enforce しない問題修正**
- **verbose output 設定が再起動後に persist しない問題修正**（これ自体は `/config` 永続化の副次的バグ修正）
- **PR が git worktree 作業時にセッションにリンクされない問題修正**
- **`/doctor` が higher-precedence scope で override された MCP サーバーエントリを警告する問題修正**（過剰警告を抑制）
- **false-positive "GitHub API rate limit exceeded" ヒント**: `gh` 出力に "rate limit" を含む PR タイトルがあるとヒントが誤発火する問題を修正

#### Vertex AI / Windows

- **Vertex AI でツール検索がデフォルト無効**: unsupported beta header エラー回避のため、opt-in は `ENABLE_TOOL_SEARCH`
- **Windows**: 誤検出の "Windows requires 'cmd /c' wrapper" MCP config 警告を削除

#### VSCode

- **VSCode: 音声ディクテーションの初回録音が macOS でマイクロフォン permission prompt 表示中に何も生成しない問題修正**

### 2. Anthropic Engineering Blog: "An update on recent Claude Code quality reports"（2026-04-23 公開）📢 Layer 2 公式

**情報源**: [Anthropic Engineering Blog - April 23 postmortem](https://www.anthropic.com/engineering/april-23-postmortem) / [The Register - Anthropic admits it dumbed down Claude with 'upgrades'](https://www.theregister.com/2026/04/23/anthropic_says_it_has_fixed/) / [VentureBeat - Mystery solved: Anthropic reveals changes to Claude's harnesses and operating instructions likely caused degradation](https://venturebeat.com/technology/mystery-solved-anthropic-reveals-changes-to-claudes-harnesses-and-operating-instructions-likely-caused-degradation)

#### 要点

Anthropic が **Claude Code / Claude Agent SDK / Claude Cowork の品質低下について公式 postmortem を公開**。**Claude API は影響なし**。原因は意図的劣化ではなく 3 件の独立した変更・バグ。

#### 3 つの根本原因

| # | 変更内容 | 発生日 | 修正日 | 影響範囲 |
|---|---------|-------|-------|--------|
| **1** | デフォルト努力レベルを `high` → `medium` に引き下げ | 2026-03-04 | **2026-04-07（v2.1.94）** | Claude Code（Sonnet 4.6、Opus 4.6） |
| **2** | Thinking Cache 最適化バグ（1 時間アイドル後に thinking をクリアする意図が、毎ターンごとに clearing） | 2026-03-26 | **2026-04-10（v2.1.101）** | Claude Code（Sonnet 4.6、Opus 4.6） |
| **3** | システムプロンプトに「keep text between tool calls to ≤25 words」制約を追加 | 2026-04-16 | **2026-04-20（v2.1.116）** | Claude Code（Sonnet 4.6、Opus 4.6、Opus 4.7） |

##### 詳細

**Issue 1: 努力レベル引き下げ（3/4〜4/7）**
- 高努力モードで一部ユーザーの UI がフリーズしたように見える長レイテンシ問題を緩和するため、デフォルト努力を `high` → `medium` に変更
- 結果としてユーザーから「賢さが落ちた」との報告が相次ぎ、公式に "wrong tradeoff" と認定
- **v2.1.94**（4/7）で API キー / Bedrock / Vertex / Foundry / Team / Enterprise 向けに `medium` → `high` に巻き戻し
- **v2.1.111**（4/17）で Opus 4.7 のデフォルトを `xhigh` に
- **v2.1.117**（4/22）で Pro/Max サブスクライバー向けにも `medium` → `high` 適用

**Issue 2: Thinking Cache クリアバグ（3/26〜4/10）**
- 1 時間アイドル後に古い thinking セッションをクリアする意図のキャッシュ最適化が、**会話のすべてのターンで cached thinking をクリアしてしまう**バグに
- 結果として Claude が「**forgetful and repetitive**」に見え、使用量が加速消費される
- **v2.1.101**（4/10）で修正

**Issue 3: システムプロンプトの冗長性制限（4/16〜4/20）**
- モデルを冗長でなくするため、システムプロンプトに「ツール呼び出し間のテキストを ≤25 語に抑える」制約を追加
- その後の ablation テストで **Opus 4.6 / 4.7 ともにコーディング性能が 3% 低下**と判明
- **v2.1.116**（4/20）で即時巻き戻し

#### 再発防止コミット

- **本番ビルドを使った社内テスト拡充**
- **Code Review ツールのマルチリポジトリコンテキスト対応強化**
- **システムプロンプトの変更管理をより厳格化、モデル別評価を拡充**
- **インテリジェンス影響の可能性がある変更には soak 期間と段階ロールアウト導入**
- **全ユーザーの使用枠リセット**（Issue 2 のキャッシュバグで枠が加速消費された補償）

#### 意義

- Simon Willison、Fortune、VentureBeat、AMD の Stella Laurenzo（6,852 セッション分析で 67% 性能低下を定量化）、Microsoft 研究者らが **3 月後半〜4 月前半に亘って指摘してきた「Claude がダムになった」現象の公式説明と確定**
- **Anthropic の透明性ブランドにとって重要な修復イベント**（IPO 噂・プロプラン試験削除事件の真っ只中）
- **API・Bedrock/Vertex/Foundry 経由の Messages API は影響なし**が確定（エンタープライズユーザーには重要）
- Claude Code / Agent SDK / Cowork のハーネス層でのみ発生した問題である点は、**ハーネス層の品質管理が独立した運用ガバナンス領域**であることを示す

### 3. Memory for Claude Managed Agents 公開β（2026-04-23）🧪

**情報源**: [Claude Platform Release Notes – April 23, 2026](https://platform.claude.com/docs/en/release-notes/overview) / [Built-in memory for Claude Managed Agents（Claude blog）](https://claude.com/blog/claude-managed-agents-memory) / [Using agent memory - Claude API Docs](https://platform.claude.com/docs/en/managed-agents/memory)

#### 要点

- **ステータス変化**: 🔬 **Research Preview**（4/8 Managed Agents ローンチ時から）→ 🧪 **Public Beta**（2026-04-23〜）
- **ベータヘッダー**: 標準の **`managed-agents-2026-04-01`**（既存ヘッダーでそのまま有効化、新しいヘッダーは不要）

#### 機能概要

- **ファイルシステムベースのメモリ**: エージェントがセッション間で情報を保持・学習できる仕組み
- **メモリ = ファイル**: メモリ内容は **ファイルとして表現**され、**API 経由でエクスポート・独立管理可能**
- **クロスエージェント共有**: 権限スコープ制御付きで複数エージェント間で知識共有可能
- **エンタープライズコンプライアンス**: すべての変更に **詳細な監査ログ**、バージョンロールバック、コンテンツリダクション対応
- **並行アクセス**: 書き込みオーバーライトなしで並行アクセスをサポート

#### 早期採用事例

- **Rakuten**: First-pass エラー **97% 削減**（より低コストで）
- **Wisedocs**: ドキュメント検証が **30% 高速化**

#### 前回時点からの確度変更

- 既存 `CURRENT_FEATURES.md` では「Memory: セッション間記憶（研究プレビュー、要アクセス申請）」と記載されていたため、**🔬 Research Preview → 🧪 Public Beta** に昇格記録が必要
