# 調査レポート: Claude Code v2.1.198（Claude in Chrome GA・BG エージェント自動 PR）+ Fable 5 / Mythos 5 アクセス復旧

**調査日:** 2026-07-02

## 概要

- **調査モード**: フルモード（新 CLI リリース **v2.1.198**〔7/1 20:45〕あり、前回レポートの v2.1.197〔6/30〕以降の差分）。
- **Layer 1 — CLI v2.1.198**: 目玉は **「Claude in Chrome が一般提供（GA）」の告知**。加えて **`claude agents` バックグラウンドエージェントの通知（`Notification` フック新ペイロード `agent_needs_input` / `agent_completed`）**、**新スキル `/dataviz`**（チャート/ダッシュボード設計 + カラーパレット検証器）、**BG エージェントがコード作業完了時に自動 commit / push / ドラフト PR 作成**、**Explore エージェントがメインセッションのモデルを継承（opus 上限）**、**サブエージェント/コンパクションが extended thinking 設定を継承**、**Gateway に Claude Platform on AWS（anthropicAws）をアップストリームプロバイダ追加**、**`/agents` ウィザード廃止**、多数のネットワーク耐性・BG タスク・エージェントチーム修正。**新 CLI フラグ・新 Hook イベント（`Notification` は既存イベントの新ペイロード）なし**。
- **🆕🚨 Layer 2/3 — Claude Fable 5 / Mythos 5 のアクセス復旧（7/1）**: 米商務省が **6/30 夜に輸出管理規制を解除**し、**Fable 5〔公開版〕は 7/1 に Claude Platform / Claude.ai / Claude Code / Cowork でグローバル復旧**。**Mythos 5 は 6/26 承認済みの米国機関限定で復旧**。**本リポジトリは前回 7/1 レポートで「Fable 5 は 7/1 も ⛔ 停止継続」と記録していたため、本レポートで『7/1 復旧』へ訂正・ステータス更新**。
- **🆕 Layer 2 — Model Entitlements（Enterprise ベータ・7/1）**: Enterprise 管理者がユーザーの利用可能モデルと effort レベル設定を制御できる新機能（Help Center 記載）。

---

## 主要な新機能

### 1. Claude in Chrome が一般提供（GA）✅ 📢
- **リリース状態**: ✅ GA（v2.1.198 changelog の筆頭項目「Claude in Chrome is now generally available」で告知）。
- **対象プラン**: Pro / Max / Team / Enterprise（有料プラン）。
- **内容**: ブラウザサイドパネルで Claude が Web ページを読む・クリック・ナビゲートする Chrome 拡張。**Claude Code 連携**（ターミナルでビルド → ブラウザで検証、Claude がコンソールエラー・DOM 状態を直接読んでデバッグ）が可能。Team/Enterprise では管理者が組織横断で有効/無効化・サイト allowlist/blocklist を設定可能。
- **経緯**: 2025-08 リサーチプレビュー → 2025-11〜12 有料プランへベータ拡大 → **2026-07-01（v2.1.198）で GA 告知**。プロンプトインジェクション防御を強化（攻撃成功率 23.6%→11.2%、追加緩和でブラウザ特化型攻撃 0%）。
- **注意**: マーケティングページ（claude.com/blog/claude-for-chrome）は本調査時点で GA 表記へ未更新だが、公式 CLI changelog が GA を明記。
- **情報源**: [Claude Code CHANGELOG](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) / [Claude for Chrome](https://claude.com/claude-for-chrome) / [Get started with Claude in Chrome（Help Center）](https://support.claude.com/en/articles/12012173-get-started-with-claude-in-chrome)

### 2. バックグラウンドエージェント通知（`Notification` フック）✅
- **リリース状態**: ✅ GA（v2.1.198）。
- **内容**: `claude agents` のバックグラウンドセッションが **入力を要求したとき / 完了したとき**に `Notification` フックが発火（新ペイロードタイプ `agent_needs_input` / `agent_completed`）。長時間の BG エージェントの状態を外部通知（デスクトップ通知・Slack 等）へ橋渡し可能。

### 3. 新スキル `/dataviz` ✅
- **リリース状態**: ✅ GA（v2.1.198）。
- **内容**: チャート・ダッシュボード設計のガイダンススキル。**実行可能なカラーパレット検証器**を同梱し、配色の妥当性（コントラスト等）を検証できる。

### 4. BG エージェントの自動 commit / push / ドラフト PR ✅
- **リリース状態**: ✅ GA（v2.1.198）。
- **内容**: `claude agents` から起動した BG エージェントが **worktree 内でコード作業を完了した際、停止して確認を求める代わりに自動で commit・push し、ドラフト PR を作成**する。完全自律のコード納品フローが標準挙動に。

### 5. Explore エージェントのモデル継承 ✅
- **リリース状態**: ✅ GA（v2.1.198）。
- **内容**: 組み込みの **Explore エージェント**が、従来の haiku 固定から **メインセッションのモデルを継承（ただし opus を上限にキャップ）**するよう変更。探索タスクの品質向上。

### 6. サブエージェント/コンパクションの extended thinking 継承 ✅
- **リリース状態**: ✅ GA（v2.1.198）。
- **内容**: サブエージェントとコンテキストコンパクションが**セッションの extended thinking 設定を継承**し、委任タスクの出力品質を改善。

### 7. Gateway に Claude Platform on AWS を追加 ✅
- **リリース状態**: ✅ GA（v2.1.198）。
- **内容**: Gateway のアップストリームプロバイダに **Claude Platform on AWS（`anthropicAws`）**を追加。model-not-found 応答時はフェイルオーバーチェーンを次へ進めるよう改善。

### 8. `/agents` ウィザード廃止 ⚠️
- **リリース状態**: ✅ GA（v2.1.198・破壊的変更）。
- **内容**: `/agents` の対話ウィザードを削除。今後はサブエージェントの作成/管理を **Claude に依頼**するか、**`.claude/agents/` を直接編集**する運用に変更（`/agents` コマンド自体はエージェント管理として残存）。

### 9. サブエージェントのメッセージ扱いの明確化 ✅
- **リリース状態**: ✅ GA（v2.1.198）。
- **内容**: サブエージェントは**起動元エージェントからのメッセージを通常のタスク指示として扱う**。ただしエージェントのメッセージは**決してユーザーの承認としては扱わない**（権限の分離を維持）。

## Layer 2/3: プラットフォーム・外部動向

### 🚨 Claude Fable 5 / Mythos 5 のアクセス復旧（7/1）— ステータス訂正
- **リリース状態**: **Fable 5〔公開版〕= ✅ グローバル復旧（7/1）** / **Mythos 5 = 🔬 承認済み米国機関限定で復旧**。
- **経緯**: 6/10 公開の Fable 5 jailbreak（Amazon 研究者がセーフガード回避を報告）を発端に **6/12 米商務省の輸出管理指令で全面停止**していたが、**商務省が 2 週間の分析・承認を経て 6/30 夜に規制を解除**。**Fable 5 は 7/1 に Claude Platform / Claude.ai / Claude Code / Cowork でグローバル復旧**。**7/7 まで Pro / Max / Team / 一部 Enterprise は週次上限の最大 50% までアクセス**、以降は usage credits へ移行。**Mythos 5 は 6/26 承認の米国機関限定**にとどまり、Anthropic は国内外パートナーへの拡大を継続調整中。
- **安全対策**: 新しい安全分類器が問題の jailbreak を **99% 超でブロック**（商務省 CAISI の独立テスト）。Anthropic は「セキュリティリスクの能動的検知・対処」「将来リリースのプロトコルで政府と協働」「悪意ある活動の報告」に合意。
- **Claude Code への影響**: Fable 5 復旧により `/model fable`・`claude --model claude-fable-5` が再び利用可能に（要 v2.1.170+）。デフォルトは Sonnet 5（v2.1.197〜）のため通常ワークフローへの影響は限定的。
- **本リポジトリの訂正**: 前回 7/1 レポートは「Fable 5 は 7/1 も ⛔ 停止継続」と記録していたが、Help Center・複数報道により **7/1 グローバル復旧が確定**。ステータスを更新。
- **情報源**: [CoinDesk（7/1）](https://www.coindesk.com/tech/2026/07/01/anthropic-restores-ai-models-fable-mythos-after-the-u-s-lifts-export-controls) / [9to5Mac（7/1）](https://9to5mac.com/2026/07/01/claude-fable-5-cleared-to-return-as-us-lifts-anthropics-export-control-restriction/) / [CIO（7/1）](https://www.cio.com/article/4191550/us-reverses-export-restrictions-on-anthropics-fable-5-mythos-5-ai-models.html) / [Release notes（Help Center）](https://support.claude.com/en/articles/12138966-release-notes)

### 🆕 Model Entitlements（Enterprise ベータ・7/1）🧪
- **リリース状態**: 🧪 ベータ（Enterprise 限定・7/1、Help Center 記載）。
- **内容**: Enterprise 管理者が**ユーザーの利用可能モデルと使用できる effort レベル設定を制御**できる。既存の `availableModels` / `enforceAvailableModels` 系ガバナンス（v2.1.173-175 で CLI 側実装済み）を組織コンソールのエンタイトルメントとして統合する動き。
- **情報源**: [Release notes（Help Center）](https://support.claude.com/en/articles/12138966-release-notes)

## バグ修正・パフォーマンス改善（v2.1.198 抜粋）

- **ネットワーク耐性**: 応答途中の一時的な切断（`ECONNRESET` 等）でターンを中断せず、バックオフ付きリトライに変更。
- サンドボックス化プロセスが同一ホストへ繰り返しアクセスした際の過剰な BG 分類器リクエストを修正。
- web / desktop / VS Code のタスクパネルで、完了・再開後も「Running」に固着する BG タスクを修正。
- エージェントチーム: API エラーで死んだチームメイトがリードへ「failed」を報告、スタックしたチームメイトへのメッセージで即時リトライを起こすよう修正。
- `/diff` パネルがブランチ切替・セッション外コミットで更新されない問題、フルスクリーンでの markdown テーブル右枠のオーバーフロー、Warp（macOS）での Cmd+click URL オープン、ダブルクリック選択で URL 全体（スキーム含む）を選択、を修正。
- Claude Platform on AWS / Mantle セッションで STS トークン失効時に「Please run /login」で行き詰まる問題を `awsAuthRefresh` 自動実行で修正。
- macOS BG エージェントのローカルネットワーク「no route to host」を Local Network entitlements 宣言で修正、`~52 秒`ごとの「Reconnecting…」を修正。
- `claude --bg` と `--print`/`-p` の同時指定を事前に拒否（従来は attach 不能セッションを無言生成）。
- highlight.js 11 へアップグレードしコードブロック/diff/プレビューのシンタックスハイライト精度向上。
- Mac から SSH 接続時にキーボードショートカットヒントを opt/cmd 表記（alt/super の代わり）に。
- API 過負荷時にスピナーのヒントをステータスページリンクに置換、2 回目の試行後にエラー理由を表示。
- `.claude/rules/` 条件付きルールが symlink 経由のパスで読み込まれない問題、`/branch` の既定フォーク名がコンパクション要約由来だった問題、plan mode 開始時に読み取り専用ツールが自動許可されない問題、`/desktop` の作業ディレクトリ判定失敗、などを修正。
