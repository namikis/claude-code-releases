# 調査レポート: Claude Code v2.1.86 & プラットフォーム更新（2026年3月下旬）

**調査日:** 2026-03-28

## 概要

前回レポート（v2.1.76〜v2.1.85）以降、CLIは**v2.1.86**（3月27日）がリリースされた。主にバグ修正・安定性向上が中心だが、MCP OAuth の RFC 9728 準拠、セッションIDヘッダー追加、Read ツールの最適化など実用的な改善が含まれる。

プラットフォームレベルでは、**1Mトークンコンテキストの一般提供（GA）**、**Extended Thinkingの表示制御**、**Models APIの機能フィールド追加**がAPIリリースノートで発表された。また、Claude Apps（モバイル）向けにインタラクティブアプリ対応が開始された。

---

## 詳細

### 1. Claude Code v2.1.86（2026年3月27日）

#### 新機能・改善

| 項目 | 説明 |
|------|------|
| **X-Claude-Code-Session-Id ヘッダー** | APIリクエストにセッションIDヘッダーを追加。プロキシがリクエストボディを解析せずにセッション単位で集計可能に |
| **VCS ディレクトリ除外追加** | `.jj`（Jujutsu）と `.sl`（Sapling）をVCSディレクトリ除外リストに追加 |
| **スケジュールタスクのタイムスタンプ** | `/loop` や `CronCreate` によるスケジュールタスクのトランスクリプトにタイムスタンプマーカーを追加 |
| **ディープリンク拡張** | クエリが最大5,000文字に対応、長い場合はスクロール確認警告を表示 |
| **MCP OAuth RFC 9728 準拠** | Protected Resource Metadata ディスカバリーに対応 |
| **組織ポリシーによるプラグインブロック** | ブロックされたプラグインがマーケットプレイスから非表示に |
| **PreToolUse フック拡張** | `AskUserQuestion` に対して `updatedInput` + `permissionDecision: "allow"` で自動応答可能に |
| **メモリファイル操作改善** | "Saved N memories" のファイル名がホバーでハイライト、クリックで開く |
| **スキル表示改善** | `/skills` メニューがアルファベット順ソート、説明文は250文字上限 |
| **Read ツール最適化** | コンパクトな行番号フォーマット、変更なし再読み込みの重複排除 |
| **`@` メンションのトークン削減** | ファイル内容がJSONエスケープ不要な生文字列として送信 |
| **Auto Mode 表示改善** | プラン未対応時に「temporarily unavailable」→「unavailable for your plan」に変更 |

#### 主要バグ修正

- `--resume` が v2.1.85 以前のセッションで「tool_use ids were found without tool_result blocks」エラーとなる問題を修正
- 条件付きスキル/ルール設定時にプロジェクトルート外のファイルで Write/Edit/Read が失敗する問題を修正
- スキル呼び出しごとに不要な設定ディスク書き込みが発生する問題を修正（Windows パフォーマンス/破損対策）
- 長いセッションで `/feedback` が OOM クラッシュする問題を修正
- `--bare` モードで MCP ツールが脱落し、ターン中メッセージが無視される問題を修正
- OAuth ログイン URL の `c` ショートカットコピーが約20文字に切り詰められる問題を修正
- マーケットプレイスプラグインスクリプトが macOS/Linux で「Permission denied」となる問題を修正（v2.1.83 以降の回帰）
- 複数インスタンス実行時にステータスラインが別セッションのモデルを表示する問題を修正
- Markdown/ハイライト描画キャッシュによるメモリ増大を修正
- 終了後にターミナルが enhanced keyboard モードに残る問題を修正（Ghostty, Kitty, WezTerm）
- **[VSCode]** 長時間操作中に「Not responding」が表示される問題を修正
- **[VSCode]** Max プランユーザーが OAuth トークンリフレッシュ後に Sonnet にデフォルトされる問題を修正

### 2. プラットフォーム API 更新（2026年3月）

前回レポートではCLIリリースに焦点を当てたが、同期間のプラットフォームレベルの重要な更新を補足する。

#### 1Mトークンコンテキスト — 一般提供（GA）（3月13日）

- **リリース状態**: GA
- Claude Opus 4.6 と Sonnet 4.6 の 1M トークンコンテキストウィンドウが正式に一般提供開始
- ベータヘッダー不要。200k 超のリクエストは自動的に対応
- 専用レート制限が撤廃され、標準アカウント制限が全コンテキスト長に適用
- メディア制限が 100 → **600 画像/PDFページ**に引き上げ（1Mコンテキスト使用時）

（出典: [Claude Platform Release Notes](https://platform.claude.com/docs/en/release-notes/overview)）

#### Extended Thinking 表示制御（3月16日）

- `thinking.display: "omitted"` を設定することで、レスポンスのストリーミングからthinkingコンテンツを省略可能
- thinkingブロックは空の `thinking` フィールドで返され、`signature` は保持されマルチターン継続に利用可能
- 課金は変更なし

（出典: [Claude Platform Release Notes](https://platform.claude.com/docs/en/release-notes/overview)）

#### Models API 機能フィールド追加（3月18日）

- `GET /v1/models` と `GET /v1/models/{model_id}` が `max_input_tokens`、`max_tokens`、`capabilities` オブジェクトを返すように
- プログラマティックにモデルの対応機能を照会可能に

（出典: [Claude Platform Release Notes](https://platform.claude.com/docs/en/release-notes/overview)）
