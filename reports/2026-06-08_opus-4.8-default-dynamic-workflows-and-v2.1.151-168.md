# 調査レポート: Claude Code v2.1.151〜v2.1.168 + Opus 4.8 デフォルト化・Dynamic Workflows・security-guidance プラグイン

**調査日:** 2026-06-08

## 概要

前回調査（5/26、v2.1.150 継続）以降の約 2 週間で **18 連続の CLI リリース（v2.1.151〜v2.1.168）** が出荷され、Claude Code は **Opus 4.8 をデフォルトモデルに切り替え**た。最大の節目は **5/28 の Claude Opus 4.8 一般提供**（SWE-bench Verified 88.6%、SWE-bench Pro 69.2%、Terminal-Bench 2.1 74.6%、fast mode は 2.5x 速・3 倍安価）と、同日 Claude Code 全面に投入された **Dynamic Workflows（🔬 研究プレビュー）**。加えて、無料の **security-guidance プラグイン（✅ 全プラン GA）**、**`claude plugin init` によるプラグイン scaffolding**、**`.claude/skills` 自動ロード**、**`fallbackModel`（最大 3 段フォールバック）**、**Windsurf → Devin Desktop 改称**、**`requiredMinimum/MaximumVersion` 管理設定** など多数の機能が追加された。Mythos クラスモデルの一般提供は「coming weeks（数週間以内）」と Opus 4.8 発表内で公式に言及され、噂段階から **公式の near-term アナウンス（📢）** へ確度が上がった。

- **対象期間**: 2026-05-27 〜 2026-06-08
- **新規 CLI バージョン**: 18 件（v2.1.151〜v2.1.168）
- **新モデル**: Claude Opus 4.8（5/28 GA）
- **主要新機能**: Dynamic Workflows（🔬）、security-guidance プラグイン（✅）、プラグイン scaffolding、fallbackModel、auto mode の Bedrock/Vertex/Foundry 対応

## 主要な新機能

### 1. Claude Opus 4.8 — Claude Code のデフォルトモデルに（✅ GA）

- **リリース状態**: ✅ GA（2026-05-28）
- **対象**: claude.ai / Claude API（`claude-opus-4-8`）/ Claude Code（CLI・Desktop・VS Code）/ Amazon Bedrock / Vertex AI / Microsoft Foundry
- **デフォルト化**: Max / Team Premium / Enterprise pay-as-you-go / Anthropic API でデフォルト。**デフォルト effort は「high」**
- **ベンチマーク**（Opus 4.7 比）:
  - SWE-bench Verified: **88.6%**（4.7 は 87.6%）
  - SWE-bench Pro: **69.2%**（4.7 は 64.3%、GPT-5.5 / Gemini 3.1 Pro を上回る）
  - Terminal-Bench 2.1: **74.6%**（4.7 は 66.1%、ただし terminal-coding は GPT-5.5 が首位）
  - エージェント型コーディング総合: 64.3% → 69.2%、ツール併用の多分野推論: 54.7% → 57.9%
  - Super-Agent ベンチで全ケースを end-to-end 完遂した唯一のモデル
- **価格**: 標準は据え置き（$5/$25 per MTok）。**Fast mode は $10/$50 per MTok で従来モデル比 3 倍安価、かつ 2.5x 高速**
- **アライメント**: 「prosocial traits で新たな最高水準」、コード欠陥の見落としが前世代比**約 4 分の 1**。VentureBeat は「near-Mythos level alignment」と評
- **Claude Code 連携**: v2.1.154 で Opus 4.8 を高 effort デフォルトで導入、v2.1.156 で thinking block 改変による API エラーを修正
- **情報源**: [Anthropic - Introducing Claude Opus 4.8](https://www.anthropic.com/news/claude-opus-4-8) / [TechCrunch](https://techcrunch.com/2026/05/28/anthropic-releases-opus-4-8-with-new-dynamic-workflow-tool/) / [MacRumors](https://www.macrumors.com/2026/05/28/anthropic-claude-opus-4-8/) / [9to5Mac](https://9to5mac.com/2026/05/28/anthropic-upgrades-claude-with-new-opus-4-8-model-heres-whats-new/)

### 2. Dynamic Workflows（動的ワークフロー）（🔬 研究プレビュー）

- **リリース状態**: 🔬 研究プレビュー（2026-05-28、v2.1.154 で同時投入）
- **対象プラン**: Enterprise / Team / Max（Opus 4.8 利用環境）
- **対象プラットフォーム**: Claude Code CLI / Desktop App / VS Code 拡張 / Claude API / Amazon Bedrock / Vertex AI / Microsoft Foundry
- **概要**: Claude が単一セッション内でタスクを複数サブタスクに分解し、**数十〜数百の並列サブエージェント**にオーケストレーション。実行・検証してから最終回答を提示する。コードベース移行などの大規模作業向け
- **アクセス方法**: effort メニュー経由の **`ultracode`** 設定（effort を xhigh にし、ワークフロー利用判断を Claude に委ねる）。`/config` の「Workflow keyword trigger」設定でプロンプト中の「workflow」語によるトリガを抑止可能
- **情報源**: [Anthropic Opus 4.8 発表](https://www.anthropic.com/news/claude-opus-4-8) / [InfoQ - Dynamic Workflows](https://www.infoq.com/news/2026/06/dynamic-workflows-claude-code/) / [ITBrief](https://itbrief.news/story/anthropic-launches-dynamic-workflows-in-claude-code)

### 3. security-guidance プラグイン（✅ GA / 無料）

- **リリース状態**: ✅ GA（2026-05-26、全プラン無料）
- **要件**: Claude Code CLI **2.1.144+** / Python 3.8+
- **概要**: コード編集・モデル出力・コミットをリアルタイムに自律レビューし、本番到達前に脆弱性を検出する 3 層防御:
  1. **パターンマッチ層**（毎編集、モデル呼び出しなし・課金ゼロ）: `eval()` / `new Function()` / `os.system()` / `child_process.exec()` / pickle デシリアライズ / DOM インジェクション等の危険構文をフラグ（公称 25 パターン）
  2. **モデルレビュー層**（各ターン終了時）: git diff 全体を別 Claude モデルがレビューし、認可バイパス / IDOR / SSRF / 弱い暗号などロジックレベルの脆弱性を検出
  3. **エージェント型レビュー層**（commit / push 時）: 周辺コード文脈を読み込み誤検知を低減
- **インストール**: `/plugin install security-guidance@claude-plugins-official`。組織は `.claude/settings.json` で強制可能
- **効果**: 内部テストでセキュリティ関連 PR コメントを 30〜40% 削減
- **情報源**: [CybersecurityNews](https://cybersecuritynews.com/free-security-plugin-for-claude-code/) / [Let's Data Science](https://letsdatascience.com/news/anthropic-releases-claude-code-security-plugin-for-real-time-a21ee55b)

### 4. プラグインシステムの拡張（✅ v2.1.157）

- **`.claude/skills` 自動ロード**: marketplace 不要でディレクトリ内プラグインを自動読み込み
- **`claude plugin init <name>`**: プラグインの scaffolding コマンド
- **`/plugin` 引数の autocomplete 拡張**、`settings.json` の `agent` フィールドを dispatch セッションで尊重
- **`EnterWorktree`**: セッション途中で Claude 管理 worktree 間を切替可能
- **`/plugin list`**（v2.1.163）: `--enabled` / `--disabled` フィルタ付きプラグイン一覧
- **default-disabled 宣言**（v2.1.154）+ カレントディレクトリに応じたプラグイン提案マッチング

### 5. fallbackModel（最大 3 段フォールバック）（✅ v2.1.166）

- **`fallbackModel` 設定**: プライマリモデルが過負荷・利用不可のとき順に試す**最大 3 つ**のフォールバックモデルを構成
- Claude Code は API が予期しない non-retryable エラーを返したとき、フォールバックモデルで 1 回リトライ（auth / rate-limit / request-size / transport エラーは即時表面化）

### 6. その他の機能追加

- **Auto mode の Bedrock / Vertex / Foundry 対応**（v2.1.158）: Opus 4.7/4.8 で `CLAUDE_CODE_ENABLE_AUTO_MODE=1` opt-in
- **Windsurf → Devin Desktop へ全面改称**（v2.1.162）
- **Remote Control を常設フッターリンク化**（v2.1.162）
- **`requiredMinimumVersion` / `requiredMaximumVersion` 管理設定**（v2.1.163）: 指定範囲外での起動を拒否
- **thinking 無効化**（v2.1.166）: `MAX_THINKING_TOKENS=0` / `--thinking disabled` / モデル別トグルで、デフォルト thinking モデルでも無効化可能
- **glob パターンを deny ルールのツール名位置で利用可能**（v2.1.166、`"*"` で全ツール拒否）
- **クロスセッションメッセージの権限ハードニング**（v2.1.166）: 他セッションからのメッセージがユーザー権限を持ち回らないよう修正
- **Hooks: `Stop` / `SubagentStop` が `hookSpecificOutput.additionalContext` を返却可能**（v2.1.163）
- **`/btw` の「c to copy」ショートカット**（v2.1.163）+ markdown 整形の保持
- **lean system prompt がデフォルト**（v2.1.154、Haiku / Sonnet / Opus 4.7 以前を除く）
- **streaming tool execution が常時有効化**（v2.1.154）
- **`claude agents` で `! <command>` によるバックグラウンドセッション起動**（v2.1.154）

### 7. レガル領域の大規模拡張（✅）

- **20+ の新しい法務 MCP コネクタ + 12 の実務分野プラグイン**: リサーチ・契約・ディスカバリ・案件管理・リーガルエイドを横断。法律事務所・社内法務向けにワークフロー支援とアクセスを拡大
- **情報源**: [Releasebot - Anthropic Release Notes (June 2026)](https://releasebot.io/updates/anthropic)
