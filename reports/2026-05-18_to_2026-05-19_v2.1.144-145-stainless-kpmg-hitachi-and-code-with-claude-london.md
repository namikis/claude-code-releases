## Layer 1: 公式チェンジログ — v2.1.144（2026-05-19 00:48 UTC）

**位置づけ**: v2.1.143（5/15）から **3 営業日空けて投下された大型リリース**。`/resume` のバックグラウンドセッション対応、`/model` 動作変更、「extra usage」→「usage credits」rename、60+ バグ修正を含み、**5 月後半最大規模の Claude Code リリース**の 1 つ。

### `/resume` / バックグラウンドセッション統合

#### `/resume` がバックグラウンドセッションに対応 ✅

- **新規追加**: `/resume` ピッカーに **`claude --bg` や agent view 経由のバックグラウンドセッションが `bg` マーカー付きで表示**されるように
- **影響**: 「終了したつもりだったがバックグラウンドで継続中」のセッションを `/resume` から復帰可能に。Agent View（5/11 launch）の運用が `/resume` ピッカーと完全統合
- **情報源**: [GitHub Releases v2.1.144](https://github.com/anthropics/claude-code/releases/tag/v2.1.144) / [Claude Code Changelog](https://code.claude.com/docs/en/changelog)

#### バックグラウンド subagent completion 通知に elapsed duration 追加 ✅

- バックグラウンド subagent が完了した際の通知に **「N 秒/分かかった」**情報を追加

### `/model` / 設定関連の重大変更

#### `/model` がセッション単位スコープに ✅（**動作変更注意**）

- **変更**: `/model` で変更したモデルは **現在のセッションのみ**に適用
- **default 設定**: `/model` のピッカーで **`d` キー**を押すと「新規セッションのデフォルト」として保存
- **背景**: 従来は `/model` がグローバル default を変更し、他セッション（特に並行 Agent View 内）に思わぬ影響を与えていた
- **互換性**: 既存スクリプトで `/model` を使っている場合、デフォルト変更が走らないため要確認

#### 「extra usage」→「usage credits」rename ✅

- **rename**: Claude Code CLI 全体で「extra usage」呼称を **「usage credits」**に統一
- **コマンド**: `/extra-usage` → **`/usage-credits`**（旧名称はエイリアスとして残存）
- **背景**: 5/14 のサブスクリプション課金構造変更（third-party agent harness の別建てメーター化、6/15 施行）に向けた呼称統一の伏線。**「extra usage」が「メータでクレジット消費」の事実をぼかしていたのを正名化**

### バックグラウンドセッション macOS / Linux 修正

#### macOS Full Disk Access 保護下での crash 修正 ✅

- **症状**: macOS で **Full Disk Access 保護フォルダ（Documents / Desktop / Downloads / 外部ボリューム）配下**でバックグラウンドセッション起動時に **「exit 1 before init」**で crash
- **修正**: v2.1.143 で発生していた regression を解消（v2.1.143 のサンドボックス強化の副作用）
- **影響**: macOS で `~/Documents/Projects/` 等を主開発ディレクトリにしている開発者の救済

### Startup / 安定性

#### `api.anthropic.com` unreachable 時の起動 hang 修正 ✅

- **症状**: `api.anthropic.com` がネットワーク的に到達不能な場合、Claude Code 起動が最大 **75 秒**ハング
- **修正**: タイムアウトを短縮 + リトライロジック改善
- **影響**: 企業ネットワーク / プロキシ環境 / オフライン環境での UX 改善

#### `head`/`tail`/`egrep` の spurious tool errors 削減 ✅

- **修正**: `head` / `tail` でファイル一部を見るだけで **read-before-edit check** を満たすように（以前は full read が必要で誤った "needs Read" エラー）
- `egrep` / `fgrep` / `git grep` / `git diff` の **「no matches」** が tool エラー扱いされない（正常結果として処理）

#### Bedrock/Vertex の「Opus (1M context)」選択不能 修正 ✅

- v2.1.129 regression で Bedrock / Vertex ユーザーが `/model` ピッカーで **"Opus (1M context)"** を選択できなかった問題を修正

## Layer 2: Claude Managed Agents の Self-hosted sandboxes / MCP tunnels 大型機能（2026-05-19）

### Self-hosted sandboxes 🧪 + MCP tunnels 🔬

**発表媒体**: [Anthropic Blog - New in Claude Managed Agents: self-hosted sandboxes and MCP tunnels (5/19)](https://claude.com/blog/claude-managed-agents-updates) / [Self-hosted sandboxes (Docs)](https://platform.claude.com/docs/en/managed-agents/self-hosted-sandboxes) / [MCP tunnels (Docs)](https://platform.claude.com/docs/en/agents-and-tools/mcp-tunnels/overview) / [Claude Platform Release Notes - May 19](https://platform.claude.com/docs/en/release-notes/overview) / [InfoQ](https://www.infoq.com/news/2026/05/claude-mcp-tunnels/) / [The New Stack](https://thenewstack.io/anthropic-mcp-tunnels-sandboxes/) / [9to5Mac](https://9to5mac.com/2026/05/19/anthropic-enhances-claude-managed-agents-with-two-new-privacy-and-security-features/)

#### Self-hosted sandboxes（公開ベータ）🧪

- **リリース状態**: 🧪 **公開ベータ**（同日リリース、5/19）
- **概要**: Claude Managed Agents の **ツール実行サンドボックスをユーザー側インフラに移行**できる機能
- **アーキテクチャ**: 
  - **Anthropic 側に残る**: agent loop（orchestration / context management / error recovery）
  - **ユーザー側に移動**: ツール実行 + ファイル / パッケージ / サービスへのアクセス
- **対応プロバイダ**: **Cloudflare / Daytona / Modal / Vercel**（managed sandbox プロバイダ経由）/ または完全カスタム自己ホスト
- **意義**: 
  - **データ漏洩防止**: 機密ファイル / パッケージ / サービスが Anthropic インフラを通らない
  - **コンピュート制御**: CPU / メモリ / capacity をユーザー側で sizing。長時間ビルド / 画像生成等の compute-heavy ワークロード対応
  - **既存セキュリティスタック活用**: network policies / audit logging / security tooling が既に動いている perimeter 内で動作
- **対象ユースケース**: 金融 / ヘルスケア / 政府 / 防衛 / 大企業 R&D で、データを外部 SaaS に流せない組織

#### MCP tunnels（研究プレビュー）🔬

- **リリース状態**: 🔬 **研究プレビュー**（同日リリース、5/19、限定アクセス、要申請）
- **概要**: Managed Agents や Messages API が **プライベートネットワーク内 MCP サーバー**に、**インバウンドファイアウォール開放なし**で接続できる機能
- **アーキテクチャ**: ユーザーが軽量 gateway を deploy → gateway から Anthropic infrastructure への **outbound 暗号化接続を 1 本**確立 → そのトンネル経由で MCP リクエスト
- **暗号化**: end-to-end 暗号化
- **意義**: 
  - **内部 DB / API / Ticketing / Knowledge base**を agent ツールとして公開可能、外部に公開せず
  - 従来のような **「IP allow-listing + Web Application Firewall + リバースプロキシ」**構成より大幅に簡素化
  - パブリッククラウド / オンプレ / VPC を問わず統一手順

#### その他 Managed Agents 改善（5/19）

- **active session 中の MCP server / tool config 更新サポート**: 長時間走行する agent session の途中で MCP 設定を変更可能（以前は session 再起動が必要）
- **100K トークン超の tool output 自動 spill**: `agent_toolset` と MCP tools の output が 100K トークンを超えると、自動で sandbox 内ファイルに spill。モデルには **truncated preview + file path** を渡し、必要に応じて full content を read

#### Claude Code への含意

- **Claude Code は Managed Agents（クラウド harness）とは別系統**だが、**MCP tunnels は Claude Code から内部 MCP サーバーへ接続するパターンとも構造的に対応**可能（将来的に Claude Code が tunnel client 機能を内蔵する蓋然性）
- **Self-hosted sandboxes は KPMG Digital Gateway や PwC Center of Excellence の "自社インフラ内 Claude" 要件と直結**

#### 概要

- **発表日**: 2026-05-18
- **リリース状態**: ✅ 買収完了 / 統合進行中
- **対象**: **Stainless**（2022 年創業、CEO Alex Rattray は ex-Stripe エンジニア、Sequoia / a16z バックド）
- **推定金額**: **$300M+**（The Information スクープ、Anthropic 公式は金額非開示）
- **チーム**: Rattray を含む全 Stainless チームが Anthropic 入社、Platform Engineering 部門（Head of Platform Engineering: Katelyn Lesse）配属

#### Stainless とは

- **API → SDK / CLI / MCP server 自動生成基盤**（TypeScript / Python / Go / Java 他）
- **Anthropic の全公式 SDK の生成基盤**（API 初期から、つまり Stainless の主要顧客の 1 つ）
- **OpenAI / Google / Cloudflare 等のライバル AI ラボも顧客**
- 顧客企業数: **数百社**

#### Hosted 製品の wind down

- **Stainless の hosted 製品（SDK generator 含む）は wind down**
- **既存顧客は生成済み SDK の所有権 + 修正権を完全保持**（オンプレでビルドプロセスを動かす形に移行可能）
- **新規顧客の受け入れ停止**

#### 戦略的位置付け

##### Anthropic 側
- **「Agent は接続先の質で決まる」**（Katelyn Lesse, Head of Platform Engineering）
- **MCP（Model Context Protocol）強化**の文脈で、SDK + CLI + MCP server の生成パイプラインを内製化
- **ライバル AI ラボの依存基盤を入手する戦略的価値**: TechCrunch は「**競合の手から重要インフラサプライヤーを取り上げた**」と評
- **6 ヶ月で 4 件目の買収**:
  - **2025-12**: **Bun**（JavaScript runtime / tooling）
  - **2026-02**: **Vercept**（computer use agent startup）
  - **2026-04**: **Coefficient Bio**（AI 創薬バイオテック）
  - **2026-05**: **Stainless**（agent connectivity / SDK 生成）
- 各買収が **emerging agent stack の異なるレイヤー**を狙う（ランタイム / コンピュータ操作 / 科学応用 / 接続レイヤー）

##### 競合への影響
- OpenAI / Google / Cloudflare は Stainless の hosted 製品終了に伴い、**SDK 生成パイプラインの内製化 or 代替ベンダー（Speakeasy 等）への移行**が必要
- **OpenAI への打撃が最大**（Stainless が OpenAI SDK 生成の中核）

##### Claude Code への含意
- **MCP server の自動生成・標準化が Anthropic 内製で加速**
- **Claude Code から SAP / Salesforce / ServiceNow 等の MCP server に直接接続できるエコシステム拡張**の蓋然性向上
- 5/12 Claude for Legal（20+ MCP コネクタ + 12 practice-area plugin）型の **業界別 MCP パッケージ**が他業界（金融 / ヘルスケア / 製造）にも横展開する開発加速材料
