# 調査レポート: Claude Code v2.1.199 / v2.1.200 — デフォルト権限モード「Manual」化・リトライ堅牢化・BG エージェント安定化

**調査日:** 2026-07-04

## 概要

- **調査モード**: フルモード（新規 CLI リリース 2 本 = **v2.1.199〔7/2〕**・**v2.1.200〔7/3〕**）。前回 7/3 レポート時点の Layer 1 最新 v2.1.198〔7/1〕から 2 バージョン進行。
- **⚙️ 挙動変更 — デフォルト権限モードを「Manual」に改称（v2.1.200）**: CLI・`--help`・VS Code・JetBrains で従来の「default」モードを **「Manual」** と表示・呼称するように統一。`--permission-mode manual` と `"defaultMode": "manual"` が `default` と併せて受理される。**機能上の破壊的変更ではなく用語統一**だが、UI・ドキュメント・設定値に影響。
- **⚙️ 挙動変更 — `AskUserQuestion` ダイアログが既定で自動継続しなくなった（v2.1.200）**: 応答待ちで放置しても勝手に進まず、アイドルタイムアウトは `/config` でオプトイン。
- **🔁 リトライ堅牢化（v2.1.199）**: サブスクライバー向けに**利用上限とは無関係の一時的 429 を自動リトライ（バックオフ付き）**するようになり、`CLAUDE_CODE_RETRY_WATCHDOG` は非容量系の一時エラーのデフォルトリトライ回数を **300** に引き上げ、`CLAUDE_CODE_MAX_RETRIES` の**上限 15 を撤廃**。
- **🧩 スタック済みスラッシュスキル（v2.1.199）**: `/skill-a /skill-b do XYZ` のように連結したスラッシュスキル呼び出しで、先頭の**最大 5 スキル**をすべてロード（従来は先頭 1 つのみ）。
- **🩹 信頼性・回復性の集中改善**: 両リリースともバックグラウンドセッション／BG エージェントデーモン／サブエージェントのエラー伝播に関する多数の修正が中心。ストリーミング中のエラーで部分出力を保持、サブエージェントの API エラーを親へ正しく報告、Linux デーモンの ~50 秒自己 kill 修正、macOS SSH コールドスタート回帰修正、tmux 3.4+ のちらつき修正、スクリーンリーダー出力改善など。
- **新スラッシュコマンド・新 Hook イベントの追加はなし**。新しい設定値は `--permission-mode manual` / `defaultMode: manual`（改称）と、`CLAUDE_CODE_RETRY_WATCHDOG`／`CLAUDE_CODE_MAX_RETRIES` の挙動変更のみ。
- **Layer 2/3**: 7/3〜7/4 に Claude Code 固有の新機能・新モデル・新ティアの公式発表なし。公式ニュースルームの直近投稿は 7/2「More details on Fable 5's cyber safeguards and our jailbreak framework」（Fable 5 の安全フレームワーク解説で、Claude Code CLI とは別軸）。Fable 5 グローバル復旧（7/1）は前回 7/2 レポートで既出。

## 主要な変更

### 1. デフォルト権限モードを「Manual」に改称 ⚙️（v2.1.200）
- **リリース状態**: ✅ GA（v2.1.200・2026-07-03）。
- **内容**: 従来「default」と呼んでいた権限モードを、CLI・`--help`・VS Code・JetBrains 全体で **「Manual」** と表示・呼称するように統一。`--permission-mode manual` および設定 `"defaultMode": "manual"` が、既存の `default` と**並行して**受理される（後方互換維持）。
- **含意**: 権限モードの分類は Manual / Auto / Plan / bypassPermissions などがあるが、最も基本の「毎回確認する」モードの名称が「default」→「Manual」に整理された。スクリプト・ドキュメント・チーム内手順で `default` を明示指定している箇所は当面そのまま動くが、今後は `manual` 表記が正となる。
- **情報源**: [CHANGELOG.md（v2.1.200）](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) / [Releases](https://github.com/anthropics/claude-code/releases)

### 2. `AskUserQuestion` ダイアログの自動継続を既定オフ化 ⚙️（v2.1.200）
- **リリース状態**: ✅ GA（v2.1.200）。
- **内容**: `AskUserQuestion`（Claude がユーザーに選択を尋ねるダイアログ）が、**既定で自動継続しなくなった**。放置してもタイムアウトで勝手に既定選択へ進まず、アイドルタイムアウトを使いたい場合は `/config` でオプトインする。
- **含意**: 無人セッションや長時間放置時に意図しない選択が確定するのを防ぐ、より保守的な既定挙動。自動進行を前提にしていたワークフローは `/config` での明示設定が必要。
- **情報源**: [CHANGELOG.md（v2.1.200）](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)

### 3. リトライ機構の堅牢化 🔁（v2.1.199）
- **リリース状態**: ✅ GA（v2.1.199・2026-07-02）。
- **内容**:
  - **一時的な 429（利用上限とは無関係のレート制限）を、サブスクライバー向けにバックオフ付きで自動リトライ**するようになった（従来はターンが失敗）。
  - `CLAUDE_CODE_RETRY_WATCHDOG` が、非容量系の一時エラーに対するデフォルトリトライ回数を **300** に引き上げ、`CLAUDE_CODE_MAX_RETRIES` の**上限 15 を撤廃**（v2.1.186 で導入された上限 15 の緩和）。
  - **SSL 証明書エラー**（TLS 検査プロキシ・`NODE_EXTRA_CA_CERTS` 欠落・期限切れ証明書）が、リトライを無駄に消費せず**即座に失敗して修正ヒントを提示**するように。
- **含意**: 無人・長時間セッションでの一時的障害への耐性が向上。特に企業プロキシ環境の証明書問題が「原因不明のリトライ地獄」から即時のアクショナブルなエラーに変わった。
- **情報源**: [CHANGELOG.md（v2.1.199）](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)

### 4. スタック済みスラッシュスキルの複数ロード 🧩（v2.1.199）
- **リリース状態**: ✅ GA（v2.1.199）。
- **内容**: `/skill-a /skill-b do XYZ` のように**連結したスラッシュスキル呼び出し**で、先頭に並ぶスキルを**最大 5 つまで**すべてロードするようになった（従来は先頭 1 つのみ有効）。
- **情報源**: [CHANGELOG.md（v2.1.199）](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) / [AI Coding Roundup, July 3 2026（oday-bakkour）](https://oday-bakkour.com/blog/ai-coding-roundup-july-3-2026)

### 5. サブエージェント／ストリーミングのエラー伝播修正 🩹（v2.1.199 / v2.1.200）
- **リリース状態**: ✅ GA。
- **内容**:
  - ストリーミング応答が**ミッドストリームの overloaded／サーバーエラー**で破棄される問題を修正 → 部分出力を「incomplete-response」注記付きで保持（v2.1.199）。
  - レート制限・サーバーエラーで打ち切られたサブエージェントが**サイレント失敗せず、部分成果を親へ返す**ように（v2.1.199）。
  - サブエージェントが **API エラー（例: usage limit reached）を成功結果として報告**していた問題を修正 → エラーを親エージェントへ正しく報告（v2.1.199）。
  - レート制限で**テキスト出力前に打ち切られたサブエージェント**が空結果を返す問題をクリーンに失敗するよう修正（v2.1.200）。
- **情報源**: [CHANGELOG.md](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)

### 6. バックグラウンドエージェント／デーモンの安定化 🩹（v2.1.199 / v2.1.200）
- **リリース状態**: ✅ GA。
- **主な修正**:
  - Linux の BG エージェントデーモンが、不正終了で破損したワーカーレコードにより **~50 秒ごとに自身と全稼働エージェントを kill** する問題（v2.1.199）。
  - macOS で **SSH 越しの BG エージェント コールドスタート失敗**（"Could not switch to audit session"、v2.1.196 の回帰）を修正（v2.1.199）。
  - スリープ／ウェイク後や停止セッション再開時に BG セッションが**サイレントに中断**する問題（v2.1.200）。
  - **PID を OS に再利用された古い `daemon.lock`** が残り BG エージェントが二度と起動しない問題、再インストールされた古いビルドがデーモンを乗っ取れてしまう問題（ビルドの埋め込みタイムスタンプで新しさを判定）を修正（v2.1.200）。
  - BG エージェントロースターの破損・ソケット認証トークン喪失など複数修正（v2.1.200）。
- **情報源**: [CHANGELOG.md](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)

### 7. アクセシビリティ・端末描画の改善 ♿（v2.1.200）
- **リリース状態**: ✅ GA（v2.1.200）。
- **内容**: **tmux 3.4+ での描画ちらつき**を synchronized terminal output 有効化で解消。スクリーンリーダー出力改善（装飾グリフを非表示化・トランスクリプト記号を短いラベルで読み上げ・ネストテーブルを `Header: value.` 形式で読み上げ）。`/mcp` サーバーリストのスクリーンリーダー／拡大鏡向けフォーカス追従修正。音声ディクテーションが無音録音時に誤って「Voice connection failed」と表示する問題の修正。インストールスクリプトが OOM で強制終了された際の説明改善。
- **情報源**: [CHANGELOG.md（v2.1.200）](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)

### 8. その他の修正（抜粋）
- **プランモード**: 状態変更を伴うブラウザツール呼び出しがプロンプトされない問題を修正。読み取り専用の `browser_batch` 呼び出しは正しく自動許可（v2.1.199）。
- `claude stop` が BG エージェント respawn と競合してサイレントに取り消される問題を修正（v2.1.199）。
- `.claude.json` の `disabledMcpServers`／`enabledMcpServers` が非配列値のとき起動クラッシュする問題を修正（v2.1.200）。
- `claude agents --plugin-dir <dir>` を `agents` の後ろに置くとプラグインの agents/skills が表示されない問題（v2.1.200）、git worktree からの project-scoped プラグイン誤ロード（v2.1.200）を修正。
- `claude agents` セッション行の PR リンクが冗長な「PR」ラベルなしで **`#N`** 表示に（v2.1.199）。
- `SessionStart` / `Setup` / `SubagentStart` フックが exit code 2 で stderr をサイレントに隠す問題を修正（トランスクリプトに表示、v2.1.199）。
