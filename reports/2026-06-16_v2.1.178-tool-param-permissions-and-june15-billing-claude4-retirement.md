# 調査レポート: Claude Code v2.1.178（ツールパラメータ権限ルール / ネスト .claude 対応）+ 6/15 サブスク課金変更・Claude 4 リタイア施行

**調査日:** 2026-06-16
**調査モード:** フルモード（Layer 1〜3）

## 概要

6/14 レポート（v2.1.176 / v2.1.177）以降の差分を Layer 1〜3 で調査。

**Layer 1** では CLI **v2.1.178（6/15 21:35 UTC）** を 1 件確認（v2.1.177 → 178 が新規、6/16 当日の新規 CLI なし）。目玉は **権限ルールのツールパラメータマッチ構文 `Tool(param:value)`**（`*` ワイルドカード対応、例: `Agent(model:opus)` で Opus サブエージェントをブロック）と **ネスト `.claude/` ディレクトリ対応**の 2 系統。ネスト `.claude/skills` は作業対象ファイルがそのディレクトリ配下のとき自動ロードされ（名前衝突は `<dir>:<name>` で解決）、ネスト `.claude/` 配下の agent / workflow / output-style は**作業ディレクトリに最も近いものが優先**される。加えて **Auto mode がサブエージェント生成前に分類器で評価**、**`/doctor` のフラットツリー表示・ステータスアイコン改善**、Workflow キーワード UI の紫シマー化（明示フレーズのみトリガー）、Remote Control の永続赤色 `/rc failed` エラー表示など。多数のバグ修正（サブエージェント transcript 表示、`--fallback-model` を尊重しない compaction、credential refresh 後の stale config、`claude agents workers` の 401 など）。**新スラッシュコマンド・新 Hook イベントの追加はなし**。**v2.1.171 は引き続き欠番**、Layer 1 最新は **v2.1.178**。

**Layer 2/3** では予告どおり **6/15 に 2 つの大型変更が施行**された。① **サブスクリプション課金構造変更**: programmatic 利用（Claude Agent SDK / `claude -p` ヘッドレス / Claude Code GitHub Actions / サードパーティ Agent アプリ）が flat-rate サブスクプールから完全に分離され、**ユーザー単位のドル建て monthly credit メーター**（標準 API レート課金・ロールオーバーなし・チーム間プールなし、Pro 約 $20・Max 5x 約 $100・Max 20x 約 $200/月）へ移行。クレジット枯渇時は overflow billing を手動有効化しない限り自動リクエスト停止。**ターミナル対話的 Claude Code・claude.ai web/desktop/mobile チャット・Cowork はサブスク内で継続**。② **旧世代 Claude 4 の API リタイア**: `claude-sonnet-4-20250514` / `claude-opus-4-20250514` を**米国太平洋時間 6/15 9:00 AM に猶予なしでリタイア**（以後 API 呼び出しはエラー、silent fallback なし）。移行先は Sonnet 4.6 / Opus 4.8、研究者は External Researcher Access Program で継続アクセス申請可。**Claude Fable 5 / Claude Mythos 5 は 6/16 時点も全面アクセス停止のまま**（6/12 米政府輸出管理指令、復旧日未定）。

## 主要な新機能（v2.1.178）

### 権限ルールのツールパラメータマッチ構文 `Tool(param:value)` ✅
- **リリース状態**: ✅ GA
- **概要**: 権限ルールでツールの**入力パラメータ**にマッチできる `Tool(param:value)` 構文を追加。`*` ワイルドカード対応。例: `Agent(model:opus)` で **Opus サブエージェントの生成をブロック**するなど、ツール名だけでなく引数値に基づくきめ細かい allow/deny が可能に。`availableModels` / `enforceAvailableModels`（v2.1.174-176）に続くモデル・ツールガバナンス強化の系譜。
- **初出**: v2.1.178 (2026-06-15)
- **情報源**: [Claude Code changelog](https://code.claude.com/docs/en/changelog)

### ネスト `.claude/skills` ディレクトリの自動ロード ✅
- **リリース状態**: ✅ GA
- **概要**: サブディレクトリ配下の `.claude/skills` にあるスキルが、**そのディレクトリ配下のファイルを扱うとき**に自動ロードされるように。トップレベルのスキルと名前が衝突する場合は `<dir>:<name>` の形で区別表示。モノレポ等でディレクトリごとにスキルを分けやすくなる。
- **初出**: v2.1.178 (2026-06-15)
- **情報源**: [Claude Code changelog](https://code.claude.com/docs/en/changelog)

### ネスト `.claude/` ディレクトリの優先度（最近接優先） ✅
- **リリース状態**: ✅ GA
- **概要**: agent / workflow / output-style の名前が衝突する場合、**作業ディレクトリに最も近い `.claude/` のものが優先**されるように。プロジェクトスコープの workflow 保存も、最も近い既存 `.claude/workflows/` をターゲットにする。
- **初出**: v2.1.178 (2026-06-15)
- **情報源**: [Claude Code changelog](https://code.claude.com/docs/en/changelog)

### Auto mode のサブエージェント事前評価 ✅
- **リリース状態**: ✅ GA
- **概要**: Auto mode で**サブエージェントの生成（spawn）を起動前に分類器で評価**するように。不要・危険な並列起動を抑止し、Auto mode の安全性・効率を改善。
- **初出**: v2.1.178 (2026-06-15)
- **情報源**: [Claude Code changelog](https://code.claude.com/docs/en/changelog)

### `/doctor` 改善・Workflow キーワード UI・Remote Control エラー表示 ✅
- **リリース状態**: ✅ GA
- **概要**:
  - **`/doctor`**: 一貫したフラットツリー表示、セクションステータスアイコンの明確化、コマンド名のハイライト。
  - **Workflow キーワード UI**: 紫色のシマーハイライトに変更し、「run a workflow」「workflow:」など**明示的なフレーズでのみトリガー**（誤発火を抑制）。
  - **Remote Control エラー**: 接続失敗時に永続的な赤色 `/rc failed` インジケータを表示し、エラー説明を改善。
- **初出**: v2.1.178 (2026-06-15)
- **情報源**: [Claude Code changelog](https://code.claude.com/docs/en/changelog)

## バグ修正・パフォーマンス改善（v2.1.178 主要分）

- **OOM クラッシュ修正** — CLI が stale な websocket / OAuth ファイルディスクリプタ環境変数を継承した際のメモリ枯渇クラッシュを修正。
- **Claude in Chrome 接続修正** — OAuth アカウント不一致時にサイレント接続失敗していた問題を修正。
- **ネストスキルの権限プロンプト誤ブロック修正** — ディレクトリ修飾名（`<dir>:<name>`）のネスト `.claude/skills` が権限プロンプトでブロックされる問題を修正。
- **サブエージェント transcript 表示修正** — ツール結果・ライブ進捗が表示されるように。
- **サブエージェント完了中の送信メッセージ脱落修正** / **実行中サブエージェントの ctrl+b バックグラウンド化が最初からやり直しになる問題修正**。
- **`claude agents workers` の `401 Invalid bearer token` 修正**。
- **compaction が `--fallback-model` を尊重しない問題修正** / **credential refresh 後も stale なキャッシュ設定でモデルリクエストが失敗し続ける問題修正**。
- **バックグラウンドセッションが `/bg` 後にターン終了後も「Working」表示のままになる問題修正**（v2.1.176 の同種修正に続く）。
- **`CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE=1` が新規マーケットプレイスインストールを妨げる問題修正**。
- **サブエージェント `disallowedTools` 内の MCP サーバーレベル指定がサイレント無視される問題修正**。
- **vim モードの undo が複数コマンドをマージしてしまう問題修正**（1 ステップずつ戻るように）。
- **[VSCode]** CJK IME 候補ウィンドウを Esc で閉じると実行中の Claude タスクがキャンセルされる問題修正。

## Layer 2/3: プラットフォーム・コーポレート動向

### 6/15 サブスクリプション課金構造変更が施行 ✅（施行済み）
- **確度**: ✅ 公式（5/14 発表 → 6/15 施行）
- **ステータス変更**: 📢 発表のみ → **✅ 施行済み**
- **内容**: 6/15 より **programmatic な Claude 利用**（**Claude Agent SDK / `claude -p` ヘッドレス CLI / Claude Code GitHub Actions / Agent SDK ベースのサードパーティアプリ**）がサブスクの flat-rate プールから分離され、**ユーザー単位のドル建て monthly credit メーター**（標準 API レート課金）に移行。
- **クレジット詳細**: 目安は **Pro 約 $20/月・Max 5x 約 $100/月・Max 20x 約 $200/月**。**クレジットはユーザー単位**（チーム間プールなし）、**請求サイクルで月次リフレッシュ・未使用分のロールオーバーなし**。枯渇後は overflow billing を手動有効化しない限り自動リクエストが完全停止（自動フォールバックなし）。
- **影響なし**: **ターミナルでの対話的 Claude Code 利用・claude.ai web/desktop/mobile チャット・Claude Cowork** は通常どおりサブスクの利用上限内で継続。
- **情報源**: [The New Stack](https://thenewstack.io/anthropic-agent-sdk-credits/) / [Codersera](https://codersera.com/blog/anthropic-june-2026-billing-change-claude-code/) / [TechTimes](https://www.techtimes.com/articles/317625/20260602/anthropic-ends-subscription-subsidy-agents-june-15-credit-pool-replaces-flat-rate-access.htm) / [UsageBox](https://usagebox.com/articles/anthropic-june-15-agent-sdk-credit-split-claude-4-retirement)

### 6/15 旧世代 Claude 4（Sonnet 4 / Opus 4）API リタイアが施行 ✅（施行済み）
- **確度**: ✅ 公式（Claude Platform リリースノート 6/15 付で明記）
- **ステータス変更**: 📢 リタイア予告 → **✅ リタイア済み**
- **内容**: **米国太平洋時間 6/15 9:00 AM** に `claude-sonnet-4-20250514`（Claude Sonnet 4）と `claude-opus-4-20250514`（Claude Opus 4）を API からリタイア。**以後これらのモデル ID への全リクエストはエラー応答**（猶予期間・後継モデルへの silent fallback なし）。4/14 に廃止予告済み。
- **移行先**: **Claude Sonnet 4.6 / Claude Opus 4.8**。API のフォーマット・認証・レスポンス構造は不変で、モデル ID 文字列の更新のみ。コードベース内の `20250514` を全文検索するのが露出箇所特定の最短手段。**研究者は External Researcher Access Program 経由で継続アクセスを申請可能**。
- **情報源**: [Claude Platform release notes（公式、6/15 付）](https://platform.claude.com/docs/en/release-notes/overview) / [Model deprecations（公式）](https://platform.claude.com/docs/en/about-claude/model-deprecations) / [MindStudio 移行ガイド](https://www.mindstudio.ai/blog/claude-sonnet-4-opus-4-deprecation-migration-guide)

### Claude Fable 5 / Claude Mythos 5 — 6/16 時点もアクセス停止継続 ⛔
- **確度**: ✅ 公式（6/12 停止、復旧アナウンスなし）
- **ステータス**: **⛔ アクセス停止のまま継続**（6/12 米政府輸出管理指令）。**6/16 時点で公式の復旧日は未定**。Anthropic は「できる限り早く復旧する」と表明しているが、両モデルとも全ユーザー向けに依然利用不可。AWS Bedrock 上の Fable 5 / Mythos 5 も同様に revoke 済み。
- **影響範囲**: **Opus 4.8 を含む他の全モデルは引き続き無影響**。Claude Code のデフォルトは Opus 4.8 のため通常ワークフローへの直接影響は限定的（`/model fable` は当面エラー/フォールバック）。
- **情報源**: [Anthropic 公式声明](https://www.anthropic.com/news/fable-mythos-access) / [AWS（Fable 5 on AWS、revoke 注記）](https://aws.amazon.com/blogs/aws/anthropic-claude-fable-5-on-aws-mythos-class-capabilities-with-built-in-safeguards-now-available/) / [TechCrunch（6/9）](https://techcrunch.com/2026/06/09/anthropics-claude-fable-5-is-a-version-of-mythos-the-public-can-access-today/)
