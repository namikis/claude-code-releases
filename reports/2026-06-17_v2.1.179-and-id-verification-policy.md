# 調査レポート: Claude Code v2.1.179 と本人確認ポリシー（7/8 発効）

**調査日:** 2026-06-17

## 概要

- **CLI v2.1.179**（6/16 リリース）は **バグ修正・パフォーマンス改善のみの安定化リリース**。新機能・新フラグ・新スラッシュコマンド・新 Hook の追加はなし。v2.1.171 は引き続き欠番。
- 主な修正は **ストリーム中の接続断時に部分応答を保持**（生エラーやスピナー固着を解消）、**WSL2（Windows Terminal / VS Code）のマウスホイールスクロール回帰修正**（v2.1.172 で混入）、Linux でのサンドボックス glob による Bash ツール説明肥大化の修正、サブエージェント transcript の Ctrl+O 表示修正、リモートセッションのバックグラウンドタスク状態表示改善、プラグインロード性能改善など。
- **【新規・重要】消費者向けプラン（Free / Pro / Max）に本人確認・年齢確認ポリシーを導入**（プライバシーポリシー改定、6/15 公表 → **7/8 発効**）。年齢確認は **Yoti**、本人確認は **Persona**（政府発行写真 ID + ライブセルフィー）。Team / Enterprise / Platform（商用条件）は対象外。
- **Claude プラットフォーム障害が継続**。6/16 で「12 日間に 10 回目」の重大障害（Opus 4.8 / Haiku 4.5 エラー）。Anthropic は前例なき需要によるインフラ逼迫を認める。
- **Claude Fable 5 / Claude Mythos 5 は 6/16 時点も引き続きアクセス停止**（6/12 の米政府輸出管理指令、復旧日未定）。**Opus 4.8 含む他全モデルは無影響**。

## 主要な新機能・変更

### 1. CLI v2.1.179（6/16）— バグ修正・安定化リリース 🧩

新機能の追加はなく、信頼性とプラットフォーム互換性の改善に特化したリリース。

**主な修正:**
- **ストリーム中の接続断ハンドリング改善**: 中断時に生のエラーを表示する代わりに**部分応答を保持**。スピナーが "running tool" で固着する不具合も解消。
- **WSL2 マウスホイールスクロール回帰修正**: Windows Terminal および VS Code 上での WSL2 スクロール不具合（v2.1.172 で混入した回帰）を修正。
- **Linux サンドボックス glob 修正**: 大規模ディレクトリツリーに対する `denyRead`/`allowRead` glob が Bash ツール説明を肥大化させ、セッションを使用不能にする問題を修正。
- **フィードバックサーベイ修正**: ターン完了直後の 1 桁の返信を誤ってセッション評価として記録する問題を修正。
- **ウェルカム画面のプロモバナー制限**: 複数のプロモバナーが積み重なる問題を修正し、**1 セッションあたり最大 1 件**に制限。
- **Ctrl+O サブエージェント transcript 表示修正**: サブエージェント閲覧時に transcript が表示されない不具合を修正。
- **プロンプト入力フォーカス修正**: サブエージェント／フッターパネルからプロンプト入力をクリックしてもフォーカスが戻らない問題を修正。
- **リモートセッションのバックグラウンドタスク状態表示改善**: ターン間で "still running" のまま固着して見える問題を解消。
- **リモートセッションのプラグインロード性能改善**。

- **リリース状態**: ✅ GA（npm 一般公開）
- **新コマンド/フラグ/Hook**: なし
- **情報源**: [CHANGELOG.md](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) / [code.claude.com changelog](https://code.claude.com/docs/en/changelog) / [Releasebot](https://releasebot.io/updates/anthropic/claude-code)

### 2. 本人確認・年齢確認ポリシー導入（消費者向けプラン、7/8 発効） 📢

Anthropic はプライバシーポリシーを改定し、**Claude Free / Pro / Max の消費者向けアカウント**に対し、必要に応じて年齢・本人確認を求める権利を明文化した（6/15 公表）。

- **発効日**: **2026-07-08**
- **対象プラン**: Claude **Free / Pro / Max**（消費者向け）。**Team / Enterprise / Platform（商用条件）は対象外**。
- **検証プロバイダ・方式**:
  - **年齢確認 = Yoti**（セルフィーによる年齢推定、パスポート／運転免許スキャン、または「18 歳以上」クレデンシャル）。Anthropic がユーザーを 18 歳未満と疑う兆候を検知した場合に発動。
  - **本人確認 = Persona**（政府発行写真 ID + 顔形状解析を伴うライブセルフィー）。「特定のユーザーおよび特定のユースケース」に適用。明確なトリガー条件はポリシー上非開示。
- **データ取り扱い**: ID 画像は **Persona / Yoti 側で保持され Anthropic の自社サーバには保存されない**。検証データは送受信・保管時に暗号化され、**広告・マーケティング・モデル学習には使用しない**。アクセスはアカウントレビュー・異議申し立てに限定。
- **Anthropic の説明**: 「Claude の能力が高度化する中でサービスを安全かつセキュアに保つため」と位置づけ。
- **リリース状態**: 📢 発表のみ（7/8 発効予定。Claude Code の CLI 機能ではなく、消費者向け Claude アカウント全体に対するポリシー変更）。なお Claude Code のサブスク認証で消費者プランを利用するユーザーには間接的影響の可能性。
- **情報源**: [SQ Magazine](https://sqmagazine.co.uk/anthropic-age-checks-id-verification-claude/) / [The Register](https://www.theregister.com/ai-and-ml/2026/06/15/anthropic-reserves-right-to-check-id-for-claude-subs/) / [KuCoin](https://www.kucoin.com/news/flash/anthropic-announces-identity-verification-for-claude-users-starting-july-8) / [Cyberpress](https://cyberpress.org/anthropic-updates-privacy-policy/)

### 3. Claude プラットフォーム障害の継続 ⚠️

6/16 に Claude が「12 日間（6/5 以降）で 10 回目」となる重大障害を記録。Opus 4.8 / Haiku 4.5 のエラーが 6/16 14:00 ET の修正試行後も持続。Anthropic は前例なき需要によりインフラが逼迫していることを認めている。背景として年商が 2025 年末の約 $9B から 2026 年 4 月初旬に $30B 超へ急伸、年間 $1M 超を支出するエンタープライズ顧客が 2 ヶ月足らずで 500 → 1,000 社へ倍増したことが挙げられる。

- **リリース状態**: 事象（障害）。CLI 機能とは独立。
- **情報源**: [TechTimes](https://www.techtimes.com/articles/318514/20260616/claude-outage-tenth-disruption-12-days-exposes-anthropic-infrastructure-strain.htm)
