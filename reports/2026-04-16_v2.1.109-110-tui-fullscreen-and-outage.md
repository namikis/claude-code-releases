# Claude Code 調査レポート: v2.1.109-110・TUIフルスクリーン・大規模障害

**調査日:** 2026-04-16
**対象期間:** 2026-04-15〜2026-04-16
**対象バージョン:** v2.1.109〜v2.1.110

---

## 概要

2バージョンのCLIリリース（v2.1.109-110）が4月15日にリリースされた。v2.1.110ではフリッカーフリーのフルスクリーンTUIモードが正式コマンド化（`/tui`）されたほか、プッシュ通知ツール、プラグイン管理の改善、複数のバグ修正が含まれる。プラットフォーム面では4月15日に大規模障害が発生し、Claude.ai・API・Claude Code全体で約3時間のサービス中断が発生。また、Claudeのパフォーマンス低下問題に対するユーザーの反発が主要メディアで報じられた。

---

## Layer 1: CLIリリース

### v2.1.110（2026年4月15日）

#### 新機能
- **`/tui` コマンドと `tui` 設定**: `/tui fullscreen` でフリッカーフリーレンダリングに切り替え可能。セッション中にモード変更できるようになった（以前は環境変数 `CLAUDE_CODE_NO_FLICKER=1` で起動前に設定が必要だった）
- **`autoScrollEnabled` 設定**: フルスクリーンモードで会話の自動スクロールを無効化可能
- **プッシュ通知ツール**: Remote Controlと「Push when Claude decides」設定が有効な場合、モバイルデバイスにプッシュ通知を送信
- **外部エディタ（`Ctrl+G`）にClaude最終応答をコメント付きコンテキストとして表示するオプション**

#### 変更
- **`Ctrl+O`**: ノーマル↔バーボストランスクリプトのトグルに変更（以前はFocus Viewもトグルしていた）
- **`/focus`コマンド**: Focus Viewの独立トグルに分離
- **`--resume`/`--continue`**: 期限切れでないスケジュールタスクを復活させるように改善

#### 改善
- `/plugin` Installedタブ: お気に入りと注意が必要なアイテムが上部に表示
- `/doctor`: 複数の設定スコープで定義されたMCPサーバーに対する警告を追加
- `/reload-plugins`: プラグイン依存関係の更新も反映

#### バグ修正
- MCPツール呼び出しがサーバー接続切断時にハングする問題を修正
- 非ストリーミングリトライがハングする問題を修正
- CLI再起動後にキーストロークがドロップされる問題を修正
- Remote Controlのセッション名変更が永続化されない問題を修正
- Focus Viewレンダリングの修正
- プラグイン依存関係の処理修正
- 権限ハンドリングの修正

### v2.1.109（2026年4月15日）

- **Extended Thinking インジケータ改善**: ローテーションするプログレスヒントを表示（待機中に「Analyzing code structure...」「Planning changes...」のような動的メッセージ）

## Layer 3: ニュースメディア

### Claude.ai 大規模障害（2026年4月15日） ✅ 解決済み

| 項目 | 詳細 |
|------|------|
| **発生日時** | 2026年4月15日 10:53 AM ET |
| **解決日時** | 2026年4月15日 1:42 PM ET（約3時間） |
| **影響範囲** | Claude.ai、Claude API、Claude Code |
| **症状** | ログイン障害、チャット中断、使用制限の表示不具合、部分的サービス低下 |
| **影響ユーザー数** | 15,000件超の報告 |
| **経緯** | 10:53 AM ETに確認→11:03 AM ETに一時修正→11:40 AM ETに再発（「Claude.ai and Platform are down」）→1:42 PM ETに完全解決 |
| **原因** | ユーザー需要の急増（公式声明） |

**注記**: 3月〜4月にかけて繰り返される障害パターン（4月6-7日の2日連続障害に続く）。Anthropicは需要増加を原因としている。

**情報源**: [CNBC](https://www.cnbc.com/2026/04/15/anthropic-outage-elevated-errors-claude-chatbot-code-api.html) / [TechRadar](https://www.techradar.com/news/live/claude-anthropic-down-outage-april-15-2026)

### Claudeパフォーマンス低下問題とユーザーの反発

| 項目 | 詳細 |
|------|------|
| **確度** | ✅ 確認済み（主要メディア複数が報道、Anthropic幹部が一部認める） |
| **時期** | 2026年2月後半〜3月に開始、4月に報道が集中 |
| **核心的問題** | Anthropicがデフォルトの「努力レベル」をmediumに引き下げ、トークン消費を節約。ユーザーへの透明な通知なし |

**具体的な症状**:
- 指示への追従頻度が低下
- 複雑なワークフローでのミス増加
- コンテキストの読み込み量減少
- ユーザー介入の必要性増加

**主要な批判者**:
- **Stella Laurenzo**（AMD AI部門シニアディレクター）: 6,852セッション・17,871思考ブロック・234,760ツール呼び出しを分析し、67%のパフォーマンス低下を文書化。「Claude Codeは複雑なエンジニアリング作業に信頼できないレベルに退行した」
- **Dimitris Papailiopoulos**（Microsoft研究者）: 「effortをmaxに設定しても極めて杜撰で、指示を無視し、同じミスを繰り返す」

**Anthropicの対応**:
- Boris Cherny（Claude Code責任者）がTeam/Enterpriseユーザーのデフォルトを「high effort」に変更するテストを認める
- v2.1.94でデフォルト努力レベルをmedium→highに変更（API-key、Bedrock/Vertex/Foundry、Team、Enterpriseユーザー向け）
- 透明性の欠如に対する批判は認めつつ、意図的な品質低下を否定

**IPOへの影響**: 年間収益$30B超でIPOを目指す中、透明性ブランドの毀損が投資家の評価に影響する可能性が指摘されている。

**情報源**: [Fortune](https://fortune.com/2026/04/14/anthropic-claude-performance-decline-user-complaints-backlash-lack-of-transparency-accusations-compute-crunch/) / [VentureBeat](https://venturebeat.com/technology/is-anthropic-nerfing-claude-users-increasingly-report-performance) / [The Register](https://www.theregister.com/2026/04/13/claude_outage_quality_complaints/) / [Cryptonomist](https://en.cryptonomist.ch/2026/04/13/claude-code-performance/)
