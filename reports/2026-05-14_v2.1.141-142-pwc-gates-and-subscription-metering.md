## Layer 1: 公式チェンジログ — v2.1.141（2026-05-13 23:19 UTC）

**位置づけ**: v2.1.140（5/12 21:09 UTC）の翌日。**v2.1.139 / 140 の連続ローンチで判明したエンタープライズ / プラットフォーム課題への中規模機能追加 + 40+ バグ修正**を投入。新規機能カテゴリ複数（hook / federation / `/feedback` / Rewind）。

### Hook / Permission / Auto Mode

#### `terminalSequence` hook 出力フィールド ✅

- **新規追加**: hook の JSON 出力に **`terminalSequence`** フィールドを追加。デスクトップ通知 / ウィンドウタイトル / ベル音などを、現在ターミナルを制御していないバックグラウンド hook からも送信可能に
- **背景**: v2.1.139 で投入された Agent View / バックグラウンドセッションで、background hook が前景ターミナルへの通知を出せず「完了したか分からない」という UX 不足を補強
- **情報源**: [Claude Code Changelog](https://code.claude.com/docs/en/changelog) / [GitHub Releases v2.1.141](https://github.com/anthropics/claude-code/releases)

#### Auto mode 権限ダイアログが `permissions.ask` ルール起因を明示 ✅

- Auto mode で permission prompt が出た際、**「`permissions.ask` ルールが原因」**であることをダイアログに明示。サイレントな Auto mode 中断の原因が分かりやすく
- v2.1.136 の `autoMode.hard_deny` 階層と組み合わせ、Auto mode の判断パスが視覚化される方向に

#### Background agents の permission mode が `/bg` / `←←` で保持 ✅

- `/bg` または `←←` 経由でバックグラウンドに移動する際、**permission mode（`plan` / `acceptEdits` / `bypassPermissions`）が元のセッションから引き継がれる**ように修正（以前は default に戻っていた）
- v2.1.139 で導入されたバックグラウンドホスティングの permission 引き継ぎ regression を解消

### Workload Identity Federation / Enterprise

#### `ANTHROPIC_WORKSPACE_ID` 環境変数 ✅

- **新規追加**: ワークロード ID フェデレーション（WIF）のスコープ指定用に `ANTHROPIC_WORKSPACE_ID` 環境変数を追加
- **用途**: AWS IAM / Azure AD / GCP Workload Identity 経由でフェデレーション認証する際、Anthropic 側ワークスペースを ID で明示
- **背景**: 5/11 Claude Platform on AWS GA 後、IAM ベース統合運用での「どのワークスペースに認証するか」を環境変数で明示できる必要性が顕在化

#### Plugin の HTTPS clone オプト ✅

- **新規追加**: `CLAUDE_CODE_PLUGIN_PREFER_HTTPS` 環境変数で、GitHub plugin の clone を **SSH ではなく HTTPS** に強制
- **背景**: エンタープライズ環境で SSH key 配布が制限されている / プロキシ越え HTTPS が必須の組織向け

### `/feedback` / Rewind / `claude agents`

#### `/feedback` に recent sessions 添付オプション ✅

- **新規追加**: `/feedback` 実行時に **直近 24 時間または 7 日間のセッション履歴を添付**できるオプション
- ユーザーがバグ報告する際、Anthropic 側がコンテキスト復元しやすくなる（Anthropic 社内サポート効率化）

#### Rewind menu に「Summarize up to here」追加 ✅

- **新規追加**: Rewind menu に **「Summarize up to here」** メニュー項目を追加。指定位置までの履歴を圧縮（compaction）してコンテキスト効率を改善
- 既存の rewind（巻き戻し）と組み合わせ、長いセッションの中盤を圧縮しつつ最新部分は残す運用が可能に

#### `claude agents` UX 改善 ✅

- **agents finishing but leaving background shell** が完了状態として **Completed** に遷移するように（以前は曖昧）
- **agents 起動時に pre-warmed worker が unhealthy なら failover**（クラッシュせずに継続）
- 5 分間 idle の **空セッションは daemon が自動 retire**（リソース節約）

### IDE 統合 / Permission Prompt UI

#### File-edit permission prompt の「view diff in your IDE」復活 ✅

- IDE 接続時、ファイル編集の permission prompt に **「IDE で差分を見る」**ボタンを復活（過去の regression で消失）
- VS Code / JetBrains 接続時に編集前に IDE 上で差分プレビュー可能

#### スピナーの長時間思考フィードバック改善 ✅

- 10 秒以上 thinking が続くと **amber 色のステータスメッセージ**を表示。「Claude が長く考えている」のを視覚的に明示

## Layer 1: 公式チェンジログ — v2.1.142（2026-05-14 22:55 UTC）

**位置づけ**: v2.1.141 の翌日。**`claude agents` の機能完全化（8 つの新フラグ）+ Fast mode のデフォルトモデル切替（4.6 → 4.7）+ プラグイン UX 改善**を投入。バックグラウンドセッション関連の macOS sleep / wake / daemon upgrade ライフサイクル issue を集中修正。

### `claude agents` フラグ拡張（重要）

#### `claude agents` に 8 つの新フラグ ✅

| フラグ | 用途 |
|--------|------|
| `--add-dir <path>` | セッションに追加ディレクトリを動的追加（既存セッションの `/add-dir` と同等を起動時に） |
| `--settings <path>` | カスタム settings.json を読み込み |
| `--mcp-config <path>` | カスタム MCP 設定ファイル |
| `--plugin-dir <path>` | プラグインディレクトリを指定 |
| `--permission-mode <mode>` | `plan` / `acceptEdits` / `bypassPermissions` を起動時指定 |
| `--model <name>` | モデルを起動時指定（`claude-opus-4-7` 等） |
| `--effort <level>` | effort レベル指定（`low` / `medium` / `high` / `xhigh`） |
| `--dangerously-skip-permissions` | bypass permission mode を起動時に明示 |

- **背景**: v2.1.139 で投入された `claude agents` 単独サブコマンドは初期は最小限のフラグセットだった。**v2.1.142 で `claude` 本体と同等のフラグセットに揃え**、Agent View をスクリプト / CI から自由に起動できるレベルに到達
- **影響**: CI/CD でのバックグラウンドセッション自動起動（特に Linux サーバー上での remote `claude agents` 起動）が現実的に
- **情報源**: [GitHub Releases v2.1.142](https://github.com/anthropics/claude-code/releases) / [Claude Code Changelog](https://code.claude.com/docs/en/changelog)

### Fast mode

#### Fast mode のデフォルトモデルを Opus 4.7 に切替 ✅

- **オプトアウト**: `CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE=1` 環境変数で Opus 4.6 fast mode に pin 可能
- **背景**: 5/12 の **API 側 Fast mode for Opus 4.7 launch（platform release notes）から 2 日遅れで CLI が追従**。5/13 レポート時点では「Claude Code 経由で Opus 4.7 fast mode を使えるか不明」だったが、**v2.1.142 で明確にデフォルト**

#### 戦略的含意

- **Anthropic の "Responsible Scaling" / RSP の社会的延長**: モデル能力の上昇に伴う社会価値拡大を **philanthropic 配信経路**で具体化
- **IPO ロードショー前哨**: $30B / $900B 評価額調達ラウンド（5/22 Bloomberg「来週クローズ可能性」）の文脈で、**社会的責任を可視化することで規制当局 / 公的機関調達への信頼性向上**
- **OpenAI との差別化**: OpenAI も非営利財団から営利化転換に伴う philanthropic コミットを行うが、Anthropic は **Public Benefit Corporation（PBC）構造を活用した明確な beneficial deployment 戦略**を打ち出す

## Layer 3: Claude サブスクリプション課金構造の重大変更（2026-05-14）

### 6/15 から third-party agent harness を別建てメーター化 📢

**発表媒体**: [Anthropic Blog - Subscription updates (5/14)](https://claude.com/blog/subscription-updates-may-2026) （リンク要確認） / [Axios "Anthropic tightens Claude limits as OpenAI courts agent users" (5/14)](https://www.axios.com/2026/05/14/anthropic-claude-price-openai-tokens) / [InfoWorld "Anthropic puts Claude agents on a meter across its subscriptions"](https://www.infoworld.com/article/4171274/anthropic-puts-claude-agents-on-a-meter-across-its-subscriptions.html) / [DevToolPicks Analysis](https://devtoolpicks.com/blog/anthropic-splits-claude-subscriptions-agent-sdk-credit-june-2026)

#### 概要

- **発表日**: 2026-05-14
- **施行日**: **2026-06-15**（約 1 ヶ月の移行期間）
- **変更内容**: 有料 Claude プラン（Pro / Max / Team）の **third-party agent harness（OpenClaw、Codex 系外部 SDK、その他 Agent SDK 経由の自作ハーネス等）利用が、Claude 本体 / Claude Code の rate limit とは別建ての monthly credit メーター制に分離**
- **建前**: 「多数派ユーザーの使い方をサポートする」（Anthropic 公式コメント）
- **実態**: **エージェント heavy ユーザー（人間が数百 prompt/日に対しエージェントは数千 req/日 + テスト連続実行 + Web ブラウズ + 再帰モデル呼び出し）の経済性適正化**

#### 経済的背景（Axios / InfoWorld 報道）

- **「自律エージェントは人間とは桁違いにリソースを消費」** — Anthropic 公式コメント
- **ServiceNow / Uber が「年間 AI トークン予算を使い切った」** と報道。エージェント時代の "unlimited" 終焉の象徴
- **エンタープライズ顧客（80% 売上比率）** にとっては unlimited サブスク継続だがコスト管理可視性が改善

#### OpenAI の対抗策

- OpenAI は **「new business 顧客に Codex 2 ヶ月無料」** を同期間に発表。Anthropic の制限強化で離脱を狙うユーザーを取り込む戦略
- ただし OpenAI も中長期的には同様のメーター化を実施する蓋然性が高い（エージェントエコノミクスの構造的問題）

#### Claude Code への影響

- **Claude Code 本体（`claude` CLI / Claude Code SDK 経由のセッション）は影響なし**（既存サブスクリプション内のレート制限が継続適用）
- **OpenClaw / Codex 系の外部ハーネスから Claude API を叩く運用は別メーター**（6/15 から）
- **Claude Code 内部で `--api-key` 経由で外部 API key を使う場合**は、その API key の課金が直接 metered（既存通り）
- **Agent SDK 経由の自作エージェント**: monthly credit 適用範囲。OpenClaw / SST 等のサードパーティ harness ユーザーは要注意
- 4/22 の **Pro プラン試験的 Claude Code 削除事件**（compute crunch 対応）の延長戦上。**Claude Code 単独プラン構想**が再加速する可能性（IPO ロードショーでの商品差別化材料として）

#### 戦略的位置付け

- **Amodei "avg developer 20 hours/week with Claude Code"** コメント（5/8 Fortune）/ **Compute crunch ナラティブ**（5/6 SpaceX Colossus 1 / 5/8 Akamai $1.8B 等のコンピュート確保ラッシュ）の延長戦上
- IPO 前のマージン改善材料: **「使う人だけ払う」化により unprofitable な heavy user 補助コストを切り離し**
